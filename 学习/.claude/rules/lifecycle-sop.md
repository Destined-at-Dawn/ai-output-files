# 项目生命周期六阶段 SOP

> 纵轴 = 时间。每阶段给：**目标产出 / 标准动作 / 出口门禁 / 反模式**。
> 第三层工程法则规定「每个阶段都要守什么」，本层规定「按时间顺序做什么」。

---

## 使用方式

- 新项目启动时通读三条根本信条定调
- 每阶段开工前看对应阶段 SOP
- 每天/每轮对话对照工程法则自检
- 要宣告完成时跑门禁

---

## 阶段 1　立项与调研

**目标产出**：对任务规格的精确理解 + 合规与约束边界清单 + 调研 brief（数字带来源与可信度）+ 显式的范围排除清单。

**标准动作**：
1. 逐条对照任务书/规范/官方答疑，做**三档合规核对表**（🔴 致命 / ⚠️ 重要 / ✅ 合规）
2. 调研覆盖**学术文献 + 工业实践 + 商业产品**三栖；每个引用的数字标 `数值 (来源) [可信度 A/B/C/D]`，最低档不进正式产物
3. 显式写下「本项目**不做**什么」（范围排除清单）

**出口门禁**：合规核对表无致命级未决项；命名/接口/工具/约束已锁定。

**反模式**：
- 把规范当背景、未做合规核对就动手
- 引用数字不标来源
- 不写「不做什么」

---

## 阶段 2　方案空间探索

**目标产出**：候选方案集 + 各自粗筛指标 + 已淘汰方案的明确理由。

**标准动作**：
1. 先**铺开方案空间**，不过早收敛到一个
2. 优化方向用**实测扫描**找 sweet spot——绝不假定单调
3. 用**多保真漏斗**——粗筛只用来排序，不用来下结论

**出口门禁**：每个进入下阶段的候选都有粗筛数据；淘汰的方案有明确理由并入负结果日志。

**反模式**：
- 直接取「最紧/最多/最快」的那一端
- 用粗筛数字宣告达标
- 过早砍光候选不留对照基线

---

## 阶段 3　多 Agent 并行攻坚

**目标产出**：各并行小队的攻坚结果 + 可独立复验的证据。

**标准动作**：
1. 把任务拆成**正交**子空间；每队独占一个写入目录
2. 他队目录**只读**；复用一律 Read → Write 回本队
3. 共享只读资源走路径引用，不复制
4. 总指挥强制每队交付**证据路径**，不收只给结论的汇报

**出口门禁**：子空间确认正交（无重叠劳动）；无跨队文件污染；每个结果可被独立重现。

**反模式**：
- 两个 agent 同时写同一文件互相覆盖
- 总指挥只看汇总不要证据
- 跨队拉模块靠 import 路径顺序自然解析

---

## 阶段 4　分层验证与签核

**目标产出**：每个结果都带**完整口径标注**的签核数据。

**标准动作**：
1. 显式约定**签核层级**并在每个结果旁标注
2. 每个数字绑定**六元组**：`输入版本 + 验证环境 + 命令 + 报告路径 + 时间戳 + 工具版本`
3. 跨 Agent 横向对比前，先逐项核验各自流程完整性

**出口门禁**：每个「通过」带口径标签；任何高分都经过防假象审计。

**反模式**：
- 拿快速校验结果直接当最终结论
- 跨人比较不核验各自流程口径是否一致

---

## 阶段 5　复盘与知识结晶

**目标产出**：本轮的坑/教训沉淀进规则库；死路归档。

**标准动作**：
1. 每轮对话/调试结束**立即**把过程凝练成可复用规则
2. 用户纠正 ≥2 次、或同类踩坑 ≥2 次 → 升级为规则甚至独立 Skill
3. 死路写进**负结果日志**
4. 设**反过度沉淀阀门**：单次未重复的 feedback 暂不入库

**出口门禁**：本轮新坑已入规则库；本轮死路已归档。

**说明**：本阶段是其他五个阶段的「免疫系统」——它的产出保证**同类坑不被踩第二次**。

---

## 阶段 6　交付与边界声明

**目标产出**：自含、可复现的交付包 + 边界声明。

**标准动作**：
1. 每个声称的指标/结论对应一个**包内可打开的证据文件**
2. 随交付配三类边界清单：**可写入材料 / 不可写入材料 / 尚不能声称**
3. 交付包**自含**——复现步骤一键可跑
4. 数字与证据**100% 一致**

**出口门禁**：包内证据齐全 + 边界声明齐全 + 数字与证据一致。

**反模式**：
- 交付包里数字写了但证据文件缺
- 过度声称把演示结果当真实验证
- 交付包不自含必须额外配置才能跑

---

## 阶段流转示意

```
┌─────────┐   合规核对通过   ┌─────────┐   候选有数据   ┌─────────┐
│ 1. 立项  │ ─────────────→ │ 2. 探索  │ ─────────────→ │ 3. 并行  │
│  调研    │   锁定约束      │  方案    │   淘汰有理由   │  攻坚    │
└─────────┘                 └─────────┘                 └─────────┘
                                                             │
    ┌──────────┐   新坑入规则库  ┌──────────┐  口径标签齐全  │
    │ 5. 复盘  │ ←──────────── │ 4. 签核  │ ←──────────────┘
    │  结晶    │               │  验证    │
    └──────────┘               └──────────┘
         │
         │ 门禁全过 + 边界声明齐全
         ▼
    ┌──────────┐
    │ 6. 交付  │
    │  声明    │
    └──────────┘
```
