# 偏好记忆规则

## 核心原则

**用户纠正你的每一次，都是在教你如何更好地为他工作。**

---

## 必须记录的偏好类型

### 1. 信息密度偏好

**记录内容**：
- 用户喜欢简洁版还是详细版
- 用户能接受的输出长度
- 用户对技术细节的需求程度

**记录格式**：
```markdown
## 信息密度偏好
- 默认模式：[简洁/标准/详细]
- 最大输出长度：[300字/500字/不限]
- 技术细节需求：[按需提供/主动提供/不需要]
```

**应用方式**：
- 下次对话开始时，检查是否有记录
- 如果有，主动询问"需要我按照之前的风格输出吗？"
- 根据用户回答调整输出风格

### 2. 技术水平偏好

**记录内容**：
- 用户的专业背景
- 用户熟悉的技术栈
- 用户不熟悉的领域

**记录格式**：
```markdown
## 技术水平
- 专业背景：[教授/工程师/学生/非技术人员]
- 熟悉领域：[芯片设计/软件开发/数据分析/...]
- 不熟悉领域：[需要解释的术语/概念]
```

**应用方式**：
- 根据用户的技术水平调整术语使用
- 在不熟悉的领域主动解释术语
- 避免在熟悉的领域过度解释

### 3. 输出格式偏好

**记录内容**：
- 用户喜欢的输出结构
- 用户喜欢的呈现方式
- 用户对格式的要求

**记录格式**：
```markdown
## 输出格式
- 首选结构：[结论先行/背景先行/问题先行]
- 表格使用：[喜欢/中性/不喜欢]
- 列表使用：[喜欢/中性/不喜欢]
- 代码块使用：[喜欢/中性/不喜欢]
```

**应用方式**：
- 默认使用用户喜欢的格式
- 如果不确定，询问用户的偏好
- 记住用户的纠正并应用

### 4. 交互风格偏好

**记录内容**：
- 用户喜欢的沟通方式
- 用户对主动性的接受程度
- 用户对确认的偏好

**记录格式**：
```markdown
## 交互风格
- 沟通方式：[直接/委婉/正式/非正式]
- 主动性：[喜欢AI主动建议/只在被问时回答]
- 确认偏好：[每次确认/关键点确认/不确认]
```

**应用方式**：
- 根据用户的沟通方式调整语气
- 根据用户的主动性偏好决定是否主动建议
- 根据用户的确认偏好决定是否询问

### 5. 痛点和禁忌

**记录内容**：
- 用户最讨厌的输出方式
- 用户明确禁止的行为
- 用户特别在意的点

**记录格式**：
```markdown
## 痛点和禁忌
- 最讨厌：[信息过载/技术黑话/缺乏结论/...]
- 禁止行为：[使用某些术语/输出某些内容/...]
- 特别在意：[准确性/完整性/简洁性/...]
```

**应用方式**：
- 严格避免用户的痛点
- 绝不触碰用户的禁忌
- 特别关注用户在意的点

---

## 记录时机

### 必须记录的情况

1. **用户明确纠正时**
   - 用户说"太详细了" → 记录"偏好简洁"
   - 用户说"看不懂" → 记录"需要解释术语"
   - 用户说"这不是我要的" → 记录用户真正想要的

2. **用户表达偏好时**
   - 用户说"我喜欢用表格" → 记录"表格使用：喜欢"
   - 用户说"给我结论就行" → 记录"默认模式：简洁"
   - 用户说"我不懂这个术语" → 记录"不熟悉领域：[术语]"

3. **用户重复同一要求时**
   - 如果用户多次要求"简单点" → 记录"偏好简洁"
   - 如果用户多次要求"详细点" → 记录"偏好详细"
   - 如果用户多次使用同一格式 → 记录"首选格式：[格式]"

### 不需要记录的情况

1. **一次性的特殊要求**
   - 用户说"这次给我详细版" → 不记录（可能只是这次需要）
   - 用户说"这个术语我不懂" → 不记录（可能只是这个术语）

2. **不确定的偏好**
   - 用户没有明确表达偏好
   - 用户的要求可能是临时的
   - 用户的反馈可能是误解

## 应用流程

### 对话开始时

1. **检查是否有用户偏好记录**
   - 如果有，读取记录
   - 如果没有，进入下一步

2. **主动询问**
   ```
   我注意到我们之前合作过。
   需要我按照之前的风格输出吗？
   还是你有新的偏好？
   ```

3. **根据回答调整**
   - 如果用户说"是" → 应用记录的偏好
   - 如果用户说"不用" → 询问新的偏好
   - 如果用户说"有调整" → 更新记录

### 对话过程中

1. **观察用户的反馈**
   - 用户是否在纠正你的输出风格
   - 用户是否在表达偏好
   - 用户是否在重复同一要求

2. **及时调整**
   - 如果用户说"太详细了" → 立即切换到简洁模式
   - 如果用户说"看不懂" → 立即解释术语
   - 如果用户说"这不是我要的" → 立即确认需求

3. **记录调整**
   - 将用户的反馈记录到偏好文件
   - 更新对应的偏好设置
   - 为下次对话做准备

### 对话结束时

1. **总结用户反馈**
   - 用户是否纠正了你的输出风格
   - 用户是否表达了新的偏好
   - 用户是否对某些方面不满意

2. **更新偏好记录**
   - 将新的反馈记录到偏好文件
   - 更新对应的偏好设置
   - 确保记录的准确性

3. **准备下次对话**
   - 确保偏好记录是最新的
   - 确保知道如何应用这些偏好
   - 确保能够主动询问用户

## 特殊情况处理

### 情况1：用户偏好冲突

**场景**：
- 用户之前说"我喜欢详细版"
- 但这次说"给我简洁版"

**处理方式**：
```
我注意到你之前喜欢详细版，但这次需要简洁版。
我会按照这次的要求输出。
需要我更新你的偏好记录吗？
```

### 情况2：用户偏好变化

**场景**：
- 用户之前是"入门级"技术水平
- 但现在已经是"中级"了

**处理方式**：
```
我注意到你之前对[领域]不太熟悉，但现在看起来已经很了解了。
需要我调整技术深度吗？
```

### 情况3：不确定用户偏好

**场景**：
- 用户没有明确表达偏好
- 用户的反馈可能是误解
- 用户的要求可能是临时的

**处理方式**：
```
我不确定你的偏好，想确认一下：
你希望我[选项A]还是[选项B]？
```

## 记录管理

### 记录存储

- 每个用户一个独立的偏好文件
- 文件名格式：`user-preferences-[user-id].md`
- 存储位置：项目目录或用户目录

### 记录更新

- 每次对话后检查是否需要更新
- 只记录明确的偏好，不猜测
- 保持记录的简洁和准确

### 记录验证

- 定期检查记录是否仍然准确
- 如果发现记录不准确，及时修正
- 如果不确定，询问用户确认

---

**记住**：偏好记忆是AI成为真正助手的关键。用户纠正你的每一次，都是在教你如何更好地为他工作。你的目标是记住这些纠正，并在下次对话中应用，让用户感觉"这个AI懂我"。
