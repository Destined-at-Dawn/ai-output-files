# SOP-04 学习复盘

> **触发条件**：用户说"复盘/总结/回顾/哪里薄弱/学得怎么样"
> **认知科学支撑**：WYSIATI（010-思考快与慢）、元认知监控（023-认知觉醒）、执行意图（051-WOOP）

---

## 流程

### Step 0: 进水管

1. Read 课程 `project.md` → 查整体进度
2. Read `memory/MEMORY.md` → 查历史教训
3. Read 最近 3 天的 `memory/{日期}.md` → 查近期学习记录

### Step 1: 数据采集

收集以下数据（问用户或从 memory 读取）：

| 维度 | 数据 |
|------|------|
| 本周学了什么 | 从 memory/日期 文件汇总 |
| 做了多少题 | 练习题/作业题数量 |
| 正确率 | 做对/做错比例 |
| 薄弱章节 | project.md 中标记的 ⚠️/❌ |
| 学习时长 | 用户自报或记录 |
| 情绪状态 | "感觉如何"/"有没有焦虑" |

### Step 2: 诊断分析

**WYSIATI 对抗**（你只看到你想看到的）：
- 不要只看"学了什么"，也要看"没学什么"
- 不要只看"做对的题"，也要分析"做错的题"

**输出诊断报告：**

```
📊 本周学习诊断

【进度】
- 计划完成：{X}/{Y} 章节
- 实际完成：{X}/{Y} 章节
- 进度偏差：超前/落后 {Z} 章

【质量】
- 练习正确率：{X}%
- 薄弱章节：{列表}
- 需要紧急补的：{列表}

【效率】
- 平均每日学习时长：{X}h
- 有效学习比例：{高/中/低}
- 最大时间黑洞：{哪个章节花了最多时间}

【建议】
1. {最重要的改进点}
2. {第二重要的}
3. {第三重要的}
```

### Step 3: 制定下周计划

基于诊断，制定下周学习计划：
1. 补薄弱章节（优先级最高）
2. 推进新章节（按 project.md 顺序）
3. 间隔重复（复习标记了需要复习的内容）

写入课程 `project.md` 的下周计划段落。

### Step 4: 记录复盘结果

1. 写入 `memory/{YYYY-MM-DD}.md`：复盘结论 + 下周计划
2. 更新 `memory/MEMORY.md`：如有新的跨课程经验
3. 如有有效学习方法 → `自我进化/做得好的鼓励/`
4. 如有学习踩坑 → `自我进化/做得差的避免/`

## 复盘频率

| 频率 | 内容 | 深度 |
|------|------|------|
| 每日 | 学了什么、掌握程度、明日计划 | 简要（5分钟） |
| 每周 | 本周诊断 + 下周计划 | 详细（15分钟） |
| 期中 | 全面诊断 + 策略调整 | 深度（30分钟） |
| 期末 | 全学期总结 + 下学期改进 | 深度（30分钟） |

---

## 反模式

| 错误做法 | 正确做法 |
|---------|---------|
| 只总结"做了什么" | 也要分析"没做什么"和"为什么没做" |
| 复盘完不改计划 | 复盘必须产出下周计划变更 |
| 每天花30分钟复盘 | 日复盘5分钟，周复盘15分钟 |
| 只看正确率数字 | 分析错题的模式（同类错误还是随机错误） |
| 复盘时自我批评过度 | 基于数据客观分析，不带情绪判断 |
