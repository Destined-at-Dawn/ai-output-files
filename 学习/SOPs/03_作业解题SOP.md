# SOP-03 作业解题

> **触发条件**：用户说"作业/题目/解题/怎么做/这个题/帮我做"
> **认知科学支撑**：合意困难（009-认知天性 §概念1）、脚手架理论（025-认知负荷理论）

---

## 流程

### Step 0: 进水管

1. Read 课程 `project.md` → 确认用户当前学到哪里
2. 了解题目所属章节 → 判断涉及哪些知识点

### Step 1: 理解题目

1. 让用户描述题目（拍照/文字/口述）
2. 确认题目要求：求什么？已知什么？限制条件？
3. 判断题目难度和所属知识点

### Step 2: 引导思路（关键步骤——不直接给答案）

**脚手架式引导：**

```
Level 1 提示：这道题考的是哪个知识点？
  ↓ 用户回答
Level 2 提示：这个知识点的核心公式/方法是什么？
  ↓ 用户回答（或答不上来）
Level 3 提示：第一步应该怎么走？
  ↓ 用户尝试
Level 4 提示：第二步呢？
  ↓ ...
```

> **⚠️ 铁律**：只有当用户连续 3 次 Level 提示后仍无法推进时，才给完整解答。
> **例外**：用户明确说"直接给答案"/"赶时间"/"快" → 给完整解答 + 解题思路标注

### Step 3: 完整解答（需要时）

1. 分步骤写出完整解答
2. 每步标注：
   - **为什么这样做**（思路说明）
   - **用了什么知识点**（公式/定理名）
   - **易错点**（常见错误）

### Step 4: 变式练习

1. 基于同一知识点，出 1-2 道变式题（改变条件/问法）
2. 让用户独立完成
3. 验证是否真正掌握（而不只是记住了这道题的答案）

### Step 5: 记录解题模式

将本题的解题模式写入 `memory/{YYYY-MM-DD}.md`：

```
**题目类型**：{类型，如"求极限/0/0型"}
**解题模式**：{核心步骤}
**易错点**：{常见错误}
**关联知识点**：{其他相关章节}
```

---

## 解题模式库（各课程通用）

| 题型 | 解题模式 | 适用课程 |
|------|---------|---------|
| 证明题 | 反证法/数学归纳法/构造法 | 高数、物理 |
| 计算题 | 定义→公式→代入→化简 | 全部 |
| 应用题 | 建模→方程→求解→检验 | 高数、物理、电路 |
| 分析题 | 列条件→逐条分析→综合结论 | 全部 |

---

## 反模式

| 错误做法 | 正确做法 |
|---------|---------|
| 直接给完整答案 | 先引导思路，给提示 |
| 只做这道题就完 | 加变式练习验证掌握 |
| 做完不总结解题模式 | 记录解题模式到 memory |
| 用户说不会就直接答 | 先降低难度，给更基础的提示 |
| 一道题讲30分钟 | 超过3轮提示就给答案，标注"需重学该知识点" |
