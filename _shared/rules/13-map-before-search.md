# R13：地图优先于盲搜规则

> 版本：v1.0 | 创建：2026-05-25
> 维护方：mutual 工作区
> 适用：所有工作区
> 来源：CodeGraph 项目分析（GitHub 7天1.4万Star，上下文质量优化工具）

---

## 一、核心原则

**模型能力不是瓶颈，上下文质量才是。**

AI Agent 处理任务时，token 浪费的主因不是模型不够强，而是没有"地图"——每次都从盲目探索开始。给模型更好的输入（结构化索引），比换更强的模型更快、更便宜。

---

## 二、优先级链（强制）

任何需要查找信息的任务，必须按以下优先级执行，**禁止跳过前面的步骤直接用后面的**：

```
1. MCP工具查询（project-tools、skill-handler 等已有工具）
2. 注册表/总索引/SOP总索引（知识中枢/00-注册表/、SOPs/SOP总索引.md）
3. 记忆索引（memory/MEMORY.md、memory/long-term.md）
4. 精确路径 Read（已知路径时直接读取）
5. grep/Explore 子Agent（以上都找不到时，才用模糊搜索）
```

---

## 三、触发信号

出现以下情况时，说明你正在"盲目探索"，必须停下来查地图：

- [ ] 连续 grep 3 次以上还没找到目标文件
- [ ] Read 了 5 个以上文件还没定位到需要的信息
- [ ] 派出了 Explore 子Agent 但结果不理想
- [ ] 不确定某个文件/目录是否存在就开始搜索
- [ ] 不确定某条规则/记忆是否存在就开始翻文件

**正确做法**：停下来，先查注册表/索引/SOP总索引，用一次结构化查询替代十次盲目探索。

---

## 四、"工具装了但指令没到位 = 没装"铁律

| 检查项 | 说明 |
|--------|------|
| **MCP 工具** | 每个可用的 MCP 工具必须在 CLAUDE.md 中有明确的使用指令 |
| **索引系统** | 每个索引文件（注册表/SOP总索引/MEMORY.md）必须在 CLAUDE.md 路由表中列出 |
| **新工具上线** | 安装新 MCP 工具后，必须同步更新 CLAUDE.md 的路由表/快速参考 |
| **验证方式** | 执行任务时观察 AI 是否走了索引查询而非直接 grep |

## 六、维护

- **新增 MCP 工具时**：同步更新 CLAUDE.md 路由表
- **发现 AI 绕过索引盲目探索时**：强化对应 CLAUDE.md 的优先级指令
- **每月检查**：各工作区 CLAUDE.md 是否列出了所有可用的索引和 MCP 工具
