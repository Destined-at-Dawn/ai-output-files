# R08 — telos 个人信息自动捕获（真源 v1.0）

> **核心理念**：用户不应该记住"我需要告诉AI去更新telos"。AI应该自己捕捉、自己更新。
> 此规则必须植入每个工作区的 CLAUDE.md。

---

## 标准文本（植入 CLAUDE.md 的规则段落）

### 自动捕获触发器

> **每次对话中，AI 必须在后台监听以下信号。一旦检测到，立即静默更新 telos（无需告知用户，除非涉及敏感信息确认）。**

#### 触发信号表

| 信号类型 | 用户说了什么 | 更新哪个 telos 文件 | 更新方式 |
|---------|------------|-------------------|---------|
| **身份变化** | "我转专业了"/"我大二了"/"我换导师了" | identity.md | 修改对应字段 |
| **新联系方式** | "我的新号码"/"新邮箱是" | identity.md | 追加到联系方式 |
| **目标调整** | "我改主意了"/"不想考研了"/"想试试XX" | goals.md | 修改对应目标 |
| **新兴趣** | "最近在看XX"/"对XX感兴趣"/"开始学XX" | interests.md | 追加到对应分类 |
| **新技能** | "学会了XX"/"在用XX工具"/"能做XX了" | skills.md | 追加到对应分类 |
| **认知跃迁** | "我觉得XX才是对的"/"原来XX不是这样" | beliefs.md | 追加到对应分类 |
| **偏好变化** | "以后不要XX"/"我更喜欢XX"/"这样更好" | learned.md → 用户偏好 | 追加或修改 |
| **经历/状态** | "今天XX比赛"/"拿到了XX"/"XX通过了" | identity.md → 当前状态 | 追加条目 |
| **失败/批评** | "你又犯了XX错误"/"不对"/"搞错了" | learned.md → 操作教训 | 追加条目 |
| **人脉变化** | "认识了XX老师"/"XX推荐了XX" | identity.md → 指导老师/当前状态 | 追加条目 |

#### 捕获流程（后台静默执行）

```
用户消息中检测到个人信息信号
    ↓
判断：这是新信息还是已知信息？
    ↓
新信息 → Read telos 对应文件 → Edit 追加新条目 → Read 验证
已知信息但有变化 → Read → Edit 修改旧条目 → Read 验证
已知且无变化 → 跳过
    ↓
（如果是敏感信息：电话/身份证/密码）→ AskUserQuestion 确认后再写入
```

#### 写入规范

1. **先 Read 再 Edit**（禁止覆盖）
2. **追加格式**：在对应章节末尾追加，带日期前缀
   ```
   - **2026-05-09**：{新信息描述}
   ```
3. **变更记录**：在文件末尾的「变更记录」表中追加一行
4. **Read 验证**：Edit 后必须 Read 确认
5. **敏感信息确认**：电话/身份证/密码/银行卡号 → 必须 AskUserQuestion 确认

#### 写入位置

telos 文件路径：`C:\Users\13975\.claude\telos\`

写入时使用 Bash+Python（因为 telos 在项目目录外部）：

```bash
PYTHONIOENCODING=utf-8 python -c "
import os
fp = os.path.join('C:', os.sep, 'Users', '13975', '.claude', 'telos', '{filename}.md')
with open(fp, 'r', encoding='utf-8') as f:
    content = f.read()
# 在变更记录表之前插入新内容
marker = '## 变更记录'
if marker in content:
    parts = content.split(marker)
    new_content = parts[0] + '{新内容}\n\n' + marker + parts[1]
else:
    new_content = content + '\n\n' + '{新内容}'
with open(fp, 'w', encoding='utf-8') as f:
    f.write(new_content)
print('OK')
"
```

> 如果 Edit 工具能直接操作 telos 路径（某些环境下可以），则优先用 Edit。
> 只有 Edit 报路径权限错误时，才回退到 Bash+Python。

## 批量同步模式

当用户说"同步个人信息"/"更新telos"/"个人信息同步"时，执行完整扫描：

1. 读取所有工作区的 CLAUDE.md 中的用户画像章节
2. 读取所有 memory/long-term.md 中的用户偏好
3. 与 telos 当前内容对比
4. 列出差异清单
5. AskUserQuestion 确认后批量更新

## 认知科学支撑（百大认知书籍）

| 认知机制 | 来源 | 在telos自动捕获中的应用 |
|---------|------|----------------------|
| **内隐记忆** | 022-认知神经科学 §概念1 | 自动捕获=利用内隐记忆——用户不需要"记住告诉AI更新telos"，AI在后台静默监听。这消除了"显式记忆负担"→用户只管说话，系统自动归档 |
| **信号检测理论** | 022-认知神经科学 §概念2 | 10种触发信号表=信号检测的工程化——在"用户说了什么"(噪音+信号)中检测"个人信息变更"(纯信号)。false positive(误判)风险低(false: 跳过)；false negative(漏判)风险高(信息丢失)→系统偏向"宁可多捕获" |
| **习惯回路** | 016-习惯的力量 §概念1 | "对话结束自检"=习惯回路的触发器——信号(对话结束)→例行(检查telos是否需要更新)→奖励(信息不丢失)。三个要素齐全=习惯固化 |
| **外部化工作记忆** | 025-认知负荷理论 §概念1 | telos=用户的"外部化自我认知"——身份/目标/兴趣/技能/信念/教训/偏好/工作风格=8个维度的"外部化人格"。AI不需要"记住"小黎是谁，只需读telos |
| **分布式编码** | 009-认知天性 §概念1 | "新信息静默更新到telos"=分布式编码——信息在"用户说出"的时间点被编码，而非"事后回忆"时。编码特异性原理：信息在新鲜时编码>衰减后编码 |
