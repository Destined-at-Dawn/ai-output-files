# R04 — 记忆文件操作铁律（真源 v1.0）

> **历史教训**：多个工作区出现过 AI 用覆盖方式重写 memory 文件导致早期记录全部丢失。所有工作区统一规则。
> Source: 交叉学习自竞赛区（Verilog FPGA + 皮影戏 CLAUDE.md）

---

## 标准文本

| # | 规则 | 说明 |
|---|------|------|
| **4.1** | **先 Read 再 Write** | 更新任何 memory 文件前，必须先 Read 读取当前全部内容，在末尾追加新内容。**禁止直接覆盖整个文件** |
| **4.2** | **追加格式** | 新内容以 `## YYYY-MM-DD HH:MM` 的日期时间分隔开头，加在文件末尾 |
| **4.3** | **Write 后必须 Read 验证** | Write 之后必须在同一轮用 Read 验证追加的行确实存在于末尾 |
| **4.4** | **禁止空文件/占位符** | 写入的内容必须有实际信息（做了什么、发现了什么、验证结果），不能写"今日无更新" |

## 认知科学支撑（百大认知书籍）

| 认知机制 | 来源 | 在记忆操作中的应用 |
|---------|------|-------------------|
| **记忆巩固** | 009-认知天性 §概念1 | 追加式写入=保留记忆的"时间线"——每个时间点的记忆都是独立的"编码事件"。覆盖式写入=用一个"编码事件"替代所有历史→旧记忆永久丢失 |
| **编码特异性** | 009-认知天性 §概念1 | 记忆在哪个时间/情境下被记录，就会在相似情境下最容易被检索。追加时保留日期标签=保留"编码情境"→日后检索更精确 |
| **WYSIATI** | 010-思考快与慢 §概念3 | "先Read再Write"的目的是打破"我大概知道文件内容"的WYSIATI陷阱。实际Read→发现与"记忆中的"不同→避免覆盖 |
| **反馈回路** | 016-习惯的力量 §概念1 | "Write后Read验证"=即时反馈回路→确认操作成功→建立信任→习惯固化。不验证→不确定→焦虑→下次操作更谨慎→效率下降 |
