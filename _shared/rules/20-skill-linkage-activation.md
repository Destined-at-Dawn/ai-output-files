# R20 -- 技能联动激活协议（v1.0）

> **核心原则：联动优先 > 单打独斗。** 一个任务能触发 2 个 skill 就不只调 1 个。用户说的是需求，路由是 AI 的责任。

---

## 一、三条铁律

| 铁律 | 内容 | 违反后果 |
|------|------|---------|
| **联动优先** | 一个任务能调 2 个 skill 就不只调 1 个（除非极简单 <3 步） | 产出浅薄，用户不满 |
| **探索性强制** | 用户方向不明确时禁止原生裸跑，必须走 li-analyze/li-research/li-devil | 回答泛泛，无深度 |
| **不满即学习** | 用户对结果不满时，当轮记录+调整联动链，下次同类自动应用新链 | 同类问题重复踩坑 |

---

## 二、三层路由优先级链

```
收到用户消息
  |
  v
Layer 0: 技能路由表命中？（skill-routing-table.json 的 triggers/linkage）
  |-- 命中 -> 主 skill + linkage 联动 skill 一起调度
  |
  v（未命中）
Layer 1: SOP 索引命中？（li-intent Phase 0 扫描工作区 SOP）
  |-- 命中 -> 按 SOP 流程调度
  |
  v（未命中）
Layer 2: 触发词语义匹配（li-local-search 语义搜索）
  |-- 命中 -> 推荐 skill + 联动
  |
  v（全部未命中）
Layer 3: 任务复杂度判断
  |-- 简单（<3步，明确答案）-> 原生能力（唯一允许不用 skill 的情况）
  +-- 探索性/复杂 -> 强制 li-analyze 或 li-research（禁止原生裸跑）
```

---

## 三、预定义联动链

| 任务类型 | 主 Skill | 联动 Skill | 联动逻辑 |
|----------|---------|-----------|----------|
| 内容创作 | li-xhs / blog-post-writer | voice-dna + li-devil | 写 -> 文风 -> 质检 |
| 深度研究 | li-research | li-analyze + li-devil | 研究 -> 结构化 -> 质疑 |
| 商业分析 | li-dbs | li-analyze + li-devil | 战略 -> 行业 -> 挑战 |
| 学习新概念 | li-study | li-analyze + li-memory | 学 -> 深化 -> 存 |
| 任务规划 | li-plan | li-devil + li-manage | 规划 -> 质疑 -> 归档 |
| 系统诊断 | li-diagnose | li-improve + li-infra | 诊断 -> 记录 -> 修复 |
| 探索性问题 | li-analyze | li-research + li-devil | 分析 -> 补盲区 -> 反驳 |
| 数字资产 | li-asset | li-xhs + li-devil | 采集 -> 分发 -> 质检 |

联动链也写在路由表的 `linkage` 字段中，供程序化调度。

---

## 四、探索性任务判定

| 信号 | 判定 |
|------|------|
| 用户说"帮我想想""看看怎么搞""研究一下" | 探索性 |
| 用户没给具体方向，只给了主题 | 探索性 |
| 任务没有明确的"正确答案" | 探索性 |
| 涉及决策（A 还是 B？做不做？） | 探索性 |

探索性任务必须至少调用：li-analyze / li-research / li-devil / li-dbs / li-diagnose 之一。

---

## 五、自我学习闭环（5 步）

```
Step 1: 检测不满信号（"不对""太浅""再深入""不是我要的"）
Step 2: 分类（路由错误 / 联动不足 / 深度不够 / 风格不对）
Step 3: 确定修正方案
Step 4: 当轮应用修正（立即重做）
Step 5: 写入学习记录 + 更新路由表 linkage/confidence
```

| 事件 | 动作 |
|------|------|
| 同一 skill 连续 3 次不满 | confidence 降 0.15 |
| 用户说"再深入/再想想" | 下次同类强制加 li-analyze + li-devil |
| 同类任务联动成功 3 次 | 联动链固化为预定义联动 |
| 用户主动说"用 XX 技能" | 立即添加新路由（路由失败） |

---

## 六、动态联动（预定义未覆盖时）

AI 自主判断三类联动至少选 1：
1. **增强型**：谁能让这个产出更好？
2. **质疑型**：谁能挑出这个产出的问题？（首选 li-devil）
3. **归档型**：谁能帮这个产出沉淀？（首选 li-memory/li-manage）
