> AI生成的简历有一个通病：正确但空洞。HR一眼就能看出来。

## AI味的典型特征

### 黑名单词汇（出现=立即替换）

**空泛形容词：**
- 积极参与 → ❌ 删掉，写具体做了什么
- 深入了解 → ❌ 删掉，写你了解到了什么
- 全面负责 → ❌ 改成"负责XX模块的YY工作"
- 高效完成 → ❌ 改成具体时间/效率指标
- 有效提升 → ❌ 改成"从X提升到Y"

**AI套话：**
- 综上所述 → 简历里根本不需要这种过渡句

- 不可或缺 → ❌ 删掉
- 致力于 → ❌ 改成"做/负责/设计"

## 替换原则

**用一件具体的事代替一个形容词。**

```
❌ "具备良好的沟通能力和团队协作精神"
✅ "在5人团队中担任系统集成负责人，协调硬件/算法/测试3个环节的联调"

❌ "学习能力强，能够快速掌握新技术"
✅ "大一自学FPGA，3个月完成从UART到AES-128的完整模块实现"

❌ "工作认真负责，注重细节"
✅ "自写Testbench覆盖200+测试向量，包括正常/边界/异常三类场景"

❌ "对嵌入式开发有浓厚兴趣"
✅ "日常使用Claude Code做项目Debug，维护GitHub上的FPGA学习笔记"
```

## AI生成内容的检查流程

### Step 1: 标记

把AI生成的初稿打印出来（或在编辑器中），逐句标记：
- 🟢 具体、可验证 → 保留
- 🟡 有点空但可改 → 需要替换
- 🔴 纯废话/AI味 → 删除

### Step 2: 替换

每个🟡和🔴的地方，问自己：
- "这句话去掉后，信息量有没有减少？" → 如果没有，直接删
- "能不能用一个具体的数字/事实替换？" → 能就换

### Step 3: 口语化检查

把简历读出来（真的读出声）。如果某句话你正常说话时不会这么说，它就有AI味。

```
读出来别扭："致力于推动嵌入式技术的发展和应用"
读出来自然："做FPGA开发，最近在研究AES加密"
```

### Step 4: 面试验证

对每段经历问自己："如果面试官追问我这句话，我能答上来吗？"

- ✅ "你用什么工具做仿真？" → "Icarus Verilog + GTKWave" → 能答
- ❌ "你怎么有效提升团队协作效率？" → ……答不上来 → 删掉这句话

## Before/After 完整对比

### Before（AI生成版）

```
项目经历：
本人积极参与全国大学生集成电路创新创业大赛，全面负责FPGA模块设计与验证工作。
深入研究了UART、AES-128、CRC-16等核心模块的设计与实现，熟练运用Vivado、
ModelSim等EDA工具进行仿真验证。通过不断优化设计方案，有效提升了系统性能，
最终成功完成了全部模块的功能验证，取得了优异的成绩。
```

### After（人工修改版）

```
项目经历：

全国大学生集成电路创新创业大赛（数字电路模块设计赛道），负责人

- 独立手写UART收发器RTL代码（115200bps），未调用IP核
- 实现AES-128加密模块，延迟从30+周期优化至≤12时钟周期（S盒查找表+轮密钥预计算）
- 设计CRC-16-CCITT校验，标准测试向量100%通过
- 使用Icarus Verilog + GTKWave仿真，自写Testbench覆盖200+测试场景
```

**差距：** Before有100个字但信息量约等于零。After有80个字但每句都是可验证的事实。
