# 03 | 证明自己：项目 > 学校 > GPA

> "面试官不问你学校几本，问你代码能不能跑。"

---

## 残酷的现实

先把丑话说在前面：**学校牌子确实重要。**

很多大厂的简历筛选系统会自动过滤非985/211。这不是你的错，但这是你要面对的现实。

但是——**这不代表你没有机会。**

关键在于：当学历不能替你说话时，你得让别的东西替你说话。

**什么东西最能替你说话？能跑的代码、能看的波形、能验证的结果。**

---

## 什么才算"证明"

面试官心里有一杆秤。按说服力排序：

```
能跑的代码 demo > 波形截图 > 技术报告 > 竞赛获奖 > 项目描述 > GPA > 学校名字
```

**注意这个排序。** 大多数学生把精力花在最后两项（刷GPA、纠结学校），但面试官最看重的是前两项。

### 我的真实经历

面试FPGA岗位时，面试官问："你做过什么项目？"

**差的回答：** "我在学校实验室做过FPGA相关的项目，使用Vivado和ModelSim。"（空泛，谁都能这么说）

**我的回答：** "我手写了UART收发器的RTL代码，不是调用IP核。波特率115200bps，自己写的Testbench验证，波形截图在简历里。还做了AES-128加密模块，把延迟优化到12个时钟周期以内。"

然后我打开了电脑，给他看了波形截图和代码。

**面试官的反应：** "你这是自己写的？不是调IP？" — 就这一句话，后续的面试流程顺畅了很多。

## 怎么从零开始建立"证明体系"

### 场景1：你什么项目都没有

**路径：竞赛 → 课程大作业 → 个人项目**

1. **找一个竞赛参加**
   - 集创赛、电子设计大赛、数学建模、ROS竞赛……
   - 不需要拿国奖，参加了就有东西写
   - 组队时主动承担技术核心角色

2. **把课程大作业做成项目**
   - 不是"完成作业"，而是"把作业做成可以展示的作品"
   - 加一个README，写清楚你做了什么
   - 把代码传到GitHub

3. **做个人小项目**
   - FPGA方向：实现一个UART/I2C/SPI控制器
   - 软件方向：写一个解决实际问题的小工具
   - 硬件方向：画一块PCB，打板验证

### 场景2：你有项目但不会展示

**展示三件套：代码 + 波形/截图 + 技术文档**

| 展示物 | 怎么准备 | 放在哪 |
|--------|---------|--------|
| 代码 | 传GitHub，写README | 简历上放GitHub链接 |
| 波形/截图 | 用GTKWave/示波器截图 | 简历里直接放图，或面试时展示 |
| 技术文档 | 写清楚：背景→方案→实现→结果 | 作为附件或面试时讲解 |

### 场景3：你有项目但不确定够不够好

**检验标准：你能不能在面试中回答以下问题？**

1. 这个项目你具体做了什么？（不是"参与了"，是"我做了XXX"）
2. 最难的技术问题是什么？你怎么解决的？
3. 为什么选这个方案？考虑过其他方案吗？
4. 如果重新做一遍，你会怎么改进？

如果4个问题你都能流畅回答，这个项目就够了。
如果有回答不上来的，回去补课，把细节搞清楚。

## "从0到1"比"锦上添花"更有价值

面试官看过的简历里，"熟练掌握Verilog"出现过1000次。但"独立手写UART收发器RTL，未调用任何IP核"只出现过几次。

**从0到1做一件事，比在已有框架上改参数，更能证明你的能力。**

举个例子：

| 一般水平 | 有区分度 |
|---------|---------|
| 使用Vivado调用AES IP核 | 手写AES-128 RTL实现，延迟优化到12周期 |
| 跟着教程跑通ROS导航 | 从0搭建导航框架，自己调PID参数 |
| 完成课程要求的实验 | 在实验基础上加了额外功能并写技术报告 |
| 学过C语言 | 用C写了一个嵌入式项目并实际烧录运行 |

## 学术背书：用好你的老师

很多人忽略了一个强力武器：**你的指导老师。**

如果你的老师有相关研究方向，这就是天然的学术背书。面试时说"我的指导老师研究方向是FPGA+AI图像处理，我的小论文方向是基于FPGA的无人机巡检系统"——这比"我自学了FPGA"有分量得多。

**怎么拿到学术背书：**

1. 主动找老师聊，了解研究方向
2. 争取参与实验室项目（哪怕只是打杂）
3. 申请大学生创新创业项目（大创）
4. 如果老师愿意，让他在推荐信上签字

## AI工具：你的能力放大器

用AI工具不是作弊，而是**和用计算器做数学题一样正常**。

关键在于：AI是放大器，不是替代品。你得有基本功，AI才能帮你放大。

**我实际用过的AI工具和场景：**

| 工具 | 场景 | 效果 |
|------|------|------|
| Claude Code | 逐文件比对代码与比赛规则 | 省了3天手动检查时间 |
| Gemini Pro | 快速补齐PID调参逻辑 | 2小时搞定原本需要1天的调试 |
| NotebookLM | 拆解导航框架原理 | 把论文变成可操作的步骤 |

**面试时怎么说AI工具的使用：**

✅ "我用Claude Code逐文件比对代码和比赛规范，发现了3个不一致的地方。"
✅ "PID参数调试时，我用Gemini快速验证了几组参数的理论效果，缩小了实机调试的搜索空间。"

❌ "我用AI写了整个项目。"（这等于说你什么都没做）
❌ 不提AI（不诚实，而且面试官可能会问）

## 认知科学支撑：证明自己的认知原理（百大认知书籍）

| 认知机制 | 来源 | 在证明自己中的应用 |
|---------|------|--------------------------|
| **信号理论** | 010-思考快与慢 §概念1 | 证明自己=发送可信信号——"项目经历""开源贡献""技术博客"=信号→信号越难伪造→越可信→核心策略=创造"高成本高可信"的信号 |
| **锚定效应** | 010-思考快与慢 §概念4 | 量化成果=锚定——"提升了30%性能"比"提升了性能"更有说服力→数字=锚点→面试官的评价被锚定在具体数值上→感知能力↑ |
| **叙事身份** | 042-情绪大脑 §概念1 | 证明自己=构建叙事身份——不是"罗列事实"而是"讲述一个能力持续提升的故事"→故事=身份→面试官看到的是"一个有成长轨迹的人"而非"一份简历" |
| **社会证明** | 010-思考快与慢 §概念1 | "导师推荐""团队认可""开源社区star"=社会证明——人类倾向于"看别人怎么评价"→社会证明=第三方可信信号→比自我陈述更有说服力 |
| **自我效能感** | 023-认知觉醒 §概念1 | 证明给别人的前提是"先证明给自己"——自我效能感="我相信我能做"→面试中传递的信心→面试官感知到→评分↑→正反馈循环 |
