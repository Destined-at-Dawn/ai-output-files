# 04 | 面试攻略：面试是双向的信息交换

> 面试官在挑你，你也在挑他们。

---

## 面试的本质

很多学生把面试当成考试——对面坐着考官，你负责答题，答对了就通过。

**这是错的。**

面试是一场**双向的信息交换**。面试官在评估你能不能干活，你也在评估这个团队值不值得去。

带着这个心态去面试，你的表现会完全不同：
- 你不会紧张到大脑空白（因为你也在"面试"他们）
- 你会主动提问（因为你需要信息来做判断）
- 你不会为了拿到offer而过度包装（因为不合适的工作拿了也痛苦）

---

## 面试前：30分钟准备

### 5分钟公司速查

| 查什么 | 怎么查 | 为什么 |
|--------|--------|--------|
| 公司做什么的 | 官网/百度百科 | 回答"你对我们公司了解多少" |
| 主要产品/技术栈 | 官网产品页 | 让你知道面试会聊什么 |
| 最近新闻 | 百度搜公司名 | 面试时提一句显得你做了功课 |
| 面试官背景 | LinkedIn/脉脉 | 了解他的技术方向 |

### 自我介绍准备

**30秒版（电话/初筛用）：**

```
"我叫XXX，上海电力大学集成电路专业大一，GPA 3.9。
我主要做FPGA开发，手写过UART、CRC16、AES-128的RTL代码，
也在ROS竞赛中负责系统集成。对嵌入式/硬件方向很有兴趣。"
```

**2分钟版（正式面试用）：**

```
"我叫XXX，上海电力大学集成电路专业大一学生。

在技术方面，我有两个主要项目经历：
一是集创赛的FPGA项目，我作为负责人，手写了11个Verilog模块，
包括UART收发器、CRC16校验、AES-128加密，都是从0开始写的RTL，
没有调用IP核。其中AES-128的延迟优化到了12个时钟周期以内。

二是ROS机器人竞赛，我负责系统集成，从0搭建了基于move_base的
导航框架，调了分段PID控制实现动态避障。

我平时会用Claude Code和Gemini辅助开发，但核心逻辑都是自己写的。

我对贵公司的[具体方向]很感兴趣，因为[具体原因]。"
```

**关键原则：** 自我介绍不是背简历，而是用30秒/2分钟告诉对方"我最值得聊的点是什么"。

## 面试中：技术面策略

### 遇到不会的问题

**绝对不要编。** 面试官见过太多编答案的人了，一眼就能看出来。

**正确回答模板：**

```
"这个我还没有深入研究过。但根据我的理解，[说出你知道的部分]。
如果要深入学习的话，我会先[说出你的学习思路]。"
```

这个回答告诉面试官：
1. 你诚实（不装懂）
2. 你有基础（知道相关的东西）
3. 你会学习（有学习方法）

**比编一个错误答案强100倍。**

### 技术问题回答框架

```
面试官：你做过XXX吗？

回答结构：
1. "做过/了解过"（一句话确认）
2. "具体是这样的……"（技术细节，越具体越好）
3. "当时遇到了XX问题，我是这样解决的……"（展示解决问题的能力）
4. "如果现在再做一遍，我会XX改进"（展示反思能力）
```

### 行为面试问题（STAR法）

**但不要像背课文一样用STAR。** 自然一点，像讲故事：

```
面试官：说说你遇到最大的技术难题？

"在做AES-128加密模块的时候（Situation），
我需要把延迟控制在12个时钟周期以内（Task），
一开始的实现用了30多个周期，差了一倍多。
我分析了关键路径，发现是S盒查找导致的延迟，
后来把组合逻辑改成了流水线结构（Action），
最终延迟降到了11个周期，满足了要求（Result）。"
```

## 面试中：你该问的问题

面试最后，面试官一定会问"你有什么想问的吗？"

**不要说"没有了"。** 这是浪费了一个获取信息的机会。

### 好问题（展示你思考过）

- "这个岗位日常的工作内容是什么？会参与哪个阶段的项目？"
- "团队的技术栈是什么？新人入职后会有mentor带吗？"
- "实习生有机会参与核心项目吗？还是主要做支持性工作？"
- "您觉得这个岗位最需要的能力是什么？"（收集市场信号）
- "团队目前在做什么方向的产品？"（了解业务）

### 不好的问题

- ❌ "加班多吗？"（太直接，换个说法）
- ❌ "工资能再高点吗？"（初面不要谈钱）
- ❌ "你们公司是做什么的？"（说明你没做功课）

## 面试中的红旗

面试不只是他们在挑你，你也在观察他们。以下信号说明这个公司/团队可能有问题：

| 红旗 | 说明 |
|------|------|
| 面试官不了解岗位细节 | 说明团队管理混乱 |
| 全程不让你提问 | 不尊重候选人 |
| 问你"能接受加班吗？"时态度很强硬 | 加班文化严重 |
| PUA式提问："你觉得你凭什么能来我们公司？" | 管理风格有问题 |
| 急着让你当场签offer | 可能留不住人 |
| 说不清楚你能学到什么 | 他们自己都不知道怎么培养新人 |

## 我的教训：微信沟通纪律

这部分是我在求职过程中踩过的坑，必须记录下来：

### 错误示范

给一位工程师连发了三条几乎一模一样的自我介绍消息。对方回复了"好的"+"嗯"之后，我又发了一条确认消息。

**这给对方的印象：** 这个人不看回复，只顾自己说。

### 正确做法

1. **发了一条消息后等回复。** 不要追发。
2. **对方回复了"好的"/"嗯"= 已收到。** 不需要再确认。
3. **下次发消息时带新内容。** 比如代码链接、项目文档，不要重复自我介绍。
4. **备注名要专业。** "朱工-皮赛FPGA"而不是系统默认的昵称。

| 认知机制 | 来源 | 在面试攻略中的应用 |
|---------|------|--------------------------|
| **首因效应** | 010-思考快与慢 §concept1 | 面试前30秒=首因效应——"第一印象"一旦形成很难改变→攻略中的"开场策略"=最大化首因效应的正面影响→起跑线领先 |
| **STAR法则** | 010-思考快与慢 §概念1 | 每个回答按STAR编码=因果推理的结构化——Situation(背景)→Task(任务)→Action(行动)→Result(结果)→面试官读到的是"完整因果链"→可信度↑ |
| **压力脱敏** | 042-情绪大脑 §概念1 | 模拟面试=压力脱敏——反复在"面试情境"中暴露→焦虑阈值↑→正式面试时焦虑↓→表现=真实水平而非"紧张后打折的水平" |
| **元认知监控** | 023-认知觉醒 §概念3 | 面试中的"自检"=元认知监控——"我是否回答了问题？""是否说得太多？"→实时监控→实时调整→比"说完才发现跑题"好10倍 |
| **认知灵活性** | 023-认知觉醒 §概念1 | 面试中"换角度回答"=认知灵活性——"面试官追问时不是重复刚才的话而是从新角度回答"→灵活性=适应性→高适应性=高评分 |
