# 百大认知书籍 x Digital Oracle 框架-数据映射表

**版本：v1.0 | 创建日期：2026-05-03**
**目的：** 将46本百大认知书籍的永恒决策框架，与Digital Oracle的14个实时金融数据源进行精确映射，实现"框架×数据"的超级决策引擎。

---

## 设计原理

```
百大认知书籍 = 道（为什么这样决策 / 人性底层规律）
Digital Oracle = 器（此刻市场在定价什么 / 实时数据信号）
整合产出 = 道×器（用永恒框架解读实时数据 → 科学决策）
```

**核心原则：**
- 数据服务于框架，不是取代框架
- 每个决策必须同时包含"框架引用"和"数据证据"
- Digital Oracle的5步管线（理解问题→选择信号→路由筛选→获取数据→分析报告）嵌入百大认知的决策流程

## 三、7条核心协同链路

以下链路说明：多个书籍框架可以串联使用，形成从"识别信号"到"执行决策"的完整管线。

### 链路1：恐惧识别→群体分析→逆向投资
```
042《情绪大脑》（识别杏仁核劫持）
  → 039《社会动物》（确认群体极化）
  → 010《思考快与慢》（排除认知偏差）
  → Digital Oracle: FearGreed<25 + put/call>1.5 + VIX飙升
  → 决策：逆向建仓
```

### 链路2：权力追踪→结构洞→信息套利
```
035《权力》（权力流向分析）
  → 037《结构洞》（识别信息桥接者）
  → Digital Oracle: SEC内部人交易+CFTC机构持仓
  → 决策：跟随smart money的方向
```

### 链路3：混沌识别→时机套利→反脆弱建仓
```
041《垃圾桶模型》（识别系统混沌态）
  → 046《反脆弱》（杠铃策略建仓）
  → Digital Oracle: Polymarket概率突变+VIX极端值
```

## 五、使用规范

### 5.1 强制三步引用流程

1. **先查映射表**：根据决策场景，找到对应的书籍+数据源组合
2. **获取实时数据**：用Digital Oracle的Python API获取最新数据
3. **框架解读数据**：用书籍的理论框架解读数据含义，而非直接看数字

### 5.2 输出格式要求

每个决策报告必须包含：
- **框架层**：引用的书籍原文 + 理论概念（Source: 百大认知书籍/0XX-书名.md#行号）
- **数据层**：Digital Oracle获取的实时数据（标注获取时间）
- **融合层**：框架如何解读数据 → 得出什么结论 → 概率估计
