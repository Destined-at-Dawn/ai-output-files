# 小黎成长历程 —— 长期记录项目

## 项目目标
长期记录小黎的成长历程——包括过往写的文章、当时的心境感受、关键决策和里程碑事件。这是一个「活着的档案」，持续更新、永不封存。

## 目录结构
```
成长历程/
├── 索引.md           # 全部条目的时间线索引
├── 文章/              # 用户写的文章（完整保留）
│   └── YYYY-MM-DD-标题.md
├── 札记/              # 当时的感受、反思、心境记录
│   └── YYYY-MM-DD-标题.md
└── 里程碑/            # 关键节点（竞赛获奖、offer、重要决策等）
    └── YYYY-MM-DD-标题.md
```

## 使用方式
- **小黎主动添加**：随时往对应目录下加文件即可
- **内容类型**：
  - 文章：任何写过的文章、作业、文案等
  - 札记：当时的情绪、困惑、顿悟、日常随笔
  - 里程碑：竞赛结果、offer、成绩、重要决定
- **我帮你做的事情**：每次添加新内容时，我会帮你做关联分析——这篇新内容跟过去的哪些经历有关联，能看到什么成长线索

## 核心原则
1. **真实至上**：保留当时的原貌，不要为了"好看"而修改过去的文字
2. **不设KPI**：这是成长记录，不是任务清单。不需要定期更新
3. **关联分析**：每次添加新条目，我会帮你找跟过去的连接——你会在回看中发现自己的成长轨迹
4. **隐私本地存储**：所有数据都在本地，不外传

## 跨项目 Memory 互通（必读）

> **共享索引**：`../../memory/MEMORY.md`（工作区根目录）
> **进入本项目时**：先读共享索引，了解其他项目最近的教训/决策，避免重复踩坑。

### 写入检查清单（每次写 memory 时执行）

| 步骤 | 操作 | 为什么 |
|------|------|--------|
| ① | 写入 `memory/{日期}.md` | 本项目的原始记录 |
| ② | 如果是重要教训/决策 → 追加到 `../../memory/MEMORY.md` 的"跨项目教训汇总" | 让其他项目也能看到 |
| ③ | 检查 MEMORY.md 是否已有同类教训 → 有则不重复 | 防止索引膨胀 |

### 触发条件（什么时候必须查跨项目索引）

- 用户问"之前遇到过这个问题吗？" → 查 MEMORY.md
- 要做重要决策时 → 查 MEMORY.md 看其他项目有无类似决策
- 写教训/经验时 → 查 MEMORY.md 防止重复记录
