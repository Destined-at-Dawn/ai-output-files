# 05 | 复盘迭代：每次面试都是免费的市场调研

> 面试失败不亏，面试白面才亏。

---

## 复盘的真正价值

大多数人面试后的反应：
- 拿到offer → 开心 → 不想了
- 没拿到 → 难过 → 不想了

**两种反应都浪费了最有价值的东西：信息。**

每次面试，无论结果如何，你都在免费获取以下信息：
- 市场需要什么技能？
- 面试官最关心什么？
- 你的简历上哪些点被反复追问？
- 你回答不上来的问题暴露了什么短板？

如果你不复盘，这些信息就白白流失了。下次面试，你会犯一模一样的错误。

---

## 复盘模板

每次面试结束后，花15分钟填这个模板（完整版见 `interview/面试复盘模板.md`）：

```markdown
## 面试复盘 — [公司] [岗位] [日期]

### 被问了什么？
（尽量回忆每一个问题）

### 我怎么回答的？
（诚实记录，包括卡壳的地方）

### 面试官的反应
（他对哪些回答感兴趣？对哪些不满意？）

### 我答不上来的
（这些就是你需要补课的方向）

### 面试官关注的点
（这反映了市场需求）

### 我的简历上被追问最多的
（说明这些内容是亮点，要强化）
```

---

## 从复盘中提取行动项

复盘不是写日记。每次复盘必须产出**至少一个可执行的行动项**：

| 发现 | 行动 |
|------|------|
| "面试官追问了AES的实现细节，我答得不够深" | 回去重读AES算法，准备更详细的讲解 |
| "被问到项目管理经验，我没有" | 在下一个项目中主动承担组织协调角色 |
| "面试官对Claude Code很感兴趣" | 在简历中更突出AI工具的使用场景 |
| "被问到为什么选这个方案而不是另一个" | 以后做技术方案时，必须做对比分析并记录 |

## 我的真实案例：面试如何改变了我的简历

### 第一版简历（面试前）

```
项目经历：
- 参与国家级电子设计竞赛
- 负责FPGA模块设计与验证
- 熟练使用Vivado、ModelSim等工具
```

### 经过3次面试后

```
项目经历：
- 国家级电子设计竞赛（数字电路模块设计赛道），负责人
- 独立手写通信接口模块RTL代码，自写Testbench验证波形
- 实现数据处理模块，延迟优化至满足设计要求
- 设计数据校验模块，仿真100%通过率
```

**变化原因：** 面试官反复追问"你具体做了什么"和"怎么证明"。泛泛的描述完全不够用，必须写到模块级细节。

## 市场信号收集

每次面试后，问自己：

1. **这个岗位最看重什么技能？** → 记录下来，指导学习方向
2. **面试官对哪些经历最感兴趣？** → 在简历中放大这些内容
3. **哪些技能被忽略了？** → 也许不需要在这上面花更多时间
4. **行业趋势是什么？** → 面试官聊的话题反映了行业方向

举个例子：我面试了几家FPGA公司后发现，**"手写RTL"比"调用IP核"受重视程度高10倍**。这个信号直接影响了我的学习策略——我开始刻意练习从零手写模块，而不是依赖Vivado的IP核。

## 素材库复利效应

每次面试后，把新发现写入素材库。素材库会越来越厚，简历会越来越好。

```
第1次面试 → 知道要写模块细节 → 更新素材库
第2次面试 → 知道要写量化结果 → 更新素材库
第3次面试 → 知道AI工具是加分项 → 更新素材库
...
第N次面试 → 你的素材库已经非常完善 → 简历质量远超同龄人
```

**这就是复利。每次面试都在给你的求职能力"充值"。**

## 面试问题库

建议你维护一个面试问题库，按类别整理：

| 类别 | 示例问题 | 你的回答 |
|------|---------|---------|
| 项目细节 | "加密模块你是怎么实现的？" | [你的回答] |
| 技术原理 | "通信接口的时序是怎么设计的？" | [你的回答] |
| 方案选择 | "为什么用FPGA不用MCU？" | [你的回答] |
| 困难解决 | "调试时遇到最难的bug是什么？" | [你的回答] |
| 行为面试 | "你怎么看待加班？" | [你的回答] |
| 职业规划 | "你未来3年的规划是什么？" | [你的回答] |

**面试前30分钟，翻一遍这个库，比看任何面经都有效。**
