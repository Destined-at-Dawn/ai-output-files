# 06 | 职场人脉：真诚 > 套路

> "人脉不是你认识多少人，而是多少人愿意帮你。"

---

## 为什么学生也需要人脉

很多学生觉得"人脉"是职场老油条的事，跟自己没关系。

错了。

对一个双非学生来说，人脉可能是**缩小信息差最快的途径**：

- 学长内推 → 跳过简历筛选
- 老师推荐 → 直接拿到面试机会
- 行业前辈的指点 → 知道该学什么
- 同行的信息共享 → 知道哪些公司在招人

**我拿到实习机会的过程中，有两个关键人脉节点：**

1. **我的指导老师** — 她的研究方向与我的目标岗位高度相关，这成了我面试时最强的学术背书。
2. **一位HR** — 他帮我联系到了对口的工程师，让我跳过了技术筛选环节直接进入面谈。

这两次人脉的建立，都不是靠"社交技巧"，而是靠**真实的项目和真诚的态度**。

## 怎么和行业前辈打交道

### 原则1：先给价值，再要帮助

不要一上来就"求内推"、"求指导"。先想想你能给对方什么。

| ❌ 错误 | ✅ 正确 |
|---------|---------|
| "学长你好，能帮我内推吗？" | "学长你好，我最近在做FPGA项目，和你的方向相关，有些问题想请教，方便吗？" |
| "老师，我想做您的项目" | "老师，我看了您的论文，对XX方向很感兴趣，我做了一个相关的demo，您看有没有机会参与您的项目？" |
| "HR你好，你们还招人吗？" | "HR你好，我对贵公司的XX岗位很感兴趣，这是我的项目经历，不知道是否符合要求？" |

### 原则2：展示真实能力，不要吹

行业前辈见过太多"包装过度"的简历了。**真实 > 完美。**

说"我大一，自学FPGA，目前做到了数据处理模块的完整设计"比"精通FPGA设计"更有说服力。因为前者是事实，后者是吹牛。

### 原则3：不要怕被拒绝

发了消息没回复？正常。投了简历没消息？正常。

**被拒绝是求职的常态，不是你的问题。** 10条消息有2条回复就是好比例了。

## 微信沟通的教训

我踩过的坑，希望你不要重蹈覆辙：

### ❌ 教训1：信息轰炸

给一位工程师连发了三条几乎一模一样的自我介绍消息。对方已经回复了"好的"+"嗯"，我又追发了一条确认。

**给对方的印象：** 这个人不看回复，只顾自己输出。

**正确做法：**
- 发一条消息后，等回复
- 对方回了"好的"/"嗯" = 已收到，不追发
- 下次发消息带新内容（代码/文档），不重复

### ❌ 教训2：备注名不改

加了微信后没改备注名，系统显示的是对方之前的个人昵称。这在工作沟通中非常不专业。

**正确做法：** 加完微信立刻改备注——"X工-公司名+方向"。

### ✅ 做对了的事

发了真实的代码仓库链接和波形截图给工程师，而不是只发文字描述。**代码说话比嘴说话有效100倍。**

## 不同场景的沟通模板

### 场景1：加行业前辈微信

```
XX老师/前辈好，我是[学校名]的[你的名字]，做FPGA方向的。
在XX活动/通过XX了解到您，对您的研究方向很感兴趣。
我在做[具体项目]，有些问题想请教，方便的话加个微信？
```

### 场景2：找HR了解岗位

```
XX HR好，我对贵公司的[岗位名]非常感兴趣。
我是[学校+专业+年级]，主要做[方向]，有[具体经历]。
想了解一下这个岗位的具体要求和实习安排，方便聊一下吗？
```

### 场景3：面试后跟进

```
XX老师好，感谢今天的面试，和您聊了[具体内容]之后收获很大。
我对[某个技术点]特别感兴趣，回去之后会深入学习。
期待后续的沟通，祝工作顺利！
```

### 场景4：求内推

**前提：** 你和这个人已经有过至少一次有效沟通，不是突然冒出来求帮忙。

```
XX学长好，上次和您聊的[话题]收获很大。
我最近在找[方向]的实习，看到贵公司在招[岗位]，
和我的方向很匹配（[简要说明]），不知道方便内推吗？
附上我的简历，如果觉得不合适也没关系！
```

## 建立你的个人品牌

人脉不只是"认识人"，也是"被人发现"。

### 技术博客/笔记

把你学到的东西写出来。不需要多专业，能把一个技术点讲清楚就行。

平台选择：
- **CSDN/博客园** — 国内技术社区，搜索引擎收录好
- **知乎** — 适合写技术科普和经验分享
- **GitHub** — 代码+README就是最好的技术名片
- **微信公众号** — 适合长文和系列文章

### GitHub主页

一个有README、有项目、有commit记录的GitHub主页，比任何简历都有说服力。

### 朋友圈策略

你不需要每天发鸡汤。但偶尔分享：
- 完成了一个项目的截图
- 参加了一个竞赛/活动
- 学到了一个有意思的技术点

**这些内容会让别人在想到"做FPGA的人"时想起你。**

---

## 长期主义

人脉是长期投资。今天帮你的学长，三年后可能是你同事。今天指导你的老师，五年后可能是你的合作者。

**不要用"有没有用"来衡量每一段关系。真诚地交流、力所能及地帮助别人，时间会给你回报。**
