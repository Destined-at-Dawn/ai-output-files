# 贡献指南

感谢你愿意为这个项目做出贡献！

## 你可以做什么

### 📄 提交简历模板
- 新的行业方向（金融/医疗/教育/设计……）
- 新的语言版本（日语/韩语/法语……）
- 新的风格（创意型/学术型/极简型……）
- 放在 `resume/templates/` 对应目录下

### 📝 分享成功案例
- 把你的求职经历匿名化后提交到 `community/success-stories/`
- 参考 `community/success-stories/000-example.md` 的格式
- **必须脱敏**：去掉真实姓名、电话、邮箱、公司全称

### 🔧 改进方法论
- 补充你的实战经验
- 修正不准确的建议
- 添加新的行业/岗位视角

### 🐛 修复问题
- 文档错别字
- Markdown格式问题
- 链接失效

## 提交流程

1. Fork 本仓库
2. 创建分支：`git checkout -b feature/你的改动`
3. 提交更改
4. 发起 Pull Request

## 注意事项

- **不要提交个人信息**（真实姓名、电话、身份证号等）
- **不要提交未脱敏的公司信息**
- 中文内容用中文写，英文内容用英文写，不要机翻
- 保持和其他文档一致的风格（直接、实用、无废话）

---

## 5. 成功案例提交详细要求

### 文件命名规则

```
[序号]-[方向]-[学校类型]-[结果].md
```

示例：
- `001-嵌入式-双非-实习offer.md`
- `002-前端-211-秋招offer.md`
- `003-产品-海归-社招转行.md`

序号从 001 开始，按提交时间递增。维护者（项目作者）会在合并 PR 时确认序号。

### 必填字段

以下字段**必须填写**，缺少任何一个 PR 会被打回：

| 字段 | 说明 |
|------|------|
| 背景（学校/专业/年级） | 学校请用"某双非/某211/某985"，不要写真实校名 |
| 求职数据（投递/面试/offer数量） | 真实数字，不需要精确到个位 |
| 什么起了作用 | 至少 2 条具体策略（不是"努力"这种空话） |
| 踩过的坑 | 至少 1 条，这是对后来人最有价值的部分 |
| 给后来人的建议 | 至少 2 条可操作建议 |

### 隐私红线（违反即拒绝合并）

| ❌ 禁止 | ✅ 替代 |
|---------|--------|
| 真实姓名 | 化名、首字母、"我" |
| 手机号/邮箱 | 删除或用占位符 |
| 身份证号 | 绝对禁止 |
| 公司全称 | "某XX公司"/"A公司" |
| 面试官姓名 | "面试官"/"技术负责人" |
| 学校全称 | "某双非院校"/"某211" |
| 老师/同学姓名 | "我的指导老师"/"室友" |

### 审核标准

维护者审核 PR 时会检查：

1. **隐私安全** — 用 `grep` 扫描常见个人信息模式
2. **格式合规** — 文件命名正确、必填字段齐全
3. **内容质量** — 是否有真实细节、是否对他人有参考价值
4. **无广告/引流** — 不允许在案例中插入商业推广链接
