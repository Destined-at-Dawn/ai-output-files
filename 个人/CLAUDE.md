### F3. 记忆归档铁律（2026-05-16 · 四区统一）

> **根因**：write_project_file 是全量覆写，不是追加。对话中往 long-term.md 追加内容需要"读→拼接→写"三步，对话越长上下文越乱，覆盖全文的风险越高（已发生上百次）。改为每日归档制，彻底消除此风险。

| 规则 | 说明 |
|------|------|
| **禁止对话中写 long-term.md** | 任何对话中**禁止**直接写入 `memory/long-term.md`，只写 `memory/{YYYY-MM-DD}.md` |
| **当日记忆随写随记** | 对话中有重要发现/决策/教训，**立即**写入 `memory/{YYYY-MM-DD}.md`（短文件，风险低） |
| **long-term.md 每日归档** | 由 `longmemory` 技能每日自动归档（cron 00:05），读取每日记忆 → 提取长期内容 → 合并到 long-term.md |
| **归档格式** | 归档时必须先 Read 完整 long-term.md → 在对应章节插入 → 全量 Write → Read 验证 |
| **Write 后 Read** | 任何记忆文件写入后必须 Read 验证 |

> **创建 long-term.md**：本工作区首次归档时，由 longmemory 技能自动创建。

> **问题**：各项目的 memory 互相隔离，项目A的经验项目B看不到。
> **方案**：工作区级共享 memory 索引 + 各项目 memory 可互相引用。

| 规则 | 说明 |
|------|------|
| **共享索引** | `memory/MEMORY.md`（工作区根目录）= 跨项目 memory 总索引 |
