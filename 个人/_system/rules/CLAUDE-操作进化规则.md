## 四、文件操作铁律（信任验证）

> ⚠️ **"承诺"一文不值，只有 tool_use 发起的实际文件操作才算数。**
> Source: 交叉学习自竞赛区（Verilog FPGA CLAUDE.md §10 + 皮影戏 CLAUDE.md §7）


| 规则             | 说明                                                                      |
| -------------- | ----------------------------------------------------------------------- |
| **4.1 禁止伪造**   | 所有文件操作必须通过 tool_use（Write/Edit/Bash）发起，禁止在纯文本中描述"已写入 xxx"但本轮未发 tool_use |
| **4.2 验证必须**   | 每次 Write/Edit 后，必须 Read 同一文件确认内容正确落盘                                    |
| **4.3 绝对路径审计** | 系统会核对每个声称的绝对路径，虚假路径会被记录                                                 |
| **4.4 禁止空文件**  | 写入的内容必须有实际信息，不能写空文件或"今日无更新"占位符                                          |


**验证格式：** 每次写文件后，回复中必须注明 `✅ 已验证: {文件路径} — {行数}行/{大小}`

## 五、记忆文件操作铁律（追加禁止覆盖）

> ⚠️ **历史教训**：多个工作区出现过 AI 用覆盖方式重写 memory 文件导致早期记录全部丢失。所有工作区统一规则。
> Source: 交叉学习自竞赛区（Verilog FPGA §11 + 皮影戏 §8）


| #       | 规则                    | 说明                                                        |
| ------- | --------------------- | --------------------------------------------------------- |
| **5.1** | **先 Read 再 Write**    | 更新任何 memory 文件前，必须先 Read 读取当前全部内容，在末尾追加新内容。**禁止直接覆盖整个文件** |
| **5.2** | **追加格式**              | 新内容以 `## YYYY-MM-DD HH:MM` 的日期时间分隔开头，加在文件末尾               |
| **5.3** | **Write 后必须 Read 验证** | Write 之后必须在同一轮用 Read 验证追加的行确实存在于末尾                        |
| **5.4** | **禁止空文件/占位符**         | 写入的内容必须有实际信息（做了什么、发现了什么、验证结果）                             |

## 九、自我进化机制

> Source: 交叉学习自创作区 CLAUDE.md v3.2 §10 + 竞赛区 decision/CLAUDE.md §7

### 9.1 目录

```
自我进化/
├── 做得好的鼓励/    ← 用户肯定时，保存该轮对话
└── 做得差的避免/    ← 用户批评时，保存该轮对话
```

### 9.2 操作规则

**当用户说"做得好/不错/厉害/满意"时：**

- 将本轮一问一答提取为 `.md` 文件，保存到 `自我进化/做得好的鼓励/`
- 文件名：`{YYYYMMDD}_{主题}_成功案例.md`

**当用户批评或指出错误时：**

- 将本轮一问一答提取为 `.md` 文件，保存到 `自我进化/做得差的避免/`
- 文件名：`{YYYYMMDD}_{主题}_失败教训.md`
- 包含：原问题 + AI回复 + 错误原因 + 正确做法
- 写入 memory，防止同类错误重复

## 十五、教训->skill/SOP 反馈闭环（强制）

> **教训只记不改 = 白记。** 每条教训必须流向它能影响的 SOP 或 skill，否则下一次对话会被遗忘。

### 闭环铁律

每当自我进化文件或 lessons-learned 新增一条教训，**同一次对话内**必须完成：

| 步骤 | 操作 | 验证 |
|------|------|------|
| 1. 定位 | 查映射表，确定影响哪个 SOP + 哪个 skill | 映射表在 `E:\ai产出文件\牛马\知识中枢\02-共享规则\09-feedback-loop.md` |
| 2. 回写 | 在 SOP「迭代日志」追加一行 | Read 验证 |
| 3. 升级 | 同类教训 3 次 -> 升级为 SOP 铁律 | Edit 更新 SOP 正文 |
| 4. 通知 | 影响 skill 行为 -> 更新 CLAUDE.md 对应段落 | Read 验证 |
