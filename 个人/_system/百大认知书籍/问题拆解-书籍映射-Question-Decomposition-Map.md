# 问题拆解 → 书籍映射表（Question Decomposition → Book Mapping）

> **用途：** 每次用户提问，先拆解问题为N个主题维度，再查本表定位对应书籍。
> **设计原理：** 任何复杂问题都不是单维度的。本表将55本书按10大主题维度 + 子问题类型归类，支持快速交叉定位。
> **使用流程：** 拆解问题 → 识别涉及的主题维度 → 查本表锁定书籍编号 → Read原书提取原句 → 回答。
>

> **书籍总数：** 62本（v3.6新增062《冲浪不确定性》——预测处理/自由能原理/精度权重操控/主动推断/马尔可夫毯/认知觅食）

## 零、快速使用指南

### AI内部操作流程（每次回答前执行）

```
1. 识别用户问题类型 → 看§一的"问题类型快速匹配"
2. 拆解为N个子问题 → 每个子问题对应§二的1个主题维度
3. 查本表§三~§十二 → 每个维度下找到对应的书籍编号
4. 引用追踪去重 → 检查CITATION_TRACKING.md，优先选冷门书（❄️未激活）
5. Read原书 → 提取精确原句（禁止凭记忆复述）
6. 回答 → 标注Source: 百大认知书籍/0XX-书名.md#行号
```

## 十六、维护规则

1. **新增书籍时**：在§十三的跨维度矩阵中新增一行，在对应维度章节中按子问题插入书籍引用，检查§十五是否有新的协同链路
2. **发现新子问题时**：在对应维度下新增一行，匹配已有书籍——子问题必须是用户真实提出过的或高度可能提出的
3. **发现有书籍更适合某维度时**：直接修改映射，不删除旧记录（保留追溯）
4. **与本目录其他文件的协同**：
   - `000-总索引-Index.md`：书籍概览和场景速查（详细内容查询入口）
   - `CITATION_TRACKING.md`：引用频率控制（防止"高频可用性偏差"）
   - 本文件：问题拆解逻辑（AI内部使用的映射表）
5. **版本号递增规则**：大改（新增维度/重组结构/新增§章节）+0.1；小改（新增子问题/调整映射/新增场景）+0.01
6. **每新增10本或每月一次**：全量审查§十四的场景覆盖是否仍然匹配小黎当前身份状态
