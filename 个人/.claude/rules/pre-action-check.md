# 动手前检查规则（Pre-action Check）

> 来源：2026-06-14 教训闭环（4条教训共同根因）
> 优先级：高（与 No Blind Overwrite 同级）

## 触发条件

收到以下类型任务时，**必须先检查再动手**：

### 1. 写脚本/代码
- ❌ 直接开始写
- ✅ 先查 `script-library/INDEX.md` 和 li-script skill
- ✅ 先确认有没有现成的可复用

### 2. 生成图片
- ❌ 直接调用默认工具
- ✅ 先确认工具选择：
  - 脚本图（matplotlib/PIL）→ 仅用于线稿、结构图、示意图
  - image_gen/image2 → 用于高质量彩图、人物、比赛级视觉

### 3. 多图任务
- ❌ 分别独立生成
- ✅ 先锁定一致性参数（视角/透视/风格/镜头）
- ✅ 同一幕多用途图 → 先生成三联母版

### 4. 模板填充
- ❌ 重新制作文档
- ✅ 先确认"只填内容，不改结构"
- ✅ 保留原表格/段落样式/字体体系

### 5. Git 忽略/清理/取消跟踪
- ❌ 看到 `.tmp/`、`**/.tmp/`、`*.pyd` 就默认加入 `.gitignore` 或 `git rm --cached`
- ❌ 把"临时文件/二进制依赖"等同于"无价值垃圾"
- ✅ 先列清单：路径、大小、修改时间、是否已被 Git 跟踪、可能用途
- ✅ 先保证可恢复：已有归档 commit/tag，或单独归档包/恢复脚本
- ✅ 再分级决策：必须保留 / 可再生成但需记录生成方式 / 真垃圾
- ✅ 对 `.pyd` 等运行依赖，只有在确认有安装脚本、锁文件或明确可复现来源后，才允许忽略或取消跟踪

## 验证清单（每项任务收尾前必过）

- [ ] 已检索相关 skill/SOP
- [ ] 已确认工具选择正确
- [ ] 已确认边界（什么能改什么不能改）
- [ ] 已执行交付前自检
