# 2026-05-07 Recordly 大文件分片导出 — 成功案例

## 用户原问题

用户在 Recordly 中录制了一段 2 小时+ 的屏幕视频（7.5GB），导出时因 Electron 内存溢出（OOM）崩溃。用户发来了一段详细的故障分析文本，要求找到原始录制文件并用 ffmpeg 无损分段保存。

## 我的回复

1. **定位原始文件**：直接去 `%APPDATA%\Recordly\recordings\` 找到 4 个关联文件（7.5GB mp4 + 180MB mic wav + 1GB system wav + 17MB cursor json），一步到位
2. **制定分段方案**：用 `ffmpeg -c copy -segment_time 1200 -f segment` 无损切片，每段约 20 分钟/1.3-1.6GB
3. **执行切片+验证**：切出 5 段（part000~part004），用 ffprobe 验证每段时长和完整性，总计 1:35:00
4. **移动到用户指定目录**：用户说"E:\ai产出文件\牛马\创作\创作\讲座\video"，立即 mv 过去
5. **简洁汇报**：表格展示 5 段文件名/时长/大小/状态，一句话说明技术要点

## 为何满意

- **反应快**：用户说"我全部都需要"，我直接给出分段方案并执行，没有多余确认
- **零损失**：`-c copy` 无损切片，不重新编码，画质完全一致
- **带验证**：每段都用 ffprobe 验证了时长+码率，不是"已完成了"空话
- **一句话总结要点**：用户不需要懂 ffmpeg 参数，我用"无损切片/独立可播放/拼接=完整"说清楚

## 可复用模式

**大文件处理三步法**：
1. 定位源文件（按文件大小+扩展名快速筛选）
2. 分段处理（`ffmpeg -c copy -f segment`，避免重编码）
3. 验证完整性（ffprobe 检查每段时长+码率+编码）

**用户指定路径时**：不要问"要不要复制还是移动"，直接 mv + 确认。
