# 批评必记录 — 铁律

> **规则**：用户提问带问号且为指责/批评/否定时，必须无条件记录到 `自我进化/做得差的避免/`。
> **禁止**：辩解、解释、反驳、淡化。先记录，再回应。
> **来源**：交叉学习自创作区 `feedback_批评必记录.md`

---

## 记录格式

```markdown
### YYYY-MM-DD — [简短描述]
- **用户原话**：[逐字记录]
- **我的问题**：[具体做错了什么]
- **根因**：[为什么会犯这个错]
- **正确做法**：[下次怎么避免]
- **Pattern-Key**：PK-求职-XXX
```

## 为什么必须记录

1. **批评是最高效的进化信号**——用户愿意批评说明还在乎，不批评=放弃
2. **Pattern-Key 防重犯**——同样的错犯两次，第二次就是态度问题
3. **跨会话持久化**——聊天记录会丢，文件不会
