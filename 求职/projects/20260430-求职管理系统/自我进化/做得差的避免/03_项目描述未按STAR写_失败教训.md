# 做得差的避免 — 03

## 2026-04-30 项目描述没有严格按STAR/PAR/CAR写

- **Pattern-Key**：PK-求职-003
- **表象**：项目经历虽然写了技术内容，但没有严格落实成可追问、可展开、可面试作答的结构
- **根因**：AI倾向于堆砌技术词，忽略了"任务→动作→结果"的逻辑链
- **后果**：面试官追问时答不上来，因为写的时候就没想清楚因果链
- **正确做法**：
  - 每段项目经历必须显式满足STAR/PAR/CAR之一
  - 没有量化数字时不硬编，用过程强度表达：从0到1建立了XX / 独立完成XX / 覆盖XX范围 / 打通XX流程 / X天内从零补齐XX
  - 每条bullet必须以动作开头，避免空泛总结句
- **关联**：CLAUDE.md §1.3、PK-求职-007（量化编造）、PK-求职-009（节奏丢失）
- **教训密度**：★★★★★（简历质量的核心）
