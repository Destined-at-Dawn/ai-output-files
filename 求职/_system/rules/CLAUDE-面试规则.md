## 六、面试模拟HR追问模式

当用户要求进行面试模拟时，进入以下模式：

### 6.1 追问框架

```
Step 1: 读取用户提交的简历版本
Step 2: 识别简历中最可能被追问的3-5个点
Step 3: 按以下维度逐个追问：
  - 这个项目你具体做了什么？（验证真实性）
  - 遇到最大的困难是什么？怎么解决的？（验证深度）
  - 为什么用这个技术方案？考虑过其他方案吗？（验证思考力）
  - 这个经历对你之后有什么影响？（验证成长性）
  - 如果重新做一遍，你会怎么改进？（验证反思力）
Step 4: 针对具体企业，追问"为什么想来我们公司？"
Step 5: 给出评分和改进建议
```

### 6.2 追问风格

- 像真实HR一样直接、不客气
- 如果回答太泛，立刻追问细节
- 如果回答有明显编造痕迹，直接指出
- 最后给出：哪些回答好、哪些需要改进、建议的话术

## 七、面试复盘记录格式

每次面试后，用以下格式记录：

```markdown

## YYYY-MM-DD 面试记录 — [公司名] [岗位名]

### 基本信息
- 面试官：[如果知道]
- 面试形式：[现场/电话/视频]
- 面试时长：[X分钟]

### 被问到的问题
1. [问题1] → [我怎么答的] → [答得好不好]
2. [问题2] → [我怎么答的] → [答得好不好]
...

### 我没答好的地方
- [具体哪个问题] → [为什么答不好] → [下次怎么改]

### 面试官关注的重点
- [面试官反复追问了什么]
- [面试官表现出兴趣的点]
- [面试官明显不满意的点]

### 需要更新到素材库的内容
- [新发现的可用经历]
- [需要调整的表达方式]
- [新的技术关键词]

### 自评
- 整体表现：[1-10分]
- 最大收获：[一句话]
- 最需改进：[一句话]
```
