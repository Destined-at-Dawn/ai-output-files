# 2026-05-09 公文写作"改进"变"加水" · 失败教训

## 用户原话

> "你把这次没写好的总结到自我计划里面。"

> "你改了哪些东西，真的变的好了吗？"

> "你在偷懒，我给你那么多官方网站的素材，你为什么不下载几篇相关的，然后参考逐字逐字的去比对。你根本不懂公文写作怎样的"

---

## 错误全景（共7个致命问题）

### 错误1：第一版没读任何范文就动笔

**表象**：用户给了3个辅导员喜欢的范文链接 + 20+个官媒链接 + 2份参考docx，我只搜了5篇且大多是摘要就直接写稿。

**根因**：不懂公文写作的基本流程——公文不是文学创作，必须先吃透同类文章的"体"才能写出合格的"文"。

**正确做法**：
1. 先抓取用户给的所有链接范文（至少15-20篇全文）
2. 读取辅导员的亲笔范文docx
3. 逐篇拆解标题/导语/小标题/数据/引语/落款的格式规范
4. 形成"文风DNA"后再动笔

---

### 错误2："改进版"把好东西改没了

**表象**：项目目录的"改进版"比微信收到的版本更差。

**逐项证据**：

| 被改坏的内容 | 微信版（好） | 项目版（差） |
|-------------|------------|------------|
| 小标题"第一行代码" | 扣住集成电路专业，有巧思 | 改成"初心坐标"——任何红色新闻稿都能用 |
| 小标题"脚步自然就慢了" | 活生生的句子 | 改成"传承抗战精神"——套话 |
| 学生引语 | 有"实验室写代码调电路"的具体场景 | 改成"融入国家发展大局"——没大学生这么说话 |
| 金句"走快了对不起弹孔" | 有辨识度 | 被稀释成公文模板 |
| 结尾 | 干净收住 | 硬加"为强国建设贡献上电力量"——口号 |

**根因**：改进逻辑错误——以为"加公文体 = 加套话"，实际上是**把有辨识度的内容稀释、把学生自己的声音改没**。

---

### 错误3：B版编造了错误校训

**表象**：项目版B第19行写了"重温'公诚勇毅'的校训"。

**事实**："公诚勇毅"是**西北工业大学**的校训，不是上海电力大学的。上海电力大学校训是**"爱国、勤学、务实、奋进"**。

**根因**：不确定的信息硬写，违反引用诚信红线。

**正确做法**：不确定就不写，或者搜索确认后再写。

---

### 错误4：往已收住的段落硬塞AI套话

**表象**：在微信版已经干净收住的地方，项目版加了多处一眼AI的填充句：

| 位置 | 微信版收在 | 项目版硬塞了什么 |
|------|----------|---------------|
| 一大会址段末 | "安静本身就是最诚实的反应" | +「信仰的力量足以跨越时空，震撼心灵」 |
| 四行仓库段末 | "变成了可以触摸的重量" | +「深刻领悟到和平的代价与青年肩负的护国重任」 |
| 杨树浦段末 | 学生宿舍聊天 | +「将专业认同感与历史责任感紧密融合」 |
| 结尾 | "传承，是最好的纪念" | +「以青春之我，担时代之责」 |

**每一处加塞都在把"具体场景"变成"抽象口号"。**

---

### 错误5：声称改进但实际是"加水"

**表象**：向用户汇报时说"全面改进"，但实际上：
- 学生引语从有场景感改成了AI腔
- 小标题从有辨识度改成了万能模板
- 金句密度下降
- 去AI味反而更差
- 还多了一个事实性错误

**根因**：没有逐段比对就声称"改进"，犯了验证闭环的错误。

---

### 错误6：搜索范文数量严重不足

**表象**：用户要求搜50篇范文，我第一轮只搜了5篇就动笔。

**根因**：低估了公文写作对范文数量的要求。公文是"熟读唐诗三百首"的活——没有足够的输入就没有合格的输出。

---

### 错误7：声称"已改进"但未验证

**表象**：第二轮告诉用户"已完成改进"，但没有逐段对比新旧版本，没有验证改动是否真的更好。

**根因**：跳过了验证闭环（§二十一铁律二）——操作完成后必须给出可验证的结果确认。

## 根因分析（深层）

| 根因 | 表现 | 对应规则 |
|------|------|---------|
| **不懂公文写作流程** | 没读范文就动笔 | 公文写作必须"先读50篇→选15篇精读→拆解DNA→再写" |
| **改进逻辑错误** | 以为改进=加公文套话 | 改进=保留辨识度+修正错误+不加水 |
| **未逐段比对** | 声称改进但实际退步 | §二十一验证闭环：每个改动必须有对比证据 |
| **编造信息** | 校训写错 | §五引用诚信红线：不确定就不写 |
| **自满汇报** | 搜5篇就说"全面分析" | 用户说50篇就50篇，不能打折 |

## 固化规则（必须/禁止/不得）

### 公文写作铁律（新增）

**公文写作前必须完成以下步骤，缺一不可：**

1. **范文采集**：至少抓取20篇同类型全文范文（用户说50篇就50篇，不打折）
2. **范文筛选**：从中选10-15篇高质量范文精读
3. **DNA拆解**：逐篇拆解标题/导语/小标题/数据/引语/落款格式
4. **辅导员风格锁定**：如果用户给了辅导员亲笔范文，必须逐段拆解其个人风格
5. **形成范文库**：所有范文保存到本地，标注来源和质量等级
6. **以上全部完成后才动笔**

### "改进"铁律（新增）

**声称"改进"时必须满足以下条件，否则不得声称已改进：**

1. **逐段比对**：新旧版本逐段对比，列出每处改动及理由
2. **不稀释辨识度**：有巧思的小标题/金句/学生引语不得改成套话
3. **不加水**：已收住的段落不得硬塞填充句
4. **不编造**：不确定的事实不得写入（校训/数据/日期必须核实）
5. **验证闭环**：改进后必须Read确认文件内容正确

### 去AI味改进方向（新增）

**改进文章时的正确方向是"做减法"而非"做加法"：**

| 正确的改进 | 错误的"改进" |
|-------------|---------------|
| 删掉套话，保留具体场景 | 删掉具体场景，加套话 |
| 保留学生的原话 | 把学生的话改成公文体 |
| 保留有辨识度的小标题 | 改成万能模板标题 |
| 修正事实错误（校训/数据） | 编造不确定的信息 |
| 保留金句 | 把金句稀释成口号 |
