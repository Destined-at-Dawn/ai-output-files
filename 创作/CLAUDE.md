## 十五、AI 工程法则（2026-05-25 副教授方法论）

> **三条根本信条 + 十条工程法则 + 六阶段生命周期**，详见 `.claude/rules/` 目录（每次对话自动加载）。

### 三条根本信条（最高准则，不可违反）

1. **诚实决定价值**：口径不实的高分是负资产。永远先问"这个数字是怎么来的"
2. **效率来自并行与不重复**：死路必须归档，不被重复踩
3. **边界自觉**：成熟的交付 = 准确知道做到了哪、什么不能声称

### 十条工程法则

| 法则 | 核心 | 触发信号 |
|------|------|---------|
| 1. 证据分层 | `<指标> = <值> @ <口径>` | 要写下任何 PASS/达标/分数 |
| 2. 防假象审计 | 高分先审产生方式（五连问） | 出现明显高于预期的指标 |
| 3. 实测优先 | 扫描验证，不假定单调 | 要调一个连续参数 |
| 4. 工具自由度 | 克制人工 hint | 想用"硬约束"帮工具 |
| 5. 跨边界校验 | 格式/编码/单位/命名/口径 | 数据在两个系统间传递 |
| 6. 隔离契约 | 多 Agent 写入区严格隔离 | 启动并行工作 |
| 7. 负结果归档 | 死路必须归档 | 方案被验证为无效/更差 |
| 8. 门禁文化 | 完成 = 过全部门禁 | 想宣告"做完了" |
| 9. 识别结构墙 | 平台期换角度，天花板就停 | 同一目标多次失败 |
| 10. 规则结晶 | 经验 → 规则 → Skill | 每轮对话结束/用户纠正 |

### 六阶段生命周期

```
立项调研 → 方案探索 → 并行攻坚 → 分层验证 → 复盘结晶 → 交付声明
```

**每阶段开工前读对应 SOP；每天/每轮对话对照工程法则自检。**
