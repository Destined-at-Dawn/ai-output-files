# C: case-encrypt（隐私脱敏）

> 对标 `.claude/rules/privacy-sanitization.md` 的案例特化版本。

---

## 一、问题域

### 1.1 解决什么问题

案例文件中可能包含：

- 用户的真实姓名、学校、公司
- 具体的对话细节（可能暴露用户身份）
- 内部定价策略、收入数据
- 飞书/微信链接等私密信息

如果经验库文件需要公开（如推送到 GitHub），必须先脱敏。

### 1.2 行家怎么做

- **自动化扫描**：每次文件写入时自动检查敏感词
- **白名单机制**：只有通过脱敏的文件才能进入公开区
- **分级处理**：致命级（禁止公开）vs 重要级（需要人工判断）vs 注意级（自动替换）

### 1.3 做错了什么样

- 案例文件包含真实姓名，推送到 GitHub 后被搜索引擎收录
- 定价策略泄露给竞争对手
- 用户投诉隐私泄露

---

## 二、使用方式

### 2.1 触发条件

- 案例文件写入经验库后自动触发
- 涉及的模块经验库路径如果在 Git 仓库中 → 必须脱敏

### 2.2 执行流程

```
案例文件写入
    │
    ▼
Step 1: 扫描敏感词
  - 对照 privacy-sanitization.md 的扫描清单
  - 分三梯队：致命级/重要级/注意级
    │
    ▼
Step 2: 分级处理
  - 致命级：自动替换为脱敏标记
  - 重要级：标记为"需人工判断"，在文件中用 {TODO:人工审核} 标注
  - 注意级：记录到日志，不阻断
    │
    ▼
Step 3: 生成脱敏版本
  - 原始版本保留在私有目录（不进入 Git）
  - 脱敏版本写入公开经验库
    │
    ▼
Step 4: 输出扫描报告
```

### 2.3 脱敏替换规则

| 原始 | 替换为 | 级别 |
|------|--------|------|
| 小黎/李兰源 | [用户名] | 致命 |
| 具体学校名 | [学校] | 致命 |
| GPA 数字 | [GPA] | 致命 |
| 具体公司名 | [公司] | 致命 |
| 微信/飞书 ID | 删除 | 致命 |
| 定价金额 | [定价] | 重要 |
| 具体日期（非版本日期） | [日期] | 注意 |

## 三、配置

### 3.1 可配置参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `自动脱敏` | `true` | 致命级敏感词自动替换 |
| `人工审核标记` | `{TODO:人工审核}` | 重要级敏感词的标记格式 |
| `扫描清单路径` | `.claude/rules/privacy-sanitization.md` | 扫描清单来源 |

---

## 四、不可做清单

- ❌ 跳过脱敏直接写入公开经验库
- ❌ 只替换姓名不替换路径/数字
- ❌ 脱敏后不验证（可能替换不完整）
- ❌ 用 PowerShell 操作中文路径
