# 个人信息检索协议（Personal Info Finder）

> **版本**：v1.0
> **创建日期**：2026-06-10
> **来源**：教训029——面试记录搜索失败，只搜memory/遗漏了projects/直播笔记/

---

## 触发条件

任何需要查找"小黎经历了什么/说过什么/做过什么"的任务，包括但不限于：
- 写个人经历相关文章（心得体会、简历、自我介绍）
- 查找面试问答记录
- 查找竞赛/实习/项目经历细节
- 查找直播/分享中的具体内容

---

## 核心原则

**先搜全貌，再读细节。搜文件路径 > 读文件内容 > 凭记忆推断。**

---

## 强制执行步骤

### Step 0: 绘制信息地图（不可跳过）

```
必须执行以下全部Glob/Grep：
1. Glob **/*直播* → 找所有直播相关文件
2. Glob **/*分享* → 找所有分享相关文件  
3. Glob **/*面试* → 找所有面试相关文件
4. Glob **/*实习* → 找所有实习相关文件
5. Glob **/*竞赛* → 找所有竞赛相关文件
6. Grep "关键词" -l → 找所有含关键词的文件
```

### Step 1: 按信息密度排序

| 数据类型 | 存放位置 | 信息密度 | 优先级 |
|---------|---------|---------|--------|
| 原始经历记录 | `**/直播笔记/` `**/txt/` | 最高（逐字稿级） | ⭐⭐⭐⭐⭐ |
| 结构化知识 | `**/SOP/` `**/私域认知/` | 高（方法论级） | ⭐⭐⭐⭐ |
| 加工后版本 | `**/outputs/` `**/分享稿/` | 中（不同角度） | ⭐⭐⭐ |
| 概括性记忆 | `**/memory/` | 低（一句话级） | ⭐⭐ |
| 推断（AI猜测） | — | 最低 | ⚠️ 标注置信度 |

### Step 2: 读取+交叉验证

```
规则：
- 至少读取3个不同来源
- 发现矛盾 → 优先采信直播笔记（原始记录）
- 推断内容必须标注"⚠️推断，非原始记录"
- 推断内容禁止直接使用，必须用 mcp__ask-user__ask_user 请求用户确认
```

---

## 禁止模式

| 禁止操作 | 原因 |
|----------|------|
| 只搜memory/目录就声称"找不到" | memory是最低信息密度来源 |
| 找不到就直接推断 | 推断≠事实，必须标注置信度 |
| 假设"某类信息一定在某目录" | 信息可能分布在多个目录 |
| 用推断内容写入正式文章 | 违反"绝对不能有事实错误"铁律 |
