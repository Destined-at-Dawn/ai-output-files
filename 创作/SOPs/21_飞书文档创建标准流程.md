# SOP-21 飞书文档创建标准流程（v1.0）

> **优先级：最高**（违反即事故）
> **创建日期：** 2026-06-07
> **来源：** 飞书操作 v1→v2→v3 演进——8个坑合并为可执行SOP
> **触发条件：** 上传飞书 / 创建飞书文档 / 写入飞书 / 飞书文档 / 知识库迁移 / 一比一复刻
> **关联规则：** `.claude/rules/feishu-api-protocol.md`（API铁律）、`经验库/做得差的/飞书文档全流程永不再犯检查清单`（教训来源）

## §〇 写前强制 GATE（30 秒，不过不动手）

**每个飞书文档创建任务开始前，必须回答以下 3 个问题。全部通过才动手。**

| GATE | 问题 | 不通过怎么办 |
|------|------|-------------|
| G1 | 目标文档是独立文档还是 Wiki 知识库？ | 确认后决定走 §一 还是 §二 |
| G2 | 当前用 tenant_access_token（App）还是 user_access_token（用户）？ | tenant = 只能创建独立文档；user = 可写 Wiki |
| G3 | 源内容是否完整提取？（所有段落、表格、代码块、引用） | 不完整 → 先补全再创建，禁止边提边建 |

**违规信号：** 收到飞书任务但没过 GATE 就动手。

## §一 独立文档创建流程（6 步闭环）

### Step 1：创建文档

```
POST /docx/v1/documents
body: { "title": "文档标题" }
```

- ✅ 不设任何权限参数（默认 tenant_readable = 最安全默认值）
- ❌ 禁止 `link_share_entity: closed`（管理员看不到）
- ❌ 禁止 `external_access_entity`（App 无权设置）
- 记录返回的 `document_id`

### Step 2：解析源内容为 Blocks

**Block Type 映射表（已验证）：**

| Block Type | block_type 值 | 用途 | 注意事项 |
|---|---|---|---|
| TEXT | 2 | 普通段落 | 默认段落 |
| H1 | 3 | 一级标题 | heading 从 3 开始，不是 1 |
| H2 | 4 | 二级标题 | — |
| H3 | 5 | 三级标题 | — |
| Ordered List | 13 | 有序列表要点 | T13 = 飞书文档看起来专业的关键 |
| QuoteContainer | 34 | 引用容器 | 飞书 v2 API 支持 |
| Code Block | 14 | 代码块 | 需要 code_language 参数 |
| Divider | 22 | 分割线 | 独立文档支持，Wiki 不支持 |
| Image | 27 | 图片 | 需要 file_token，Wiki 不支持 |

**解析规则：**
1. 加粗文本 → 在 block 的 `style.bold` 字段设置，不用 `**markdown**`（飞书不渲染）
2. 行内代码 → 在 text_run 的 `style.code_block` 字段设置
3. 表格 → 需要特殊处理（Table block + Cell blocks），逐行验证
4. 链接 → text_run 的 `style.link` 字段
5. 代码块 → Code Block（type 14），必须带 `code_language` 字段

### Step 3：测试写入（1 个 block）

```
POST /docx/v1/documents/{id}/blocks/batch_create
body: { "children": [测试block] }
```

- ✅ 验证返回值：block_id 存在 = 格式正确
- ❌ 返回 `invalid param` → 检查 block_type 映射
- ❌ 返回 `403` → 权限问题，停止，走 §三 退出判据

### Step 4：批量写入

- 逐批写入（建议每批 50 个 blocks）
- **每个返回值必须检查**，不要只看汇总统计
- 失败的 block 记录到临时列表，最后统一重试
- 连续 3 个同类 block 失败 → 检查格式，不要盲目重试

### Step 5：浏览器验证（铁律6，不可跳过）

```
browser_open → 文档 URL
→ 确认页面可访问（无"没有权限"错误）
→ 确认内容完整（首尾段落都在）
→ 确认格式正确（标题层级、加粗、列表）
```

**API 返回成功 ≠ 用户能看到。** 最后一步永远是浏览器验证。

### Step 6：记录文档 URL

- 写入 memory/ 当日记忆
- 输出文档链接给用户
- 如需移入知识库：告知用户在飞书 UI 手动移动

## §二 Wiki 写入流程（3 步 + 阻塞切换）

> **前置条件：** 需要 user_access_token 或 Wiki space 已添加 App 为编辑者。

### Step 1：测试 Wiki 写入权限

```
向 wiki 页面发 1 个最小化 block
→ 成功 → 继续
→ 403 / 131006 / 1770032 → 立即停止，走 §二 Step 2
```

### Step 2：权限不足时的切换路径（按优先级）

```
方案 A: user_access_token（需 OAuth 流程，用户授权）
方案 B: 浏览器操作（browser_open → 新建文档 → 粘贴内容）
方案 C: 创建独立文档 → 用户手动移入 Wiki
❌ 禁止: 反复尝试不同 API 路径（2 轮规则——见 §四）
```

### Step 3：Wiki 可用 Block 白名单

| ✅ 支持 | ❌ 不支持 |
|---------|----------|
| TEXT(2), H1(3), H2(4), H3(5) | Divider(14) |
| Ordered(13), Bullet(12) | Image(27) |
| QuoteContainer(34) | Table（未确认） |
| — | Callout（未确认） |

## §三 退出判据（该停手了）

| 场景 | 退出信号 | 切换到 |
|------|---------|--------|
| 权限 API 返回 1063002 | 第 1 次就停 | 用户在 UI 操作 |
| Wiki 写入返回 403/131006 | 第 2 次确认后停 | user_access_token 或浏览器 |
| Block 写入返回 invalid param | 连续 3 个同类失败 | 检查 block_type 映射 |
| 内容提取不完整 | 第 2 次补全仍缺失 | 切换提取方式（API↔浏览器） |
| 文档创建后用户打不开 | 第 1 次就停 | 重建文档（不设权限参数） |
| 同一底层障碍第 3 次失败 | 🛑 强制停止 | 记录为结构墙，切换完全不同的方案 |

## §四 2 轮规则（方案切换决策树）

```
操作失败了？
├── 第 1 次失败
│   → 分析错误码含义
│   → 参数/格式错误 → 修正后重试（最多 2 次）
│   └── 权限/架构限制 → 直接进入方案切换
│
├── 第 2 次失败（同类错误）
│   → 确认是否同一个底层障碍
│   → 是 → 立即切换到完全不同的方案
│   └── 不是 → 修正后给最后一次机会
│
└── 第 3 次失败（同类错误）
    → 🛑 停止。这是结构墙，不是平台期
    → 切换到替代方案
    → 记录为负结果
```

**铁律：同一底层障碍最多探测 2 轮。第 3 轮必须切换方案。35 轮 → 4 轮，这就是退出机制的价值。**

## §五 权限架构速查

> 飞书权限三层架构——每一层独立，不能互相替代。

```
第一层：App scope（开放平台配置）→ 决定 API 能不能调
  → 不足 → HTTP 400 / 401

第二层：文档级权限（owner/成员）→ 决定谁能访问这个文档
  → 不足 → 错误码 1063002

第三层：Wiki space 级权限（知识库设置）→ 决定 App 能不能操作这个知识库
  → 不足 → 错误码 131006 / 1770032
```

**关键认知：**
- scope 齐全 ≠ 能操作文档 ≠ 能操作 Wiki
- `tenant_access_token` 创建的文档，创建者 = App ≠ 用户
- `open_id` 按 app 维度分配，不同 app 的 open_id 不同

## §六 已知陷阱速查表

| 坑 | 根因 | 预防措施 |
|------|------|---------|
| 设 `closed` → 管理员看不到 | 创建者 ≠ owner | 不设权限参数（默认 tenant_readable） |
| 孤岛文档——不在云盘 | API 创建不挂云盘 | 告知用户手动移入 |
| open_id 跨 app 不一致 | 不同 app 不同 open_id | 用 email 不用 open_id |
| tenant_token 无权管理协作者 | App ≠ owner | 权限操作全部交用户 UI |
| Wiki 写入 403 | Wiki space 未添加 App | §二 切换路径 |
| heading 从 3 开始 | block_type 映射不同 | 用 §一 Step 2 映射表 |
| Wiki 不支持 Divider/Image | Wiki 可用 block 更少 | §二 Step 3 白名单 |
| 内容提取首尾截断 | 浏览器懒加载 | API 分页 + 滚动提取 |
