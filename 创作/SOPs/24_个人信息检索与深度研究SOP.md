# SOP-23：个人信息检索与 li-research 深度研究 SOP

> **版本**：v1.0 | **创建日期**：2026-06-10
> **来源**：教训032——形势与政策v4面试题推断事故（AI只搜memory/，漏掉直播笔记中的真实记录）
> **适用场景**：任何需要引用小黎真实经历/对话/决策的内容创作任务
> **关联技能**：personal-info-discovery、li-research、niuma-voice-dna、deep-research

---

## 〇、核心铁律

**个人经历事实错误 = A级事故（零容忍）。宁可标注"未找到记录，待确认"，也不用推断冒充事实。**

---

## 一、触发条件

以下任一条件触发本 SOP：

| 触发信号 | 示例 |
|---------|------|
| 内容需要引用"小黎做了什么/说了什么/经历了什么" | 写心得体会、嘉宾笔记、社群分享、简历优化 |
| 需要找到小黎的面试/实习/竞赛/项目经历细节 | 面试被问了什么、项目踩了什么坑、实习做了什么 |
| 任何"基于个人经历"的创作任务 | 朋友圈真实故事、公众号个人经历、简历项目描述 |
| li-research 需要研究小黎本人（而非外部信息） | 用户说"用 li-research 再迭代"且涉及个人经历 |

---

## 二、五步流程

### Step 1：个人信息发现（强制第一步）

**调用 `personal-info-discovery` Skill**，输入关键词，输出所有相关文件路径+信息密度排序。

```
mcp__skill-handler__Skill skill: "personal-info-discovery" args: "关键词"
```

**不调用 personal-info-discovery 就动笔 = 违规。**

### Step 2：按信息密度读取（L1→L5）

| 优先级 | 来源 | 信息密度 | 读取顺序 |
|--------|------|---------|---------|
| L1 | 直播笔记/原始记录/逐字稿 | ★★★★★ | 第一优先 |
| L2 | 行动手册/方法论文档/私域认知 | ★★★★ | 第二优先 |
| L3 | 课程笔记/学习记录/经验库 | ★★★ | 第三优先 |
| L4 | 概括性记忆（memory/） | ★★ | 仅作线索，不作事实来源 |
| L5 | 其他/待确认 | ★ | 标注置信度 |

**铁律**：L4 及以下的信息**不能作为事实直接写入内容**，只能作为搜索线索。

### Step 3：交叉验证

**同一件事至少找两个独立来源验证：**

```
来源A：直播笔记说"面试官问：你是研一还是研二？"
来源B：分享稿说"董事长以为我是研究生"
→ 交叉验证通过 ✅，事实可信度 = 高

来源A：直播笔记提到"EEPROM地址扫描"
来源B：memory/里没有
→ 单一来源 ⚠️，事实可信度 = 中，标注"来源：6.6直播"
```

### Step 4：置信度标注

| 置信度 | 条件 | 处理 |
|--------|------|------|
| ✅ 确定 | ≥2个独立来源一致 | 直接使用 |
| ⚠️ 推断 | 1个来源 + 合理推断 | 标注"推断"，用户确认后使用 |
| ❌ 未找到 | 搜索无结果 | 标注"未找到记录"，**禁止推断冒充** |

**内容中引用个人经历的格式：**
- 确定事实：直接叙述，不加标注
- 推断内容：括号标注"（注：此条来自XX，待确认）"
- 未找到：告知用户"没有找到XX的记录，需要你补充"

### Step 5：li-research 深度研究（需要外部信息时）

当内容不仅需要个人经历，还需要外部研究支撑时（如形势与政策需要政策原文、行业数据），调用 li-research：

```
mcp__skill-handler__Skill skill: "li-research" args: "研究主题"
```

**li-research 用于外部信息研究，personal-info-discovery 用于个人信息发现。两者互补，不替代。**

---

## 三、li-research 在本 SOP 中的职责边界

| 场景 | 用谁 | 理由 |
|------|------|------|
| "小黎面试被问了什么" | personal-info-discovery | 个人信息，内部文件 |
| "二十届四中全会公报原文" | li-research | 外部政策，需要网络搜索 |
| "FPGA建立时间违例的原理" | li-research | 外部技术知识 |
| "小黎FPGA项目踩了什么坑" | personal-info-discovery | 个人经历，RTL教训文件 |
| "形势与政策怎么写" | li-research → 公文素材库 | 方法论，可能在内部也可能需要外部 |
| "用 li-research 再迭代" | personal-info-discovery 先找个人素材 → li-research 补外部研究 | 两者串联 |

---

## 四、与 F8 技能链的集成

本 SOP 在 F8 技能链中的位置：

```
任何涉及个人经历的内容创作：
  → personal-info-discovery（Step 1，找个人素材）
  → li-research（如需外部研究）
  → li-analyze（内容诊断 Mode A 五维诊断）
  → ...（按平台技能链继续）
  → niuma-voice-dna（文风注入）
  → 自审（含事实核查）
```

**关键：personal-info-discovery 在 li-analyze 之前运行**——先有真实素材，才能做内容诊断。

## 五、违规检测信号

| 信号 | 说明 | 严重度 |
|------|------|--------|
| 写了个人经历但没调 personal-info-discovery | 可能用了推断冒充事实 | 🔴 A级 |
| 只搜了 memory/ 就开始写 | 漏掉直播笔记等高密度来源 | 🔴 A级 |
| 引用了"面试被问了XX"但来源不是直播笔记/原始记录 | 可能是AI推断 | 🔴 A级 |
| 内容中出现具体数字/问答但没有来源标注 | 无法验证 | 🟡 B级 |
| 交叉验证只找到一个来源但没标注置信度 | 单一来源应标注 | 🟡 B级 |
