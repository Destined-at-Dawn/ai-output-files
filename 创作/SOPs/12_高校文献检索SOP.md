## 三、执行步骤

### Step 1: 明确检索需求

在搜索之前，先回答三个问题：

| 问题 | 示例 | 决定什么 |
|------|------|----------|
| 我要找什么类型的文献？ | 综述 / 实验 / 理论 / 方法 | 选数据库 |
| 我的检索词是什么？ | 「power electronics fault diagnosis」 | 构造检索式 |
| 我需要多新的文献？ | 近3年 / 近5年 / 不限 | 设时间范围 |

**关键词裂变方法：**
```
核心词 → 英文同义词 → 上位词（更宽）→ 下位词（更窄）→ 相邻领域词

示例：
故障诊断 → fault diagnosis / fault detection / anomaly detection
电力电子 → power electronics / power converter / inverter
深度学习 → deep learning / CNN / transformer / neural network
```

### Step 2: 选择检索平台（决策树）

```
我要搜中文文献？
  ├─ 学位论文/硕博论文 → 知网（CNKI）
  ├─ 中文期刊 → 知网 + 万方 + 维普交叉验证
  └─ 快速定位下载源 → 百度学术

我要搜英文文献？
  ├─ 电气/电子/IC方向 → IEEE Xplore（首选）
  ├─ 综合理工科 → ScienceDirect / Springer
  ├─ 需要SCI/影响因子 → Web of Science
  └─ 不确定哪个库有 → Google Scholar / Semantic Scholar

我要用AI辅助？
  ├─ 需要AI扩展关键词 + 图谱 → Scienceing（科应）
  ├─ 需要TLDR快速预判 → Semantic Scholar
  ├─ 需要可视化文献关联 → Connected Papers
  └─ 需要自动提取论文信息 → Elicit

我要看最新未发表的？
  └─ arXiv（AI/ML/CS/物理/数学方向）
```

### Step 3: 构造检索式

**布尔逻辑检索（通用）：**
```
("power electronics" OR "power converter") AND ("fault diagnosis" OR "fault detection") AND ("deep learning" OR "CNN")
```

**高级技巧：**

| 技巧 | 说明 | 示例 |
|------|------|------|
| 精确匹配 | 双引号包裹 | `"fault diagnosis"` |
| 通配符 | `*` 匹配任意字符 | `diagnos*` 匹配 diagnosis/diagnostic |
| 同义词扩展 | OR 连接 | `"anomaly detection" OR "fault detection"` |
| 限定字段 | 指定标题/摘要/关键词 | `ti=("power electronics")` |
| 时间范围 | 设发表年份 | 2021-2026 |
| 排除干扰 | NOT 排除 | NOT "review"（排除综述） |

**Scienceing（科应）AI检索：**
- 输入自然语言描述 → AI自动扩展关键词 → 生成优化检索式
- 适合：不确定精确关键词时的探索性搜索

### Step 4: 检索结果筛选

**快速筛选四步法：**

1. **看标题** → 排除明显不相关的（10秒/篇）
2. **读摘要** → 判断是否与研究问题相关（30秒/篇）
3. **查引用** → 被引次数 > 10 的优先关注
4. **看年份** → 近3年优先，经典文献不限年份

**文献质量判断：**

| 指标 | 高质量信号 | 低质量信号 |
|------|-----------|-----------|
| 期刊 | SCI一区/二区、IEEE Trans系列 | 掠夺性期刊、无影响因子 |
| 会议 | 顶会（CVPR/ICML/AAAI） | 无peer review的会议 |
| 被引 | 高被引（领域前5%） | 零引用或自引为主 |
| 作者 | 领域知名课题组 | 无法查到的机构 |

### Step 5: 论文阅读与笔记

**三遍阅读法：**

| 轮次 | 时间 | 读什么 | 目的 |
|------|------|--------|------|
| 第一遍 | 5-10分钟 | 标题+摘要+引言最后一段+结论 | 判断值不值得精读 |
| 第二遍 | 30-60分钟 | 全文通读，标注关键图表和公式 | 理解方法论和结果 |
| 第三遍 | 1-2小时 | 逐行推导，尝试复现 | 深入理解技术细节 |

**结构化笔记模板：**
```markdown
## [论文标题]
- **作者：** xxx et al.
- **期刊/会议：** xxx, 年份
- **DOI/链接：** xxx
- **一句话总结：** 这篇论文用xx方法解决了xx问题
- **研究问题：** ...
- **核心方法：** ...
- **关键结果：** ...
- **创新点：** ...
- **局限性：** ...
- **与我的研究关联：** ...
- **可借鉴的点：** ...
- **引用格式（GB/T 7714）：** ...
```

### Step 6: 文献管理

**推荐工具：**

| 工具 | 优势 | 适用场景 |
|------|------|----------|
| Zotero（免费） | 开源、插件丰富、浏览器一键保存 | 日常文献管理首选 |
| EndNote | Word集成好、学校常提供 | 写论文时的引用管理 |
| Mendeley | PDF标注 + 社交功能 | PDF阅读+标注 |
| Notion/飞书 | 灵活、支持数据库视图 | 笔记+文献混合管理 |

**Zotero + 浏览器插件工作流：**
1. 安装Zotero Connector（浏览器插件）
2. 在任意数据库看到论文 → 点击插件 → 一键保存元数据+PDF
3. 在Word中用Zotero插件插入引用 → 自动生成参考文献列表

## 五、高校学生专属技巧

### 5.1 善用学校图书馆

- **VPN/校外访问**：多数学校提供校外VPN，放假回家也能用数据库
- **馆际互借**：学校没有的文献，可通过CALIS/CASHL馆际互借免费获取
- **学科馆员**：图书馆有专门的学科馆员，可以预约一对一检索指导
- **培训讲座**：图书馆定期举办数据库使用培训，建议新生必听

### 5.2 Scienceing（科应）特别提醒

- **试用期**：至2026年7月15日，过期需确认学校是否续订
- **校园IP注册**：首次注册需在校园网内（或VPN连接校园网）
- **核心功能优先级**：
  1. **AI检索** — 不确定关键词时，用自然语言描述让AI帮你扩展
  2. **智能图谱** — 选定一篇核心论文，展开图谱找到关联文献
  3. **知识订阅** — 设置感兴趣的关键词/期刊，自动推送新文献
  4. **AI帮读** — 英文论文读不动时，AI提炼精华要点
  5. **多篇问答** — 写综述时批量对比多篇论文

### 5.3 检索效率提升

- **先综述后论文**：先搜「review/survey/综述」类文章，快速了解领域全貌
- **滚雪球法**：从一篇核心论文的参考文献出发，顺藤摸瓜
- **反向滚雪球**：在WoS/Google Scholar查「谁引用了这篇」，追踪后续研究
- **关注课题组**：找到领域核心团队后，直接搜其近期发表

## 八、认知科学支撑（百大认知书籍）

### 文献检索的信息觅食理论

| 认知机制 | 来源 | 在文献检索中的应用 |
|---------|------|-------------------|
| **信息觅食** | 060-稀缺 §概念3 | 文献检索=认知版的"觅食行为"——在"信息景观"中搜索"信息补给"。关键词裂变（Step 1）=扩展觅食范围；数据库决策树（Step 2）=选择最优觅食路径 |
| **满足化 vs 最大化** | 010-思考快与慢 §概念3 | "满足化"策略（找到够用的就停）vs"最大化"策略（一定要找到最好的）。开题调研用最大化策略→可能陷入"永远在搜"的陷阱；日常追新用满足化策略→效率更高 |
| **注意力分配** | 020-专注力管理 §概念1 | 文献检索时的认知资源是有限的。快速筛选四步法（Step 4）=注意力量化分配——标题10秒/摘要30秒→将最多注意力留给"值得精读"的论文 |
| **确认偏误** | 010-思考快与慢 §概念5 | 检索时大脑会倾向于搜索"支持自己假设"的文献，忽略"反驳自己假设"的文献。Step 3检索式应包含"反驳词"（如NOT "effective"、AND "limitation"）→主动打破确认偏误 |
| **可得性启发** | 010-思考快与慢 §概念1 | 最先想到的关键词≠最相关的关键词。关键词裂变方法的底层逻辑是打破"可得性启发"——用同义词/上位词/下位词/相邻领域词系统性扩展，而非依赖"想到什么搜什么" |

### 三遍阅读法的认知科学

| 阅读轮次 | 认知原理 | 来源 | 科学依据 |
|---------|---------|------|---------|
| **第一遍**（5-10分钟） | 快速模式匹配 | 010-思考快与慢 §概念2 | 系统1快速扫描标题/摘要→与已有图式匹配→判断"相关/不相关"。这一遍是杏仁核快速通路驱动的，不需深度分析 |
| **第二遍**（30-60分钟） | 图式建构 | 025-认知负荷理论 §概念4 | 全文通读→构建论文内部的"知识图谱"——哪些变量相关、哪些假设支撑哪些结论。这一遍是工作记忆驱动的主动建构过程 |
| **第三遍**（1-2小时） | 精细复述 | 009-认知天性 §概念4 | 逐行推导→强制"用自己的话重新表达"→深层编码→长期记忆。这一遍的本质是"测试效应"——推导=自我测试→记忆巩固 |

### 文献管理的认知优化

| 管理策略 | 来源 | 应用 |
|---------|------|------|
| **外部化工作记忆** | 008-协同智能 §概念1 | Zotero/文献管理工具=外部化的工作记忆。大脑负责"理解与判断"，工具负责"存储与检索"→认知分工→将有限的工作记忆释放给核心思考 |
| **滚雪球的认知原理** | 037-结构洞 §概念1 | "滚雪球法"参考文献的本质是在文献引用网络中发现"结构洞"——两组引用了不同文献的研究团队之间的连接点往往是创新机会 |
| **渐进式筛选漏斗** | 025-认知负荷理论 §概念3 | 文献检索结果不可能全部精读→需要"漏斗式筛选"——100篇标题→30篇摘要→10篇全文→3篇精读。每层漏斗的筛选标准递增→认知负荷可控 |
| **时间窗口管理** | 060-稀缺 §概念1 | 高校学生的文献检索时间是稀缺资源。应利用"时间盒"策略——给检索设定明确时限（如2小时）→避免"信息贪婪"→在时间内达到最大信息收益 |
| **AI辅助的认知杠杆** | 008-协同智能 §概念2 | AI检索工具（科应/Semantic Scholar）=认知杠杆——用AI扩展关键词和筛选结果→人类负责最终判断→人机协同效率>>纯人工检索 |
