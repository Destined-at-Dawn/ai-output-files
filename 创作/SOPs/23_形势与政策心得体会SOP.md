## 〇、GATE（写前强制检查，不过不动笔）

### GATE 1：政策文件精准定位

| 检查项 | 标准 | 验证方式 |
|--------|------|---------|
| 全会名称 | 必须确认是第几届第几次（如"二十届四中全会"而非"三中全会"） | 搜索"二十届X中全会 时间 通过 审议" |
| 通过的文件 | 必须确认文件全称（如《十五五规划建议》而非《深化改革决定》） | 读公报或新华社全文 |
| 时间 | 必须确认召开日期（如2025年10月20-23日） | 公报首段 |

### GATE 2：个人信息事实核查

| 检查项 | 标准 | 验证方式 |
|--------|------|---------|
| 专业身份 | 确认当前专业（不是"想转"，是"现在在哪"） | 读 telos/identity.md |
| 经历归属 | 每条经历必须能追溯到具体文件 | personal-info-discovery skill |
| 竞赛/实习状态 | 确认是进行中还是已完成 | 读竞赛区/求职区 memory |

### GATE 3：格式规范确认

| 检查项 | 标准 |
|--------|------|
| 字数要求 | 1500字以上（实际写到1800-2200最安全） |
| 文档格式 | .docx（不是 .md、不是 .pdf） |
| 排版 | A4、宋体、正文12pt、标题18pt加粗、1.5倍行距、首行缩进2字符 |
| 署名 | 姓名+班级+学号 |

## 一、道——这类作业的本质

### 1.1 这不是论文，是"个人叙事+政策对接"

形势与政策心得体会的核心结构是：

```
政策原文引用（精确到文件名+条款号）
    ↕ 对接
个人真实经历（具体到时间、地点、数据）
    ↕ 升华
认知转变（从"跟我没关系"到"原来跟我有关"）
```

**关键区别**：
- ❌ 论文写法：大量文献综述+理论分析+数据论证
- ✅ 心得写法：以小见大——一个小细节折射大政策

### 1.2 评分者的视角

评分者（通常是辅导员或思政课老师）看什么：

| 看什么 | 怎么判断 | 你的对策 |
|--------|---------|---------|
| 政策引用是否准确 | 文件名、会议名称、时间是否正确 | GATE 1 强制核查 |
| 是否有真实个人经历 | 细节是否具体到"只有本人写得出" | 注入直播笔记/竞赛教训中的原始细节 |
| 是否结合专业 | 提到的东西是否跟你的学科相关 | 从自身专业出发选切入点 |
| 是否有认知转变 | 从不懂→理解的过程 | 写"说实话以前觉得跟我没关系" |
| 是否AI代写 | 排比句堆砌、过渡词太顺、缺乏具体细节 | 去AI味+注入真实踩坑故事 |

### 1.3 为什么不能让AI"推断"你的经历

**教训032的根因**：AI推断的面试题（"STM32高级定时器PWM互补输出+死区"）听起来专业，但跟真实被问的题（"你最大的缺点是什么"→背模板翻车）完全不同。真实经历的说服力远超"合理推断"。

> 「你只看到你想看到的（WYSIATI）。」——《思考，快与慢》010

AI推断 = WYSIATI——用已有信息编一个"合理"的故事，但这个故事不是真的。

## 二、法——标准写作流程（7步）

### Step 1：政策定位（GATE 1 展开）

1. 确认全会名称、时间、通过的文件全称
2. 搜索公报全文（至少读前三段+最后一段）
3. 搜索《建议》或《决定》全文，找到与你专业相关的条款
4. **记录8-10条带引号的原文表述**（后面写作时选用）

**搜索指令**：
```
"二十届X中全会 公报 全文"
"《文件全称》全文 新华社"
```

### Step 2：个人素材盘点（GATE 2 展开）

用 `personal-info-discovery` skill 扫描全部工作区：

1. Glob 扫描所有"面试/实习/竞赛/直播/分享"相关文件
2. 按信息密度排序：直播笔记（最高）> 分享稿 > memory（最低）
3. 提取具体细节：
   - 时间锚点（"今年3月""大一下学期"）
   - 数据锚点（"GPA 3.9""100多人""3场直播"）
   - 踩坑细节（"卡了三天""面试官沉默了"）
   - 认知转变（"以前觉得…现在发现…"）

### Step 3：选定主题和切入角度

从考试要求的选项中选择**跟你的经历关联最紧密的主题**。

**选题原则**：
- 你的经历能"以小见大"的 → 选
- 需要大量外部调研才能对接的 → 不选
- 你的专业能提供独特视角的 → 优先选

### Step 4：结构设计

**推荐结构**（来自已认可范文DNA）：

```
开篇：政策引语 + 个人认知转折（"说实话，以前觉得跟我没关系"）
    ↓
第一段：一个具体经历 + 政策对接（如：实习→新质生产力）
    ↓
第二段：另一个经历 + 政策对接（如：竞赛→科技自立自强）
    ↓
第三段：未来规划 + 政策呼应（如：五年时间线对齐）
    ↓
收束：克制收尾，不做排比升华
```

### Step 5：初稿写作

**写作铁律**：
1. 每个政策引用必须有出处（文件名+条款号）
2. 每个个人经历必须有具体细节（时间/数据/踩坑）
3. 过渡不用"正是在这个过程中""这让我深刻体会到"——用场景切换自然过渡
4. 结尾克制——"方向是清楚的，手头的事一件一件在推进"比"我将以青春之我建设青春之中国"更真实

### Step 6：去AI味检查

用 `chinese-natural-voice-revision` skill 诊断：

| 检查项 | AI味信号 | 修正方式 |
|--------|---------|---------|
| 过渡词 | "正是""这让我""在这个过程中" | 删掉，用场景直接切换 |
| 排比句 | 三句以上结构相同的句子 | 拆开，只保留最有力的一句 |
| 升华句 | "影响深远""最生动的注脚" | 改为朴素陈述 |
| 结尾 | "我将以…贡献自己的力量" | 改为"方向清楚，手头的事一件件推进" |
| 文学腔 | "理论有了重量""轮廓是具体的、可触碰的" | 改为具体场景描写 |

### Step 7：事实核查 + 格式输出

1. **逐条核查政策引用**：每条引号内容搜索确认出处
2. **逐条核查个人经历**：每条能追溯到具体文件
3. **检查专业身份**：当前专业是否写对
4. **字数统计**：确认过线
5. **生成 .docx**：A4宋体12pt、1.5倍行距、首行缩进

## 三、术——技法库

### 3.1 开篇技法：认知转折

```
❌ AI腔："党的二十届四中全会是在全面建设社会主义现代化国家关键时期召开的一次重要会议"
✅ 人话："说实话，以前看到'中国式现代化'这种表述，我的反应就是跳过——太大了，跟我有什么关系。但今年春天的几件事让我改变了看法。"
```

**来源**：辅导员李骁DNA（口语化引语）+ 已认可范文技法（情绪转折开篇）

### 3.2 政策引用技法：嵌入式引用

```
❌ AI腔："四中全会指出，要因地制宜发展新质生产力。新质生产力是……（展开定义）"
✅ 人话："全会提出'因地制宜发展新质生产力'。去了公司之后我才真正理解这句话——技术不是引进来就能用的，得在真实场景里反复调、反复改。"
```

**原则**：政策原文不超过15个字就嵌入句子，超过15个字才独立引用。

### 3.3 个人经历技法：失败叙事

```
❌ AI腔："通过这次面试，我深刻认识到了自身的不足，也更加坚定了提升自我的决心"
✅ 人话："面试官问'你最大的缺点是什么'，我说'太追求完美'——说完就知道完蛋了，对面秒懂是模板。"
```

**原则**：写失败比写成功更真实。踩坑→意识到问题→改变做法，这个弧线比"取得了优异成绩"更有说服力。

### 3.4 结尾技法：克制收束

```
❌ AI腔："作为新时代青年，我将以更加饱满的热情投身中国式现代化建设，为实现中华民族伟大复兴贡献青春力量"
✅ 人话："我是一个大一学生，能做的事情确实有限。但方向是清楚的，手头的事一件件在推进。"
```

**事实核查（🔴 不过不交）**：
- [ ] 全会名称、时间、文件全称是否正确
- [ ] 每条政策引用是否能追溯到公报/文件原文
- [ ] 专业身份是否写对（当前专业，不是下学期要转的专业）
- [ ] 每条个人经历是否能追溯到具体文件
- [ ] 面试题/竞赛细节是否来自真实记录，不是AI推断

**格式核查**：
- [ ] .docx 格式（不是 .md）
- [ ] A4宋体12pt、1.5倍行距、首行缩进
- [ ] 姓名+班级+学号
- [ ] 字数 > 1500

**风格核查**：
- [ ] 零英文引号（全部中文全角）
- [ ] 零排比句堆砌（三句以上结构相同=AI味）
- [ ] 零机械过渡词（"正是在这个过程中"=AI味）
- [ ] 结尾克制（不做排比升华）
- [ ] 有失败叙事（踩坑比成功更真实）

## 五、事故复盘：v1→v15 的教训

### 5.1 致命错误时间线

| 版本 | 致命错误 | 根因 |
|------|---------|------|
| v1-v3 | 政策引用张冠李戴（三中全会文件当四中全会用） | GATE 1 缺失——没有确认"第几次全会通过什么文件" |
| v4 | 面试题全为AI推断 | 没有调用 personal-info-discovery 扫描原始记录 |
| v5 | 专业身份搞反（写成"转入电气工程"，实际还在集成电路） | GATE 2 缺失——没有确认当前专业 |
| v6-v15 | 迭代只改表述，不改事实 | 没有做事实核查就开始迭代 |

### 5.2 为什么迭代15个版本还是错的

**根因：迭代的是"文风"而不是"事实"。**

v6→v15 的迭代方向全部是：
- 换结构（并列→时间线→递进）
- 换技法（范文DNA→已认可技法→极简版）
- 换素材密度（少→多→精）

**没有一轮做了**：
- 事实核查（政策引用是否正确）
- 身份核查（专业是否写对）
- 来源核查（面试题是否真实）

> 「规则10：经验 → 规则 → Skill。没有结晶的经验，会随上下文丢失。」——十条工程法则

这15个版本的迭代经验结晶为本SOP的GATE机制——**先确认事实，再优化表达**。

### 5.3 关键教训

| 教训 | 规则 |
|------|------|
| 政策引用必须先确认全会名称+文件全称 | GATE 1 |
| 个人经历必须从原始记录提取，不能AI推断 | GATE 2 + personal-info-discovery |
| 专业身份必须确认"现在"而非"将来" | GATE 2 |
| 迭代方向：事实>结构>文风 | Step 7 前置到 Step 1 |
| docx才是最终交付物，不是md | GATE 3 |
| 15个版本不等于15次改进——没有事实核查的迭代是无效迭代 | 工程法则8 |
