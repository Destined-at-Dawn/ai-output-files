## 二、消息路由规则

> 每次收到消息时，按以下顺序匹配：

### 2.1 输入类型优先（强制）

```
IF 消息包含链接/图片/文件/引号内容 → 输入分发SOP（路由到对应主题SOP）
ELSE → 进入主题匹配
```

### 2.2 主题匹配（10类）

```
IF 说"写/创作/灵感/公众号/小红书" → SOP-01 内容创作
IF 说"链接/文章/采集/文风/down.mptext/提取公众号/公众号链接" → SOP-02 文章采集（v2.0 API优先）
IF 说"图/封面/配图/信息图" → SOP-03 视觉素材
IF 说"PPT/答辩/汇报/演讲" → SOP-04 PPT制作
IF 说"视频/运镜/分镜/Seedance" → SOP-05 视频内容
IF 说"skill/机制/映射" → SOP-06 系统管理
IF 说"保存/整理/归档" → SOP-07 文件管理
IF 说"学习/认知/拆解" → SOP-08 知识学习
IF 说"论文/文献/arXiv" → SOP-09 科研论文
IF 说"自我介绍/规划/竞赛/面试" → SOP-10 个人发展
IF 检测到个人信息变动信号 → SOP-11 个人信息同步
IF 说"搜论文/找文献/知网/文献检索/数据库/AI检索" → SOP-12 高校文献检索
IF 说"github/仓库/release/开源/推送/同步/发布/脱敏/隐私扫描" → SOP-14 GitHub项目生命周期管理
IF 说"社群分享/在别人群分享/嘉宾分享/群内宣传/社群引流/群分享准备/社群运营" → SOP-16 社群分享
IF 说"obsidian/知识库/vault/笔记/论文笔记/实习记录/术语表/MOC/双向链接" → SOP-15 Obsidian知识库管理
IF 说"学员/教学/上课/备课/教学计划/1v1/小班/私教/教小红书/教做号/期末突击/竞赛备赛/考研备考" → SOP-17 学员教学系统（→ 路由到 17a/17b/17c）
IF 说"技能治理/注册表/引用图/配置同步/功能重叠/整合skill/合并skill/弃用skill/重构skill" → SOP-18 技能生态治理
IF 说"写脚本/开发工具/工具迭代/文件迁移/死路径/扫描器/v1/v2/v3" → SOP-20 工具开发与迭代
IF 说"上传飞书/创建飞书文档/写入飞书/知识库迁移/一比一复刻/飞书wiki" → SOP-21 飞书文档创建标准流程
IF 说"写心得/公文/形势与政策/学习心得/心得体会/考试论文/课程作业" → SOP-23 公文写作全流程（personal-info-discovery + 公文素材库 + 去AI味 + 打印版docx）
ELSE → 回退到 CLAUDE.md §十二 skill映射表
```

### 2.3 SOP-00 拦截层

> **SOP-00包裹所有SOP**。在匹配到任何SOP后，先过SOP-00清晰度检查。

- 清晰度≥6分 → 直接执行匹配的SOP
- 清晰度<6分 → 生成3个改进提示词变体，让用户选择
- **绕过条件**：含链接/图片/文件/引号文本/用户说"快"/"急"/"马上"/消息>50字且有明确动词

## 三、执行后标准收尾

每次SOP执行完毕后：

1. 记录本次执行路径（与SOP决策表对比）
2. 如有偏差，记录到SOP的迭代日志
3. 同一偏差出现3次 → 触发SOP更新
4. 更新 `memory/{YYYY-MM-DD}.md` 当日记录
