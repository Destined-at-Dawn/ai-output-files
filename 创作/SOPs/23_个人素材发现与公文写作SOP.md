# SOP-23 个人素材发现与公文写作

> **版本**：v1.0（2026-06-10）
> **优先级**：🔴 最高（含事实零容错门禁）
> **适用场景**：写心得、思想汇报、形势与政策、入党申请、实习报告、课程作业等一切需要引用个人真实经历的正式写作
> **来源**：形势与政策作业 v1→v5 迭代事故（6次推断编造 + 面试题张冠李戴 + 政策引用错误），教训编号待定

---

## ⚠️ 本SOP的诞生背景

这篇形势与政策作业经历了 v1→v2→v3→v4→v4.5→v5 共6个版本。前5版存在以下问题：

| 版本 | 致命问题 | 根因 |
|------|---------|------|
| v1 | 压根没执行F8，不调技能链，不读品牌档案 | F8铁律被跳过 |
| v2 | 面试题全是AI编的 | 从记忆推断而非原始文件 |
| v3 | 真实面试经历被当作不重要一笔带过 | 个人独特价值未识别 |
| v4 | 引用了三中全会内容冒充四中全会 | 政策引用未交叉验证 |
| v4.5 | 课外学习资料DNA零应用 | 已有素材未扫描 |
| v5 | 面试细节仍来自推断 | 没搜到6.6直播笔记原始文件 |

**本SOP的唯一目标：杜绝以上全部问题再次发生。**

## §〇 铁律层（违反任何一条 = 任务失败）

### 铁律1：事实零容错

**任何涉及小黎个人经历的细节——面试题、项目细节、课程名、时间节点、竞赛经历——必须来自已落盘的文件，不得从记忆推断。**

违反 = 事故。推断 = 编造。"我觉得应该是" = 不知道。

### 铁律2：政策引用双源验证

**政策表述必须同时满足：① 来自权威来源（新华社/人民网/共产党员网）的完整原文 ② 与考试要求的主题、全会名称、文件名称三重匹配。**

单源验证 = 可能出错（v4教训）。三中全会的内容不能安到四中全会头上。

### 铁律3：六轮修订不可跳过

**初稿完成后必须走完6轮修订。用户说"不用改了"可提前终止，但AI不得自行决定跳过任何一轮。**

## §一 Phase 0：需求消解（3分钟）

### Step 1：提取任务规格

用精确格式提取：

```
📋 任务规格
- 文档类型：[心得体会/思想汇报/入党申请/实习报告/课程作业]
- 主题：[从文件中提取]
- 字数要求：[最低字数]
- 截止日期：[YYYY-MM-DD]
- 格式要求：[字体/字号/行距/其他]
- 核心约束：[逐字提取，如"不得AI代写""结合自身实际"]
- 自选题目：[如有多选一，列出选项]
```

### Step 2：匹配DNA模板

| 文档类型 | DNA来源 | 核心技法 |
|---------|--------|---------|
| 心得体会/思想汇报 | 辅导员李骁DNA | 第一人称+具体数字+口语化引语+四字板块标题 |
| 入党申请 | 同上 + 高频政策词词库 | 个人叙事+政策对照+行动计划 |
| 实习报告 | 技术文风DNA | 工程细节+数据说话+问题→方案→结果 |
| 课程作业 | 学术文风DNA | 论点→论据→个人案例→结论 |

### Step 3：确认选题方向

- 如有多选一：基于个人经历与主题的"天然对接度"推荐（不做作 > 每个观点都有数据支撑 > 专业对口）
- 推荐理由用一句话说清（不超3句分析）
- **禁止长篇大论铺垫后才给出结论**

### Step 4：确认后立即进入Phase 1

## §二 Phase 1：个人素材发现（最关键阶段——决定文章质量上限）

> **核心原则：文章质量上限 = 素材质量上限。AI组织语言的能力人人都有，只有你的个人经历是不可替代的。**

### Step 1：搜索已有记忆（3分钟）

用 Glob/Grep/Read 在以下位置搜索关键词（如"面试""实习""竞赛""项目"）：

**搜索路径优先级**：

| 优先级 | 路径 | 信息密度 | 说明 |
|--------|------|---------|------|
| 🔴 最高 | `projects/20260425-内容创作系统/社群运营/直播笔记/` | 逐字稿级 | 直播分享的完整记录，包含最具体的经历细节 |
| 🔴 最高 | `projects/20260425-内容创作系统/社群运营/产出/嘉宾分享笔记/` | 结构化笔记 | 整理后的嘉宾/自己的分享笔记 |
| 🟡 高 | `outputs/` 目录下的分享稿 | 不同角度 | 同一经历的不同版本叙述 |
| 🟡 高 | 竞赛区/求职区/个人区的 memory/ | 概括性 | 长期记忆中的精炼版 |
| 🟢 中 | `自我进化/经验库/` | 反思性 | 成功/失败案例总结 |
| 🟢 中 | `SOPs/` | 方法论 | 可引用的做事方法 |

**🔴 关键：必须搜索全部路径后再开始写。不得在memory没找到就认为"没有素材"。**

**铁律：素材缺口 > 30%（经历少于3段+缺乏深度细节）= 向用户提3个问题。**

### Step 2：素材缺口评估

| 缺口程度 | 判断标准 | 动作 |
|---------|---------|------|
| 低（<30%） | 已有≥3段具体经历，每段有时间/地点/人物/数据 | 直接动笔 |
| 中（30-60%） | 有经历但缺数据、缺细节 | 追问用户 |
| 高（>60%） | 经历<2段，或全部经历过于笼统 | 必须追问，不得硬写 |

### Step 3：补充提问（如需要）

**提问方向**：最骄傲的事、最大挫折、与主题相关的经历、数据支撑。

**提问方式**：用 `mcp__ask-user__ask_user` 工具。

**禁止**：自行编造经历。信息不足时向用户要，不得脑补。

## §三 Phase 2：政策素材发现

### Step 1：确认权威来源

```
📋 政策确认
- 全会名称：[精确名称，如"中国共产党第二十届中央委员会第四次全体会议"]
- 召开时间：[YYYY年M月]
- 通过文件：[精确名称]
- 核心部署：[列出3-5条与主题相关的部署]
```

### Step 2：搜索补充素材

| 来源 | 搜索关键词 | 用途 |
|------|-----------|------|
| 百大认知书籍 | 主题相关关键词 | 认知框架 |
| 公文素材库 | 高频政策词、已认可范文 | DNA模板、范文技法 |
| 课程学习资料 | 公开课/时政学习 | 课堂实例 |

### Step 3：交叉验证（强制）

**每条政策表述必须满足双源验证**：

```
验证清单：
- [ ] 全会名称、时间、文件名称三重匹配
- [ ] 来自权威来源的完整原文（非记忆转述）
- [ ] 与考试要求的主题完全对应
```

**常见错误**：
- 三中全会 vs 四中全会内容混用
- "公报"和"决定/建议"内容混用
- 把前几次全会的表述安到本次全会

## §四 Phase 3：写作

### Step 1：选择文风DNA

根据文档类型加载对应DNA模板（见Phase 0 Step 2）。

**核心公式**：政策做骨架，经历做血肉。

**禁区**：
- ❌ 大段引用政策原文（政策是"骨架"，不是"血肉"）
- ❌ 堆砌形容词和排比句（典型AI腔）
- ❌ 用空泛概念代替个人经历（"让我深刻认识到"后面没有具体内容）
- ❌ 复述政策内容充字数

### Step 2：注入个人经历

**经历注入铁律**：
- 每段经历必须包含：时间+地点+具体做了什么+结果/教训+数据
- 经历之间必须有逻辑串联（不是简单罗列）
- 每段经历必须回应对应的政策表述（政策骨架+经历血肉）

### Step 3：标注信息来源

**关键细节必须标注来源文件路径**：

```
标注格式：
- [来源：直播笔记/6.6-如何AI面试-小黎.md §五次面试-第5次]
- [来源：竞赛区/rtl_code_lessons.md §EEPROM调试]
```

**AI无法找到来源的细节 = 不写入。** 宁可留空让用户补充，不得编造。

## §五 Phase 4：事实核查（强制门禁——不过不交）

> 作业可以文笔差、可以字数刚过线、可以观点浅显。但不能有任何一个事实错误。一个错误 = 整篇文章的信用归零。

### Step 1：事实核查

| 核查项 | 方法 | 判定标准 |
|--------|------|---------|
| 政策引用 | 双源验证 | 全会名称、时间、文件名称完全匹配 |
| 个人经历 | 逐条比对原始文件 | 时间线、人物、结果一致 |
| 技术细节 | 查原始记录 | 不得推断 |

### Step 2：4层自检

| 层级 | 检查内容 | 通过标准 |
|------|---------|---------|
| L1 | 硬性规则 | 零英文引号、零禁用词、字数过线、政策引用正确 |
| L2 | 风格一致性 | 像小黎写的、第一人称、有口语穿插 |
| L3 | 内容质量 | 每个论点有真实经历支撑 |
| L4 | 活人感 | 有犹豫、有踩坑、有自嘲、有不完美 |

### Step 3：信息来源审计

| 审计项 | 标准 | 不合格处置 |
|--------|------|-----------|
| 来自记忆的推断 | 0条 | 标注⚠️并替换 |
| 无来源的细节 | 0条 | 删除或向用户确认 |
| 来自原始文件的事实 | ≥80% | 最佳 |

## §六 Phase 5：六轮修订

> 来源：中国日报7个专业编辑5小时打磨400词初稿的故事。六轮独立聚焦，一轮只解决一个问题。

### 修订协议

| 轮次 | 聚焦点 | 方法 | 起点时间 |
|------|--------|------|---------|
| 第1轮 | 主题深化 | 逐段检查"与主题关联是否紧密"，删除离题内容 | 用户确认初稿后 |
| 第2轮 | 语言去AI味 | 对照AI味词表逐句扫描，替换为自然表达 | 第1轮完成后 |
| 第3轮 | 内容扩展 | 每个"概念段"补齐具体案例，添加过渡 | 第2轮完成后 |
| 第4轮 | 细节打磨 | 检查每一处经历是否与原始文件一致 | 第3轮完成后 |
| 第5轮 | 结构优化 | 检查段落逻辑，开头吸引力，结尾力量感 | 第4轮完成后 |
| 第6轮 | 事实终审 | 逐条比对政策+经历+数据，零容错 | 第5轮完成后 |

**每轮修订的文件名含版本号**：`xxx-v1.md` → `xxx-v2.md` → ... → `xxx-v6-final.md`

**用户可在任意轮次说"停"提前终止。AI不得自行跳过。**

## §七 高压场景快速通道

**触发条件**：用户消息含"半小时/马上/就要/快/急" + 截止时间<2小时。

**底线**（不可跳过）：

| 底线项 | 标准 | 耗时 |
|--------|------|------|
| 素材发现 | 必须搜索全部路径，不得只搜memory | 3分钟 |
| 事实核查 | 个人经历+政策引用双重验证 | 2分钟 |
| 六轮修订 | 精简为2轮：事实终审+语言去AI味 | 3分钟 |
| Write后Read | 强制执行 | 10秒 |
