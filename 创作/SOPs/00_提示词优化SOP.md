## 〇、Phase 0：深度需求澄清（ClarityAI 外部工具）

> **核心判断**：SOP-00 的提示词优化假设用户"想得清但说不清"——只需要帮用户把想法翻译成更精确的指令。但有一类需求是用户"自己也没想清楚"——说"我想做一个XX"但 XX 的边界、目标、约束全都没定义。这类需求不能靠改进提示词解决，需要**结构化对话**帮用户先想清楚。

### ClarityAI 是什么

ClarityAI 是一个智能需求对齐工具——通过多轮交互式对话帮助用户从模糊的初步想法逐步明确为结构化项目需求。后端 Python Flask + Qwen AI API，前端 Vue3 + Element Plus。

| 资源 | 链接 |
|------|------|
| 体验地址 | https://clarity.ycxhl.top/ |
| 前端开源 | https://github.com/YCXHL/ClarityAI-Frontend |
| 后端开源 | https://github.com/YCXHL/ClarityAI-Backend |

**核心能力**：用户输入模糊想法 → AI 生成 5-10 个针对性问题 → 用户回答（单选/填空/开放题）→ 多轮交互 → 生成结构化需求分析报告（支持 MD/PDF 导出）。

### 何时建议使用 ClarityAI

| 信号 | 说明 | 示例 |
|------|------|------|
| 用户说"我想做一个XX"但连 XX 是什么都没定义 | 需求本身未被定义 | "我想做一个自动化工具"——做什么的？输入输出？ |
| SOP-00 生成的 3 个改进版本用户都无法选择 | 用户自己也没想清楚 | 三个版本都说"不是这个意思" |
| 需求涉及跨模块/跨工作区的新系统设计 | 复杂度超出单次对话可覆盖 | "帮我设计一个全自动内容管线" |
| 用户说"你帮我设计一个XX流程/系统"但没给任何约束 | 需要先明确边界条件 | "帮我设计一个知识管理系统" |
| 需求涉及新 Skill/新 SOP 的从零创建，且用户只有方向没细节 | 需要结构化需求挖掘 | "我要一个能自动分析竞品的 skill" |

### 使用流程

```
用户提模糊需求
  ↓
SOP-00 评分 → 清晰度 < 4？
  ├── 否 → 走正常 SOP-00 三步流程
  └── 是 → 生成改进版本 + 同时判断是否为"深度澄清"场景
       ├── 不是深度场景 → 用户选一个版本，继续
       └── 是深度场景 → 建议用户先用 ClarityAI
            ↓
       用户在 ClarityAI 完成多轮对话 → 获得结构化需求文档
            ↓
       用户将结构化需求发回 → AI 直接进入对应 SOP 执行
```

### AI 建议话术

```
你的需求属于"需要先想清楚再动手"的类型。建议先用 ClarityAI 做一轮需求澄清：

1. 打开 https://clarity.ycxhl.top/
2. 输入你的想法（即使很模糊也行）
3. ClarityAI 会通过多轮问答帮你把需求结构化
4. 拿到需求报告后发给我，我直接按你的 SOP 执行

大概 5-10 分钟能完成一轮对齐。比我们来回猜你的意思高效得多。
```

### 与 SOP-00 的关系

| 场景 | 用什么 | 为什么 |
|------|--------|--------|
| 用户知道要什么但说不清 | SOP-00 三步流程 | 改进提示词即可 |
| 用户自己也不知道要什么 | ClarityAI → SOP-00 | 先想清楚 → 再优化表达 |
| 用户给了参考/链接/文件 | 直接进入对应 SOP | 有参考=需求已经足够具体 |

## 一、触发条件

| 触发信号 | 示例 | 优先级 |
|----------|------|--------|
| 模糊指令 | "帮我写点东西"/"弄个内容" | P0 |
| 不完整指令 | "写一篇文章"（没说主题/风格/长度） | P0 |
| 口语化指令 | "那个啥，帮我搞一下" | P1 |

---

## 二、清晰度评分标准（0-10分）

| 维度 | 0分 | 5分 | 10分 |
|------|-----|-----|------|
| **主题** | 完全没说 | 说了大方向 | 明确主题+角度 |
| **风格** | 没提 | 说了"公众号风格" | 给了参考文章/文风DNA |
| **长度** | 没提 | 说了"短一点" | 明确字数/段落数 |
| **受众** | 没提 | 说了"给学生看" | 明确画像+场景 |
| **输出格式** | 没提 | 说了"markdown" | 明确格式+结构要求 |

**总分 = 5项之和 / 5**（满分10分）

---

## 三、执行步骤

### Step 1: 评分
- **操作**：对用户指令进行5维度评分
  - 总分 ≥ 6 → 直接执行对应SOP
  - 总分 < 6 → 进入澄清流程

### Step 2: 绕过条件检查
- **操作**：以下条件直接绕过拦截
  - 消息包含链接/图片/文件/引号文本
  - 用户说"快"/"急"/"马上"
  - 消息 > 50字且有明确动词
  - 用户已经给了参考文章/文风DNA

### Step 3: 生成改进提示词
- **操作**：为模糊指令生成3个改进版本
  ```
  版本A（简洁版）：{主题} + {风格} + {长度}
  版本B（详细版）：{主题} + {角度} + {受众} + {风格} + {格式}
  版本C（创意版）：{主题} + {创意角度} + {参考风格}
  ```

### Step 4: 用户选择
- **操作**：
  - 展示3个版本供用户选择
  - 用户可选择任一版本或自行修改
  - 选择后执行对应SOP

---

## 四、决策表

| 清晰度分数 | 处理方式 |
|-----------|---------|
| ≥ 6分 | 直接执行对应SOP |
| 4~5分 | 生成2个改进版本供选择 |
| 0~3分 | 生成3个改进版本 + 追问确认 |
