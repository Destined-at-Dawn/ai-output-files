## §〇 写前强制GATE（🔴 所有社群分享任务必须先过，不完成不动笔）

> **铁律**：本GATE覆盖所有SOP-16场景。收到任何"社群分享"任务时，必须先完成GATE 0-6，才能动笔。跳过任何一道GATE = 违规。没有"这次很简单不用过"的例外。
> **来源**：教训026（8大错误）+ 教训029（4轮迭代9个坑 + 道法术器深挖）。核心发现：53个教训文件没阻止本轮犯错，因为教训只是软约束。GATE = 硬约束。
> **认知支撑**：「决定你成败的不是你设定的目标有多宏大，而是你日常遵循的系统有多稳固。」——《原子习惯》006

### GATE -1：跨平台适配检查（🔴 v2.2 新增 · 分享Skill类内容强制执行）

> **来源**：SkVM论文（上交大，arXiv:2604.03088）——11万Skills分析发现：15%的任务加Skill后性能下降，87%的任务至少在一个模型上无提升。根因是Skill隐含了对特定模型能力、Agent Harness、本地环境的默认假设。
> **触发条件**：社群分享内容涉及「Skill/技能/工具/安装/使用」时强制执行。纯经验分享类内容可跳过。
> **认知支撑**：「把Skill当作程序，把大模型当作处理器。」——SkVM核心类比

☐ 这次分享是否涉及推荐/演示/安装某个Skill？
  → **是** → 必须完成以下三项检查，不完成不动笔
  → **否** → 跳过本GATE，进入GATE 0

**检查1：模型能力适配**

☐ 这个Skill对模型能力有隐含假设吗？
  ☐ 需要强推理能力（多步逻辑链/工具选择判断）→ 必须标注"适合Claude 4+/GPT-5+"
  ☐ 需要工具调用 → 必须标注"需要Function Calling支持"
  ☐ 需要长上下文 → 必须标注"需要≥128K上下文"
  ☐ 仅需基础文本生成 → 标注"弱模型可用"

☐ 如果群友用弱模型（Qwen等），会出现什么问题？
  ☐ 指令过于繁复造成误导（→ 在分享中给出简化版指令）
  ☐ JSON格式输出失败（→ 提供纯文本替代方案）
  ☐ 工具调用选错命令（→ 提供手动操作步骤）

**检查2：环境依赖声明**

☐ 这个Skill依赖什么外部环境？
  ☐ Python包：______（列出，标注版本）
  ☐ CLI工具：______（列出，标注安装方式）
  ☐ 系统配置：______（列出）
  ☐ 无环境依赖

☐ 如果群友环境缺依赖，Skill会怎么死？
  ☐ 自动诊断+安装（强模型可行，但消耗额外Token）
  ☐ 直接失败（弱模型常见，→ 在分享中给出手动安装命令）

**检查3：降级策略**

☐ 如果群友的环境不完全匹配，有没有Plan B？
  ☐ 提供简化版Skill（跳过高级功能，保留基础功能）
  ☐ 提供手动替代方案（不依赖Skill的手动操作指南）
  ☐ 提供"不装Skill怎么做"的并行方案（让用不了的群友也能带走东西）

☐ **降级策略写进分享稿了吗？** 不是放在评论区，是正文里一段话——"如果你用的是XXX模型/环境，可能会遇到XXX，这时候可以XXX"。

> **为什么放在GATE 0之前**：Skill适配问题应该在决定"要不要分享这个Skill"之前就想清楚——如果发现这个Skill适配成本太高（要对3种模型3种框架做适配），可能就不适合在这个群分享，或者只推荐给特定群友。GATE -1 = 分享决策的"可行性评估"。

### GATE 0：触发确认（5秒）

☐ 这是群主分享还是嘉宾分享？
  → **群主分享**：必须过 GATE -1（如适用）+ GATE 1-6 + §三 8条铁律 + 周回顾串联（见GATE 1）
  → **嘉宾分享**：走 §四 嘉宾分享流程，GATE 1-6 按需执行

### GATE 1：角色判断（15秒）

☐ 群主分享 ≠ 嘉宾分享。以下全部不同：语气、结构、开场、互动、收尾
☐ 群主语气："我跟你们站在一起"，不是"我来教你"
☐ 群主必须做周回顾串联（如果本周有前序分享）
☐ 群主不能用"讲课感"开场，不能用"今天分享XX方法论"

### GATE 1.5：周回顾串联（30秒——群主分享强制执行）

> **来源**：教训029·v3——第一版稿子没有串联前4天嘉宾内容，用户手把手给出模板："周一建立认知→周二打破滤镜→周三提效upup→今天从学到用"。
> **为什么需要**：群主分享不是孤立事件，是本周分享链条的最后一环。串联=建立叙事弧线=群友感知"这一周干货在层层递进"。

☐ 读本周前序嘉宾分享笔记（GATE 3已读的），提取每个嘉宾的核心贡献
☐ 用"周回顾串联模板"生成过渡段落：

```
串联模板（填入即可）：
周一，[嘉宾名]帮大家[核心收获——一句话]；
周二，[嘉宾名]帮大家[核心收获——一句话]；
周三，[嘉宾名]帮大家[核心收获——一句话]；

```

☐ 确认：串联段落有没有歪曲嘉宾分享内容？（对照笔记核实）
☐ 确认：串联有没有建立"层层递进"的叙事弧线？（不是简单罗列）

### GATE 2：受众画像（30秒）

☐ 确认：**最不相关的人能从今天的分享中带走什么？**
☐ 主题必须泛化——让不求职的人也能带走东西

☐ 经验库/做得差的/（读最近5条教训，确保不重复踩坑）

### GATE 5：三版差异化标准（如需三版）

> **来源**：教训029·v3——用户要求"三版起码都是自我迭代3到5次的，每一次都要有对比表格"。
> 本GATE定义三版骨架差异化标准 + 每版自迭代流程 + 三版对比表格模板。

#### 5.1 三版骨架差异化（三版不能是同稿换语气）

| 骨架类型 | 核心结构 | 开场方式 | 适合场景 |
|---------|---------|---------|---------|
| **A 故事驱动** | 真实经历→提炼方法→桥接到群友 | 数据悬念（"一个月3个offer"） | 群友爱听故事 |
| **B 框架驱动** | 方法论先→经历佐证→反面案例 | 数据+预告结构（"今天分三幕"） | 群友想学方法 |
| **C 互动驱动** | 每段先抛悬念/投票→再给答案→互动引导 | 投票悬念（"你猜是几个？扣选项"） | 活跃群气氛 |

☐ 三版分别选了不同骨架类型？vA骨架 ≠ vB骨架 ≠ vC骨架
☐ 三版共用同一套事实数据（GATE 3.5 已校验）

#### 5.2 每版自迭代流程（≥3轮）

| 轮次 | 检查内容 | 自问 |
|------|---------|------|
| 第1轮 | 骨架完整性 | 结构完整吗？事实准确吗？核心故事到位吗？ |
| 第2轮 | 文风注入 | 有"我"的锚点吗？长短句交替吗？口语缓冲有吗？ |
| 第3轮 | 受众适配 | 不求职的人能带走什么？每个回答有桥接句吗？ |
| 第4轮 | 精简 | 每段回答≤4句话？删掉多余解释了吗？ |
| 第5轮 | 自审 | 事实/术语/emoji/消息块/周回顾/预告全部复查？ |

☐ 每版附迭代记录（改了哪几轮，每轮改了什么地方和原因）

## 二-B、嘉宾笔记飞书文档设计标准（🔴 嘉宾分享笔记制作必读）

> **来源**：2026-06-06 用户严厉批评——wiki 里的笔记格式、结构、视觉效果远超独立文档。
> **适用范围**：所有嘉宾分享笔记上传到飞书时。

### Block Type 选用

| 用途 | Block Type | 说明 |
|------|-----------|------|
| 每个核心观点 | **T13（block_type=13）** | 加粗标题 + 破折号 + 正文。这是飞书文档看起来专业的关键 |
| 章节标题 | H2（block_type=4） | 道/法/术/器/行动清单 |
| 子标题 | H3（block_type=5） | 子章节 |
| 补充说明 | TEXT（block_type=2） | 背景信息、过渡段落 |
| 资源导航 | QTCONT（block_type=34） | 工具链接、参考资源卡片 |
| 多列对比 | GRID | 工具对比、方案对比 |

### 结构模板

```
H2: 道 — 核心认知
  T13: 核心判断 — 一句话提炼 + 数据/案例支撑
  T13: 第二个判断 — 同上

H2: 法 — 方法论
  T13: 方法名 — 具体步骤
  T13: 关键公式 — 用数据说话

H2: 术 — 具体操作
  T13: 操作步骤1 — 怎么做（动词开头）
  T13: 操作步骤2 — 怎么做

H2: 器 — 工具与资源
  T13: 工具名 — 能做什么 + 一句话评价

H2: 认知武器
  T13: 金句1 — 出处/背景

H2: 行动清单
  T13: 今天做 — 具体动作
  T13: 本周做 — 具体动作
```

### 视觉规范

- **禁止多余 emoji**：只在标题层偶尔使用，正文一律不用
- **用 T13 而非 TEXT 做要点**：T13 自带有序列表格式 + 加粗标题，视觉层次远优于普通段落
- **每个观点独立 T13**：不要把 3 个观点塞进一个 TEXT block
- **Markdown 格式不进飞书**：`**bold**` 不渲染，用 API style 字段或不加
- **每个 T13 必须"深挖一层"**：不只是转录嘉宾原话，而是解释为什么成立 + 数据支撑
- **🔴 Deep-Research 强制**：嘉宾笔记整理必须走 SOP-20（Deep-Research 集成 SOP）——道层和法层的每个 T13 必须有学术/行业数据支撑，没有数据的断言 = 罗列观点 = 不及格。详见 `SOPs/20_Deep-Research集成SOP.md` §二 三步法 + §三 质量门禁

### Wiki 注意事项

- Wiki 不支持 divider（block_type=14），用 heading 替代
- Wiki 不支持 callout（block_type=19），用 QTCONT 或 TEXT 替代
- Wiki children API 不支持 CTNR/GRID，需逐个测试

## 三、自己在别人社群分享（核心流程——8条铁律）

> **血的教训**：2026-05-31 在向阳集分享犯了8个错误，被朋友逐一指出。
> 本节是那次教训的直接结晶。每条铁律后标注违反后果。

### 3.1 分享前（必须在分享开始前完成）

**铁律1：预设结构，不打无准备的仗**

分享前必须写好「分享脚本」——不是逐字稿，是结构大纲：

```
分享脚本模板：

段1：镇场（自我介绍+战绩+图片）
  - 我是谁（一句话）
  - 我做过什么（3个成果，配截图）
  - 预计时长预告（"今天聊10分钟，三块内容"）

段2：核心故事（有开头有转折有结果）
  - 故事本身（不超过10条消息）
  - 桥接句："这跟你的关系是……"

段3：方法论/干货
  - 用数字序号分段
  - 每个术语加一句解释
  - 每个方法配一个具体例子

段4：行动号召
  - "如果你想……可以来我的群"
  - 不用红包，用价值吸引

段5：收尾金句
  - 有力、简洁、让人记住你
```

> 违反后果：想到什么说什么→结构混乱→读者划走

**铁律2：提前协调分工**

分享前必须跟邀请人确认：
- 文案谁写？（你写还是对方写介绍你的文案）
- 你发什么内容？对方发什么内容？
- 节奏怎么配合？（对方介绍完你再出场，还是同时）
- 有没有时间限制？

> 违反后果：临时准备→错别字（"如何没有吉洋"）→分工不明确→结尾潦草

**铁律3：准备素材库**

分享前准备好以下素材，存在手机备忘录或收藏夹里：
- 成果截图（GitHub项目、实习数据、GPA等）
- 关键数据（"20+投递→3周2个offer"）
- 3-5张配图（项目截图、海报、数据图）
- 预设好的自我介绍段（3-5条消息）

> 违反后果：现场打字→消息拆太碎→80条信息轰炸

### 3.2 分享中（执行时必须遵守）

**铁律4：先镇场，正主必须出来**

对方介绍完你后，**立刻**发自己的战绩/成果——不是等别人问，是主动亮：
- 第1条：感谢邀请+一句话定位（"我是小黎，大一电气在读，靠AI拿到了2个实习offer"）
- 第2条：成果截图1（GitHub项目）
- 第3条：成果截图2（实习数据/Obsidian系统）
- 第4条：成果截图3（社群/开源项目）
- 间隔30秒后开始正文

> 违反后果：别人介绍完你不说话→说服力不够→白介绍

**铁律5：3-5段节奏法，不刷屏**

内容必须整理成3-5个大段，每段之间留间隔：

```
段1：镇场（3-5条消息+图片）→ 间隔30秒
段2：核心故事（不超过10条消息）→ 桥接句 → 间隔1分钟
段3：方法论/干货（数字序号分段）→ 间隔30秒
段4：行动号召（"如果你想……可以来"）→ 间隔30秒
段5：收尾金句（有力、简洁）
```

**绝对禁止**：
- ❌ 把一个故事拆成18条消息
- ❌ 10分钟内发80条消息
- ❌ 没有间隔的连续刷屏

> 违反后果：信息轰炸→读者直接划过→什么都不记住

**铁律6：术语必解释**

每个非通用术语第一次出现时，必须加一句括号解释：
- ❌ "我做了一个Harness Engineer的工作流"
- ✅ "我做了一个Harness Engineer（驾驭AI的工程师，简单说就是用AI干活的人）的工作流"

需要解释的术语清单（高频）：
- Harness Engineer、本地Agent、配置包、SOP、结构洞
- Obsidian、双向链接、MCP、Skill、知识库
- SAB分级、arXiv、预印本

> 违反后果：群里大部分人是学生→术语=黑话→看不懂→划走

**铁律7：每段故事必须桥接**

讲完自己的经历后，**必须**加一句桥接句，告诉读者"这跟你有什么关系"：
- ❌ "然后我就拿到了offer"（句号结束）
- ✅ "然后我就拿到了offer。如果你现在也在找实习，我后面会讲具体怎么用AI改简历——这个方法你今天就能用"

桥接句模板：
- "这跟你的关系是……"
- "如果你也遇到过这个问题，你可以在……"
- "这个方法你今天就能用，具体是……"

> 违反后果：读者不会自己脑补→"他讲的故事跟我没关系"→划走

### 3.3 分享后（收尾必须有力）

**铁律8：结尾要有力量，不用红包引流**

**好的结尾**：
- 一句有力的金句收尾（"强者互相帮助，弱者才相互贬低"）
- 简洁的行动号召（"想一起玩的，私聊我发'互助'"）
- 留一个开放问题引发讨论

**禁止的结尾**：
- ❌ 用红包引流（"发个红包，进群领更多"）
- ❌ 虎头蛇尾（前面讲了一大堆，最后两个词收尾）
- ❌ 前面讲"利他真诚"，后面用红包拉人——前后矛盾

> 违反后果：红包不是利他，是引流手段→破坏人设→读者感到被套路

## 四、嘉宾在自己社群分享

> 你的同辈互助群每天有嘉宾分享。你需要帮嘉宾准备好，同时引导群友互动。

### 4.1 分享前（提前1天）

**Step 1：发嘉宾须知**

给嘉宾发一份简短的须知（不是长文档，3-5条就够）：

```
嘉宾须知模板：

1. 时间：X月X日（周X）晚上 XX:00-XX:30
2. 形式：群内文字分享，不用做PPT，就当跟学弟学妹聊天
3. 建议结构：
   - 自我介绍（你是谁、做过什么）
   - 核心经验（3-5个要点）
   - 踩过的坑（最值钱的部分）
   - 互动环节（群友提问）
4. 提醒：消息不要太碎，建议3-5段大段，每段之间留30秒间隔
5. 结尾：可以推荐自己的公众号/社群，但不要硬广
```

**Step 2：提前预热**

分享当天上午在群里发预告：

```
预告模板：

今晚 XX:00 嘉宾分享 🔔

🎤 [嘉宾名] — [方向]
📌 主题：[一句话主题]
💡 你将收获：[3个要点]

有问题想问的可以提前丢群里，嘉宾会优先回答。
```

### 4.2 分享中

**Step 3：开场引荐**

嘉宾出场前，你先发一段引荐（3条消息）：
1. "今晚的嘉宾来了——"
2. 嘉宾的亮点介绍（战绩+数据）
3. "有请 [嘉宾名] 👏"

**Step 4：过程引导**

嘉宾分享过程中：
- 适时用 emoji 或短句回应（"这个太真实了""记下来了"）
- 如果嘉宾消息太碎，私聊提醒"可以合并成一段发"
- 如果冷场，主动提一个问题引导讨论

**Step 5：收尾总结**

嘉宾分享结束后，你发一段总结：
- 3个核心收获（帮群友提炼）
- 感谢嘉宾
- 预告明天的嘉宾和主题

### 4.3 分享后

**Step 6：存档**

将嘉宾分享内容整理成知识库条目：
- 存入 Obsidian Vault（如果有的话）
- 或存入飞书文档（历次分享知识库）
- 格式：日期 + 嘉宾 + 方向 + 核心要点 + 群友精华提问

**Step 7：反馈收集**

私聊嘉宾：
- 感谢参与
- 问体验如何
- 有没有推荐的朋友来分享

## 六、社群引流

### 6.1 引流漏斗设计

```
公域（朋友圈/QQ/小红书/别人社群）
  ↓ 钩子内容（故事/数据/海报）
私域主群（同辈互助群）
  ↓ 每日嘉宾分享制造价值
项目群（AI赋能实习/竞赛/考研…）
  ↓ 深度实战
付费群（未来）
```

### 6.2 引流内容原则

「把自己做轻，把阵营做大」——《063-小陶创业精华》

- 你不是"教大家用AI的老师"
- 你是"帮优秀的人互相认识的连接者"
- 嘉宾教内容，群友互相学，你做平台
- 引流话术的重点是"这里有谁"，不是"我能教你什么"

### 6.3 引流话术模板

**朋友圈版**：
- 用真实故事开头，不用"你是否知道"的营销号模板
- 用 kaomoji 和 #话题标签，保持你自己的文风
- 评论区放群二维码或"私聊我发XX"

**私聊版**：
- 一句话说明群在做什么
- 一句话说明对方为什么适合
- "你有时间来看看嘛？"

**别人社群版**：
- 先贡献价值（分享干货），再引流
- 引流只在分享结尾提一句，不要通篇推销
- 提前跟群主沟通

## 九、认知科学支撑（百大认知书籍）

| 认知机制 | 来源 | 在社群分享中的应用 |
|---------|------|-------------------|
| **WYSIATI** | 010-思考快与慢 | 你只看到自己想讲什么，没看到读者想听什么。桥接句就是对抗WYSIATI的武器——强制把"我的故事"映射到"你的利益" |
| **认知负荷** | 025-认知负荷理论 | 80条消息=认知超载。3-5段节奏法=控制工作记忆负荷在7±2个组块以内 |
| **检索式练习** | 009-认知天性 | 读者需要"这跟我有什么关系"才能把信息编码进长期记忆。没有桥接句=信息无法提取 |
| **峰终定律** | 010-思考快与慢 | 读者记住的是峰值（最精彩的故事）和结尾（收尾金句）。虎头蛇尾=破坏峰终体验 |
| **稀缺** | 060-稀缺 | 80条消息占用读者全部注意力带宽，反而什么都记不住。Less but better |
| **结构洞** | 037-结构洞 | 你的社群价值=结构洞——连接不同圈子的人。引流的本质不是"拉人"，是"当桥梁" |
| **利他互惠** | 002-自私的基因 | 真正的利他=提供价值（嘉宾分享、工具、信息差），不是发红包。互惠是长期博弈，红包是一次性交易 |
| **做题家→出题人** | 063-小陶创业精华 | 分享不是"回答问题"，是"定义问题"——你告诉群友"这个问题值得关心"，才是真正的价值 |
