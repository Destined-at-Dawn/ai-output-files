# 教训031 — HTML生成前不充分读参考文件导致质量低下

> 日期：2026-06-10
> 涉及 skill：html-to-notes
> 教训等级：严重

---

## 一、完整事件链

1. **用户需求**：将已生成的 `12篇文章信息差总结-道法术器版.md` 转为 HTML 文件
2. **第一轮尝试**：助手生成深色主题 HTML（#0f1117 背景 + 基础卡片），用户说"你让我很失望；你的一切效果远远不如参考文件"
3. **第二轮尝试**：助手读取参考文件 ~200 行 CSS，提取了颜色变量、卡片样式、Hero 结构，但只做了部分借鉴（如 Hero + Nav + 卡片），漏掉了 fade-up 动画、section-divider 分节符、comparison 对比卡片、scene-grid 等组件。用户再次纠正："我们之前不是有参考的风格吗？为什么你不用？"
4. **第三轮**：完全复制参考文件的设计语言 → 通过

---

## 二、根因分析（道法术器四层）

### 道（底层认知）
- **锚定效应**：第一个想到的方案（"深色主题适合技术内容"）被固化为锚点，第二轮只在锚点基础上修补
- **WYSIATI**「你只看到你看到的东西」——《思考快与慢》：读了 CSS 就以为读完了设计系统，实际上 CSS 只是设计系统的一个子集
- **think-before-act 跳过**：没有先做"参考品质量化分析"，直接进入了执行模式

### 法（方法论问题）
- 读参考文件时只读了前 500 行（CSS + 部分 HTML），没有看完整结构
- 没有提取"设计语言清单"作为自检依据
- 第一轮和第二轮之间没有做差异分析（"我的版本缺了什么"）

### 术（具体操作问题）
- 参考文件 903KB，只读了 200 行就停 = 看了封面就写书评
- 没有用 IntersectionObserver 的 JS 逻辑（只用 CSS 不能实现滚动触发动画）
- 自己的版本和参考版本的差距没有量化评估

### 器（工具使用）
- 没用 grep 扫描参考文件有哪些设计组件（fade-up / section-divider / comparison 等）
- 没有先建 checklist 再对照执行

## 三、正确流程（已固化到 html-to-notes skill v2.0）

### HTML 生成前参考品质量化分析（新增强制子流程）

```
Step A: 用户要求生成 HTML → 先读 1-2 个参考文件（至少读完前 1000 行）
Step B: 提取设计语言清单（11 项必查组件）:
  □ Hero 首屏（渐变背景 + 圆形光斑 + 滚动箭头）
  □ Sticky Nav（模糊效果 + 圆角按钮）
  □ section-divider（分节符 ✦）
  □ fade-up 动画（IntersectionObserver + CSS transition）
  □ comparison 对比卡片（旧 vs 新 路径）
  □ cognitive-callout（虚线边框认知卡片）
  □ scene-grid（emoji + 标题 + 描述场景卡片）
  □ pitfall-card（左侧彩条 + 大号数字）
  □ card（白色卡片 + icon 图标 + hover 浮起）
  □ tool-grid / cost-grid（工具网格 / 数据网格）
  □ quote-block（引用块）
Step C: 确认四层色系分配（道/法/术/器各有独特色系）
Step D: 开始写 HTML
Step E: 对照 Checklist 逐项确认 → 全部打勾才交付
```
