## 零、方法论前置：为什么用道法术器拆解"AI犯错"

「人们往往无法达到他们设定的目标高度，而是会跌落到他们构建的系统水平。」——《原子习惯》006

AI犯错不是"这次没做好"的问题，是**系统没有设计拦截机制**的问题。道法术器四层拆解的本质是：不在"行为层"解决问题（"下次注意"），而在**架构层**解决问题（"让系统自动拦截"）。

**四层的对应关系**：

```
道 → 为什么"记录教训"永远不够？（认知科学层面的根因）
法 → 防复发需要遵循什么原则？（系统设计法则）
术 → 具体怎么拦截？（可操作的拦截技术）
器 → 用什么工具实现？（CLAUDE.md / SOP / Skill / 脚本）
```

每一层都引用百大认知原文，不是"AI觉得应该这样"，是"认知科学已经证明应该这样"。

## 一、道层：为什么"记录教训"永远不够——四条认知铁律

### 道1：AI的"失忆结构"——每次新对话 = 新大脑

**现象**：53个教训文件躺在经验库里，但每次新对话的AI还是会踩同样的坑。

**根因**：AI没有"学习"——每次对话启动时，AI是一张白纸，它只加载 CLAUDE.md + .claude/rules/ + SOP。经验库里的教训文件**不会自动加载**。

**认知支撑**：

「人类工作记忆仅能同时处理4-7个信息组块，且只能维持数秒——这一物理瓶颈是所有学习困难的终极根因。」——《认知负荷理论》025

AI的"工作记忆"是什么？是 system prompt + CLAUDE.md + .claude/rules/。经验库里的53个教训文件 = 在AI的"工作记忆"之外 = 不会生效。

**道层判断**：**只有嵌入CLAUDE.md/SOP/Skill的规则，才是AI的"工作记忆"。写在经验库文件里的，是"长时记忆"——但AI的"长时记忆"没有自动检索机制。**

### 道2：WYSIATI——AI只能看到"当前任务"的窄管

「系统1会自动构建一个尽可能连贯的故事，但它只使用当前可用的信息。它不问'我还缺什么信息'——它直接基于已有信息跳结论。这就是WYSIATI：What You See Is All There Is.」——《思考快与慢》010

当AI收到"社群分享"这个指令时：
- System 1（快速直觉）：这是内容创作任务 → 写一篇分享稿 → 动笔
- System 2（慢速理性）应该做的：先确认角色（群主还是嘉宾？）、先读群公告、先做受众画像、先读前序分享笔记——但System 2没有被激活。

**为什么System 2没有被激活？**因为CLAUDE.md的路由表里，"社群分享"触发的是"读SOP-16"——但SOP-16没有强制写前检查步骤。AI的System 1看到"读SOP-16"，读了，觉得"我知道了"，就动笔了。它不会主动想"我还缺什么"。

「稀缺会激活大脑的生存防御回路...将注意力像激光一样锁定在'稀缺物'上...大脑的全部算力被后台常驻的稀缺焦虑进程挤占，丧失了处理'管子之外'信息的能力。」——《稀缺》060

AI的"稀缺物"是什么？是"写出分享稿"这个目标。在"要快点出稿"的压力下，AI的注意力被锁定在"写"上——受众分析、事实核对、格式确认全部在"管子"之外。

**道层判断**：**不是AI不想做写前检查，是WYSIATI + 稀缺隧道效应让AI根本没有"看到"这些检查项。唯一的解法是在AI的"默认路径"上放置强制停止点——GATE——强迫它看到。** 

### 道3：软约束 vs 硬约束——提醒自己 ≠ 改变系统

「如果你挣扎于改掉坏习惯，这并不意味着你有什么问题，而是因为你使用的'系统'出了问题。」——《原子习惯》006

**AI犯错的本质不是"忘了检查"，是"系统没有拦截"。**

| 教训类型 | 示例 | 效果 |
|---------|------|------|
| 软约束 | "预防措施：创建文件前先确认目录结构" | 依赖AI主动想起来，54%概率被跳过 |
| 硬约束 | CLAUDE.md F2：Write后必须Read验证 | 每次对话自动加载，不依赖记忆 |

**教训026（社群分享8大错误）就是软约束的反面教材**——它被写在了经验库里，但本轮对话中AI还是在犯同样的错（消息太碎、术语不解释、故事不桥接）。为什么？因为教训026的经验没有被"硬化"——它没有变成CLAUDE.md的一条铁律，也没有变成SOP-16的一个强制步骤。

「精要主义者按设计生活，而不是按默认模式被推着走。如果你不设计自己的优先级，别人会替你设计。」——《精要主义》013

**AI的"默认模式"是"读到指令→直接执行"。如果不设计强制拦截点，AI就会被默认模式推着走。**

### 道4：反脆弱——让每一次犯错都成为系统加固的燃料

「风会熄灭蜡烛，却能使火越烧越旺。你要成为火，渴望风的吹拂。」——《反脆弱》046

**蜡烛模式（当前）**：踩坑 → 记录教训 → 下次新对话 → AI没读到教训 → 再踩 → 再记录 → 循环

**火模式（目标）**：踩坑 → 分析根因 → 识别系统缺口 → 在缺口中嵌入硬约束 → 下次自动拦截 → 永远不再犯

**关键差异**：蜡烛模式在"记录"后就停了。火模式必须走到"硬约束生效"才停。

「反脆弱性则需要并利用压力、混乱、波动和不确定性来实现自我进化与变得更强。」——《反脆弱》046

**本次对话的4轮迭代、9个坑、5次用户纠正——不是失败，是反脆弱系统的"压力测试数据"。这些数据如果不用于加固系统，就是浪费；如果用于加固系统，每一次踩坑都让系统更强。**

## 二、法层：防复发的三条根本法则

### 法则1：硬约束优先法则

**原则**：任何经验，如果不能变成"下次对话自动加载的硬约束"，就等于不存在。

**判据**：
```
这条规则在下次新对话中会自动生效吗？
├── 是（在CLAUDE.md / .claude/rules/ / SOP / Skill中）→ 硬约束 ✅
└── 否（只在经验库文件中 / 只在memory中）→ 软约束 ❌
```

**为什么53个教训没阻止本轮犯错**：它们全是软约束。

**应用**：GATE体系必须嵌入SOP-16（让它变成新对话自动加载），不能只存在于经验库文件中。

### 法则2：信号→拦截映射法则

**原则**：不是"记住以后要检查X"，而是"当AI做Y时，系统自动弹出Z"。

**反模式**：
```
"预防措施：下次写分享稿前先读群公告" ← 依赖AI主动想起来
```

**正确模式**：
```
触发信号："社群分享" ← AI必然读取路由表
  → 路由表触发 SOP-16
    → SOP-16 §一 强制 GATE 0：必须先读群公告
      → 不读 = 违规 = 任务失败 ← 系统自动拦截
```

**本轮9个坑的信号→拦截映射表**：

| 坑 | 触发信号 | 拦截点 | 拦截方式 |
|---|---|---|---|
| P1 kaomoji | 写入非基础emoji | GATE 4 emoji策略 | 写前确认场景→选择emoji策略 |
| P2 不写MD | 内容长度>800字且不在MD | GATE 4 输出格式 | 先Write再聊天摘要 |
| P3 受众偏差 | 收到"社群分享"指令 | GATE 2 受众画像 | 先读群公告确认群定位 |
| P4 不串联 | 用户说"结合前面X天" | GATE 3 信息收集 | 读了笔记必须提取主题线 |
| P5 事实不准 | 写下具体数字 | GATE 3 源文件校验 | 数字必须追溯到源文件 |
| P6 三版不够差异 | 写完vA写vB时 | GATE 5 三版标准 | 每版独立骨架设计 |
| P7 角色混淆 | 收到"我的社群分享" | GATE 1 角色判断 | 群主≠嘉宾，切换语气 |
| P8 未预告直播 | 写完收尾时 | GATE 6 预告复查 | 检查稿中是否有遗漏活动 |
| P9 教训不深 | 写"预防措施"时 | 法则3（下条） | 必须追问到"触发信号+拦截点" |

### 法则3：五问深度法则（写教训时的强制追问）

**原则**：写每一条教训时，必须连续追问5次，直到触及系统层面。

**模板**：
```
发生了什么？
  ↓ 为什么会发生？
系统缺了什么？
  ↓ 为什么缺？
这个缺口在哪一层？
  ↓ 怎么补？
补上后，下次会自动拦截吗？
  ↓ 如果不会，缺什么？
还需要加固什么？
```

**反例（v1版教训）**：
```
P1：用了kaomoji → 因为没做受众判断 → 预防措施：只用基础emoji
```
只问了2层。停在了"行为层"。

**正例（v3版）**：
```
P1：用了kaomoji
  → 为什么？因为AI照搬了朋友圈文风DNA的kaomoji列表
  → 系统缺什么？缺少"场景识别→emoji策略切换"的机制
  → 在哪一层？GATE 4，输出格式确认
  → 怎么补？GATE 4 增加"确认场景→选择emoji策略"步骤
  → 会自动拦截吗？会——因为SOP-16 §一强制GATE 0-6，不完成不动笔
```

## 三、术层：具体的拦截技术

### 术1：GATE前置 —— 不读完这些文件，不允许动笔

**核心机制**：在SOP-16的触发条件中，增加"社群分享写前强制GATE"作为第一道门。AI读取SOP-16时，GATE 0-6 是第一节，必须先完成。

**完整GATE清单**（6道门，已设计，待嵌入SOP-16）：

```
GATE 0 ☐ 触发确认：这是群主分享还是嘉宾分享？
GATE 1 ☐ 角色判断：群主≠嘉宾 → 语气/结构/互动方式全部不同
GATE 2 ☐ 受众画像：群定位 + 群友构成 + 最不相关的人能带走什么
GATE 3 ☐ 信息收集：群公告 + 前序笔记 + 品牌档案 + 源文件
GATE 4 ☐ 输出格式：MD双轨 + emoji策略 + 6消息块上限
GATE 5 ☐ 三版标准：三版有独立骨架设计（如需三版）
GATE 6 ☐ 写后自审：8项复查（事实/受众/emoji/消息块/术语/桥接/周回顾/预告）
```

### 术2：角色锚定 —— 防止"用嘉宾模板写群主稿"

**触发时机**：看到"我的社群"+"我要分享"→ 立即切换为群主模式

**群主模式 vs 嘉宾模式速查**：

| 维度 | 嘉宾模式 | 群主模式 |
|------|---------|---------|
| 开头 | 群主代为介绍 | 自己出场 + 周回顾串联 |
| 语气 | "我来教大家" | "我踩过的坑你也能避" |
| 结构 | 独立主题 | 必须串联本周前序内容 |
| 互动 | 回答群友提问 | 主动发起投票/扣1扣2 |
| 收尾 | 感谢群主 + 联系方式 | 预告后续 + 收束整周 |

### 术3：受众反转 —— 防止"假设所有人都跟主题相关"

**核心问题**：每次写分享稿前，先问："群里最不相关的人是谁？他能从今天的分享中带走什么？"

**本轮案例**：
- 群里有不求职的人 → 主题从"怎么投简历"改为"AI帮你解决实际问题"
- 群里有文科生 → 每个技术案例配一个通用解读
- 群里有围观者 → 每条回答加一个"即使不求职，这个思路你也能用"

**检验标准**：把稿子里所有"求职""面试""简历"替换成"答辩""展示""报告"——如果替换后仍然成立，说明泛化成功。

### 术4：桥接强制 —— 防止"故事讲完就完了"

**规则**：每段回答的最后一句，必须包含"这跟你有什么关系"——具体到让读者知道"我今天就能用"。

**桥接句模板**：
- "这个方法你拿去准备答辩/展示/找老师谈项目都适用"
- "不求职的同学，这个思路你拿去写课程报告一样好用"
- "你今天就能做的一件事：打开XX，试一下YY"

### 术5：双轨输出 —— 防止"长篇分享稿丢在聊天里"

**规则**：
- 完整稿 → Write到 outputs/ MD文件 → Read验证落盘
- 聊天中 → 只发摘要（≤100字）+ 文件路径
- 禁止：把1700字稿子直接丢在聊天里

### 术6：经验库自我进化 —— 防止"写了教训不生效"

**规则**：每次教训写入后，必须执行：
1. 回答"这条教训对应系统的哪个环节？"
2. 回答"这个环节现在缺什么硬约束？"
3. 确认"加上硬约束后，下次同类错误是否会自动被阻止？"
4. 如果第3问答案是"不会"→ 说明硬约束不够硬，继续加固

## 七、下次对话的10题启动检查

> 如果下次对话中再次出现"社群分享"任务，AI必须在动笔前自问以下10个问题。任何一个答不出来 → 停，执行对应GATE。

```
1. ☐ 我读了群公告吗？（没有 → GATE 2停）
2. ☐ 我做了角色判断吗？群主还是嘉宾？（没有 → GATE 1停）
3. ☐ 我做了受众画像吗？最不相关的人能带走什么？（没有 → GATE 2停）
4. ☐ 我读了本周所有前序分享笔记吗？（没有 → GATE 3停）
5. ☐ 我读了所有与主题相关的源文件，做了事实校验清单吗？（没有 → GATE 3停）
6. ☐ 我确认了输出方式是MD+摘要双轨吗？（没有 → GATE 4停）
7. ☐ 我确认了emoji策略是"基础emoji、禁止kaomoji"吗？（没有 → GATE 4停）
8. ☐ 如果需要三版，三版有独立骨架设计吗？（没有 → GATE 5停）
9. ☐ 稿子里每个术语首次出现时都有解释吗？（没有 → GATE 6停）
10. ☐ 每个回答后面都有"这跟你有什么关系"的桥接句吗？（没有 → GATE 6停）
```

## 八、核心结论（一句话）

**AI的教训只有变成 CLAUDE.md / SOP / Skill 的硬约束，才有可能"永不再犯"。写在经验库文件里的教训，下一次新对话的AI不会自动读到——这不是AI的态度问题，是AI的架构特性。解法不是"写更详细的教训"，是"把拦截点嵌入系统启动时自动加载的文件"。**
