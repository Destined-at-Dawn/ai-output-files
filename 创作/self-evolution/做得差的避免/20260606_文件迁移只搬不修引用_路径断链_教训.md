# 教训：文件迁移只搬不修引用——路径断链

> 日期：2026-06-06
> 来源：根目录社群运营/ → 项目层社群运营/ 迁移
> 严重度：🔴 高（5个SOP/规则文件路径断链，影响后续AI自动读取）

---

## 现象

将根目录 `社群运营/` 的 html/、素材/、经验库/ 迁移到项目层 `projects/.../社群运营/` 后，发现 **5 个文件共 8 处硬编码路径引用** 指向了已不存在的旧路径。

| 文件 | 断链引用 | 影响 |
|------|---------|------|
| `SOPs/01_内容创作SOP.md` | 2处指向根目录约束文件 | SOP读取失败 |
| `SOPs/16_社群分享SOP.md` | 2处（约束文件+经验库路径） | 分享SOP断链 |
| `_system/rules/CLAUDE-写作规则.md` | 1处指向根目录约束文件 | 写作规则断链 |
| `公众号/自己/文风自审流程.md` | 1处 | 自审流程断链 |
| `自我进化/INDEX.md` | 1处 | 经验库索引断链 |

## 根因

**搬文件前只检查了 skill 层有没有硬编码路径，没检查 SOP 层。**

- Skill 层（opc-private-domain、siyuliebian）确实没有硬编码路径 → 不受影响
- 但 SOP 层的 5 个文件有硬编码路径 → 断链

教训：**迁移前的影响面扫描必须覆盖所有层：skill → SOP → rules → CLAUDE → INDEX，不能只查一层就放心。**

## 为什么看起来行其实不行

迁移执行很顺利（Python shutil 一次成功），文件都在正确位置。但"文件在" ≠"系统能找到"——AI 读 SOP 时是按路径找文件的，路径断了就等于文件不存在。

## 正确做法

1. **迁移前：全量 grep 扫描**——搜旧路径在所有 .md/.yaml/.json 文件中的引用次数
2. **迁移后：立即修正引用**——不要"以后再改"，因为"以后"就是下次对话 AI 找不到文件的时候
3. **验证：grep 旧路径确认零引用**——搜旧路径应返回 0 条结果
4. **经验库迁移也要双写验证**——模块级经验库迁移后，根级 self-evolution/ 的引用也要同步更新

## 触发的规则

- 工程法则 5（跨边界校验）：文件搬了但引用没搬，两个系统接缝处断裂
- F2（文件操作铁律）：Write 后必须 Read 验证——迁移后必须 grep 验证引用完整

## 影响范围

- 所有文件迁移/重命名操作
- 经验库/约束文件的跨目录移动
- 任何改变文件物理位置的操作
