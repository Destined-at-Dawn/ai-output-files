# 20260605_二维码区误改成图标卡片_失败教训

## 用户原反馈

> 我不要这样的动画，很不好看；以及我原本的二维码很好看，你现在这样很不好看。

## 错误原因

我把问题误判成“二维码区需要整体专业化重画”，于是建议用线性图标、占位二维码、产品卡片式布局替换原本二维码区。

实际问题只是：二维码旁边的 emoji 和可爱装饰违和。

## 正确做法

- 保留原本二维码样式、颜色、大小、位置和卡片结构
- 保留原本人物半写实校园头像风格
- 只删除二维码旁边的违和 emoji / 火苗 / 星星 / 喇叭 / 爱心等装饰
- 不新增图标，不重画二维码，不把底部改成产品 UI

## 可复用规则

当用户说“某个局部装饰违和”时，优先做局部减法，不要扩大成整区重设计。

二维码是转化入口，如果原样式好看，绝不能随意替换成占位符或图标卡片。
