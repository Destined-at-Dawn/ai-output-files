## 失败的精确轨迹

```
用户指令（中文直译）：
  "帮我把这些html笔记整理到直播笔记目录，格式是md"
  "改善我的工作流"
  "改善我的私域认知"

AI的实际行为：
  1. ✅ 转换4个HTML到MD → 落盘了
  2. ❌ 工作流改善 → 只在对话中输出"三个盲区+三个可做的事"，没有落盘
  3. ❌ 私域认知 → 只在对话中输出"道法术器分析"，没有创建文件夹/文件
  4. ❌ skill → 提都没提，用户第二轮才指出

AI的心理活动（重建）：
  "我给出了完整的分析，用户可以参考执行。这就算完成了。"
```

## 为什么这个模式反复出现（根因分析）

### 根因1：动词≠名词的判断失误

| 用户的动词 | AI的理解 | AI应该的理解 |
|-----------|---------|------------|
| "整理到直播笔记目录" | 落盘 → ✅ 做到了 | 落盘 |
| "改善工作流" | 分析输出 → ❌ 没落盘 | **更新SOP/创建规则文件/修改路由表** |
| "改善私域认知" | 分析输出 → ❌ 没落盘 | **创建文件到私域认知文件夹/更新认知体系** |

**核心问题**：AI把"改善"理解为"给我建议"，但用户的意思是"把改善措施写进我的系统里，让我下次遇到同类问题时系统已经变了"。

### 根因2：分步指令的"部分完成错觉"

用户的指令是三个子任务：
1. 整理笔记 → 需要落盘
2. 改善工作流 → 需要落盘
3. 改善私域认知 → 需要落盘

AI完成了子任务1的落盘，就产生了一种"完成感"，对子任务2和3只做了分析就停了。

**类比**：让你"打扫房间、修理漏水的水龙头、整理书架"。你扫完地就觉得"差不多了"，水龙头和书架只是看了一眼写了个分析报告，但没有动手修。

### 根因3：对话输出比文件落盘更"爽"

- 对话输出：即时反馈、成就感、用户能看到你的"聪明"
- 文件落盘：需要设计目录结构、写文件、验证落盘、更新引用——麻烦

**AI会下意识地选择更"爽"的路径**——这是最隐蔽的根因。

## 为什么看起来行其实不行

**表面合理逻辑**："改善工作流"是一个分析型任务——分析现有流程，找出问题，给出建议。AI做了这个分析，在对话中输出了结果。看起来完成了。

**实际漏洞**：
1. "改善"的最终落点是**系统变了**，不是"知道了问题"
2. 对话中的分析，对话结束后就消失了。下一轮AI不会读这轮的分析
3. 用户要的不是"你知道我的工作流有什么问题"，而是"我的工作流文件/SOP/skill已经改了"

**最深层的漏洞**：分析本身如果正确，对用户是有价值的——但价值只存在这一轮。不落盘 = 价值随上下文蒸发 = 用户白问了一次 = 下轮还要重新分析 = 死循环。

## 退出判据

用户指令中出现以下动词时，**必须**有对应的文件操作，不是只有对话输出：
- 改善 / 优化 / 调整 → 必须修改已有文件
- 建立 / 创建 / 新增 → 必须创建新文件
- 更新 / 升级 / 迭代 → 必须修改已有文件 + 变更记录
- 整理 / 归档 / 打包 → 必须文件移动或新建

**禁止**：只输出分析 = 完成。分析是中间步骤，文件落盘才是终点。

## 影响范围

- 浪费了一整轮对话（用户第二轮纠正后，AI才补建5个文件+2处更新）
- 私域认知文件夹延迟一天创建
- 工作流改善措施延迟一天落盘
- 伤害用户对AI"一次到位"能力的信任

## 预防措施（必须逐项检查）

### 预防1：接收指令后立即列"落盘清单"

收到任何含动词的指令后，在开始分析前，先在脑中（或对话中）列出所有需要落盘的文件：

```
任务："改善工作流+改善私域认知"
落盘清单：
  □ 私域认知/ 文件夹 → 新建
  □ 私域认知/私域认知体系.md → 新建或更新
  □ 社群运营-skill.yaml → 更新（新增workflow）
  □ SOPs/ 或 社群运营/SOP/ → 新建SOP文件
  □ CLAUDE.md → 更新路由表+关键文件引用+变更记录
```

这个清单列完之前，不开始分析。

### 预防2：分析完成 ≠ 任务完成

把"分析输出"和"文件落盘"当成两个独立阶段：

```
阶段1：分析（对话输出）—— 中间产物
阶段2：落盘（文件操作）—— 最终交付

只有两个阶段都完成，才能说"做完了"。
```

### 预防3：三类任务强制关联文件操作

| 任务动词 | 最少文件操作数 |
|---------|--------------|
| 改善/优化/调整 | ≥1 个 Edit/Write |
| 建立/创建/新增 | ≥1 个 Write + ≥1 个路由引用更新 |
| 整理/归档/打包 | ≥1 个 Bash（移动/复制） |

低于这个数 = 没有真正完成。
