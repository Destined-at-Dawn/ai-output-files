# 失败教训：社群分享稿5大踩坑——本轮对话逐坑纠正

## 日期
2026-06-06

## 场景
小黎在自己的「同辈互助群」做第六天分享，主题为"技术岗实习与AI实战"。AI助手生成了三版分享稿，小黎逐轮纠正，最终产出终版。过程中暴露了5个大小坑，全部在本轮对话中被用户指出并修正。

## 踩坑清单

### 坑1：使用奇怪emoji不符合社群受众
**表现**：第一版用了大量日式kaomoji——(๑•̀ㅂ•́)و✧、ฅ՞•ﻌ•՞ฅ、₍ᐢ..ᐢ₎、( ˚ཫ˚ )、(⸝⸝•‧̫•⸝⸝) 等
**用户批评**："不能有那种非常奇怪的emoji"
**根因**：没有做受众判断。群的受众以理工科学生为主，日式kaomoji显得轻佻不专业。小黎自己朋友圈可以用，但社群分享是"半正式场合"，需要收敛。
**正确做法**：只用基础emoji（数字序号、箭头、分隔符），不用装饰性kaomoji。

### 坑2：初版未落盘MD文件，直接在聊天输出
**表现**：第一响应把三版分享稿（数千字）全部写在聊天中，没有保存为MD文件
**用户批评**："稿子应该给我写进MD文件"
**根因**：把分享稿当成"临时聊天内容"处理，但分享稿是正式交付物——用户需要拿MD文件去复制粘贴到微信、需要存档、需要后续修改。
**正确做法**：先写到 outputs/ 目录的MD文件，再在聊天中摘要引用。Write后Read验证落盘。

### 坑3：受众定位偏差——假设群友都是求职者
**表现**：第一版默认所有人都在找技术岗实习，三个问题全部围绕"怎么投简历/怎么面试/怎么写简历"
**用户批评**："群里面也不是全部都是要投岗位的"
**根因**：没有做受众分析。群的定位是AI学习互助群，求职只是其中一个场景。分享主题应该是"AI帮你解决实际问题"，技术岗实习只是载体，不是全部。
**正确做法**：先确认受众画像——群里有大学生也有已工作的、有理工科也有文科、有求职需求也有纯学习需求。主题要泛化到"AI实战"，让不求职的人也能带走东西。

### 坑4：未结合前4天嘉宾分享做铺垫
**表现**：第一版直接进入三个问题，没有串联周一至周四的内容
**用户批评**：给了明确的串联模板——"周一建立认知→周二打破滤镜→周三提效→今天实战"
**根因**：忽略了社群分享的"连续性"。群公告已经明确了周一到周五的分享节奏，每天的内容应该承前启后。第六天的分享是整周的高潮收尾，必须回顾前四天做了什么。
**正确做法**：写稿前先读群公告确认本周分享节奏，开头用一段"周回顾"串联前四天，再引出今天的主题。

### 坑5：事实校验不够严格（写前未读全源文件）
**表现**：第一版有些细节可能不够准确，用户朋友之前曾指出过"与事实不符"的问题
**用户批评**：要求"特别尤其不能出现与事实不符的"，并让助手去读求职区和竞赛区的源文件
**根因**：写稿前凭记忆而非读源文件。正确的做法是先完整读取求职区所有经历文件、面试技巧文件、offer详细记录，交叉核对每一个数字和说法。
**正确做法**：写分享稿前，先做"事实校验清单"——打开所有相关源文件，逐条确认每个数字、经历、公司名的准确性。写完后再次逐条复查。

## 本轮用户纠正原话（关键摘录）

1. "不能有那种非常奇怪的emoji"
2. "你要体现出我的分享的独特之处"
3. "对应的稿子应该给我写进MD文件"
4. "群里面也不是全部都是要投岗位的"
5. "特别尤其不能出现，与事实不符的"
6. "你应该把这次对话所有采过的坑更新进去"

## 根因分析

**最深层根因：写分享稿前没有完整的"写前检查清单"。**

具体来说缺了以下步骤：
1. ❌ 没有读群公告确认本周分享节奏（跳过→导致坑4）
2. ❌ 没有做受众画像分析（跳过→导致坑3）
3. ❌ 没有先读求职区所有源文件再做交叉核对（跳过→导致坑5）
4. ❌ 没有确认输出方式是"聊天摘要+MD文件"双轨（跳过→导致坑2）
5. ❌ 没有确认emoji使用规范（跳过→导致坑1）

## 正确流程（社群分享稿写作前检查清单）

```
☐ 1. 读群公告 → 确认本周分享主题和节奏
☐ 2. 读前4天嘉宾分享笔记 → 提取可串联的内容和风格参考
☐ 3. 读受众画像 → 群成员构成（年级/专业/求职状态）
☐ 4. 读求职区/竞赛区所有源文件 → 做事实校验清单
☐ 5. 确认输出方式 → MD文件 + 聊天摘要双轨
☐ 6. 确认emoji规范 → 只用基础emoji，不用装饰性kaomoji
☐ 7. 写完后逐条复查事实准确性
```
