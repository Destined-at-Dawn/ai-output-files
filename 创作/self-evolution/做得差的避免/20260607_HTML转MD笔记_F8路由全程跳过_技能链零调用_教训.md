# 教训：HTML转MD笔记——F8路由全程跳过，技能链零调用

## 日期
2026-06-07

## 触发场景
用户连续给三个HTML文件要求整理成MD笔记：
1. `第三次直播——如何AI面试-笔记初稿.html` → "总结是md文件"
2. `5.30直播-简历与AI求职方法论.html` → "帮我继续整理成md笔记"
3. `23日分享会笔记.html` → 同上

AI三次全部直接读HTML→手动写MD，**全程零技能调用**。

## 踩坑清单（7个坑，3次重复）

### 坑1：F8路由表未触发

**现象**：收到"总结是md文件"时，AI没有识别为"内容创作"任务，直接当成了"格式转换"任务。

**根因**：AI看到"总结"和"md文件"，把任务分类为"技术操作"（HTML→MD），而不是"内容创作"（整理嘉宾笔记）。

**正确判断**：
- "HTML→MD" 是**格式**，不是**任务本质**
- 任务本质是"整理直播分享笔记" = 内容创作 = 必须走F8
- 路由表明确写了：`朋友圈/公众号/小红书/视频/海报/群公告` → F8。笔记整理虽然不在这个列表里，但它是内容产出，F8的精神是"有SOP/品牌档案/文风DNA就必须先读"

### 坑2：li-content五维诊断未调用

**现象**：F8第二步强制要求调用 `li-content` 做五维诊断（文字洁癖/封面标题/表达效率/认知落差/AI辅助），三次均未执行。

**根因**：坑1的连锁反应——没走F8 → 不知道要调li-content。

**影响**：产出的笔记没有经过质量诊断，可能有文字洁癖问题（AI腔）、表达效率低（冗余内容）、认知落差未消除（术语未解释）。

### 坑3：niuma-voice-dna未注入

**现象**：笔记风格偏"AI课堂总结"，不像"小黎自己听完后写的笔记"。

**根因**：三次都没调 `niuma-voice-dna`，也没读文风DNA速查表。

**对比**：已有的笔记整理铁律.md第四条明确规定——"笔记优化后的产出，必须调用 niuma-voice-dna 注入文风"。这条规则存在但没被读到。

### 坑4：笔记整理铁律6条全部未读

**现象**：`私域认知/笔记整理铁律.md` 专门针对这个场景写了6条铁律+9项自审checklist，AI一条都没执行。

**根因**：AI没有把"HTML转MD"关联到"笔记整理"→ 没读这个文件。

**这是最严重的坑**：规则文件已经存在（之前踩坑后建立的），但新对话的AI不知道要去读它。说明路由表缺少明确的映射。

### 坑5：已有笔记格式未参考

**现象**：AI直接凭感觉写格式，没有先读已有的好笔记（如 `5.30-简历与AI求职方法论-小黎.md`、`5.23-AI赋能实习求职-小黎.md`）来对齐格式。

**根因**：没有"先读已有产出再写新产出"的习惯。

### 坑6：SOP总索引未读

**现象**：F5铁律要求"写作前必须读SOPs/SOP总索引.md → 匹配指令 → 读对应SOP"。三次均未执行。

**根因**：同坑1——任务被分类为"技术操作"而非"写作任务"。

### 坑7：经验库更新需要用户提醒

**现象**：三份笔记写完后，AI没有主动更新经验库。用户明确提醒"你应该把这次对话所有采过的坑更新进去"才开始做。

**根因**：同20260607_经验库被动更新教训——补正动作占用全部注意力，经验沉淀被遗漏。

**这是同类错误第2次**（上一次是2026-06-07经验库被动更新教训），升级为系统性问题。

---

## 根因分析：三层根因

### 表层：任务分类错误
"HTML转MD"被当成技术操作，不是内容创作。

### 中层：路由表缺少"笔记整理"入口
路由表有"朋友圈/公众号/小红书/视频/海报/群公告"的映射，但没有"嘉宾笔记/直播笔记/参考材料整理"的明确入口。已有的 `笔记整理铁律.md` 没有被路由表引用。

### 深层：每次新对话 = 新AI
上次踩坑后建立的规则（笔记整理铁律6条）存在，但新对话的AI没有被引导去读它。规则存在 ≠ 规则生效。规则必须嵌入到路由表（每次对话必读）才能生效。

## 系统加固动作（硬约束变更）

### 加固1：路由表新增一行

**文件**：`CLAUDE.md` 路由表
**新增**：

```
| 嘉宾笔记/直播笔记/分享笔记/HTML转MD/笔记整理/参考材料 | `私域认知/笔记整理铁律.md` + `社群运营/SOP/嘉宾分享笔记整理-SOP.md` + F8路由 → li-content → niuma-voice-dna → 自审 | 笔记整理6条铁律+9项自审 |
```

**验证**：变更后，AI收到"整理笔记/HTML转MD"类需求时，会自动关联到笔记整理铁律。

### 加固2：笔记整理铁律增加"识别信号"

**文件**：`私域认知/笔记整理铁律.md`
**新增内容**（在铁律1后追加）：

```markdown
## 识别信号（自动触发本文件）

以下任何关键词出现 → 本文件生效：
- HTML转MD / 总结是md / 整理成笔记
- 嘉宾笔记 / 直播笔记 / 分享笔记
- 参考材料 / 朋友笔记 / 参考文章
- 笔记优化 / 笔记整理 / 内容总结

**关键判断**：任务表面是"格式转换"（HTML→MD），实质是"内容创作"（整理分享笔记）。格式转换只是第一步，后续的道法术器蒸馏、文风注入、质量诊断才是核心。
```

### 加固3：经验库更新纳入"每份产出完成后"自动检查

**文件**：本教训 + `防错机制-再也不会犯错.md`（规则3追加）

```
每份内容产出完成后 → 自动执行：
□ 经验库/做得差的/ 已检查是否有新坑
□ 经验库/做得好的/ 已检查是否有成功经验
□ 经验库 INDEX 已更新
□ 私域认知/ 已检查是否有可沉淀的认知
```

### 加固4：步骤1（识别任务类型）的决策树扩展

**嵌入位置**：F8路由铁律的决策逻辑

```
收到任何"产出文字"类需求 →
  1. 表面是什么？（格式转换/数据整理/代码生成...）
  2. 实质是什么？（内容创作/知识整理/信息蒸馏...）
  3. 实质是内容创作 → 走F8
  4. 有对应的笔记整理铁律/SOP → 先读
```

## 退出判据

**如何判断"这次教训是否真正被吸收"**：

1. 下次收到"HTML转MD/整理笔记"类需求时，AI是否自动走F8路由？
2. 走F8后是否自动调用 li-content + niuma-voice-dna？
3. 写完笔记后是否主动更新经验库？
4. 三个都"是" = 教训被吸收。任何一个是"否" = 拦截器没生效。
