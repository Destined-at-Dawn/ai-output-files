# 教训：分析深度不足——"再也不会犯错" vs "记录了什么错"

## 日期
2026-06-07

## 触发轮次
用户第五轮说："你总结的还不够，我要的是再也不会犯错"

## 用户原话的精确含义

"再也不会犯错" ≠ "记录清楚这次犯的错"

**用户要的是**：
- 这次踩的坑 → 变成规则 → 变成系统 → 未来自动拦截
- 不只是"我知道这个坑了"，而是"这个坑在我未来的工作流中自动被排除了"

**AI给的是**：
- 描述了踩了什么坑
- 写了教训文件
- 但没有触及"为什么这个坑会反复出现，怎么让系统自动拦截"

## 失败现象

上一轮AI写的经验库文件（20260606_分析输出只在对话没落盘、20260606_SOP放错目录和笔记混在一起等）虽然格式完整、有根因分析、有预防措施，但**用户仍然说"总结得还不够"**。

问题出在哪里？

### 对比：记录型总结 vs 预防型总结

**记录型总结（AI做的）**：
```
踩了什么坑 → 为什么踩 → 怎么避免 → 预防措施
```
这是"做一个好的记录"，但它依赖于**下一轮的AI主动读这个记录**。如果下一轮没读，或者读了但不执行——还是会踩。

**预防型总结（用户要的）**：
```
踩了什么坑 → 为什么反复踩 → 系统的哪个环节缺少拦截 → 在拦截点加固
```
这是"改变系统，让这个坑自动被拦截"，不依赖下一轮AI的记忆。

## 根因分析：为什么AI的分析总是停留在"记录型"

### 根因1：AI的认知模型是"这次→下次"

AI认为：这次记录了 → 下次读到 → 下次避免。这是一个**知识传递模型**。

但现实是：每次新对话的AI ≠ 上次对话的AI。依赖"下次读到"不可靠。

用户知道这一点——所以他要求"再也不会犯错"——意思是**建立不依赖记忆的机制**。

### 根因2：AI不区分"软约束"和"硬约束"

| 类型 | 示例 | 效果 |
|------|------|------|
| 软约束 | "记住以后要检查"、"经验库里有记录" | 依赖AI主动想起来，不可靠 |
| 硬约束 | CLAUDE.md 铁律、路由表、Skill 规则 | 每次对话自动加载，可靠 |

AI倾向于写出软约束（"预防措施：创建文件前先确认目录结构"），但这句话只在经验库文件里，没有变成硬约束。

**硬约束应该是**：在 CLAUDE.md 的"文件操作铁律"里加一条"新建文件前必须先确认目录结构（使用 Bash ls 查看父目录）"——这样每次对话自动加载，不需要AI"想起来"。

### 根因3：AI的"预防措施"往往是"提醒自己"，不是"改变系统"

看一个典型预防措施：
> "创建文件前先问：这个文件的性质是什么？→ SOP/约束类 → 专用目录"

这是一条**checklist 条目**，不是**系统变更**。

系统变更应该是：
1. 在社群运营-skill.yaml 的"创建新文件"workflow 里强制加上"先 ls 确认父目录"
2. 在 CLAUDE.md 路由表里加一条"新建SOP/约束类文件 → 先确认目录结构"
3. 写一个自动化检查脚本，检测新增文件是否在正确目录

用户要的是系统变更，AI给的是checklist提醒。

## 退出判据

**写每一份教训时，必须回答三个问题**：

1. **这条教训对应系统的哪个环节？**（路由表？Skill？SOP？CLAUDE.md？操作规则？）
2. **这个环节现在缺什么拦截？**（少了一条规则？少了一个检查步骤？少了一个自动化验证？）
3. **加上拦截后，下次同类错误是否会自动被阻止？**（不会 → 拦截不够硬，继续加固）

三个问题答不出来 = 教训写得太浅。

## 正确做法模板

教训文件不是写"以后记住要XXX"，而是写：

```
## 系统加固动作

本次教训触发以下**硬约束变更**：

1. [文件路径] 第X行 — 新增规则："XXX"
2. [Skill名称] workflow X — 新增检查步骤
3. [脚本路径] — 新增自动化验证

验证：变更后，如果AI再次尝试[错误行为]，会在[拦截点]被阻止。
```

## 关联法则

- 法则 10（规则结晶）：经验 → 规则 → Skill。经验库里的"预防措施"如果没有变成规则/Skill，结晶就没有完成
- 法则 8（门禁文化）："写完了经验库"不是门禁，"系统的拦截点生效了"才是门禁
- CLAUDE.md F6：教训必须回写 Skill/SOP——不只是"在经验库里记录"，而是"修改 Skill/SOP 本身"

## 关联认知书籍

- 「系统胜于目标。如果你有一个好的系统，你不需要每次都靠意志力。」——《原子习惯》——用户的系统（CLAUDE.md + 路由表 + Skill）应该自动拦截错误，不靠AI"想起来"
- 「不要用制造问题的思维方式去解决问题。」——AI用"写更详细的提醒"来解决"提醒被忽略"的问题，这是同一层思维。应该上升一层：不要提醒，直接拦截。
