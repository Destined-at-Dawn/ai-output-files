# 教训：经验库被动更新——用户不提醒就不做

## 日期
2026-06-07

## 触发轮次
用户第四轮说："你应该把这次对话所有采过的坑更新进去（经验库）"

## 失败现象

AI在第二轮被纠正（"没建skill和私域认知文件夹"）后，做了：
1. ✅ 补建私域认知文件夹
2. ✅ 补建嘉宾分享笔记整理SOP
3. ✅ 更新社群运营skill
4. ✅ 更新CLAUDE.md路由表

但**没有主动更新经验库**。直到用户第四轮明确提醒"把坑更新进经验库"才做。

## 为什么这个遗漏很严重

经验库（做得差的/ + 做得好的/）是整个社群运营系统的**免疫系统**：
- 它的作用是**让同类错误不再发生**
- 如果踩坑后不更新经验库 = 免疫系统失效
- 免疫系统失效 = 同样的病会再得

**类比**：你感冒了，吃了药，好了。但你没有记录"这次感冒是因为淋雨"。下次淋雨，你还会感冒。经验库就是那个"注意：淋雨导致感冒"的记录。

## 为什么看起来行其实不行

**表面合理逻辑**：纠正已经完成了——文件补建了，目录调了，引用更新了。问题解决了。经验库是"锦上添花"。

**实际漏洞**：
1. 纠正动作 ≠ 经验沉淀。纠正只解决"这次"的问题，经验解决"永远"的问题
2. 不写进经验库 = 下一轮AI不知道这次踩了什么坑 = 可能再踩
3. "用户提醒才做"本身就是一个需要记录的经验——说明缺乏主动性

**最深层的漏洞**：AI把"被用户纠正"当成一个**事件**，而不是一个**信号**。事件过去了就过去了。信号需要被分析、记录、传播。经验库就是信号传播的渠道。

## 根因分析

### 根因1：纠正动作占用全部注意力

当用户说"你本应该建立skill和私域认知文件夹"时，AI的全部注意力在"补建这些文件"上。补建完=松了一口气=自然想结束这个任务。经验库更新不在当前注意力范围内。

### 根因2：经验库更新没有自动化触发

目前经验库更新是**手动触发**的——需要AI主动想起来"哦对，应该更新经验库"。没有强制机制。

### 根因3：经验库更新的"延迟惩罚"不可见

- 不更新经验库 → 下一次同类错误 → 用户再次纠正 → 再次浪费时间
- 但这个链条太长，AI感受不到"不更新"的即时后果
- 相比之下，不补建文件的后果是用户马上指出——可见、可感受

## 退出判据

**每次被用户纠正后**，必须输出以下格式：

```
🔴 纠正归档检查

□ 补正动作完成
□ 经验库/做得差的/ 已更新
□ 经验库 INDEX 已更新（如有）
□ dual-write-verify.py 已运行
□ self-evolution/lessons.md 已更新

全部打勾 = 纠正闭环完成
```

**禁止**：被纠正后做了补正动作就停——补正只是第一步，经验沉淀是第二步。

## 预防措施

### 预防1：教训沉淀纳入"每轮对话结束前强制检查"

在每轮对话结束前，对照 checklist 检查：
- [ ] 本轮有被用户纠正吗？→ 有 → 写经验库
- [ ] 本轮有踩坑吗？→ 有 → 写经验库
- [ ] 本轮有做得好的吗？→ 有 → 写经验库

### 预防2：制定 MINIMUM 经验库更新触发器

| 触发器 | 最少写入 |
|--------|---------|
| 用户纠正（任何形式） | ≥1 个做得差的/ 文件 |
| 方案被验证为死路 | ≥1 个做得差的/ 文件 |
| 一次到位完成复杂任务 | ≥1 个做得好的/ 文件 |
| 用户说"下次别这样" | ≥1 个做得差的/ 文件 + 预防措施 |

### 预防3：经验库更新不求"完美"，先求"存在"

不需要写成教科书。可以先写50字的速记，后续再细化。关键是**有** vs **无**。

## 关联法则

- 法则 7（负结果是一等公民）：不记录 = 白踩坑
- 法则 10（规则结晶）：经验必须固化，否则蒸发
- CLAUDE.md § 升级规则：第1次记录 → 第2次高频坑 → 第3次铁律
- 如果不写经验库，这个升级链条就断了

## 关联认知书籍

- 「知识的诅咒：一旦我们知道了某件事，我们就很难想象不知道它是什么样子。」——《让创意更有黏性》——AI"知道"这个坑了，但下一轮对话的AI不知道。经验库是打破知识诅咒的工具。
- 「复利是世界第八大奇迹。」——经验库的每次更新都是复利的一笔存入。不更新 = 复利中断。
