# 教训 E36：嘉宾笔记只输出到对话，没有落盘飞书文档

## 日期
2026-06-06

## 场景
优化完三份嘉宾分享笔记后，只在对话中输出了结果，没有创建飞书文档。

## 做了什么（错误行为）
1. 优化后的笔记直接输出到聊天窗口
2. 笔记存在本地 MD 文件
3. **没有创建飞书文档**
4. 笔记不是可搜索、可分享、可协作的知识资产

## 应该做什么（正确流程）
嘉宾分享笔记 → 落盘飞书文档：

1. 用飞书 API 创建文档（铁律：只创建+写入，不碰权限）
2. 按飞书设计标准（铁律7）排版：
   - 要点用 T13（block_type=13）
   - H2 分隔道/法/术/器/行动清单
   - QTCONT 做资源导航卡片
   - 禁止多余 emoji
3. 返回文档链接给用户
4. 用户自行在飞书 UI 管权限

## 为什么犯这个错
- 对话输出 ≠ 知识资产
- 本地 MD 文件 = 只有小黎能看 = 不可协作 = 不可搜索
- 飞书文档 = 团队可看 = 可搜索 = 可迭代

## 关联铁律
- 飞书 API 协议（铁律7：飞书文档设计标准）
- SOP-16（嘉宾笔记飞书文档设计标准）

## 防再犯
嘉宾分享笔记类内容 → 默认落盘飞书。本地 MD 是副本，飞书才是主版本。
