### 法——系统加固

#### 加固1：个人信息检索协议（新增规则）

**触发条件**：任何需要查找"小黎经历了什么/说过什么/做过什么"的任务。

**强制执行步骤**：

```
Step 0: 绘制信息地图（不可跳过）
  ├── Glob **/*直播* 找所有直播相关文件
  ├── Glob **/*分享* 找所有分享相关文件  
  ├── Glob **/*面试* 找所有面试相关文件
  ├── Glob **/*实习* 找所有实习相关文件
  └── Grep "面试" -l 找所有含面试关键词的文件

Step 1: 按信息密度排序
  ├── 直播笔记（原始记录，最详细）→ 优先
  ├── 分享稿（加工后版本）→ 次优先
  ├── memory（概括性描述）→ 最后
  └── 推断（AI猜测）→ 标注置信度，不直接使用

Step 2: 读取+交叉验证
  ├── 至少读取3个不同来源
  └── 发现矛盾 → 优先采信直播笔记（原始记录）
```

**核心原则**：先搜全貌，再读细节。搜文件路径 > 读文件内容 > 凭记忆推断。

#### 加固2：个人信息数据地图

| 数据类型 | 存放位置 | 信息密度 |
|---------|---------|---------|
| 原始经历记录 | `**/直播笔记/` `**/txt/` | 最高（逐字稿级） |
| 结构化知识 | `**/SOP/` `**/私域认知/` | 高（方法论级） |
| 加工后版本 | `**/outputs/` `**/分享稿/` | 中（不同角度） |
| 概括性记忆 | `**/memory/` | 低（一句话级） |
| 教训反思 | `**/经验库/` | 中（反思级） |

#### 加固3：Glob-first搜索纪律

```
✅ 正确：glob **/*关键词* → 找到文件 → 按信息密度排序 → 逐个读取
❌ 错误：假设"经历在某区memory" → 只读memory → 没找到 → 推断
```

### 术——具体操作清单

| 场景 | 旧做法（错） | 新做法（对） |
|------|------------|------------|
| 需要个人经历细节 | 读memory/ → 找不到就推断 | Glob搜全貌 → 读直播笔记 → 交叉验证 |
| 需要技术细节 | 从技术栈推断 | Glob搜全貌 → 优先竞赛区/projects → 读原始记录 |
| 需要面试问答 | 读求职区memory | Glob `**/*面试*` → 读直播笔记 > 分享稿 > memory |
| 不确定信息来源 | "我觉得应该在X目录" | `grep -rl "关键词" .` 全工作区搜索 |

### 器——工具脚本

需要创建个人信息发现脚本，输入关键词，输出所有相关文件路径+信息密度评分。
