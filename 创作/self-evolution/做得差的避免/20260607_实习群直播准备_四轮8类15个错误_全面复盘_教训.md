## 根因链分析（追到最底层）

### 核心触发条件：「半小时就要分享」

```
触发条件：时间压力（T - 30min）
    ↓
AI内部决策：启动 "最短路径模式"
    ↓
────────────────────────────────────
五个连锁问题同时爆发：
────────────────────────────────────
    │
    ├─→ [材料层] 只读3个文件 → 产出信息不足
    │   根因：时间压力下"全面阅读"让位给"最少阅读"
    │
    ├─→ [技能层] 只调3个技能 → 质量链条断裂  
    │   根因：技能是"力倍器"但AI误判为"时间消耗器"
    │
    ├─→ [验证层] 跳过Read验证 → 声称落盘但文件不存在
    │   根因：时间压力触发"感觉完成了=完成了"错觉
    │
    ├─→ [个性化层] 文风注入不完整+工具名未替换
    │   根因：niuma-voice-dna被调用了但只是走了形式
    │
    └─→ [归档层] 经验库空着 → 同类错误下次还会犯
        根因：F6（教训回写）被"任务完成"心态架空
```

### 根本原因：高压任务缺少专项Protocol

当前规则体系有一个**致命盲区**：

| 任务类型 | 有Protocol吗 | 效果 |
|---------|-------------|------|
| 普通内容创作 | F8铁律（路由表+技能链） | ✅ 正常 |
| 快模式（简单问答） | 跳过启动序列 | ✅ 合理 |
| **高压任务（<1h截止）** | **❌ 没有** | **❌ 触发"最短路径模式"→ 质量崩塌** |

**核心矛盾**：AI的内部启发式是"时间紧 → 做少一点"。但内容是"力倍器"——**技能用得越多，单位时间产出质量越高**。时间紧应该"加速跑每一步"，不是"跳过多步"。做少=做错=重做=反而更慢。

### 逐错误根因深挖

#### 错误1：材料读取极少（3/15）

**表象**："时间不够，先做出来再说"
**中层原因**：没有"材料清单"机制——AI不知道哪些文件是必读的
**深层原因**：路由表只定义了"应该读哪些SOP/规则"，没有定义"应该读取用户哪些经历和背景文件"。对于"直播"这种需要大量个人经历的任务，**缺少一个"个人经历清单"强制读取机制**。
**认知偏误**：可用性启发式（Availability Heuristic）——最容易拿到的文件（模板）被高估了重要性，但真正重要的文件（用户亲身经历、文风DNA）需要更多步骤才能找到 → 被跳过了。

#### 错误2：技能调用极少（3/10+）

**表象**："时间紧，少调几个技能"
**中层原因**：同上——技能链被默认当作"可选"而非"强制"
**深层原因**：这是**教训027的同一类问题但触发条件不同**。027是因为"没意识到li-content是必需的"，本轮是因为"时间紧所以跳过了"。两种触发条件指向**同一个系统漏洞**：F8技能链没有"高压场景"的强制规则。
**关键洞察**：第一轮跳过了chinese-natural-voice-revision → 第二轮的文风问题（"怎么说呢"0次）本来可以在一轮就避免。**跳技能=制造问题=延长总时间。这叫"假高效"。**

#### 错误3：声称落盘但未验证（A级事故）

**表象**：在回复中说"文件已生成在XXX"，第二轮才发现文件不存在
**中层原因**：违反了F2铁律（Write后必须Read验证）
**深层原因**：「做了才说」铁律被架空——在时间压力下，AI的"完成感"压过了"验证感"。这是一个**心理层面的漏洞**：压力→想做快→"感觉自己做完了"→跳过验证→实际没做。
**关联历史**：与E35（覆盖公众号文章Git救回）同源——都跳过了Read验证环节。两次事故=系统级问题，不是偶发。

#### 错误4+5+6：路径假设 + docx/pdf参数错误

**表象**：朋友简历用了不存在的路径`D:\personal_information\send\`
**中层原因**：没先搜索/验证路径
**深层原因**：**"三查铁律"被架空**（0601教训的核心）。AI"感觉"路径应该存在 → 直接用 → 失败。这是典型的WYSIATI（What You See Is All There Is）——AI只看到了自己"感觉"到的路径，没看到实际文件系统。
**pdf参数错误**同理：没读pdf skill指南 → 猜了一个`--output-format text`→ 失败。

#### 错误7：工具名未个性化

**表象**：全文多处使用"Claude"，需第三轮用户纠正
**中层原因**：niuma-voice-dna涵盖了语气/节奏/标点，但不涵盖"工具偏好替换"
**深层原因**：文风DNA缺少**「环境上下文」维度**——用户在牛马AI生态中使用，产出的工具推荐应该默认指向牛马AI。这不是风格问题，是**身份一致性**问题。

#### 错误8：经验库未主动更新

**表象**：2-3轮纠正后经验库空着，用户第四轮说"你应该更新"
**中层原因**：纠正都写入了当日记忆，但没有走F6四步闭环
**深层原因**：**"任务完成"心态压过了"成长闭环"心态**。任务完成了（文件已生成、格式已修正、工具名已替换）→ 自然觉得"结束了"→ F6和lesson-sink-checklist被遗忘。这跟E28（双写规则空壳化）同源——**纠正≠教训=归档**这条链路一直是断裂的。

#### 错误9（本轮新增）：教训总结不够深

**表象**：初版教训文件只罗列了错误，用户说"你总结的还不够，我要的是再也不会犯错"
**中层原因**：把"教训"写成了"错误清单"，没做根因链分析和系统级修复方案
**深层原因**：**教训沉淀的质量维度被忽略了**。F6和lesson-sink-checklist定义了"必须写"，但没定义"写多深才算够"。一个只写"错误+正确做法"的教训文件=下次还会犯。「再也不会犯错」=需要追到根因链的终端=需要改系统级的规则/Protocol，不是改一句"下次注意"。

## 系统级修复方案

### 修复1：新增「高压任务Protocol」

在CLAUDE.md F8铁律旁新增高压场景分支：

```
⚠️ 高压任务（截止时间 < 1小时 或 用户说"快/急/马上/就要"且任务非简单问答）：

高压任务 ≠ 快模式。快模式用于简单问答，高压任务用于复杂交付。

高压任务强制清单：
1. [ ] 材料清单：列10秒→全读完→打勾（不是选读，是全读）
2. [ ] 技能清单：列核心技能→全跑完→打勾（不可跳过li-content+niuma-voice-dna+自审）
3. [ ] 落盘验证：每个Write后跟Read
4. [ ] 文风定量：扫描"怎么说呢">0次、禁用词=0、中文标点全角
5. [ ] 工具名个性化：全文扫描→替换为非平台通用名

高压任务铁律：
- 不可跳过li-content诊断（输出1分钟极简版但不可不跑）
- 不可跳过niuma-voice-dna注入
- 不可跳过自审
- 跳过的技能必须列在F1审计"被忽视的"并注明"高压场景跳过"
```

### 修复2：新增「落盘验证强制规则」（独立于F2）

将"Write后Read"从F2的一个子规则升级为独立强制步骤：

```
任何Write/Edit/Bash写文件操作，必须在下一次回复之前完成Read验证。
禁止在未验证的情况下说出"文件已生成/已创建/已写入"。
本条规则优先级高于"快模式"或"时间压力"——验证只花2秒，不验证花一整轮。
```

### 修复3：文风DNA新增「工具偏好」维度

在niuma-voice-dna中增加检查项：
- 工具推荐默认指向牛马AI（首选）+ Codex + ChatGPT网页端（备选）
- 平台名默认写"牛马AI/Newmax"而非"Claude"

### 修复4：F6教训沉淀新增「深度自检」

在lesson-sink-checklist.md中增加Step 6：

```
### Step 6：教训深度自检（🔴 2026-06-07新增）
写完教训文件后自问3个问题：
1. 这篇教训的"根因链"挖到第几层了？如果只到"下次注意"，重写。
2. 这篇教训有没有提议**系统级的规则/Protocol修改**？如果没有，重写。  
3. 如果我是另一个AI，读完这篇教训，**同一个坑还会不会再踩**？如果会，重写。
```

### 修复5：内容创作任务新增「个人经历清单」读取步骤

在F8技能链第一步前新增：
```
步骤0：个人经历清单读取（直播/分享/自我营销型任务强制）
1. 搜索求职区源文件（实习经历/面试复盘/简历版本）
2. 读取文风DNA完整画像
3. 读取相关的参考案例（朋友简历、往期直播笔记）
4. 列出清单→全读完→打勾→进入步骤1
```
