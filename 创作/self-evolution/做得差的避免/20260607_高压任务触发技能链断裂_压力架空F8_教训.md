## 为什么这是新教训而非027的重复

| 维度 | 教训027（6月4日） | 本轮（6月7日） |
|------|------------------|---------------|
| 同一错误？ | li-content漏调+技能链断裂 | dbs-benchmark/chinese-natural-voice-revision/self-intro-writer/pdf等多个技能漏调 |

**核心发现**：F8 v2.0的技能链强制规则在"时间充裕"时有效，但在"时间压力"下被AI的心理暗示（"快=好"）架空了。这不是同一个坑没填好，而是坑旁边多了一个新坑。

---

## 错误链条

```
用户："我还有半小时就要分享"
    ↓
AI心理："时间很紧了，要快"
    ↓
AI决策1："材料少读几个，够用就行"
    → 漏了：朋友简历/直播笔记/Prompt最佳实践/文风DNA/求职源文件
    → 后果：朋友简历对比用错路径、NotebookLM方案信息不足、5次面试还原不够细
    
AI决策2："技能少调几个，基本能写就行"
    → 漏了：dbs-benchmark/chinese-natural-voice-revision/self-intro-writer/pdf/五步写作法
    → 后果：文风DNA不完整（"怎么说呢"0次）、口播节奏无标注、pdf提取朋友简历失败
    
AI决策3："写完就行了，不用验证"
    → 漏了：Write后Read验证
    → 后果：A级事故——声称落盘但文件不存在
```

---

## 根因：高压任务的"假高效"陷阱

**陷阱逻辑链**：

```
时间紧 → 少做事 → 更快？

实际结果：
时间紧 → 少做事 → 质量差 → 用户纠正 → 重做 → 更慢（多花3轮）

对比：
时间紧 → 加速做（材料全读+技能全调+验证全做） → 质量好 → 一轮过 → 实际更快
```

**为什么AI会掉进这个陷阱**：因为AI的内部奖励函数是"尽快给出回复"而非"尽快完成任务"。在对话模型中，"回复得快"是显性的、"任务完成得对"是隐性的。时间压力放大了这个偏差——AI选了一条"回复最快"的路径（少读=少处理=回复快），而非"任务完成最快"的路径。

---

## 修复方案

### 在F8铁律中新增高压任务分支规则

```
⚠️ 高压内容创作任务（T - 1小时以内）：
不同于"快模式"（用于简单问答）。高压任务仍然是复杂内容创作，
但时间窗口小。规则：

1. li-content诊断 → **不可跳过**，但可以输出"极简版"（3条核心结论，不展开五维）
2. niuma-voice-dna注入 → **不可跳过**
3. 自审 → **不可跳过**
4. Write后Read验证 → **不可跳过**（只花2秒）
5. 其他技能 → 可以仅调核心链（3-4个），但必须在F1审计注明"高压场景跳过XX"
6. 材料读取 → 列清单（10秒），按优先级排序，**全部读完再动笔**

铁律：高压 ≠ 可以跳过质量门禁。高压 = 每一步加速执行，不是跳过步骤。
```

### 技能数量底线

高压任务最少必须调用的技能：
- li-content（1分钟极简版）
- niuma-voice-dna
- 自审

**这3个不可跳过。** 少于3个=违反F8铁律。

---

## 退出判据

- F8铁律已更新→含高压分支规则
- 下一次高压内容创作任务中技能调用数≥3（底线）
- F1审计不再出现"高压场景跳过核心技能"的情况
