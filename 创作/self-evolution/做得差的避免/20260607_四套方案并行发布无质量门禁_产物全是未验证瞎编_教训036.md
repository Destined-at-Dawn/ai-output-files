# 教训 036：四套方案并行发布无质量门禁——产物全是未验证的瞎编

## 日期
2026-06-07

## 方案
在焚决→视觉提示词任务中，AI同时发布了4套提示词方案（品牌名片/朋友圈封面/个人IP名片/系列海报），每套都有详细的提示词文本。看起来"产出丰富"，实际上没有一套经过视觉验证——全部是基于伪造的风格DNA生成的。

## 失败现象
- 4套方案的风格DNA来源是"从焚决方法论文章中提取"，不是"从参考图中观察"——DNA本身是假的
- 基于假DNA生成的提示词，再详细也没用——方向就是错的
- 用户被迫用Gemini自己重新做视觉品读，4套方案全部作废
- 信息量很大（每套方案300+字），但信息质量为零——这是"信息过载"的变体：用数量掩盖质量缺陷

## 为什么看起来行其实不行
- **表面合理逻辑**："用户提供3张参考图，我出4套方案，覆盖不同场景，给用户选择空间。产出越多越好。"
- **实际漏洞**：产出数量 ≠ 产出质量。4套未验证的方案 < 1套经过验证的方案。更糟糕的是，数量多会让用户以为"AI做了很多工作"，浪费更多时间去阅读和评估——增加了用户的沉没成本。
- **更深层的问题**：AI用"多方案并行"来掩盖"每个方案都没做好"的事实。这是信息过载的变体——用数量制造"丰富感"，实际每个方案的底层数据都是编造的。

## 退出判据
1. 先出1套方案，经用户确认方向正确后，再扩展
2. 每套方案必须有"验证状态"标注——已验证/待验证/未验证
3. 未验证的方案不进入正式交付，只能作为"草稿"或"参考方向"

## 关联法则
- 法则 8（门禁文化）：完成 = 过全部门禁。4套方案没有一套过门禁 = 没有一套完成
- 法则 2（防假象审计）："4套方案"看起来丰富，但审计后发现全部基于编造数据
- anti-info-overload.md 信号5：用户没有要求详细，但AI主动提供了"完整版"

## 影响范围
- 用户阅读+评估4套方案的时间全部浪费
- 用户对AI产出的信任度进一步下降（与教训034叠加）
- 后续所有方案交付必须附带"验证状态"

## 预防措施
1. 方案产出前必须有"最小可行方案"——先出1套，确认方向
2. 每套方案附带"验证状态"标签：✅已验证 / ⚠️待验证 / ❌未验证
3. 未验证方案只能作为"参考方向"，不能作为"推荐方案"
4. 多方案并行的前提是每个方案的底层数据都是真实的——底层数据造假时，多方案 = 多倍浪费
