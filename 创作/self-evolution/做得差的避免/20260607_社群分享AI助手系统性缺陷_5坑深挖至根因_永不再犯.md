# 失败教训：社群分享AI助手写作系统性缺陷——5坑深挖至根因，设计防复发机制

## 日期
2026-06-07（对话跨6月6日→6月7日）

## 场景
小黎在「同辈互助群」做第六天分享（主题：技术岗实习与AI实战）。AI助手经历了4轮迭代才到终版，暴露了5+4=9个坑。前5个在第一版中集中爆发，后4个在用户纠正过程中暴露。

## ⚠️ 本文不是"记住就完事了"——每条坑后面跟着"下次再犯的触发信号"，AI必须在发现信号时立刻停止并执行强制检查项。

### 第一轮就犯的5个坑（写前检查缺失）

| 编号 | 坑 | 表现 | 用户原话 |
|------|-----|------|----------|
| P1 | 日式kaomoji轰炸 | v1用了大量 (๑•̀ㅂ•́)و✧、ฅ՞•ﻌ•՞ฅ、₍ᐢ..ᐢ₎、( ˚ཫ˚ ) 等装饰性符号 | "不能有那种非常奇怪的emoji" |
| P2 | 数千字直接丢聊天不写MD | v1三版稿子全写在聊天中，没有保存文件 | "对应的稿子应该给我写进MD文件" |

|------|-----|------|----------|

## 二、深层根因分析（不是"没做检查清单"，而是"为什么没做"）

### 根因链：逐层下钻

```
现象层：用了奇怪emoji / 没写MD / 假设都求职 / 没串联 / 事实不准
  ↓ 为什么？
行为层：写前没有走完整的受众分析、源文件读取、输出格式确认
  ↓ 为什么没走？
认知层：AI不区分"社群分享"和"写一篇小红书"，都用同一个写作模式
  ↓ 为什么不区分？
系统层：CLAUDE.md 的路由表里，"社群分享"触发了 SOP-16，但没有触发"写前强制读取清单"
  ↓ 为什么没有？
架构层：SOP-16 定义了"怎么写社群分享"，但没有定义"写前必须确认什么"
```

**唯一根因：社群分享任务缺少一个自动激活的"写前GATE"——不是"建议读这些文件"，而是"不读完这些文件就不允许动笔"。**

### 每个坑的精确触发信号（检测点）

| 坑 | 触发信号（AI应该发现"不对劲"的时刻） | 当时AI在做什么 | 本应做什么 |
|-----|------|------|------|
| P1-kaomoji | 写入包含非基础emoji的字符串时 | 照搬朋友圈文风DNA的kaomoji列表 | 识别场景=社群半正式分享，切换emoji策略为基础emoji |
| P2-不写MD | 用户说"给我三版"之后、动笔之前 | 在聊天中直接长篇输出 | 先Write到outputs/ MD → 聊天中给摘要+文件路径 |
| P3-受众偏差 | 看到"社群分享"四个字时 | 假设群友=求职者 | 先读群公告确认群定位（AI学习互助群≠求职群） |
| P4-不串联 | 用户说"结合前面4天的笔记"时 | 读了笔记但只参考了风格 | 读了笔记就要提取"本周主题线"用于开头铺垫 |
| P5-事实不准 | 写下"一个月拿了3个offer"时 | 凭记忆写数字 | 先打开求职区源文件逐条核对 |
| P6-差异化不够 | 写完vA后发现vB/vC只是改了语气 | 以为"改几个词"就是新版本 | 三版必须有独立骨架设计（故事/框架/互动） |
| P7-角色混淆 | 看到"我的社群"时 | 用嘉宾分享模板 | 识别群主 ≠ 嘉宾，切换为群主语气 |
| P8-未预告直播 | 写完收尾后 | 不知道直播信息 | 用户提到"周六直播"就应该主动问"要不要在分享中预告" |
| P9-教训不深 | 写教训时停在"为什么发生" | 没继续追问"怎么保证不再发生" | 每条教训必须追问到"下次触发信号是什么+强制检查项是什么" |

## 三、防复发机制：社群分享写前强制GATE

> 以下不是"建议"，是"F级别铁律"——触发后必须逐条执行，跳过任何一条 = 违规。

### GATE 0：触发条件（自动激活）

**出现以下任一关键词组合时，GATE自动激活：**
- "社群分享" + "我要分享/轮到我了/今天讲"
- "群内分享" + 任何内容主题
- "在群里" + "分享/讲/说"
- SOP-16的触发条件（"社群分享/在别人群分享/嘉宾分享/群内宣传"）

### GATE 1：角色判断（15秒）

```
☐ 这是群主分享还是嘉宾分享？
  → 群主：语气更近、结构更稳、角色是"跟你们站在一起"
  → 嘉宾：可以讲自己的故事，群主负责介绍和收尾
☐ 群主分享不能用"讲课感"，不能用"我来讲方法论"
☐ 群主分享必须包含"周回顾串联"（如果有前序分享）
```

### GATE 2：受众画像（30秒）

```
☐ 读群公告 → 确认群定位（不是所有人都是来找实习的）
☐ 确认受众构成：大学生/已工作、理工科/文科、求职/学习/围观
☐ 确认主题泛化：让不求职的人也能带走东西
```

### GATE 3：信息收集（强制读取以下文件，缺一不可）

```
☐ 群公告（了解本周分享节奏和主题线）
☐ 前4天嘉宾分享笔记（提取可串联的内容和风格参考）
☐ 用户自我介绍/品牌档案（确认身份锚点）
☐ 与主题相关的所有源文件（求职区/竞赛区/个人区）→ 做事实校验清单
☐ 社群运营CLAUDE.md（确认社群路由规则）
☐ SOP-16 社群分享SOP（确认分享流程铁律）
```

### GATE 4：输出格式确认（动笔前）

```
☐ 输出方式 = MD文件 + 聊天摘要双轨
  → MD文件：完整稿，Write后Read验证落盘
  → 聊天摘要：每段不超过100字，引导用户打开MD文件
☐ emoji策略：社群半正式场景，只用基础emoji（数字、箭头、分隔符），禁用kaomoji
☐ 消息发送数量：控制在6条大消息块以内，每条间隔20-30秒
```

### GATE 5：三版差异化标准（如果需要三版）

```
☐ 三版必须有独立骨架设计，不能是同一篇文章改语气
☐ vA骨架 ≠ vB骨架 ≠ vC骨架（比如：故事驱动 / 框架驱动 / 互动驱动）
☐ 每版附一张差异对比表
☐ 三版共用同一套"事实数据"（源文件已读取并核对）
```

### GATE 6：写后自审（提交给用户前）

```
☐ 事实复查：每个数字/公司名/经历，最后一次对照源文件
☐ 受众复查：不求职的人能带走什么？
☐ emoji复查：grep所有非基础emoji并删除
☐ 消息块复查：总消息数 ≤ 6条
☐ 术语复查：JD/STAR法则/面经/诱饵/纵向实习——每个首次出现时都有解释
☐ 桥接复查：每个回答后面有没有"这跟你有什么关系"
☐ 周回顾复查：开头有没有串联本周前序内容
☐ 预告复查：有没有遗漏的直播/活动信息需要预告
```

## 四、信号→行动映射表（AI实时自检用）

> 这条是最关键的——下次对话中AI正在写社群分享稿时，如果出现左列信号，立即执行右列行动。

| 你正在做... | 如果发现这个信号 | 立即停下，执行 |
|-------------|------|------|
| 往稿子里加emoji | emoji不是①②③→📌❌✅ | 切换到基础emoji模式 |
| 写完后准备发给用户 | 内容长度>800字且不在MD文件里 | 先把全部内容Write到outputs/xxx.md → Read验证 → 聊天中只写摘要+文件路径 |
| 假设群友背景 | 没读过群公告 | 停下来，先读群公告 |
| 写第一句话 | 没有周回顾段落 | 检查：本周是否有前序分享？有→先写周回顾 |
| 写一个具体数字 | 数字来自记忆而非源文件 | 停下来，先Read对应的源文件核对 |
| 写完vA后写vB | vB只是改了语气词 | 重新设计vB的独立骨架 |
| 写到收尾 | 用户提过"周六/直播/活动"但稿子里没出现 | 加预告段 |
| 写了"XX坑" | 只写了"为什么发生" | 继续写："下次触发信号是什么+强制检查项是什么" |

## 九、关联认知书籍

- **010-思考快与慢**：WYSIATI（你只看到你想看到的）——AI只看到了"求职分享"，没看到群里有不求职的人。**解法：GATE 2强制读群公告做受众画像。**
- **009-认知天性**：检索式练习——串联前4天内容不是"锦上添花"，是把信息编码进群友长期记忆的核心机制。**解法：GATE 1强制周回顾串联。**
- **013-精要主义**：Less but better——基础emoji就够了，不需要花哨kaomoji；6条消息块就够了，不需要80条。**解法：GATE 4强制输出格式约束。**
- **060-稀缺**：带宽负担——80条消息占满了群友的注意力带宽，什么也记不住。**解法：GATE 4消息数量上限 ≤ 6条。**
- **007-反脆弱**：杠铃策略——左端（保守）：分享前完整准备+事实校验；右端（激进）：互动环节+直播预告。两端同时做，不走中间。**解法：GATE 6写后自审 → 左端保障，互动设计 → 右端突破。**
