# E28：双写规则空壳化——口头规则无验证

## 日期
2026-06-01

## 场景
用户要求"自我迭代20次"，AI 自检发现：CLAUDE-社群运营.md §八 写了"双写规则：模块级经验库 + 根级自我进化，两份都要写"，但运行验证脚本后发现同步率只有50%——1个文件只写了模块级没写根级。

## 用户批评原话
"经验库的「双写规则」应该在创建子模块时自动执行，不需要用户提醒。我不信你已经完成了自动化自我迭代流程，你再自我迭代20次"

## 问题
1. 规则写了但没有验证机制——"两份都要写"是纯文字，没有脚本检查
2. INDEX.md §四 规则3说"只写根级"，跟 CLAUDE-社群运营.md §八"双写"直接矛盾
3. 实际执行中只有1/2的文件真正双写到位
4. 没有 `dual-write-verify.py` 这样的自动化验证工具

## 根因
- **规则 vs 执行脱节**：写了规则但没有"怎么检查规则是否被执行"的机制
- **矛盾未发现**：INDEX.md 说"只写根级"，模块级 CLAUDE 说"双写"——两个文件说的话相反，但没有交叉校验
- **假阳性**：写了"已同步"但没有 `os.path.exists()` 验证

## 正确做法
1. 规则必须伴随验证脚本——"双写" + `dual-write-verify.py` 一起创建
2. 规则创建前必须检查是否有矛盾的旧规则
3. 每次双写后必须运行验证脚本，同步率 < 100% 时自动 `--fix`
4. INDEX.md 是唯一权威索引，其他文件的规则必须跟它对齐
