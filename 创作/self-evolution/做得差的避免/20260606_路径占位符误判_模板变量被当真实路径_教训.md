# 教训：路径占位符误判——模板变量被当真实路径

> 日期：2026-06-06
> 来源：dead-path-scanner.py 误报分析
> 严重度：🟡 中（导致扫描结果严重失真）

---

## 现象

扫描器将以下内容误判为"死路径"：

| 误报类型 | 示例 | 误报数 |
|---------|------|--------|
| 占位符文件名 | `{topic}.md`、`{YYYY-MM-DD}.md`、`xxx.md` | ~200 |
| CLI 命令 | `python scripts/xxx.py`、`/compact focus` | ~150 |
| 通配符 | `*.md`、`*.yaml` | ~50 |
| 标识符/类名 | `SkillChainNode`、`li-content` | ~300 |

## 根因

**正则表达式只检查路径是否"像路径"（含 `/` + `.md`），没有检查它是否"是真实路径"。**

`{topic}.md` 从格式上看确实像文件路径，但它是模板变量——花括号 `{}` 表示这是占位符，不是真实文件名。

`python scripts/dead-path-scanner.py` 从格式上看也像路径，但它是 CLI 命令——前缀 `python` 表示这是在执行脚本，不是引用文件。

## 正确做法

**过滤优先级（从高到低）**：

1. **花括号检测**：路径任何段含 `{` 或 `}` → 跳过（模板变量）
2. **CLI 前缀检测**：行首是 `python`/`bash`/`npx`/`pip` → 跳过（命令行）
3. **通配符检测**：路径含 `*` 或 `?` → 跳过（glob 模式）
4. **纯中文检测**：路径全是中文字符和 `/` → 跳过（路由表关键词）
5. **标识符检测**：路径段含大写字母+驼峰命名 → 可能是类名，需上下文判断

教训：**"看起来像路径"和"真的是路径"是两回事。过滤逻辑必须按优先级逐层排除，不能只做格式匹配。**

## 触发的规则

- 工程法则 2（防假象审计）：3175 条死路径中 60%+ 是假象
- 工程法则 3（实测优先）：应先小样本验证过滤逻辑

## 影响范围

- 所有从文本中提取结构化信息的脚本（路径提取、URL提取、依赖分析）
- 任何正则匹配"看起来像X"的场景
