# 飞书文档格式错误：Markdown 语法 + 日文引号

## 日期
2026-06-01

## 教训来源
飞书文档写入后用户手动修正

## 教训内容

### 错误1：飞书中出现错误的星号标记
- 现象：文章末尾出现了裸露的两个星号（Markdown 加粗语法），飞书不渲染 Markdown
- 根因：AI 写入飞书 block 时直接用了 Markdown 格式文本，没有转为纯文本
- 正确做法：写入飞书的内容必须是纯文本，不包含任何 Markdown 语法标记。加粗等格式通过飞书 block API 的 style 字段实现，或干脆不加粗

### 错误2：使用了日文引号而非中文引号
- 现象：文章中使用了日文方头括号（U+300C / U+300D）
- 根因：AI 的默认输出习惯倾向于使用日文方头括号，但中文语境下应该使用中文弯引号
- 正确做法：中文内容一律使用中文弯引号（U+201C / U+201D），不使用日文方头括号

## 影响范围
- 飞书文档写入 SOP
- 所有中文内容输出

## 预防措施
1. 写入飞书前，strip 所有 Markdown 语法标记
2. 输出后自审：扫描「」→ 替换为「」
3. 写入飞书的文本与本地 .md 文件可能需要两套格式（本地 .md 保留 Markdown，飞书用纯文本）
