# 飞书权限架构认知——三层模型与决策树

> **来源**：2026-06-01 至 2026-06-06 连续踩坑 8 次，浪费 30+ 轮
> **核心结论**：飞书权限三层独立控制，每层都可能成为瓶颈

---

## 一、三层权限模型

```
第1层：App Scope（开放平台配置）
  ▼ scope 通过 ≠ 一定能操作
第2层：Space/Resource 级权限
  ▼ 是成员 ≠ 能写
第3层：Document 级权限
```

## 二、Tenant vs User Token

| 维度 | tenant_access_token | user_access_token |
|------|--------------------|--------------------|
| 创建独立文档 | ✅ | ✅ |
| 写入 Wiki | ❌ 403 | ✅ |
| 管理协作者 | ❌ 1063002 | ✅ |
| 设链接分享 | ❌ 99992402 | ✅ |

核心：App 创建文档 ≠ App 是 owner。user token 是万能钥匙。

## 三、决策树

- 独立文档 → tenant token ✅
- Wiki → user token 必须
- 协作者管理 → user token 或用户手动
