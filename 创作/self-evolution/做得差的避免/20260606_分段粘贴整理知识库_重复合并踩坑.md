# 教训：分段粘贴逐字稿整理知识库——重复合并与路径踩坑

## 日期
2026-06-06

## 场景
用户上传 6.3 JC 分享逐字稿（何锦成 + Michelle），要求整理为知识库文档并存入经验库和私域认知。

## 踩了什么坑

### 坑1：分段粘贴导致重复整理

用户分三次粘贴内容（段1 + 段2 + 段3），最后又给了一次完整版（1608行）。前三段每段都单独整理了一次并追加到文档，最后完整版出来后又要对比补充遗漏内容。

**实际流程**：
- 段1 → 整理 §1-§4 + §5部分 → 写入文件
- 段2 → 追加 §5.6 → Edit
- 段3 → 追加 §5.7 → Edit
- 完整版 → 对比发现遗漏 §5.8/§5.9/§6.4 → 再 Edit

**正确流程**：等完整版到了再一次性整理，不做分段处理。

### 坑2：没提前确认输出路径

知识库文件写到了项目 outputs/ 目录，但用户同时要求更新经验库和私域认知——这三个目标应该在一开始就确认，而不是写完主文档后才问。

### 坑3：Read 大文件失败后没及时调整策略

Read 工具对中文长路径文件偶尔返回 "File does not exist"（实际文件存在），但一直重试同一个方法。应该早点切换到 Python 读取。

## 为什么看起来合理实际不行

- **分段整理的逻辑**："用户给一段我整理一段，效率高"——实际是每段都要重新读上下文 + 合并去重 + 格式对齐，总工作量比一次性整理多 2-3 倍
- **不提前确认路径的逻辑**："先做出来再说放哪"——结果主文档、经验库、私域认知三处要分别操作，而且经验库和私域认知的内容提取要重新读完整文档

## 退出判据

如果用户以分段方式提供内容，且内容总量 > 500 行：
1. 先问"这是完整版吗？还有后续内容吗？"
2. 确认全部内容到齐后再开始整理
3. 同时确认输出目标（本地文件 + 经验库 + 私域认知 + 飞书？）

## 正确做法

1. **等齐再动**：分段内容收到后只做缓存（存临时文件），不开始整理
2. **先确认输出矩阵**：开头就问清楚"整理完放哪里"（本地 outputs/ + 经验库/ + 私域认知/ + 飞书知识库？）
3. **一次成型**：完整版到齐后一次性整理，一次性写入所有目标位置

## 影响范围
- 所有"逐字稿/长文整理"类任务
- 所有需要"分段输入 → 结构化输出"的工作流
