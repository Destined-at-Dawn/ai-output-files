# E34 — 文件迁移完整踩坑（搬文件 + 修引用 + 版本判断）

> 搬文件是 10 秒的事，修引用是 10 分钟的事。**永远不要低估"搬完之后还有多少引用要改"。**

## 日期
2026-06-07

## 背景
用户要求把根目录 `社群运营/` 下的 html/、素材/、经验库/ 统一挪到项目层 `projects/.../社群运营/`。

## 踩坑清单（3个）

### 坑1：只搬文件不修引用
- 现象：文件搬完后，5 个 SOP 中 8 处硬编码路径全部断链
- 涉及文件：SOP-01（2处）、SOP-16（2处）、CLAUDE-写作规则.md（1处）、文风自审流程.md（1处）、INDEX.md（1处）
- 根因：迁移前只扫了 SKILL.md 有没有硬编码路径（没有），没扫 SOP 和 CLAUDE 文件
- 修复：grep 所有文件找旧路径引用，逐个 Edit 替换
- 教训：**搬文件前必须先 grep 所有引用**。迁移 = 移动文件 + 修引用 + 验证，三步缺一不可

### 坑2：旧版文件差点覆盖新版
- 现象：根目录 `约束文件/小黎文风DNA完整版.md`（1371字）vs 项目层同名文件（1454字）
- 根因：两个位置各有一份，内容不同步。根目录的是旧版
- 正确做法：比较文件大小/内容，保留新版，丢弃旧版
- 教训：**合并文件前先 diff**。同名文件在不同位置 = 不同版本，不能假设内容一样

### 坑3：html/ 已经在项目层了
- 现象：迁移脚本试图把根目录 html/ 移到项目层，但项目层已有同名目录
- 根因：之前某次对话已经迁移过了，但根目录还残留了一份
- 正确做法：迁移前先检查目标位置是否已有同名目录
- 教训：**迁移前检查目标是否已存在**。重复迁移 = 覆盖风险

## 迁移 SOP（提炼自这次教训）

```
Step 1: grep 所有文件，找出旧路径的所有引用 → 列出清单
Step 2: 检查目标位置是否已有同名文件/目录 → 有则 diff 比较
Step 3: git checkpoint（归档旧版本）
Step 4: 移动文件（Python shutil，不用 PowerShell）
Step 5: 修引用（逐个 Edit 替换旧路径为新路径）
Step 6: grep 验证旧路径引用 = 0
Step 7: Read 验证关键文件内容正确
```

## 防止再犯的规则

1. 迁移 = 移动 + 修引用 + 验证，三步缺一不可
2. 合并文件前先 diff，保留更新的版本
3. 迁移前检查目标位置是否已有同名
4. 用 Python 操作中文路径，不用 PowerShell
5. git checkpoint 后再动文件
