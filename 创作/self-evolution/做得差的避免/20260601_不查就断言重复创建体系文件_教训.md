# 教训：不查就断言，导致重复创建已有体系

## 日期
2026-06-01

## 事件
用户问为什么没有GitHub管理体系，我在项目沙箱找不到 → 断言不存在 → 重新创建。实际上文件在根工作区，而且是我自己上一轮刚建的。

## 4个子错误
1. 没查 projects/ 目录 → 没发现已有 github-publisher-skill.yaml
2. 没读 .newmax 记忆 → 没看到5月9日就踩过git历史隐私残留的坑
3. 没查 git 历史 → 没发现 scripts/ 被删过，应该先恢复
4. 直接在根目录新建同名文件 → 跟已有体系脱节

## 核心规则
> 断言任何文件不存在之前，必须三查：1) 根工作区目录 2) projects/ 子目录 3) git 删除历史。不查完不说话。
