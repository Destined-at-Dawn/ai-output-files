# 成功实践：HTML嘉宾分享批量转Markdown笔记

## 日期
2026-06-06

## 做了什么
用户给了4份HTML格式的嘉宾分享笔记，要求整理到`直播笔记/`目录，格式为Markdown。

## 执行过程
1. 一次性读取4份HTML文件（并行Read）
2. 提取每份的核心内容：主题、嘉宾背景、关键洞察、金句、行动清单
3. 统一Markdown格式：标题层级、分隔线、引用块、列表
4. 一次性写入4份文件（并行Write）
5. Read验证每份文件确认落盘

## 为什么做得好
- **批量处理**：4份笔记一次完成，不是逐个处理
- **格式统一**：所有笔记用同一套模板（主题 → 核心洞察 → 金句 → 行动清单），方便后续检索和对比
- **信息无损**：HTML中的关键信息全部提取，没有遗漏
- **落盘优先**：先产出文件，再做分析——文件是交付物，分析是附加价值

## 可复用的经验
1. **批量任务用并行**：多份同类型文件 → 并行Read → 并行Write，效率最高
2. **格式先行**：先确定统一模板，再填充内容——避免每份笔记格式不同导致后续对比困难
3. **验证闭环**：Write后Read验证，确认文件实际落盘

## 关联法则
- 法则 8（门禁文化）：文件落盘 + Read验证 = 完成
- F2（文件操作铁律）：Write后Read验证
