# 成功实践：实习群直播稿——四轮迭代从材料不足到完整交付

## 日期
2026-06-07

## 场景
小黎要在实习项目群直播分享「AI模拟面试」主题，30分钟后开始。AI助手从第一版（材料严重不足、技能链断裂）到第四版完成全部交付，过程中有几个做得对的点值得沉淀。

---

## 做得对的（按迭代秩序排列）

### 1. 第一轮被批评后没有防御，直接加速补材料

**具体表现**：
- 第一轮用户指出"你修改的和查看的文件都太少了，你用的技能也很少"
- 第二轮立即**全量拉取**所有相关文件（朋友改后简历、OPC直播笔记、Claude官方最佳实践、文风DNA、当天记忆、evolution-calendar、lessons）
- 第二轮技能从3个增加到5个

**为什么好**：承认不足→立即补→不浪费时间解释。第一轮漏了的东西在第二轮全补上了。

### 2. 朋友简历用了PDF提取方案绕过docx中文路径问题

**具体表现**：
- 第一轮用docx skill打`D:\personal_information\send\`路径失败
- 第二轮使用`pdf` skill + Python pdfplumber提取改后简历（PDF版本比docx更稳定）
- 成功获取简历改前改后对比内容

**为什么好**：工具问题不纠结，换方案。pdfplumber比python-docx更适合中文环境。

### 3. Claude官方Prompt最佳实践被完整参考

**具体表现**：
- 使用WebFetch获取了`platform.claude.com`的Prompt Engineering Best Practices全文
- 四个实战Prompt都按官方规范重写：
  - 角色设定一句话搞定（不说"你是一个专业的XX"长篇背景）
  - XML标签`<execution_steps>`/`<constraints>`包裹结构化约束
  - 正向指令（"只追问一个细节"而非"不要讨好面试者"）
  - 步骤编号2-3步具体可执行

**为什么好**：不是看了就完了，是**每条规范都映射到Prompt设计里**。用户给这个参考链接的意图被完整执行了。

### 4. 高数课逐字稿被结构化清洗

**具体表现**：
- 501句原始语音转文字
- 洗掉全部语气词（嗯/啊/那个）、重复句、杂音
- 保留所有问答原意和关键措辞
- 分六段 + 每段加「💡 注解」标注博弈点
- 提炼5条面试教训

**为什么好**：清洗标准一致——保留信息密度、删除噪音、增加分析价值。用户说"清洗逐字稿"，结果超过了"清洗"——做了语义分段+博弈分析。

### 5. 工具名称个性化在第三轮一次性全部完成

**具体表现**：
- 用户第三轮说"把首选的工具变成牛马AI和Codex"
- 全文搜索所有工具引用 → 6处替换：
  - "粘贴到Claude/GPT"→"牛马AI、Codex或ChatGPT网页端"
  - "Claude做武器"→"牛马AI（首选）或Codex"
  - "Claude免费版+豆包"→"牛马AI免费版+豆包"
  - "连Claude都不想买"→"连牛马AI都不想下载"
  - "不如Claude"→"不如牛马AI和Codex"
  - 逐字稿GPT/Claude/Copilot→牛马AI、Codex、ChatGPT

**为什么好**：不是替换关键词，是**理解了用户的牛马AI生态偏好后整体适配**。

### 6. 每次Write后都做了Read验证（第二轮起）

**具体表现**：
- 第二轮v2文件Write后 → Read验证（用Bash wc确认字节数、grep确认关键词）
- 第N轮v2追加加餐后 → Read验证（确认从499到673行完整插入）
- 第N+1轮工具名替换后 → Read验证（确认无遗漏的"Claude"）

**为什么好**：第一轮犯了"声称但未落盘"的A级事故后，后续每一轮都严格执行Read验证。**同一个错误不再出现第二次。**

### 7. 第四轮面对"再也不会犯错"的要求，写了完整的根因链分析

**具体表现**：
- 不是写"错误列表+下次注意"
- 而是追到每个错误的**认知偏误层**（如"可用性启发式导致高估模板重要性"）
- 提出5个**系统级修复方案**（高压任务Protocol、落盘验证独立铁律、文风DNA新增维度、F6深度自检、个人经历清单机制）
- 每个方案都写到了"可以写入CLAUDE.md的可执行规则"的粒度

**为什么好**：真正理解了"再也不会犯错"的含义——不是靠"下次注意"这种人工自律，是靠**系统级规则让AI不可能再走那条错误路径**。

## 可复用模式

### 模式1：「高压内容创作快速路由」

```
1. 识别高压任务（T - 1小时内 + 复杂内容创作）
2. 列必读材料清单（10秒）→ 全读完（不等确认）
3. 调核心3-4个技能（li-content极简版 + niuma-voice-dna + 自审 + 平台专项如chinese-natural-voice-revision）
4. Write → Read验证 → 回复中声明文件路径
5. F1审计注明高压场景跳过的技能
```

### 模式2：「逐字稿清洗→博弈分析」

```
1. 提取全文（docx/pdf/语音转文字等）
2. 清洗：去语气词→去重复句→保留原意和关键措辞
3. 分段：按对话语义转折切段（不是按字数切）
4. 注解：每段加「💡 注解」标注面试博弈点（为什么面试官这样问/面试者这样答）
5. 提炼：3-5条跨段通用教训
```

### 模式3：「工具名全量替换」

```
1. 搜索全文所有工具引用（grep: Claude/GPT/Copilot/Gemini等）
2. 按用户生态偏好替换：牛马AI（首选）→ Codex → ChatGPT网页端（备选）
3. 替换后重新Read确认无遗漏
4. 注意：方法论文献引用（如"Claude官方Prompt Best Practices"）不替换——那是来源标注，不是工具推荐
```

### 模式4：「三查路径」（见0601教训，本轮再次验证有效）

```
断言任何路径存在/不存在之前：
1. 先ls/Read验证
2. 先grep搜索可能的替代路径
3. 先查用户消息中是否有更精确的路径
三查完成前不说话。
```

## 量化收益

- 4轮迭代到达最终版（如果第一轮走完整F8高压Protocol可以缩到2轮）
- 参考文件数：第一轮3个 → 第四轮15+个
- 技能调用：第一轮3个 → 第二轮5个（极限下可用到10+）
- 文风验证率："怎么说呢"从0→3次、禁用词从有→0、英文引号从有→0
- 工具名替换覆盖率：6/6 = 100%
- 朋友简历对比还原度：改前vs改后完整还原+三条心法
- 高数课逐字稿：501句→6段结构化+5条教训+6处博弈注解
- A级事故恢复：第一轮文件不存在→后续每轮强制Read验证（零复发）

## 适用场景

- 高压内容创作任务（<1小时截止 + 需要多源材料）
- 直播/分享稿写作（需要结合大量个人经历）
- 任何需要多轮迭代纠正的内容交付
