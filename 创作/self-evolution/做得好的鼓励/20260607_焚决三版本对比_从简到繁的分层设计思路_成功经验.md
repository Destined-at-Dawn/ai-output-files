# 成功经验：焚决三版本对比——从简到繁的分层设计思路

## 日期
2026-06-07

## 背景
用户要求找焚决提示词。通过扫描知识中枢+工作区，找到了3个版本的焚决提示词，并做了分层对比。

## 做得好的地方
1. **快速定位**：通过 Glob + Read 扫描关键词"焚决""风格拆解"，10分钟内找到全部3个版本
2. **分层对比清晰**：简版（1句话快速出结果）→ 新版（15维度标准化）→ 深度拆解（10维度精确到色号），每个版本的适用场景、优劣势、精确度层次分明
3. **选型建议实用**：不是罗列信息，而是给"怎么选"的决策表——用户不需要知道所有细节，只需要知道"我的场景用哪个"
4. **溯源完整**：每个版本标注了来源文件路径和行号，用户可以自己去核实

## 可复用模式
- **"找→比→选"三步法**：找齐所有版本 → 横向对比差异 → 按场景给选型建议
- 这个模式适用于任何"用户有一堆类似工具/模板/SOP，不知道用哪个"的场景

## 认知支撑
「你不必事事皆知，你只需要知道在哪里能找到答案。」——认知觉醒中的"外部化记忆"策略。这次不是AI自己创造知识，而是快速检索+结构化呈现已有知识——这才是AI在知识管理中最该做的事。
