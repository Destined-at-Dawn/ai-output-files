## 最终验证通过的流程

### Step 1：创建文档（不碰权限参数）
```python
resp = requests.post(
    "https://open.feishu.cn/open-apis/docx/v1/documents",
    headers={"Authorization": f"Bearer {token}"},
    json={"title": title}  # 不传任何权限参数

```
**关键**：不传 `link_share_entity`、不传 `external_access_entity`。默认 = tenant_readable = 租户内所有人可见。

## 关键认知

### 飞书权限模型
- **tenant_access_token 创建的文档**：创建者 = 应用 ≠ 用户
- **设 `closed`** = 只有应用能看 = 管理员失联
- **不设权限** = 默认 tenant_readable = 租户内所有人可见 = **最安全的默认值**

### open_id vs user_id vs email
- `open_id`：按 app 维度分配，不同 app 看到同一用户的 open_id 不同 → **不可靠**
- `user_id`：租户内唯一 → 可用
- `email`：跨 app 通用 → **最可靠**

### Wiki 文档 vs 独立文档
- **独立文档**（API 创建）：不挂在任何人云盘下 → "孤岛"，UI 里找不到
- **Wiki 文档**（通过 Wiki API 创建）：挂在知识库里 → 有入口、有归属
