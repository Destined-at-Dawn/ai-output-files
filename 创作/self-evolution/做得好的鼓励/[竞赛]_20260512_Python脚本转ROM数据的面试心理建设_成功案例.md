# 20260512 Python脚本转ROM数据的面试心理建设

## 原问题
小黎问："logo_rom.v里写了'由logo_to_rom.py自动生成'，对方会因此怀疑我是AI生成的吗？"

## 正确回复要点
### 结论：不会，反而觉得非常专业
1. **手敲不可能**：30,000个像素点，没有人会手动提取RGB值转Hex
2. **体现工具链能力**：懂Verilog+会写Python脚本辅助开发=基本功扎实
3. **工业界常态**：音频数据、余弦表、滤波器系数全都是脚本跑出来的
4. **注释是好习惯**：说明"不要手动改这个文件，要去跑脚本"，方便维护

### 满分话术
> "因为图片像素太多了，手动弄不现实。我写了个Python小脚本，调了Pillow库，把Logo拉伸到中心区域尺寸，每个像素提取出来转成RGB888 Hex，Vivado用$readmemh一行读进去。"

## 为何满意
- 消除了用户对"AI痕迹"的焦虑
- 把"可能暴露AI"转化成了"加分项"
- 给出了直接可用的面试话术

## 核心教训
用户对"AI痕迹"产生焦虑时，用**行业标准**和**配套话术**来消除疑虑。脚本化工作流是工程师的标配，不是AI的标签。
