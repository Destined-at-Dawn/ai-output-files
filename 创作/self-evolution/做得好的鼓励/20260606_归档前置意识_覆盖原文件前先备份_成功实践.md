# 成功实践：归档前置意识——覆盖原文件前先备份

## 日期
2026-06-06

## 场景
优化嘉宾分享笔记前，先将原文件归档到 `归档/2026-06-06-直播笔记优化前版本/`。

## 做对了什么
1. 在写入优化版之前，先把三份原 MD 文件复制到归档目录
2. 归档完成后才执行覆盖式写入
3. 输出了归档报告（哪些文件归档到了哪里）

## 为什么这很重要
- 笔记优化是"覆盖式修改"——原内容会被替换
- 如果优化后发现新版本反而更差，可以从归档恢复
- 这是 F4 双归档铁律的轻量版执行（归档到工作区归档目录）

## 关联铁律
- F4（双归档铁律）
- Git 检查点规则（删前有快照）

## 后续改进
- 归档时应同时归档到全局归档目录（`E:\ai产出文件\牛马\归档\`），不只是工作区内归档
- 归档后应执行 git commit 检查点

## 适用场景
- 所有覆盖式文件修改
- 笔记优化、文档重写、版本升级
- 批量文件替换
