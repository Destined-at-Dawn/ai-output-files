# 教训沉淀强制 Checklist（Lesson Sink Checklist）

> **优先级**：最高（等同于 no-blind-overwrite.md）
> **创建日期**：2026-05-27
> **来源**：2 次真实遗漏——教训只写 memory + negative-results，漏了 self-evolution，导致进化系统断裂
> **触发条件**：本轮对话中出现「用户纠正」「方案验证为死路」「踩坑 ≥1 次」

---

## 铁律

**教训沉淀不是"写完 memory 就结束"。必须完成全部 4 步，少任何一步 = 流程未完成。**

---

## 强制 Checklist（每次教训沉淀时逐项打勾）

教训触发后，必须**按顺序**完成以下全部步骤。每步完成后在脑中标记 ✅，全部完成后才能结束对话。

### Step 1：写入当日记忆
- [ ] 路径：`.newmax/memory/{YYYY-MM-DD}.md`
- [ ] 内容：教训摘要 + 根因 + 正确做法
- [ ] 验证：Read 文件确认内容已落盘

### Step 2：写入项目负结果日志
- [ ] 路径：项目目录 `negative-results.md`
- [ ] 内容：方案 → 失败现象 → 为什么看起来行其实不行 → 退出判据
- [ ] 验证：Read 文件确认内容已落盘

### Step 3：回写自我进化系统（🔴 最容易遗漏的一步）
- [ ] 路径：`E:\ai产出文件\牛马\创作\创作\自我进化\做得差的避免\` — 追加教训文件
- [ ] 路径：`E:\ai产出文件\牛马\创作\创作\自我进化\INDEX.md` — 追加索引条目
- [ ] 验证：Python `os.path.exists()` 确认新文件存在（不用 Bash ls，不用 PowerShell——中文路径不可靠）

### Step 4：检查是否需要回写 SOP/Skill
- [ ] 查 F6 映射表：此教训影响哪个 SOP？
- [ ] 影响的 SOP 是否需要更新（追加迭代日志 / 添加高频坑标记）？
- [ ] 同类教训 ≥3 次 → 升级为铁律

### Step 5：双写验证（🔴 模块级经验库同步）
- [ ] 教训是否与某个子模块相关（社群运营/公众号/图片提示词）？
- [ ] 如果相关：写入模块级 `经验库/做得差的/`（如 `projects/20260425-内容创作系统/社群运营/经验库/做得差的/`）
- [ ] 运行验证：`python scripts/dual-write-verify.py`，确认同步率 100%
- [ ] 如果不同步：`python scripts/dual-write-verify.py --fix` 自动补全

---

## 遗漏检测信号（AI 自检）

以下任何一条出现 = Step 3 被跳过了：

- [ ] 本轮写了 memory 但没提"自我进化"
- [ ] 写完 negative-results.md 后直接结束对话
- [ ] 用户问"你为什么没更新进化文件"——说明已经遗漏了

**如果检测到遗漏**：立即停下当前任务，先补完 Step 3，再继续。

---

## 为什么 Step 3 最容易遗漏

| 根因 | 表现 | 解法 |
|------|------|------|
| self-evolution/ 在根工作区，memory 在 .newmax 层 | 两个路径不连续，写完 .newmax 自然"觉得完了" | **本 checklist 强制续行** |
| F6 铁律只说"必须回写"，没有说"怎么检查" | 规则太抽象，执行时不会每条都过 | **改为逐项打勾的 checklist** |
| 没有失败退出机制 | 遗漏了也没有自动检测 | **自检信号 + 用户纠正时立即补** |

---

## 与 CLAUDE.md F6 铁律的关系

F6 说"教训必须回写 Skill/SOP"。本规则是 F6 的**可执行展开**——把抽象要求变成逐项打勾的 checklist。

F6 = 原则层（"必须做"）
本规则 = 执行层（"怎么做、怎么检查做没做"）
