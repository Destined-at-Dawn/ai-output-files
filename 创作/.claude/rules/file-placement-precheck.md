# 文件放置写前检查规则（File Placement Pre-check）

> **优先级**：高（等同 script-safety-check.md）
> **创建日期**：2026-06-07
> **来源**：AI信源网站合集错放 outputs/（架构违规），用户指出"output是最终产出的地方，你放错位置了"
> **触发条件**：任何 Write / write_project_file 操作前

---

## 铁律

**创建任何新文件前，必须通过两个判断。没有"这次很简单不用判断"的例外。**

---

## 判断一：这个文件的性质是什么？（持续性参考 vs 一次性交付物）

| 如果是… | 放哪里 | 示例 |
|---------|--------|------|
| **持续性参考资源**（会被反复读取、更新、作为工作素材） | 按内容归属放到对应子模块目录（`公众号/自己/`、`SOPs/`、`社群运营/`、`.claude/rules/` 等） | 图片提示词参考卡、搜索速查指南、AI信源合集、文风自审流程 |
| **一次性最终交付物**（做一次就完了，后续不会反复翻出来用） | `outputs/` | 群公告v5、嘉宾分享笔记、定价方案、一次性分析报告 |
| **规则/约束/SOP** | `.claude/rules/` 或 `SOPs/` | feishu-api-protocol.md、SOP-18 |
| **记忆/教训** | `自我进化/` + 对应模块 `经验库/` | 教训文件、成功实践记录 |
| **Skill 定义** | `~/.newmax/skills/{skill-name}/` | SKILL.md |

## 判断二：根目录或目标子模块是否已有同类文件？

1. **先 grep / list_project_files 检查目标目录和相邻目录**
2. 已有同类文件 → **不重复创建**，在已有文件基础上扩展
3. 没有同类文件 → 按判断一确定位置

**自检清单（每次写文件前逐项确认）**：

- [ ] 这个文件会被反复使用吗？→ 是 = 非 outputs/，否 = 可放 outputs/
- [ ] 这个文件是规则/约束吗？→ 是 = `.claude/rules/` 或 `SOPs/`
- [ ] 这个文件是记忆/教训吗？→ 是 = `自我进化/` + `经验库/`
- [ ] 目标目录是否已有编号/分层体系？→ 有 = 先扫编号规律，匹配已有目录，禁止裸建无编号前缀的独立文件夹

## 反模式警告

- ❌ "这个产出看起来很重要，先放 outputs/ 再说" → outputs/ 是一次性交付物目录，持续性资源放这里 = 架构违规
- ❌ "放哪都一样，反正 CLAUDE.md 会引用" → CLAUDE.md 引用错了路径 = 后续 AI 找不到文件 = 失忆
- ❌ "先建了，后面再挪" → 挪文件需要同步更新所有引用（CLAUDE.md + 子模块 SKILL.md + skill 定义），成本远高于一开始放对
- ❌ "这次是小文件，随便放" → 架构违规和文件大小无关，和文件性质有关
- ❌ 🔴 "在已有编号目录体系外裸建新文件夹" → 目标目录如有 `01-XX/` `02-XX/` 的编号体系（如社群运营），新内容必须拆入已有编号目录或按编号规则新建，禁止在根目录裸建 `马原22讲/` `马原63/` 等独立文件夹。**先 `ls` 扫编号体系 → 按内容性质匹配已有目录 → 没有匹配的才按编号规则新建。**

---

## 判断三：目标目录是否有编号/分层体系？（目录结构遵从）

**触发条件**：目标目录下存在 `01-XX/` `02-XX/` 或 `道/` `法/` `术/` `器/` 等分层编号体系。

**强制流程**：
1. **先 `ls` 扫描目标目录**：列出所有一级子目录，识别编号规律
2. **按性质匹配已有目录**：内容属于什么类型 → 对应哪个编号目录？
3. **有匹配 → 写入已有目录**（可建子目录，不可裸建同级目录）
4. **无匹配 → 按编号规则依次补号新建**（如已有 `01-07`，则新建 `08-XX/`）
5. **禁止**在编号体系外裸建无编号前缀的独立文件夹

**示例**：
```
社群运营/ 已有：01-群公告/ 02-嘉宾与话术/ ... 06-咨询案例库/
新内容「马原22讲笔记」→ 知识型内容 → 归属已有 私域认知/（或按编号规则建 07-学术备考/）
❌ 禁止：社群运营/马原22讲-分讲笔记/（裸建，破坏编号体系）
```
