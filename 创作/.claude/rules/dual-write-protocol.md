# 双写协议（Dual-Write Protocol）

> **优先级**：高
> **创建日期**：2026-06-01
> **来源**：教训 E28——双写规则空壳化，口头规则无验证
> **适用范围**：所有含经验库的子模块（社群运营/公众号/图片提示词/视频提示词）

---

## 铁律

**经验教训必须同时写入根级和模块级，写完必须验证。不验证 = 未完成。**

---

## 一、双写规则

### 写入位置

| 内容类型 | 根级（所有工作区共享） | 模块级（子模块专属） |
|----------|----------------------|---------------------|
| 失败教训 | `自我进化/做得差的避免/` | `{模块}/经验库/做得差的/` |
| 成功经验 | `自我进化/做得好的/` | `{模块}/经验库/做得好的/` |
| 索引更新 | `自我进化/INDEX.md` | `{模块}/README.md`（如有） |

### 写入流程

```
1. 写入根级文件
2. 复制到模块级（Python shutil.copy2）
3. 更新 INDEX.md
4. 运行验证脚本
5. 同步率 < 100% → --fix 自动补全
```

### 验证命令

```bash
# 全量检查
python E:\ai产出文件\牛马\创作\创作\scripts\dual-write-verify.py

# 只检查一个模块
python E:\ai产出文件\牛马\创作\创作\scripts\dual-write-verify.py --module 社群运营

# 自动修复
python E:\ai产出文件\牛马\创作\创作\scripts\dual-write-verify.py --fix
```

---

## 二、模块级经验库初始化

当一个新的子模块创建时，如果还没有 `经验库/` 目录，必须：

```
1. 创建 {模块}/经验库/做得好的/
2. 创建 {模块}/经验库/做得差的/
3. 将根级自我进化中与该模块相关的文件同步过去
4. 运行 dual-write-verify.py 确认同步率 100%
```

---

## 三、禁止

- ❌ 只写根级不写模块级
- ❌ 只写模块级不写根级
- ❌ 不运行验证脚本就声称"已同步"
- ❌ 用 PowerShell 操作中文路径（用 Python）
- ❌ 规则文件中写"双写"但没有对应的验证脚本

---

## 四、自检信号

以下任何一条出现 = 双写未完成：

- [ ] 本轮写了教训但没有运行 `dual-write-verify.py`
- [ ] 验证脚本返回同步率 < 100% 但没有运行 `--fix`
- [ ] 创建了新子模块但没有初始化 `经验库/` 目录
- [ ] INDEX.md 的最后更新日期不是今天
