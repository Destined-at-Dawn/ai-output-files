## 二、自动同步机制

### 2.1 本地自动提交（防误删）

**频率**：每 2 小时自动 commit（已有 NiumaAutoCommit 定时任务）

**触发点**：
- 定时任务自动执行
- AI 删除文件前（检查点提交）
- AI 大规模修改文件前（检查点提交）
- 对话结束时（有变更则 commit）

**脚本**：`_system/scripts/smart-commit.sh`

### 2.2 远程同步推送（防物理损坏）

**规则**：

| 工作区类型 | 推送目标 | 推送频率 | 触发方式 |
|-----------|---------|---------|---------|
| A/B 类（公开项目） | GitHub Public | 每次更新后 | AI 手动推送 |
| C 类（私有工作区） | GitHub Private | 每日一次 | 定时任务 |
| C 类（关键修改） | GitHub Private | 即时 | AI 手动推送 |

### 2.3 同步推送流程

```bash
# 1. 确认本地有未推送的 commit
git log origin/main..HEAD --oneline

# 2. 如果有，推送到远程
git push origin main

# 3. 验证推送成功
git status
```

## 三、防误删机制（三层防护）

### 第一层：预防（不删）

- AI 修改文件优先用 Edit（精确替换），不用 Write（全文覆写）
- 删除文件前必须双归档（见 F4 铁律）

### 第二层：检查点（删前存）

- 删除任何文件前，先执行检查点 commit
- 超过 5 个文件的批量删除，先列出清单给用户确认

### 第三层：恢复（删后救）

- `scripts/git-recover.sh` 提供 30 秒恢复能力
- 支持：单文件恢复、批量恢复、查看已删文件列表

---

## 四、恢复脚本使用

**脚本位置**：`scripts/git-recover.sh`

```bash
# 列出最近删除的文件
bash scripts/git-recover.sh --list [目录]

# 查看文件修改历史
bash scripts/git-recover.sh --diff <文件>

# 恢复单个文件
bash scripts/git-recover.sh <文件相对路径>

# 恢复最近一次提交中所有被删文件
bash scripts/git-recover.sh --all
```

**恢复后必须**：
1. Read 文件确认内容完整
2. git add + git commit 保存恢复
3. 记录到 memory/教训

## 五、定时任务配置

### 每日远程同步（所有私有工作区）

```
任务名：Git Daily Sync
频率：每日 23:00
脚本：
  1. 遍历所有工作区目录
  2. 检查是否有未推送的 commit
  3. 如有，执行 git push
  4. 记录同步结果
```

### 每 2 小时本地提交（已有）

```
任务名：NiumaAutoCommit
频率：每 2 小时
脚本：smart-commit.sh
```

## 六、违规信号

如果出现以下任何情况，说明同步规则被违反了：

- [ ] 用户问"我的文件去哪了"时 AI 无法从 Git 恢复
- [ ] AI 说"文件已经被删了，无法恢复"（在有 Git 的情况下）
- [ ] 工作区没有 .git 目录
- [ ] 工作区有 .git 但没有远程仓库
- [ ] 超过 24 小时没有 commit
- [ ] 超过 7 天没有 push 到远程
