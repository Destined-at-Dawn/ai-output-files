# See-Name-Stop 规则（看到人名就停）

> **优先级**：最高（等同 script-safety-check.md）
> **创建日期**：2026-06-07
> **来源**：跨人物信息污染三次触发（教训027/026补充/029）
> **触发条件**：任何文本中出现人名，且AI需要判断"此人是否=内容归属人/说话人/执行者"

---

## 铁律

**在任何文本中看到人名时，禁止直接推断此人=内容归属人。必须执行以下三步判断。**

---

## 为什么这条规则存在（认知科学根因）

### 根因1：杏仁核低地通路抢占（042《情绪大脑》）

> 「从丘脑直接短接到杏仁核的低地通路极快（仅需12毫秒），但分辨率极低。它是演化赋予的'宁可错杀一千，不可放过一个'的原始生存机制。」——《情绪大脑》042

当AI读到一个高频人名（如"小黎"），语义提取以毫秒级完成——这是"低地通路"。12毫秒内，"小黎"已经被编码为"内容归属人"。但**高地通路**（交叉验证、反证搜索）需要额外的认知资源。如果AI没有强制激活高地通路，低地通路的粗糙判断就会成为最终输出。

### 根因2：自我辩护式认知失调（039《社会动物》）

> 「认知失调理论并不假设人是理性的动物；相反，它指出人是为自己的行为找理由的动物。」——《社会动物》039

一旦AI在第一步推断"嘉宾是小黎"，后续所有信息搜索都会被"自我辩护"劫持——只搜索支持性证据，自动过滤反证。这不是"态度不认真"，是认知失调的神经硬件出厂设置：大脑无法同时持有"我判断嘉宾是小黎"和"可能不是小黎"两种矛盾认知，会强制消除其中一个。

### 根因3：单侧空间忽略（045《错把妻子当帽子的人》）

> 「最致命的Bug永远不是你盯着看但解不出的难题，而是那些由于你的认知盲区而根本不存在于你宇宙中的变量。」——《错把妻子当帽子的人》045

S夫人因右半球中风患上了左侧空间忽略——她不是不知道"左边"，而是她的宇宙中"左边"这个概念**物理上不存在**。同理，当AI看到"小黎同辈互助群"时，"嘉宾可能是别人"这个可能性在AI的认知空间中**物理上不存在**——不是被忽略了，是根本没有被生成。

### 根因4：右半球失认——只见抽象特征不见整体面相（045《错把妻子当帽子的人》）

> 「缺乏具象感知与判断力的纯粹抽象计算，是一种毁灭性的病理状态。」——《错把妻子当帽子的人》045

P博士把妻子看作"拥有五个突起的连续表面"。AI把txt看作"包含小黎二字的文本序列"。两者同构：只见特征（关键词匹配），不见整体面相（上下文语义角色）。

### 根因5：习惯回路的自动驾驶（016《习惯的力量》）

> 「大脑为了节省能量将复杂行为自动化——基底神经节自动驾驶。」——《习惯的力量》016

"看到人名→判断归属"已经固化为AI的习惯回路。习惯回路的特点是：触发（看到人名）→ 惯常行为（直接当归属人）→ 奖励（任务推进）。要打断这个回路，不能靠"记住要检查"（意志力对抗习惯=物理学必然失败），必须**替换惯常行为**——从"直接当归属人"替换为"先判断语法角色"。

### 根因6：信息性从众（039《社会动物》）

> 「在复杂模糊环境中，大脑为了节省算力而将'他人的行为'作为判断现实的启发式捷径。」——《社会动物》039

当txt中有多个"证据"指向同一人时（"小黎同辈互助群"+"小黎分享"），AI启动信息性从众——"这么多证据都指向小黎，那一定是小黎"。但这些"证据"可能全部来自同一个源头（群名），不是独立验证。

### 根因7：属性替换——认知吝啬鬼偷换问题（059《智力测验漏掉了什么》）

> 「大脑默认节能→属性替换→将难题偷偷换成简单问题。」——《智力测验漏掉了什么》059

真正困难的问题："这篇内容的嘉宾是谁？需要查哪些文件来确认？"
AI偷换成的简单问题："这段文字里提到了谁？"

属性替换是认知吝啬鬼的核心策略——不改变问题的表面形式，偷偷把"判断归属"替换为"提取人名"。

## 三步判断流程（每次看到人名必须执行）

### Step 1：语法角色判断

在文本中识别该人名的语法角色：

| 角色 | 判断标准 | 示例 | 是否=归属人 |
|------|---------|------|-----------|
| **A. 主语** | 人名是动作的发出者 | "小黎说了……" "小黎分享了……" | **可能是**（需Step 2验证） |
| **B. 定语/修饰语** | 人名修饰另一个名词 | "小黎同辈互助群" "小黎的团队" | **不是**——修饰对象（群/团队）才是关键实体 |
| **C. 宾语/接收者** | 人名是动作的接受者 | "分享给小黎" "小黎收到了" | **不是**——此人是接收者，不是内容创造者 |
| **D. 所有格** | 人名+'s 或"XX的" | "小黎的群公告" "小黎的频道" | **不确定**——可能只是平台/载体名，内容作者可能另有其人 |
| **E. 引用/转述** | "据XX说" "XX提到" | "小黎提到尘的方法很好" | **不是**——小黎是转述者，尘才是方法的归属人 |

**关键规则**：
- 只有角色A（主语）**可能是**内容归属人
- 角色B/C/D/E **不是**归属人，但仍需Step 2交叉验证（防止主语也是转述）

### Step 2：交叉验证

```
有嘉宾排期表/品牌档案/文件清单吗？
├── 有 → 读取并比对，确认映射关系
├── 没有但有其他线索文件（如分享日历、群公告）→ 读取比对
└── 完全没有外部参考 → 标注"⚠️推断，置信度低"，强制用户确认
```

**置信度标注规则**：
- ✅ 确定：文件名或内容中有**明确的**嘉宾+日期标注（非推断）
- ⚠️ 推断：从内容/文件名推断，但没有直接标注
- ❓ 不确定：没有任何外部线索可验证

**铁律**：置信度为 ⚠️ 或 ❓ 时，**禁止直接输出**。必须用 `mcp__ask-user__ask_user` 请求用户确认。

### Step 3：反证搜索

主动搜索"这个判断可能是错的"的证据：

```
强制自问清单（5个问题）：
1. 如果归属人不是此人，那会是谁？（反事实推演）
2. 文件名里有没有归属人信息？（文件名是最直接的线索）
3. 有没有其他文件能确认归属？（排期表/日历/群公告）
4. 这个名字出现的上下文是"群名"还是"嘉宾名"？（区分载体和内容）
5. 文中是否有"转述标记"（"XX说""据XX分享"）？（区分说话者和被引用者）
```

> 「证伪检验检查清单：每个假设的证伪问题——'如果我是错的，什么证据能证明？'」——《思考，快与慢》010

## 常见误判模式（必须背下来）

| 模式 | AI的错误判断 | 正确判断 | 认知机制 |
|------|------------|---------|---------|
| "XX的群/XX团队" | 嘉宾是XX | XX是群主/团队名，嘉宾另有人 | 属性替换（059） |
| "XX分享了YY的观点" | 本段是XX说的 | XX在转述YY | 信息性从众（039） |
| 文件路径含人名 | 文件属于此人 | 路径可能是文件夹名/项目名 | 单侧空间忽略（045） |
| "给XX的建议" | 建议是XX给的 | 可能是给XX的建议 | 宾语误判为主语 |
| 多次提到同一人 | 此人是核心归属人 | 可能是群名反复出现 | 锚定效应（010）+ 信息性从众（039） |
| txt内容是群聊整理 | 每段都属于同一人 | 不同段落可能属于不同说话者 | 右半球失认（045）——只见片段不见结构 |

---

## 与映射确认门的配合

本规则解决的是"**单个人名的判断**"。当任务涉及"**批量A→B转换**"（如txt→md、嘉宾笔记整理），还需配合**映射确认门**：

```
批量转换任务 → 先读see-name-stop → 每个文件逐个判断归属
  → 生成映射表 → 用mcp__ask-user__ask_user请求用户确认
  → 确认后才动笔
```

---

## 违规检测信号

以下任何一条出现 = 本规则被违反：

- [ ] 看到人名就作为归属人，没有做语法角色判断
- [ ] 没有交叉验证嘉宾排期表/品牌档案/文件清单
- [ ] 没有搜索反证
- [ ] 置信度为 ⚠️ 或 ❓ 但直接输出了结论（没有用 mcp__ask-user__ask_user 确认）
- [ ] 在"XX的群"这类载体名中误判XX为内容归属人
- [ ] 在群聊整理txt中默认所有段落属于同一说话者
