# AI前沿选题日报 + 仿版内容输出 SOP 模板

> 适用对象：想把「AI前沿信息获取 → 选题判断 → 内容拆解 → 仿版输出」做成日更系统的个人创作者
> 目标：把随机刷信息，变成稳定的信息输入与内容输出闭环
> 日期：2026-04-13

---

# 一、SOP总目标

这套SOP不是为了让我“看起来很努力”，而是为了让我每天稳定完成两个结果：

1. **产出一份有判断的 AI 前沿选题日报**
2. **基于一个已验证结构，输出一条仿版内容草稿**

核心原则只有一句话：

**先稳定跑通最小闭环，再逐步提高质量。**

---

# 二、适用场景

这套SOP适用于以下内容方向：

- AI前沿资讯解读
- AI工具实操
- AI行业趋势判断
- AI对学习 / 工作 / 内容创作的影响
- 理工科知识与AI结合内容

如果当天没有重大新闻，也照样执行，只是把重点从“热点追踪”切换成“结构拆解和观点整理”。

---

# 三、每日总流程

每天固定走四步：

1. **采集**：搜集AI前沿信息
2. **筛选**：选出最值得讲的题
3. **拆解**：找到一个可复用结构样本
4. **输出**：产出一条仿版内容草稿

每天最终必须交付两个结果：

- 一份日报
- 一条草稿

如果时间不足，也必须完成简化版，不允许中断链条。

---

# 四、执行频率与时间建议

## 标准版

- 早上：30-45分钟，完成信息采集与初筛
- 中午或下午：20-30分钟，完成样本拆解
- 晚上：30-45分钟，完成仿版草稿
- 睡前：10分钟，完成复盘

## 极简版

如果当天很忙，最低执行标准：

- 采集3条信息
- 选1个题
- 拆1篇样本
- 写1个草稿提纲

原则：

**宁可缩短，不可断掉。**

# 五、Step 1：AI前沿信息采集 SOP

## 目标

每天从多个固定信息源里，拿到足够新、足够重要、足够适合我表达的AI信息。

## 信息源建议

至少固定 3 类来源：

### 1. 海外前沿源
- X / Twitter
- Reddit
- 海外AI产品发布页
- 海外技术博客

### 2. 中文内容源
- 公众号
- 小红书
- 即刻
- 知乎 / B站（按需）

### 3. 竞品与同类创作者源
- 我关注的AI博主
- 目标对标账号
- 做AI资讯 / 工具评测 / 方法论的账号

## 采集数量标准

每日最低采集：
- 海外信息 5 条
- 中文信息 5 条
- 同类创作者内容 3 条

合计最低：**13条原始信息**

如果时间不足，最低保底：**6条原始信息**。

## 采集判断标准

不是所有信息都值得记录。优先记录这4类：

1. **新模型 / 新工具 / 新功能发布**
2. **明显会影响创作者或普通用户工作流的变化**
3. **同类博主已经开始讨论，说明有传播潜力**
4. **能和我的方向结合：AI + 学习 / 内容 / 理工科 / 个人效率**

## 记录格式

每条原始信息按这个格式记：

```markdown
- 标题：
- 来源平台：
- 链接：
- 发生了什么：
- 为什么值得关注：
- 我能不能讲：能 / 不能
- 如果讲，我的角度是什么：
```

# 六、Step 2：AI前沿选题日报 SOP

## 目标

把原始信息整理成一份有判断的日报，而不是单纯搬运资讯。

## 日报输出结构

每天日报固定分四部分：

## 1. 今日最值得关注的3条AI信息

每条用以下结构写：

```markdown
### 1）信息标题
- 发生了什么：
- 为什么重要：
- 对谁有影响：
- 我自己的判断：
```

## 2. 今日最值得做的1个选题

固定输出：

```markdown
## 今日最值得讲的选题
- 选题名称：
- 选择原因：
- 最适合的受众：
- 我准备怎么讲：
```

## 3. 今日不值得做的题

这个部分很关键，用来训练判断力。

```markdown
## 今日不建议讲的题
- 题目：
- 不讲原因：信息太浅 / 不适合我 / 已经泛滥 / 难形成差异化
```

## 4. 今日输出动作

```markdown
## 今日执行动作
- 今天要仿哪一类内容：
- 今天要写哪条草稿：
- 今天的输出目标是什么：
```

## 日报质量标准

一份合格日报必须满足：

1. 不是罗列链接，而是有“为什么重要”的判断
2. 至少筛出一个明确可做选题
3. 明确指出一个不做的题，训练取舍
4. 能直接支撑晚上的内容输出

# 七、Step 3：仿版样本拆解 SOP

## 目标

找到一个结构已被验证的样本，不抄内容，只学结构。

## 样本选择标准

优先选这3类：

1. 同主题下数据已经验证过的爆款
2. 结构清晰、逻辑完整、容易看懂的内容
3. 风格和我未来方向兼容的内容

不要选：
- 纯靠人设强撑的内容
- 纯靠情绪发泄的内容
- 纯靠热点标题骗点击但正文空的内容

## 每日拆解维度

每天只拆1篇，但必须拆清楚这5件事：

### 1. 标题结构
记录：
- 它是资讯型、冲突型、判断型还是教程型？
- 有没有具体对象、具体结果、具体时效？

### 2. 开头钩子
记录：
- 前3句在干什么？
- 是制造冲突、制造焦虑、制造好奇，还是直接给结论？

### 3. 主体结构
记录：
- 它是怎么展开的？
- 是“结论先行 → 原因解释 → 例子 → 建议”
还是“问题 → 方法 → 效果”

### 4. 结尾动作
记录：
- 它最后让用户做什么？
- 是评论、关注、收藏，还是行动尝试？

### 5. 可复用骨架
最后必须提炼一版：

```markdown
## 这篇内容的结构骨架
1. 开头：
2. 中段1：
3. 中段2：
4. 结尾：
```

## 样本拆解记录模板

```markdown
# 样本拆解
- 标题：
- 平台：
- 链接：
- 为什么选它：

## 标题结构

## 开头钩子

## 主体结构

## 结尾动作

## 这篇的可复用骨架

## 我能怎么改成我的版本
```

# 八、Step 4：仿版内容输出 SOP

## 目标

基于一个已经验证过的结构，快速写出一条属于我自己的内容草稿。

## 核心规则

**仿的是结构，不是结论；仿的是节奏，不是原句。**

## 仿版输出流程

### 第一步：确定今天要讲的题
来源必须来自当天日报里的“今日最值得讲的选题”。

### 第二步：确定要借用的结构
从今天拆解的样本中，提炼一套骨架。

### 第三步：进行结构映射
把样本结构映射到我的主题上。

示例：

如果样本结构是：
- 开头先给结论
- 中间解释为什么
- 再给案例
- 最后给建议

那我今天也按这个结构写，但内容全部换成我的主题。

### 第四步：写草稿
草稿最低要求：
- 有标题
- 有开头
- 有主体结构
- 有结尾动作

## 仿版草稿模板

```markdown
# 今日仿版草稿

## 标题

## 开头
第一句直接给判断：
第二句解释为什么这件事值得关注：
第三句给读者一个继续看下去的理由：

## 主体
### 1. 这件事发生了什么

### 2. 为什么它重要

### 3. 它对谁最有影响

### 4. 如果我是读者，我下一步该怎么做

## 结尾
给一个明确动作：
- 收藏
- 关注
- 评论
- 去试
```

## 草稿质量标准

一条合格草稿必须满足：

1. 不是资讯搬运，而是有我的判断
2. 结构完整，不是碎片句子堆砌
3. 至少有一个明确受众
4. 至少给出一个明确动作建议
5. 能在后续基础上继续优化，而不是从零重写

# 九、Step 5：晚间复盘 SOP

## 目标

把每天的执行，沉淀成可迭代资产。

## 每天固定回答这5个问题

```markdown
# 今日复盘

1. 今天我看到了什么高价值信息？
2. 今天我最终选择讲什么？为什么？
3. 今天拆的样本，最值得复用的结构是什么？
4. 今天写草稿时，最卡的地方是什么？
5. 明天我要优化哪个环节？
```

## 复盘原则

- 不写空话
- 不写“今天挺好”这种无效总结
- 重点写：什么能复用、什么要修正

因为复盘的目标不是记录心情，而是：

**把今天的试错，变成明天的模板。**

# 十四、升级方向

当这套SOP跑顺之后，可以往下升级：

## 升级1：Agent化采集
把固定信息源交给Agent自动采集、整理、分类。

## 升级2：选题打分机制
给每个选题按以下维度打分：
- 新鲜度
- 传播性
- 与我定位匹配度
- 变现潜力

## 升级3：内容模板库
把不同类型样本沉淀成模板库：
- 快讯型
- 判断型
- 教程型
- 复盘型

## 升级4：复盘资产库
把每天复盘里重复出现的问题，沉淀成：
- 常见卡点清单
- 优秀开头模板
- 高转化标题模板
- 常用结尾动作模板

# 十五、最后的执行原则

这套SOP最重要的，不是格式，而是下面三条：

### 原则1
**先产出，再优化。**

### 原则2
**先仿结构，再谈原创。**

### 原则3
**先让闭环跑起来，再追求自动化和规模化。**

如果我每天都能稳定完成：
- 一份日报
- 一次拆解
- 一条草稿
- 一次复盘

那我就不是在“零散学AI”，而是在真正建立自己的内容与Agent双系统。
