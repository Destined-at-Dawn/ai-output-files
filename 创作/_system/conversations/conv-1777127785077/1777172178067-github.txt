  三、筛选规则
  ──────────────────────────────────────

  第一步：过滤非 AI 项目

  只保留项目名或描述中含以下关键词的项目（不区分大小写）：

  - 描述为空的项目
  - 今日 star 增量 < 50 的项目
  - 已在 sent_history.json 中的项目

  第二步：道法术器打分

  ┌──────┬──────┬────────────────────────────────────────────────────────┐

  按得分排序，取前 6 条，「道+法」至少 2 条。

  - 不足 2 条道法内容时，在消息顶部标注：⚠️ 今日高价值项目较少，仅 X 条道法内容

  过滤后可用项目为 0 条，立即停止，输出：
  「❌ 今日无符合质量阈值的 AI 项目，本次不发送」

  ──────────────────────────────────────
  四、提炼规则
  ──────────────────────────────────────

  每个项目写 2–3 句话，60–80 字，按以下结构：

  第一句：谁干了什么 + 最核心的数据或功能（直接给结论，不铺垫）
  第二句：为什么重要，或反常识的地方，或社区的关键反应——这句必须有，不能省
  第三句（可选）：补一个具体场景或对比，让读者感受到差距

  要求：
  - 主语必须是具体的团队/机构/模型名，禁止用「一个开源项目」「某团队」等抽象开头
  - 用反差制造张力：「只用笔记本就能跑的本地模型，推理速度不输云端 API」
  - 用具体数字：不说「性能大幅提升」，说「推理速度比 vLLM 快 3 倍、显存减少 40%」
  - 多用强动词：开源、击败、干掉、逆转、打破——而不是「实现了」「提供了」
  - 专有名词保留英文原貌：LLaMA、LoRA、RAG、vLLM、GGUF、SOTA 等
  - 引号用「」不用 ""

  禁止词：值得关注、引发热议、强烈推荐、令人震惊、非常、极其、有趣的是、总而言之、关键在于、在当今时代

  好的写法：
  ✅ ggerganov 把 DeepSeek-V3 的 4-bit 量化塞进了 llama.cpp，M2 MacBook 上跑 70B 模型推理速度达 18 token/s。这个数字以前只有 A100
  才能做到，现在一台笔记本就够了。

  ✅ vLLM 团队开源了新的 prefix caching 方案，相同 prompt 的重复推理成本直接砍掉 80%。社区炸了——有人说 RAG 应用的 API 费用从此减半。

  烂的写法：
  ❌ 一个开源项目支持了 DeepSeek-V3 的量化推理，性能表现不错。（主语抽象，没说为什么重要）
  ❌ 该框架提供了高效的 LLM 推理能力，今日新增 star 超 2k。（只有数字，没有张力）

  分类（四选一）：
  - 【道】底层原理 / 范式突破
  - 【法】方法论 / 框架 / 评测
  - 【术】实操 / 教程 / 工作流
  - 【器】工具 / 模型 / 平台
