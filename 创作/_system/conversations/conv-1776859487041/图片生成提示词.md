> 
> **通用风格要求**（请在每次生成时附加这段）：
> - 学术论文配图风格，白底，线条清晰，配色克制（蓝灰为主色，红色仅用于标注攻击/危险路径）
> - 无3D效果、无渐变、无阴影、无装饰性元素
> - 所有文字使用中英文对照（中文在上，英文小字在下），字号适合A4打印
> - 输出分辨率 ≥ 300dpi，适合打印
> - 格式：PNG或SVG

## 图3：主控状态机转换图

**在论文中的位置**：第6.4节"RTL设计"段末

**提示词**：

```
请绘制一幅Moore状态机转换图，用于学术论文配图。这是FPGA安全通信系统的主控状态机。

状态列表（6个状态，用圆角矩形表示）：
1. IDLE（初始状态，用双圈表示）
2. RECV
3. DECODE
4. CRYPTO
5. SEND
6. ERROR

转换条件（箭头上的标注）：
- IDLE → RECV：rx_done=1（收到完整帧）
- RECV → DECODE：帧CRC校验通过
- RECV → ERROR：帧CRC校验失败
- DECODE → CRYPTO：帧类型识别完成，启动加密操作
- CRYPTO → SEND：crypto_done=1（加密操作完成）
- SEND → IDLE：tx_done=1（发送完成）
- ERROR → IDLE：超时或手动复位

布局建议：
- 水平排列为主：IDLE → RECV → DECODE → CRYPTO → SEND，形成主回路
- ERROR放在主回路下方，RECV向下连到ERROR，ERROR向左连回IDLE
- 每个状态圆内写状态名，旁边小字标注该状态的输出动作：
  - IDLE：等待接收
  - RECV：锁存帧数据
  - DECODE：解析帧类型
  - CRYPTO：启动加密/解密/密钥交换
  - SEND：组帧并发送
  - ERROR：点亮错误LED

风格：学术论文配图，白底，状态用蓝色圆角矩形，转换箭头为黑色实线，条件标注为小号字体，IDLE用双圈标记。简洁清晰，无3D效果。
```
