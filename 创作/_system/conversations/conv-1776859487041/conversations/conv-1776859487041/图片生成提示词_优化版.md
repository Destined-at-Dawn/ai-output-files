# 课程论文配图提示词（教师示范优化版）

> **教师示范说明**：本版本针对学术论文配图标准进行了全面优化，适合作为教学示范。优化重点包括：1) 符合IEEE/学术出版规范；2) 增强技术细节准确性；3) 统一视觉风格；4) 提高可读性和打印质量。

> **通用风格要求**（**必须**在每次生成时附加）：
> - **学术论文配图风格**：白底（#FFFFFF），线条清晰锐利，无抗锯齿（矢量图优先）
> - **配色方案**：蓝灰色系为主（#2E5AAC, #4A6FA5, #7B9BC2），红色仅用于标注攻击/危险路径（#D32F2F）
> - **禁止元素**：无3D效果、无渐变、无阴影、无装饰性边框、无背景网格
> - **文字规范**：
>   - 中英文对照（中文在上，英文在下，括号内）
>   - 字体：Arial或Times New Roman系列
>   - 字号：标题10-12pt，标注8-10pt，小字6-8pt
>   - 所有文字必须清晰可读，A4打印后不模糊
> - **输出规格**：
>   - 分辨率：≥600 DPI（打印级）
>   - 格式：SVG（矢量图优先）或PNG（无损压缩）
>   - 尺寸：适合A4纸宽度（~170mm），高度按需调整
> - **线条规范**：
>   - 实线：正常数据流/连接（1.5pt）
>   - 虚线：虚拟/逻辑连接（1pt，间隔2pt）
>   - 点线：参考/辅助线（0.5pt，间隔1pt）
>   - 箭头：三角实心箭头，大小一致

## 图3：主控状态机转换图（优化版）

**在论文中的位置**：第6.4节"RTL设计"段末  
**学术意义**：展示Moore状态机的完整转换逻辑，体现时序控制

**优化要点**：
1. 状态机布局更符合学术规范
2. 转换条件标注更严谨
3. 状态输出明确分离

**提示词**：

```
请绘制一幅符合IEEE格式的Moore状态机转换图，用于FPGA安全通信系统主控状态机。

### 状态定义
**6个状态**（圆角矩形，蓝灰色系）：
1. **IDLE**：初始状态（双线框表示，深蓝#2E5AAC）
2. **RECV**：帧接收状态（中蓝#4A6FA5）
3. **DECODE**：帧解析状态（中蓝#4A6FA5）
4. **CRYPTO**：加密运算状态（中蓝#4A6FA5）
5. **SEND**：帧发送状态（中蓝#4A6FA5）
6. **ERROR**：错误处理状态（红色#D32F2F）

### 布局设计
- **主回路**：水平排列 IDLE → RECV → DECODE → CRYPTO → SEND → IDLE
- **错误分支**：ERROR状态置于主回路下方，RECV向下连接ERROR，ERROR向左连接IDLE
- **状态间距**：等距排列，中心距80mm

### 状态详细规范
每个状态圆角矩形（60×30mm）内包含：
- **状态名**：顶部居中，Arial 10pt加粗
- **状态编码**：右上角小字，如"S0"、"S1"（Arial 7pt）
- **输出动作**：矩形内部分区域，灰色背景（#F5F5F5），Arial 8pt
  - IDLE：等待接收，uart_rx_en=1
  - RECV：锁存帧数据，data_latch=1
  - DECODE：解析帧类型，opcode_decode=1
  - CRYPTO：启动加密/解密/密钥交换，crypto_start=1
  - SEND：组帧并发送，tx_start=1
  - ERROR：点亮错误LED，err_led=1，记录错误码

### 转换条件标注
所有转换箭头（黑色实线，1pt）上标注条件（Arial 8pt）：
- IDLE → RECV：rx_done == 1'b1 (帧接收完成)
- RECV → DECODE：crc_ok == 1'b1 (CRC校验通过)
- RECV → ERROR：crc_ok == 1'b0 (CRC校验失败) 或 timeout == 1'b1 (超时)
- DECODE → CRYPTO：frame_type_valid == 1'b1 (帧类型识别完成)
- CRYPTO → SEND：crypto_done == 1'b1 (加密操作完成)
- SEND → IDLE：tx_done == 1'b1 (发送完成)
- ERROR → IDLE：err_timeout == 1'b1 (超时) 或 manual_reset == 1'b1 (手动复位)

### 时序标注
- 状态转换时间：在每个转换箭头上方小字标注典型延迟
  - RECV→DECODE：< 10 clock cycles
  - CRYPTO→SEND：~ 1000 clock cycles (AES-128)
- 时钟域标注：图右下角标注"Clock: 12MHz (iCE40 FPGA)"

### 学术元素
- 图例框：右下角添加图例
  - 圆角矩形：状态
  - 实线箭头：正常转换
  - 虚线箭头：错误/异常转换
  - 红色状态：错误处理
- 引用标注：状态机设计参考[14] Baksi et al., 2022

### 输出要求
- 格式：SVG矢量图
- 线条：所有线条无抗锯齿，清晰锐利
- 文字：所有文字可编辑，打印后清晰
```

## ⚠️ 教师示范注意事项

### 1. 学术诚信强调
- **图4-8（波形图）**：AI生成的波形图**仅用于教学示范和占位**，学生必须理解：正式论文必须使用**真实仿真截图**。伪造数据是严重的学术不端行为。
- **引用标注**：所有参考他人工作的部分必须明确标注引用，如图1中的"[9]"。

### 2. 技术准确性验证
- 生成图片后，必须与报告中的技术参数核对：
  - 时钟频率：12MHz (iCE40 FPGA)
  - 波特率：125kbps
  - 算法参数：AES-128, SHA-256, ECDH P-256
  - 帧格式：Magic 0x55AA, CRC-16-CCITT

### 3. 图片插入规范（Word）
1. **删除占位文字**：删掉文档中的"【图X ...】"
2. **插入图片**：粘贴图片，右键→"插入题注"
3. **题注格式**：标签"图"，位置"所选项目下方"，编号包含章节号
4. **题注文字**："图X. XXX"（如"图1. 嵌入式设备安全通信应用场景拓扑"）
5. **字体**：题注五号宋体，居中
6. **图片格式**：嵌入型，居中，文字环绕方式为"上下型"

### 4. 教学示范价值
本优化版展示了：
- **道**：学术诚信和技术严谨性的核心原则
- **法**：系统化的图片优化方法论
- **术**：具体的提示词编写技巧和学术规范
- **器**：AI工具在学术写作中的合理使用边界

### 5. 学生常见错误提醒
1. **风格不一致**：不同图片使用不同配色、字体
2. **细节缺失**：缺少技术参数、单位、引用
3. **分辨率不足**：图片打印后模糊
4. **比例失调**：元素大小不合理，影响可读性
