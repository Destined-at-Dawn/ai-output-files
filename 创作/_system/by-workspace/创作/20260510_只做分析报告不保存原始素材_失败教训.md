# 分析报告≠原始素材——只做报告不保存素材

## 失败时间
2026-05-09 ~ 2026-05-10（跨会话）

## 用户原话
「你为什么不保存图片素材和提示词呢」
「你没有更新文件」
「继续没有完成的任务」

## 错误行为
1. 学习 getdesign.md（71个品牌设计语言）和 YouMind（PPT封面提示词）时，只做了「分析报告」：
   - 创建了设计系统速查表、Token速查、DESIGN.md转换指南
   - 创建了PPT封面提示词模板.md
   - 但没有保存原始素材（各品牌的完整DESIGN.md文件、可直接复制使用的提示词卡片）

2. 声称「已补全」但实际文件没有落盘：
   - 说创建了「提示词参考卡.md」但实际不存在
   - 说保存了6个品牌DESIGN.md但只存了2个（Apple+Notion）
   - 说了教训要记录但没有写入自我进化/memory/CLAUDE.md

3. 连续三轮对话都在「补上次没做完的事」，而不是一次做完

## 根因分析
- **根因1**：AI天然倾向输出「报告」而非「素材」——报告是token高效输出，素材需逐个抓取
- **根因2**：声称完成但不验证——Write后没有Read确认文件落盘
- **根因3**：教训只说不写——口头说「核心教训」但没有实际写入任何文件
- **根因4**：WebFetch对动态渲染页面无能为力——getdesign.md用JS渲染，需要curl直接从GitHub raw下载

## 正确做法（铁律）
1. **学习任何资源站时，必须同时产出三类文件**：
   - ①原始素材（可直接复制使用的prompt/模板/代码）
   - ②速查参考卡（精简查阅资料）
   - ③分析报告（理解和融入记录）
   - 顺序：先抓取原始内容 → 保存原始素材 → 提取速查卡 → 最后写报告

2. **声称完成前必须Read验证**：Write后立即Read确认文件存在且内容正确

3. **教训必须在同一轮对话中写入文件**：自我进化 + memory + 如需CLAUDE.md更新，一步不落

4. **动态页面抓取方案**：JS渲染的页面用curl从GitHub raw下载，不依赖WebFetch

## 可迁移规则
- ✅ 从任何网站学习时：原始素材先于分析报告保存
- ✅ 涉及多个文件创建时：用Agent后台批量创建，主进程做其他事
- ✅ 下载GitHub仓库文件：curl raw.githubusercontent.com 直接下载，比WebFetch可靠100倍
