# 2026-05-01 失败教训：说了但没做

## 用户原问题
用户要求：
1. "你把你的使用规则写进md文件"
2. "你先根据现有的牛马棚搭建出各个任务的使用路径"
3. "姐姐，你生产出来的成果不好时，我对你的成果表示否定，并给你改进意见的时候，你直接写进文件的那种果断。"

## 我的错误回复
我写了一大段文字，详细描述了9条Skill链式调用路径的内容——但是**没有调用任何文件创建工具**。回复里全是文字描述，没有一个 write_project_file / Write tool_use。

## 用户的反应
> "你的正式文件在哪里？你没有新建文件。我记得我在CLAUDE.md早就反映了这个问题，你怎么还犯？"

## 错误根因
**"说了但没做"**——把回复内容本身当成了交付物，而不是把文件当成交付物。

具体来说：
1. 用户说"写进md文件"→ 我理解为"在回复中描述方案"（错）
2. 用户说"直接写进文件的那种果断"→ 用户要的是 tool_use 一把落地，不要废话
3. CLAUDE.md §21.2 已经明确："每个操作必须带工具验证确认"+"禁止句式：已创建/已设置/已完成（没有验证证据）"

## 正确做法
1. 用户说"写进文件" → 立刻调 write_project_file，不要先回复文字描述
2. 文件创建后 → Read 验证落盘
3. 验证通过 → 再给用户总结改了什么
4. **顺序是：先做 → 再说。不是先说 → 再做（甚至说了不做）**

## 规则提炼
> ⚠️ **铁律：用户说"写进文件/创建文件/保存到..."时，必须在同一个回复中发起 tool_use 创建文件。纯文字描述 ≠ 文件创建。回复中没有 tool_use = 未完成。**
