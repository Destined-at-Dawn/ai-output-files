# IDEAS 管理系统

> **用途**：管理你的创作灵感、项目想法、研究思路
> **核心规则**：每次有新idea → **必须**创建或更新文件 → 旧版本**必须**移入archive

---

## 目录结构

```
IDEAS/
├── README.md             ← 本文件：系统说明
├── _version_index.md     ← 版本索引（自动维护）
├── active/               ← 当前活跃的idea
│   └── {主题简短名}/
│       └── v{数字}.md    ← 最新版
└── archive/              ← 归档的历史版本
    └── {主题简短名}/
        └── v{数字}.md    ← 旧版本
```

## 版本管理规则

| 操作 | 规则 |
|------|------|
| **新增 idea** | 在 `active/{主题}/` 创建 v1.md |
| **更新 idea** | 当前 v1 → 移入 `archive/{主题}/` → 创建 v2.md |
| **再次更新** | v2 → 移入 archive → 创建 v3.md |
| **完成/放弃** | 移入 archive，标记 `_done` 或 `_abandoned` |

**特别注意**：
- ❌ 不用 `ideas.md` 单一文件——多个idea必须分文件夹
- ❌ 不覆盖旧版本——所有旧版本必须保留在 archive 中
- ✅ 版本号递增：v1 → v2 → v3 ...
- ✅ 主题目录名用英文/拼音简短名

## 版本文件结构

每个 idea 版本文件统一结构：

```markdown
# {主题名} v{版本号}

- **创建时间**：YYYY-MM-DD
- **更新时间**：YYYY-MM-DD
- **状态**：活跃/变更中/已完成/已放弃

## 核心想法
（1-3句话描述这个idea的核心）

## 背景
（为什么有这个想法？什么触发的？）

## 具体内容
（展开描述想法，越具体越好）

## 下一步行动
（如果要推进这个idea，下一步要做什么）

## 关联资源
（相关的文件、链接、参考）
```

## 如何让AI帮你整理（强约束）

⚠️ **此规则对所有读取此文件的AI智能体具有强制约束力。以下触发词对应的操作是"必须执行"的指令，不是"可以执行"的可选操作。**

| 你说的话 | AI**必须**自动执行的操作 |
|---------|----------------------|
| **"我这有一个 idea"** / "有个想法" / "有个灵感" | ①读取 `IDEAS/active/` 查看有无已有文件 ②有→当前版移入 archive → 创建新版 ③无→在 active/ 创建新文件夹 + v1.md ④更新 `_version_index.md` ⑤同步更新 CLAUDE.md 中的相关路径引用 |
| **"帮我整理工作区结构"** / "整理文件" | ①扫描工作区所有文件夹 ②检查文件是否放错位置 ③检查空文件夹 ④给出整理报告+建议 |
| **"归档这个 idea"** | ①当前 active 内容移入 archive ②标记 `_done` 或 `_abandoned` ③更新索引 |
