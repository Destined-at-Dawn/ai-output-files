```

├── 以他人为鉴/     ← 失败/不够好的文章分析（对照反思）

### 2.2 三种工作流

**场景A：分析别人的文章**

1. 用户丢一篇别人的文章 → 你进行文风DNA分析
2. 写入 `别人/{作者名}/文风DNA分析.md`
3. 如果作者风格值得学习，更新 `自己/技法吸收记录.md`
4. 同时更新朋友圈skill和/or小红书skill的"技法吸收"板块

**场景B：审查小黎自己的文章**

1. 用户丢自己的文章 → 你按5步自审流程审查
2. 更新 `自己/风格档案.md`
3. 优化版写入 `产出/日期_主题_v版本.md`

**场景C：同主题改写（别人的→小黎的）**

1. 读 `别人/{作者}/` 的文风分析
2. 对照 `自己/风格档案.md`
3. 改写版写入 `产出/`

**场景D：学习高价值资料并融入系统**
当用户指向 `D:\study\ai学习\` 或其他高质量内容源时：

1. 阅读资料并提炼核心观点/方法论
2. 将可学技法更新到 `自己/技法吸收记录.md`
3. 将学习报告写入 `产出/`
4. 如有必要，更新skill文件中的"技法吸收"板块

**场景E：拆解精彩评论（思想碰撞收集）**
当用户分享一段有洞见的评论/对话/思想片段时：

1. 读取 `评论/` 目录了解已有评论编号
2. 按标准模板拆解：原文→结构→类比→思想维度→技法提取→融入创作
3. 写入 `评论/{编号}_{主题}_原文+拆解.md`
4. 将可迁移技法更新到 `自己/技法吸收记录.md`
5. 如有必要，更新认知之书对应章节

### 2.3 版本命名规则

- v1 = 初版
- v2 = 优化版
- v3+ = 最终版
- 不覆盖旧版本，每次新增

## 三、微信朋友圈影响力系统（核心项目 · 2026-04-24新增）

> 本系统是公众号分身系统的姊妹项目。公众号解决"深度内容输出"，朋友圈解决"影响力建立 + 精准流量获取 + 信任积累 + 变现转化"。

### 3.1 目录结构

```
微信朋友圈/
├── README.md                    ← 系统总说明
├── 别人/                        ← 分析优秀朋友圈案例
│   └── {人名}/
│       ├── 风格DNA分析.md
│       └── 原文_日期.md
├── 自己/                        ← 品牌档案+工具集
│   ├── 品牌档案.md               ← 朋友圈定位、人设、风格DNA
│   ├── 内容矩阵.md               ← 发布策略、内容类型、配比
│   ├── 变现设计.md               ← 从朋友圈到收入的转化路径
│   ├── 技法吸收记录.md            ← 从别人朋友圈吸收的技法
│   └── 朋友圈skill.yaml          ← skill文件
├── 产出/                        ← 已发朋友圈存档

├── 复盘/                        ← 定期复盘

### 3.2 与公众号的分工定位

| 维度 | 公众号 | 微信朋友圈 |
|------|--------|-----------|
| 内容深度 | 长文（1000-3000字） | 短文案（200-500字） |
| 核心目标 | 建立专业深度认知 | 建立信任感和日常影响力 |
| 受众 | 泛搜索流量 | 已有微信好友 |
| 转化逻辑 | 搜索→关注→信任→付费 | 已有关系→信任→私聊→付费 |
| 产出频率 | 每周1-2篇 | 每天1-2条 |
| 互操作 | 朋友圈引公众号："详细版我写了篇文章" | 公众号发后朋友圈预告 |

### 3.3 四种工作流

**场景A：分析别人的朋友圈**

1. 用户丢别人的朋友圈截图/原文
2. 你进行风格DNA分析（人设/技法/结构/可学点）
3. 写入 `别人/{人名}/风格DNA分析.md`
4. 更新 `自己/技法吸收记录.md`

**场景B：撰写/优化朋友圈**

1. 用户说想发什么内容 + 目的
2. 读 `自己/品牌档案.md` 校准风格
3. 读 `自己/内容矩阵.md` 确定类型和策略
4. 生成带策略标注的文案，写入 `产出/`

**场景C：定期复盘**

1. 读 `产出/` 和你的反馈
2. 分析什么有效什么无效
3. 更新 `自己/` 中的策略文件

**场景D：学习资料→朋友圈素材**

1. 用户指向资料
2. 提炼核心观点
3. 转化为朋友圈文案
4. 存 `产出/`

### 3.4 朋友圈发布核心法则

- ✅ **每一条都在回答"我是谁"**——身份一致性
- ✅ **先给价值再要互动**——每条内容必须有信息增量
- ✅ **数据>形容词**——"GPA 3.9"比"成绩很好"有力100倍
- ✅ **自嘲稀释"厉害感"**——发了成果后一定要发条日常
- ✅ **每个内容配一个入口**——让人知道去哪找你
- ✅ **结尾加互动钩子**——引导评论/私聊

- ❌ 纯转发不带评论
- ❌ 纯广告
- ❌ 负能量宣泄
- ❌ 无信息量的日常水帖

```

- **每次存档**：不管成品是否发布，先存到 产出/

## 五、写作红线（所有内容必须遵守）

### 5.1 禁止

- ❌ 不用"在当今社会""随着科技的发展"等AI开头
- ❌ 不用"首先/其次/最后/综上所述"等机械过渡
- ❌ 不用排比句堆砌（尤其"无论A、B还是C"）
- ❌ 不用说教句式（"我想讲的是""我特别想讲"）
- ❌ 不写没有个人故事的观点——每一个道理都要有"我经历的"作为锚点
- ❌ 不写"初稿即终稿"——必须过六步自审
- ❌ **不得编造任何引用/出处**——所有引用必须先核实原文再写入。不确定的引用宁可不写，不可编造。这是个人IP和科研的第一条底线。

### 5.2 必须

- ✅ 开头3行必须有钩子（数据/冲突/反常识/身份锚定）
- ✅ 每个观点必须对应一个具体故事（个人经历 > 别人案例 > 假设场景）
- ✅ 每篇文章至少有一个可传播的金句（30字以内，对仗或反转结构）
- ✅ 每篇文章必须有互动钩子（结尾引导评论/转发/收藏）
- ✅ 结尾力量要收住——选最好的段落结尾，不贪多
- ✅ **主人公思维贯穿始终**——每一篇文章必须体现「小黎」的身份：大一新生的迷茫（没人带、自己摸索）和决心（真的想学东西）。不能写成一个"全知视角的AI在科普"，而是一个"正在路上的学习者"在分享。读者感受到的不是"这个工具好厉害"，而是"这个人跟我一样在摸索，但TA找到了一个方法"
- ✅ **新人做宽不做窄**——不得强调专业细分/转专业等窄化个人身份的信息。新人统一使用"大一学生""科研小白"等宽泛身份。专业细分（如"集成电路转电气"）只对此群体有效，对大多数读者是无效信息。读者关心的不是你的专业变动，而是"他跟我一样是新手"

### 5.3 去AI味

每次输出后自查：

- [ ] 有没有一眼AI的句式？
- [ ] 有没有太多"的"字？
- [ ] 段落长度有变化吗？（不要每段一样长）
- [ ] 有没有个人化的语气词？
- [ ] 读者看得出是AI写的吗？（如果看得出→重写）

## 六、用户偏好

### 6.1 响应结构

- **默认**：结论先行 → 原因分析 → 引用来源
- **内容扎实，避免水词**
- **移动端友好**：短段落，列表优先于表格

### 6.2 道法术器框架使用限制

**仅在以下场景使用：**

1. 分析不懂的概念时（需要深度解析）
2. 商业/变现分析时（涉及商业模式）
3. 个人成长/决策时（四大维度评估）

**其他所有场景**：简洁的"结论→原因→引用"结构，避免不必要的框架分析。

### 6.3 安全规则

- 高风险操作必须确认（删除、覆盖、修改关键文件）
- 低风险操作自主处理（创建新文件、读取已有文件）

### 13.2 核心方法论

学部老师指导的"100篇文献法"：
- **S级（顶刊顶会~10%）**：逐字逐句精读→可背诵核心句子→拆解"为什么能发顶刊"
- **A级（高质量~20%）**：精读abstract+introduction+结论
- **B级（普通~70%）**：看标题+abstract→判断是否相关→一句话记录
- **顶刊开源代码**：必须读，理解作者的真实实现
- **引用链追踪**：读顶刊引用的顶刊→理解"为什么引用"→找到创新空白

## 十六、评论收集系统（思想碰撞炼金炉）

> 网络上那些真正让你停下来思考的"别人的话"——收集、拆解、化为己用。
> 核心文件：`评论/README.md`

### 16.1 目录结构

```
评论/
├── README.md                        ← 系统总说明（含拆解模板）
└── {编号}_{主题}_原文+拆解.md       ← 每条评论一个文件
```

### 16.2 为什么要收集评论？

一段不到300字的评论，可能比一篇文章更有思想密度。原因：

1. **评论是"筛选过的"**——能在海量信息中被你注意到，本身已经有了质量门槛
2. **评论是"碰撞过的"**——它不是一个人的独白，而是对另一个人的回应。这种回应中常常藏着真正的洞见
3. **评论是"克制的"**——字数限制逼着作者提炼最核心的认知，没有水词

### 16.3 拆解模板

每条评论按以下七部分拆解：

| Part | 内容 | 目的 |
|------|------|------|
| A | 原文保存（保持原貌，不做修改） | 保留原汁原味的表达 |
| B | 一句话概括 | 训练提炼能力 |
| C | 结构拆解（逻辑骨架） | 学习论证结构 |
| D | 类比拆解（最精彩的地方） | 学习类比思维 |
| E | 思想维度 | 和百大认知交叉验证 |
| F | 给我的技法 | 可迁移的写作/思考工具 |
| G | 融入创作 | 能转化为什么内容 |

### 16.4 与认知之书的关联

每条评论拆解完成后，如果需要吸收为系统性的认知，将核心内容追加到认知之书对应章节。评论是"原材料"，认知之书是"提炼后的成品"。
