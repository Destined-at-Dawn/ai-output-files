# SOP-02 考前复习

> **触发条件**：用户说"复习/考试/期末/期中/考前/冲刺"
> **认知科学支撑**：检索式练习（009-认知天性）、测试效应（009-认知天性）、元认知监控（023-认知觉醒）

---

## 流程

### Step 0: 进水管

1. Read 课程 `project.md` → 确认哪些章节已学/未学/标记了薄弱点
2. Read `memory/MEMORY.md` → 查本课程历史教训和薄弱点
3. 确认考试范围和时间 → 计算可用复习天数

### Step 1: 诊断（哪里薄弱）

**检索式练习法（不看笔记回忆）：**
1. 列出考试范围内所有章节
2. 对每章，让用户回忆核心概念和公式（不看任何材料）
3. 标记回忆程度：
   - ✅ 完全记得（跳过）
   - ⚠️ 模糊（重点复习）
   - ❌ 完全忘了（紧急复习）

输出「薄弱点热力图」：
```
章节        回忆程度    优先级
第1章 极限   ⚠️ 模糊    ★★★★
第2章 导数   ✅ 清楚    ★
第3章 积分   ❌ 忘了    ★★★★★
```

### Step 2: 制定复习计划

根据诊断结果，按优先级分配时间：
1. **紧急**（❌ 忘了）→ 重新学习 + 大量练习
2. **重点**（⚠️ 模糊）→ 检索练习 + 查漏补缺
3. **轻松**（✅ 清楚）→ 快速过一遍 + 1-2道综合题

### Step 3: 分章节复习

对每个需要复习的章节：
1. **先测**：出 3-5 道题，不给答案，用户先做
2. **再补**：做错的题 → 讲解 + 关联知识点回顾
3. **再测**：换 2 道题，验证是否真懂了

> **⚠️ 关键**：不是"重新看书"，而是"测试 → 补洞 → 再测试"。检索式练习效果 > 重复阅读。

### Step 4: 模拟考试

1. 出一份覆盖所有章节的模拟试卷（题量和难度对标真实考试）
2. 计时完成
3. 批改 + 逐题分析
4. 标记最终仍薄弱的点 → 考前最后过一遍

### Step 5: 考前最终检查

考试前最后一次：
1. 快速回忆所有公式（不看表）→ 对比公式表查漏
2. 回顾所有错题的解题模式
3. 心理建设：告诉用户已覆盖了多少内容

### Step 6: 考后复盘（SOP-04 接管）

考试结束后：
1. 对比考试题和复习覆盖度
2. 分析哪些题复习时漏掉了
3. 更新 `project.md` 和 `memory/`

## 时间分配模板（考前7天）

| 天数 | 内容 | 时间分配 |
|------|------|---------|
| Day 1 | 诊断（检索式练习找薄弱点） | 2h |
| Day 2-3 | 紧急章节重新学习 | 3h/天 |
| Day 4-5 | 重点章节查漏补缺 | 3h/天 |
| Day 6 | 模拟考试 + 批改分析 | 3h |
| Day 7 | 考前最终检查（公式+错题） | 1.5h |

## 反模式

| 错误做法 | 正确做法 |
|---------|---------|
| 从第一页开始重看教材 | 先诊断薄弱点，重点突破 |
| 只看不练 | 测试 → 补洞 → 再测试 |
| 考前熬夜临时抱佛脚 | 提前7天启动，每天3小时 |
| 做完题对答案就走 | 逐题分析错因，标记解题模式 |
| 只做简单题找自信 | 优先做难题（合意困难） |
