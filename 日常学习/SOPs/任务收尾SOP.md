# SOP-07: 任务收尾五步法

> **铁律**：mark_task_done 是收尾的最后一步，不是第一步。
> 不完成收尾五步，任务不算结束。

---

## 一、为什么需要收尾 SOP

### 设计意图
- 项目内 `outputs/` = 工作台（中间产物、草稿、过程文件）
- 根目录 `outputs/` = 成果库（终态产物 + 心得 + SOP 改进日志）

### 根因问题
任务"完成"的定义里不包含"产物归档到成果库"，导致：
- 中间产物留在项目 outputs/ 没人管
- 心得体会没有写
- SOP 改进日志没有更新
- 根目录 `outputs/` 一直是空的

---

## 二、收尾五步（强制执行）

### Step 1: 产出提炼

**动作**：从 `projects/{课程}/outputs/` 中筛选最终产物

**规则**：
- 只提炼"可交付"的成果（解题笔记、学习总结、分析报告）
- 过程草稿、调试日志、临时文件不提炼
- 每个任务提炼 1-3 个核心产物

**输出**：
```bash
outputs/{日期}-{任务名}/
├── 产物1.md          # 最终交付物
├── 产物2.md          # 可选的补充产物
└── ...
```

**命名规范**：
- 日期格式：`YYYY-MM-DD`
- 任务名：简洁、可检索（如"戴维南定理学习"、"高数期中复习"）
- 完整示例：`outputs/2026-06-22-戴维南定理学习/`

---

### Step 2: 心得沉淀

**动作**：写 `outputs/{日期}-{任务名}/心得体会.md`

**内容要求**（至少包含 3 项）：

```markdown
# 心得体会

## 踩过的坑
- [坑1]：[具体描述] → [怎么解决的]
- [坑2]：...

## 有效的方法
- [方法1]：[为什么有效] → [适用场景]
- [方法2]：...

## 下次改进
- [改进1]：[具体怎么改]
- [改进2]：...

## 时间分配（可选）
- 实际用时：X 小时
- 预估用时：Y 小时
- 偏差原因：...
```

**写作时机**：
- 可以在任务过程中随时记录到 `memory/{日期}.md`
- 收尾时从 memory 中提炼、结构化

---

### Step 3: SOP 改进日志

**动作**：写 `outputs/{日期}-{任务名}/SOP改进日志.md`

**核心问题**：
> 这个任务结束后，我们的技能/SOP 得到了哪些改善？

**内容模板**：

```markdown
# SOP 改进日志

## 本轮验证有效的 SOP
- [SOP 名称]：[哪个步骤被验证有效] → [为什么有效]
- ...

## 本轮需要修改的 SOP
- [SOP 名称]：[哪个步骤有问题] → [怎么改] → [改完后的预期效果]
- ...

## 本轮新增的规则/SOP
- [规则名称]：[具体内容] → [触发条件] → [预期效果]
- ...

## 本轮发现的死路（负结果）
- [死路描述]：[为什么走不通] → [退出判据] → [正确做法]
- ...
```

**联动机制**：
- "需要修改的 SOP" → 写入 `self-evolution/lessons.md`
- "新增的规则" → 写入 `.claude/rules/` 或对应 SOP 文件
- "死路" → 写入 `negative-results.md`（如有）

---

### Step 4: 索引更新

**动作**：更新 `outputs/_INDEX.md`

**索引格式**：

```markdown
# 成果库索引

> 最后更新：{日期}

## 按时间倒序

| 日期 | 任务名 | 课程/主题 | 关键词 | 产物数 |
|------|--------|----------|--------|--------|
| 2026-06-22 | 戴维南定理学习 | 电路分析 | 戴维南、等效电路、最大功率 | 2 |
| 2026-06-20 | 高数期中复习 | 高等数学 | 极限、导数、积分 | 3 |
| ... | ... | ... | ... | ... |

## 按课程分类

### 电路分析
- [2026-06-22-戴维南定理学习](2026-06-22-戴维南定理学习/)
- ...

### 高等数学
- [2026-06-20-高数期中复习](2026-06-20-高数期中复习/)
- ...

### 大学物理
- ...

### 大学英语
- ...
```

**更新规则**：
- 每次收尾必须更新索引
- 索引是成果库的"目录"，方便检索

---

### Step 5: mark_task_done

**动作**：调用 `mcp__project-tools__mark_task_done`

**前置条件**（必须全部满足）：
- [ ] Step 1 完成：产物已提炼到 `outputs/{日期}-{任务名}/`
- [ ] Step 2 完成：心得体会已写
- [ ] Step 3 完成：SOP 改进日志已写
- [ ] Step 4 完成：索引已更新

**调用示例**：
```json
{
  "task_id": "task-xxx",
  "output_files": [
    "outputs/2026-06-22-戴维南定理学习/产物1.md",
    "outputs/2026-06-22-戴维南定理学习/心得体会.md",
    "outputs/2026-06-22-戴维南定理学习/SOP改进日志.md"
  ]
}
```

**铁律**：
- 不填写 `output_files` 参数 = 违规
- output_files 必须包含至少 1 个产物 + 心得体会 + SOP 改进日志

---

## 三、收尾质量门禁

### 门禁检查清单（每个任务收尾前必过）

- [ ] 产物已提炼（Step 1）
- [ ] 心得体会已写（Step 2）
- [ ] SOP 改进日志已写（Step 3）
- [ ] 索引已更新（Step 4）
- [ ] mark_task_done 已调用且填写了 output_files（Step 5）

### 质量标准

| 维度 | 标准 | 违反后果 |
|------|------|---------|
| **完整性** | 五步全完成 | 缺一步 = 任务未完成 |
| **真实性** | 心得体会必须有实际内容（不能是空话） | 空话 = 教训丢失 |
| **可检索性** | 索引关键词覆盖核心概念 | 无关键词 = 找不到 |
| **可追溯性** | 产物路径在 mark_task_done 中正确填写 | 路径错误 = 产物丢失 |

## 四、与现有机制的联动

| 现有机制 | 如何联动 |
|---------|---------|
| **mark_task_done** | 从"结束动作"变成"收尾最后一步" |
| **memory/{日期}.md** | 心得体会的草稿先写这里，收尾时提炼到 outputs/ |
| **self-evolution/lessons.md** | SOP 改进日志的内容同步到这里 |
| **li-improve** | 收尾 SOP 的执行质量由 li-improve 监控 |
| **li-manage** | 成果库的结构治理由 li-manage 负责 |

---

## 五、收尾节律

### 即时收尾（任务完成后立即执行）
- 每个任务完成后，必须在**同一轮对话**内完成收尾五步
- 不允许"先标记完成，后面再收尾"

### 定期审计（每月一次）
- 检查是否有任务标记完成但没有产出文件
- 检查成果库的索引是否完整
- 检查心得体会的质量（是否有空话）

---

## 六、反模式（绝对禁止）

| 反模式 | 后果 | 正确做法 |
|--------|------|---------|
| ❌ 先 mark_task_done 再收尾 | 任务"完成"但产物丢失 | ✅ 收尾五步全完成后再 mark_task_done |
| ❌ 心得体会写空话 | 教训丢失，下次重复踩坑 | ✅ 必须有具体的坑、方法、改进 |
| ❌ 不更新索引 | 成果库无法检索 | ✅ 每次收尾必须更新索引 |
| ❌ 产物路径填写错误 | 产物"消失" | ✅ 路径必须是 `outputs/{日期}-{任务名}/xxx.md` |
| ❌ 中间产物不筛选全部提炼 | 成果库变成垃圾场 | ✅ 只提炼可交付的终态产物 |

## 八、示例：一个完整的收尾流程

### 场景：完成"戴维南定理学习"任务

**Step 1: 产出提炼**
```bash
# 从项目 outputs/ 筛选最终产物
cp projects/20260603-电路分析/outputs/戴维南定理解题笔记.md \
   outputs/2026-06-22-戴维南定理学习/产物1.md

cp projects/20260603-电路分析/outputs/戴维南定理思维导图.md \
   outputs/2026-06-22-戴维南定理学习/产物2.md
```

**Step 2: 心得沉淀**
```markdown
# 心得体会

## 踩过的坑
- [等效电阻计算]：忘记断开负载电阻 → 导致结果错误
  → 解决：画等效电路图时，先断开负载再计算

## 有效的方法
- [分步验证法]：先求开路电压，再求等效电阻，最后验证
  → 为什么有效：每步都可以独立验证，错误容易定位

## 下次改进
- 计算前先画完整电路图，标注已知量和未知量
```

**Step 3: SOP 改进日志**
```markdown
# SOP 改进日志

## 本轮验证有效的 SOP
- SOP-01 八步解题格式：Step 3（画图）被验证为关键步骤
  → 为什么有效：画图能直观发现错误

## 本轮需要修改的 SOP
- 无

## 本轮新增的规则/SOP
- 新增规则：计算戴维南等效电阻时，必须先断开负载
  → 触发条件：遇到戴维南定理题目
  → 预期效果：避免低级错误

## 本轮发现的死路（负结果）
- [死路]：不断开负载直接计算等效电阻
  → 为什么走不通：负载电阻会并联/串联影响结果
  → 退出判据：等效电阻结果与预期不符时，检查是否断开负载
```

**Step 4: 索引更新**
```markdown
# 在 outputs/_INDEX.md 中追加

| 2026-06-22 | 戴维南定理学习 | 电路分析 | 戴维南、等效电路、最大功率 | 2 |
```

**Step 5: mark_task_done**
```json
{
  "task_id": "task-1781073144163-xxx",
  "output_files": [
    "outputs/2026-06-22-戴维南定理学习/产物1.md",
    "outputs/2026-06-22-戴维南定理学习/产物2.md",
    "outputs/2026-06-22-戴维南定理学习/心得体会.md",
    "outputs/2026-06-22-戴维南定理学习/SOP改进日志.md"
  ]
}
```
