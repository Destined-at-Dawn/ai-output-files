# SOP: 复习PDF制作 Pipeline

## 结论

复习 PDF、章节图片、A4 速查和模拟卷必须走 `source-first exam mode`：先建资料索引和原题数据库，再生成原资料截图证据、题型模板、A4 速查、模拟卷、image2 逐页图片，最后用 LaTeX 合成复习 PDF。旧图片和旧 PDF 只能作为版式参考，不能作为事实来源。

**证据展示逻辑（高要求复习PDF）：原资料对应区域截图 / AI视觉融合展示 > Source 行号。** Source: 文件路径#行号 只保留为辅助追溯；真正放进 PDF 的是教材/PPT/习题/真题 PDF 对应部分的截图，或裁切/放大/标注/融合后的版本。禁止只给 Source 行号而不提供原资料截图证据。

**成品验收门槛**：高要求复习 PDF 若没有先用 image2 生成逐页图片，或页面主体没有嵌入原资料截图/AI视觉融合证据，只能标为“结构草稿/索引草稿”，不能标为最终成品。Source 行号不能替代截图证据，也不能作为页面主要内容。

## Source-First Exam Mode

### P0：资料索引先行

必须先生成或更新：

- `outputs/00_资料索引.md`
- `outputs/chapter_resource_index.md`
- `真题库/final_questions_status.md`（存在最终复习题或红线题时）

要求：

- 扫描 PDF/PPT/习题/答案/教材/往届题/图片。
- 标出资料类型、对应章节、是否在考试范围、可用程度。
- 无法确认页码/题号时写 `Source: 相对路径#位置待确认`。
- 红线题必须渲染页面目检，不能只靠 OCR。
- 对重要题目、公式、老师重点和易错判断，必须截取原资料 PDF/图片对应区域，保存到 `outputs/source_snippets/` 或章节输出目录的 `source_snippets/`。截图是核心证据，Source: 行号只是辅助追溯标签。

Source: ..\CLAUDE.md#红线题和生成稿规则

### P1：原题数据库

在任何题型模板或 PDF 页面生成前，必须先建立或更新：

- `outputs/05_教材原题与习题原题索引.md`
- 对应章节 `习题/exercise_index.md`

每道题必须记录：

```markdown
## 原题 ID：CHxx-Qxxx

- 章节：
- 来源类型：最终复习题 / PPT例题 / 习题 / 习题答案 / 教材例题 / 教材课后题 / 往届题
- 来源文件：
- 页码 / 幻灯片 / 题号：
- 原资料截图：
- 页面证据位置：
- 题目简述：
- 是否保留：保留 / 不保留 / 待确认
- 是否有答案：
- 标准解法摘要：
- 考点：
- 题型：
- 原题复现概率：高 / 中 / 低
- 易错点：
- Source:
```

Source: ..\AGENT.md#五、每道题记录格式

### P2：题型模板与陷阱表

原题数据库完成后，才能生成：

- `outputs/04_计算题题型总表.md`
- `outputs/03_选择判断易错点.md`
- `outputs/07_第11章_第13章综合大题专题.md`
- `outputs/08_第16章选择题专题.md`

题型模板按下面结构整理：

- 识别信号：看到什么条件就判断为该题型。
- 核心方法：一句话说明解法。
- 标准步骤：闭卷考试可复写的步骤。
- 必背公式：写明使用条件。
- 对应原题：必须绑定 CHxx-Qxxx。
- 易错点：从答案解析和错题模式提炼。

Source: ..\CLAUDE.md#必走流程

### P3：压缩材料

题型模板完成后，才能生成：

- `outputs/09_考前A4速查表.md`
- `outputs/10_模拟卷_含答案解析.md`
- `outputs/1小时吃透版.md`
- 章节逐页图片
- LaTeX 合成 PDF

要求：

- A4 速查只收录高频题型和小题陷阱，不写教材摘要。
- 模拟卷必须贴合选择 10 题 20 分、判断 10 题 10 分、计算 7 题 70 分。
- 章节逐页图片必须用 image2 生成，可以参考旧图的紫色分块、表格结构、红色核心提醒，但事实必须来自 P0/P1/P2。
- 每页的关键结论、公式或题型判断，必须嵌入原资料截图证据（裁切/放大/降噪/标注/融合后的版本）；空间不足时至少放”原资料局部截图 + 简短提炼”，绝不允许只放 Source 行号。
- 允许对原资料截图做 AI 视觉融合：裁切、放大、降噪、标注重点、与新排版合成；禁止改变题目条件、数值、图形关系和老师红线含义。
- 最终 PDF 必须由已审查通过的逐页图片通过 LaTeX 合成；Markdown 只允许作为内容草稿或审稿稿，不作为最终 PDF 的默认成品路径。
- 若页面只是重新排版公式、题型和 Source 文本，而没有展示原资料对应区域截图，应退回到 P0 截图步骤重做；不得用“已有 Source 可追溯”作为放行理由。

## 资料优先级

1. 最终复习题、老师划重点、明确考试说明。
2. PPT 例题和老师强调内容。
3. 习题答案。
4. 习题原题。
5. 教材例题与教材课后题。
6. 教材正文概念。
7. AI 补充解释，必须标注“补充内容，非资料原文”。

冲突时优先考试侧材料，并保留冲突说明；不要替资料强行统一。

## 旧图与旧PDF使用规则

- 旧图/旧 PDF 只参考视觉风格、表格结构、内容偏好。
- 不继承旧图里的题号、页码、结论、答案。
- 旧图中的“对应往届题位置”等信息必须回到真实资料核对后才能写入。
- 若旧图风格与当前资料冲突，以当前资料为准。

## PDF / 图片制作流程

1. 读取 `CLAUDE.md`、`AGENT.md`、本 SOP、`outputs/00_资料索引.md`。
2. 确认目标章节范围和降权内容。
3. 从教材、PPT、习题、答案中抽题，写入原题数据库。
4. 从原资料 PDF/图片中截取对应区域，建立 `source_snippets/`，并在原题数据库中绑定截图路径；截图必须覆盖题干、图形、老师红线/标注、答案关键步骤或公式出处。
5. 按题型用 image2 生成逐页图片版面；每页必须是可独立审查的图片，且主体视觉证据优先来自 `source_snippets/`。
6. 页面中优先放原资料截图或 AI 融合后的原资料局部，让用户直接看到“原来资料长什么样”；Source 只放页脚或角落作为辅助追溯。
7. 先输出逐页图片给用户审查。
8. 用户审查通过后，用 LaTeX 将图片逐页合成为 PDF。
9. PDF 生成后渲染抽检，确认无乱码、重叠、上下标错误、图片裁切和页边距错误。
10. 重要产物完成后写入 `memory/YYYY-MM-DD.md`。

## 禁止事项

- 禁止跳过资料索引和原题数据库直接做图片或 PDF。
- 禁止把旧图、旧样章、AI生成稿当事实主源。
- 禁止把 Markdown -> HTML -> PDF 作为高要求复习 PDF 的默认成品链路。
- 禁止只给 Source 行号而不提供原资料截图证据。
- 禁止把 Source 表格、路径列表、索引页包装成高要求 PDF 成品。
- 禁止 AI 融合时改写原资料的题干、数值、图形关系、红线标记和老师重点含义。
- 禁止编造页码、题号、老师重点。
- 禁止把超出考试范围内容放进重点复习页。
- 禁止把第16章做成复杂计算题重点。
- 禁止不同频率相量直接合并。
- 禁止混用最大值和有效值。
