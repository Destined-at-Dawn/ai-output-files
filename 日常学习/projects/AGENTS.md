## 高要求复习 PDF 通用继承规则

- 任何学科只要用户要求“复习 PDF / 期末冲刺 / 1小时吃透 / A4速查 / 原题截图融入 / 题型模板”，都必须读取 `SOPs\高要求复习PDF制作SOP.md`。
- 高要求复习 PDF 不是基础知识点摘要，必须同时服务：一小时吃透、概念辨析、小题速通、大题必会模板、拔高题、A4考前速查、答案附页和原资料截图证据。
- 页面顺序必须先给思维导图，再按重点难点展开：联系前置知识/后续章节/本章概念网 -> 教材定义 -> 易混辨析 -> 1-2道选择或判断自测题 -> 答案放附页。
- 题源优先级：老师最终复习/划重点 > PPT例题 > 习题答案 > 习题原题 > 教材例题/课后题 > 往届真题 > AI补充解释。越新、越贴近期末的题源权重越高。
- 原题必须以截图或视觉融合片段进入页面；Source 行号只能做页脚追溯，不能替代原题截图。
- `image2` 在本工作流中指 Codex 内置 `image_gen` 图片生成能力；高要求复习页必须由 `image_gen` 直出完整成品图，不能用 PIL/Matplotlib/普通脚本/HTML/ReportLab/LaTeX/字体层排版冒充。
- 图片页必须实际调用 `image_gen` 生成，并在产物记录中写明 image_gen 调用、页图目录、最终 PDF 和总览图路径。
- 公式页优先通过精确 prompt 控制 image_gen，降低单页公式密度；若文字或公式明显错误，优先重写 prompt 整页再生，不默认用 LaTeX/字体层修补。
- 若用户要求“先给图片确认”，则未确认图片前不能生成最终 PDF；若用户明确要求“直接生成所有图片和最终 PDF”，则可在图片生成后直接做图片装订 PDF。

## 根目录行为规则

- 根目录 `AGENTS.md` 只做入口路由，不复制各项目完整规则。
- 项目内规则与根目录入口冲突时，以项目内 `AGENT.md`、`CLAUDE.md`、SOP 为准。
- 更新项目级 `AGENT.md`、`CLAUDE.md`、SOP、memory 前，先读取原文件，使用精确追加或局部修改，避免盲目覆盖。
- 高要求复习 PDF 禁止默认走 Markdown -> HTML -> PDF；必须先用 `image_gen` 直出每一页图片，再只做图片装订 PDF。
- 高要求复习 PDF 的核心证据不是行号 Source，而是原资料 PDF/图片中对应区域的截图；能截图就截图，必要时把原资料截图用 AI 视觉方式融合进页面供用户直接看。
