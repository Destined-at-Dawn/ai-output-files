## 最近教训（按时间倒序）

### 教训001 — 跨区通用：声称完成但未验证 [严重]
- **场景**：任何执行-验证场景
- **错误/做法**：基于操作记录回复"已修复"，没有实际验证文件状态
- **后果**：用户发现实际未完成，信任度下降
- **正确做法**：任何"已完成"声明必须附带验证结果（Read/ls确认）
- **跨区通用**：是
- **来源**：2026-05-22 创作工作区实战

### 教训002 — 跨区通用：教训只记不改 [严重]
- **场景**：教训沉淀流程
- **错误/做法**：教训写入 memory 但没有回写 self-evolution/SOP
- **后果**：自我进化系统断裂，同类教训反复出现
- **正确做法**：教训沉淀四步闭环——memory → negative-results → self-evolution → SOP
- **跨区通用**：是
- **来源**：2026-05-27 创作工作区实战（2次被纠正）

### 教训003 — 跨区通用：文档代替执行 [中等]

### 教训004 — 日常学习区专属：记录 ≠ 建设 [严重]
- **场景**：用户提出新的工作规则/偏好
- **错误/做法**：只写了一个 memory 文件就声称"已记录"，没有建 SOP、没有建目录结构、没有建自我进化文件
- **后果**：用户当场抓包——"也没有建立对应的SOP也没有建立对应的文件夹，也没有建立对应的自我进化文件，你就在敷衍我"
- **正确做法**：用户提出规则时，必须同步完成四件事：
  1. memory/{日期}.md 记录
  2. SOPs/ 对应 SOP 文件
  3. self-evolution/ 更新（lessons + evolution-log + calendar）
  4. 目录结构完整（归档/、outputs/ 等）
- **跨区通用**：是（任何工作区都适用）
- **来源**：2025-06-05 日常学习工作区实战

### 教训005 — 跨区通用：并发 Agent 429 不降级 [严重]
- **场景**：需要多步骤信息收集，使用 Agent 工具并发调用
- **错误/做法**：并发 Agent 返回 429 限流后，继续等待重试，浪费整轮对话
- **后果**：1 轮对话零产出。同日连续 2 次犯同一错误（未从第 1 次学习）
- **正确做法**：
  1. Agent 返回 429 → 立即放弃所有并发 Agent
  2. 切换为顺序直接工具调用（Read/Grep/Bash）
  3. 保持任务目标不变，只变执行方式
  4. 审计中如实报告降级事件
- **关键判断**：大多数任务不需要并发 Agent，顺序直接调用 3-5 步就能完成
- **跨区通用**：是（任何工作区使用 Agent 工具时都适用）
- **来源**：2026-06-10 日常学习工作区（工作区全量扫描任务）
- **升级**：已升级为共享规则 R19（知识中枢/02-共享规则/19-agent-concurrency-fallback.md）
- **认知科学**：沉没成本谬误（等待=被已投入时间绑架）、计划谬误（默认假设"一切顺利"）

### 教训006 — 跨区通用：对外交付未脱敏 [严重]
- **场景**：打包本机 skill/SOP/文档交付给外部对象（朋友/客户/开源/上传）
- **错误/做法**：完整复制本机 li-* 的 SKILL.md 到交付包，内嵌 `C:/Users/13975`、`E:\ai产出文件`、真实姓名、考研、telos 等可反推身份信息
- **后果**：若直接发送 = 隐私泄露（脱敏扫描命中 9 个文件）
- **正确做法**：对外交付前过脱敏门禁——dry-run 扫描 → apply 替换 → 重扫 RESIDUAL=0 → 删构建临时报告 → 人工复核命中 Top3
- **跨区通用**：是（任何工作区对外交付都适用）
- **来源**：2026-06-19 日常学习区（给超算中心朋友打包高密度学习材料资料包）
- **升级**：已升级为规则 `.claude/rules/outbound-delivery-sanitization.md`
- **认知科学**：WYSIATI（只看到"复制就行"，没看到副本里的隐藏信息）、损失厌恶（先扫描确认再发送，不可逆操作前先验证）
