## 二、执行步骤

### Step 1: 信息吸收
- 读取参考素材（实验PDF/参考代码/用户提供的示例）
- **PDF读取**：先 `mcp__skill-handler__Skill skill: "pdf"` 读指南 → 用 pdfplumber 提取 → 输出到 `outputs/`
- **Word读取**：先 `mcp__skill-handler__Skill skill: "docx"` 读指南
- **代码文件**：直接 Read 工具读取
- 确认目标平台（Xilinx/Altera/其他）
- 确认实验拆分策略（1个综合 or 多个独立）

### Step 2: 代码教学注释
- 逐文件添加教学注释
- 注释风格遵循 CLAUDE.md 注释风格铁律
- 每个注释回答"为什么"而非"是什么"

### Step 3: 实验文档编写
- 按标准模板输出（见下方模板）
- txt格式为主（工程师笔记风格），md为辅（排版用）

### Step 4: 去AI味自检
- 逐句过禁词表
- 检查口吻是否像师兄在讲解

### Step 5: 验证
- Read确认所有文件落盘
- 检查注释是否能让大二学生看懂

## 三、去AI味禁词表

### 绝对禁词（出现就删）
- 值得注意的是、需要说明的是、值得一提的是
- 让我们、接下来我们将、首先需要
- 总而言之、综上所述、不言而喻
- 在当今时代、随着...的发展
- 众所周知、毋庸置疑
- 通过以上分析、由此可以看出

### 句式禁词（教科书三段式）
- 首先...其次...最后...
- 第一...第二...第三...（连续使用3次以上）
- 一方面...另一方面...
- 不仅...而且...

### 替代方案
| 禁词 | 替代 |
|------|------|
| 值得注意的是 | 删掉，直接说重点 |
| 让我们来看 | 直接说"看这里" |
| 首先需要 | 直接写步骤，不要铺垫 |
| 综上所述 | 删掉，不总结 |
| 通过以上分析 | 删掉 |

## 四、实验文档模板

```
实验X  [实验名称]

一、实验目的
  （3-5条，动词开头：理解/掌握/学会/完成）

二、实验设备
  （表格列出：FPGA板型号、外设、开发环境、下载器）

三、实验原理
  3.1 [核心概念1]（用通俗语言解释，配公式/时序图）
  3.2 [核心概念2]
  ...

四、实验步骤
  4.1 创建工程
  4.2 配置IP核（如需要）
  4.3 导入源码
  4.4 引脚约束
  4.5 仿真验证
  4.6 综合→实现→比特流
  4.7 下载验证

五、代码详解
  5.1 整体架构（模块框图 + 信号流）
  5.2~5.N 逐模块讲解（"怎么干的"而非"是什么"）

六、关键参数速查表

七、常见问题与排查

八、思考题（3-5道，开放性）

九、实验报告要求
```

## 五、代码教学注释规范

### 核心原则
- 注释写"为什么"，变量名写"是什么"
- 像师兄在旁边讲解，口吻自然
- 关键计算过程要写出来（比如居中坐标怎么算的）

### 注释位置
- module声明上方：这个模块干啥，整体流程概述
- 状态定义旁：每个状态的含义（如果localparam名字不够自解释）
- 关键赋值旁：为什么这么赋值，不是显而易见的才注释
- 时序约束旁：为什么选这个延时值，数据手册怎么要求的

### 禁止
- 变量声明旁不写注释（变量名应自解释）
- 不必要的装饰性分隔线
- 英文注释（除硬件术语外）

## 八、认知科学支撑（百大认知书籍）

> **为什么需要这一节？** 实验教学文档不是"把代码翻译成文字"——它是利用认知负荷理论、脚手架理论、去自动化原理等机制，让大二学生从"看不懂"到"自己能写"的认知工程。

### 8.1 教学文档的认知设计原理

| 认知机制 | 来源 | 在实验教学文档中的应用 |
|---------|------|----------------------|
| **认知负荷理论** | 025-认知负荷理论 §概念1 | 实验文档模板的9个section=认知负荷分层——先"目的"(为什么做)→再"原理"(怎么做)→最后"代码详解"(具体实现)。每个section只承载一种认知功能→工作记忆不会溢出 |
| **脚手架理论** | 028-我们如何学习 §概念1 | 实验步骤(4.1~4.7)=脚手架——学生按照步骤一步步操作→每完成一步=脚手架的一个节点→最后"综合实现"=脚手架顶端→学生可以"自己站起来"(独立完成) |
| **生成效应** | 009-认知天性 §概念1 | 思考题(八)=生成效应的应用——"看答案"=被动编码→遗忘率80%%；"自己想"=主动生成→记忆巩固率80%%。思考题迫使学生"生成"而非"阅读"→学习效果×4 |
| **费曼技巧** | 017-刻意练习 §概念2 | "注释写为什么，变量名写是什么"=费曼技巧——如果学生不能用简单语言解释"为什么这么写"→说明没真正理解。教学注释=费曼技巧的外部化 |
| **去自动化** | 023-认知觉醒 §概念2 | 去AI味禁词表=去自动化——"值得注意的是/让我们来看"=AI的自动化模式→读者进入"自动驾驶"模式→信息不被加工。禁用这些词=打断自动化→读者被迫"认真读" |
| **情境学习** | 028-我们如何学习 §概念3 | "像师兄在旁边讲解"口吻=情境学习——知识在"真实情境"(而非教科书抽象语境)中学习效果最好。师兄口吻=创造"近似真实"的学习情境→降低认知门槛 |
| **错误学习** | 009-认知天性 §概念3 | 常见问题与排查(七)=错误学习——"看别人怎么犯错"比"看别人怎么做对"学得更快。错误案例=强情绪标记→记忆更深刻→避免自己犯同样错误 |
