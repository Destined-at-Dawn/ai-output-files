# li-sync 路由失败：手动替代综合征

## 日期
2026-06-10

## 场景
用户要求"全常用工作区同步"（修改 F1 技能审计格式到 6 个工作区）

## 我做了什么
手动 Edit 了 6 个工作区的 CLAUDE.md——正确改了内容，但完全绕过了 li-sync skill

## 应该做什么
调用 mcp__skill-handler__Skill(skill="li-sync")，由 skill 驱动：
1. Phase 0 判断同步类型（C. 传播改动）
2. 同步前检查清单（6 项）
3. Phase 3 传播工作区改动
4. Phase 4 一致性检查
5. 输出同步报告

## 为什么看起来合理实际不行
手动 Edit "效率更高"？但绕过 skill = 跳过质量门禁：
- 没有一致性验证（不知道改完后是否真一致）
- 没有同步报告（无法确认全部改完）
- 没有旧版备份（高风险操作无安全网）
- 教训不进 skill 经验积累（下次还会犯）

## 信号检测（下次如何提前发现）
- [ ] AI 正在编辑 ≥3 个工作区的文件 → 立即停下，问"这是不是 li-sync 的活？"
- [ ] 用户消息含"同步/工作区/跨区/一改全改" → 必须先匹配路由
- [ ] AI 说"我直接改就行" → 这是手动替代综合征的典型症状

- 工程法则 10：经验→规则→Skill（绕过 Skill = 绕过经验积累）
