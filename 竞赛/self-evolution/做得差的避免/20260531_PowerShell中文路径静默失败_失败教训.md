# E04：PowerShell中文路径静默失败——假阳性陷阱

## 日期
2026-05-31

## 触发场景
用户要求将自我进化文件同步到5个工作区（知识中枢/个人/求职/竞赛），路径全部含中文

## 失败现象
- PowerShell Copy-Item 没报错但目标目录为空
- PowerShell Test-Path 返回 False，但文件实际存在
- Bash ls 返回了内容（假阳性），但 PowerShell 读不到
- 同步脚本"声称成功"但3个工作区的自我进化目录实际不存在

## 根因
PowerShell 在 Bash Shell 中执行时，中文路径编码（GBK/CP936）与 Bash 环境（UTF-8）不匹配。
编码不匹配导致路径解析静默失败——不报错、不警告、不崩溃，只是"什么都不做"。

## 为什么看起来行其实不行
- 工具返回 exit code 0（"成功"），但实际操作被编码问题吞掉了
- Bash ls 能看到文件（因为它用不同的编码方式），所以手动检查也"通过"了
- 只有用 Python os.path.exists() 或 Read 工具直接读才能暴露真相

## 正确做法
1. 任何涉及中文路径的文件操作，一律用 Python（os/shutil）
2. 验证文件是否落盘，用 Python os.path.exists() 或 Read 工具
3. 不信任 PowerShell 对中文路径的任何返回值
