# li-skills-mgmt Golden Rules

## GR-001: 场景优先于技能
先判断用户意图（What/Why），再选择底层技能（How）。不要先选技能再找场景。

## GR-002: 协调不越界
本 skill 负责编排和上下文传递，不重复底层 skill 的逻辑。

## GR-003: 反馈即进化
用户评价"好"→ 记录正模式；评价"差"→ 记录失败模式 → 更新规则。

## GR-004: 联动必须显式
调用其他 li- skill 时，明确告知用户"接下来用 li-xxx 处理..."

## GR-005: 结果可追溯
每次 skill 调用链的完整路径记录到 memory/{date}.md。
