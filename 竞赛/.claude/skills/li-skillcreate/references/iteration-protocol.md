# 迭代协议详细指南

> 来源：deep-research v4 Phase 6 + auto-evolution + 案例拆解生成器 + 本次对话教训
> 迭代不是复制粘贴。每轮必须有实质差异 + 黄金法则。

---

## 迭代前评估清单

**每次迭代前必须回答**：

1. **搜索饱和度**：还有新的参考技能可以学习吗？
   - 如果 `~/.newmax/skills/` 和 `~/.codex/skills/` 中的相关技能都读过了 → 搜索饱和
   - 如果还有未读的相关技能 → 继续学习

2. **用户反馈质量**：用户给了具体改进方向吗？
   - 具体方向（"触发词不够多""缺 YouTube 字幕流程"）→ 直接迭代
   - 泛泛反馈（"不够好""要改进"）→ 先用 jc-clarifier 澄清

3. **方向清晰度**：知道要改什么吗？
   - 清晰 → 执行迭代
   - 模糊 → 先诊断再迭代

4. **质量评分**：上次迭代的质量评分是多少？
   - ≥ 20/30 → 方向正确，继续深化
   - < 20/30 → 需要重新评估方向

## 迭代类型

### 类型 A：功能迭代（加新能力）

**触发**：用户说"加上 XXX 功能""还缺 XXX"
**流程**：
1. 读取当前 SKILL.md
2. 确定新功能放在哪个 Phase
3. 写入新功能
4. 更新 _meta.json 版本号和 changelog
5. 检查 references/ 是否需要新增

### 类型 B：质量迭代（改已有内容）

**触发**：用户说"不够好""太粗糙""参考技能太少"
**流程**：
1. 读取当前 SKILL.md
2. 诊断哪个 Phase/Section 质量不足
3. 读取更多参考技能（本轮 ≥2 个新参考）
4. 用新学到的设计模式替换/增强已有内容
5. 提炼黄金法则

### 类型 C：生态迭代（改善连接）

**触发**：用户说"路由不生效""没有更新记忆""一鱼多吃不够"
**流程**：
1. 读取路由表确认条目正确
2. 检查所有生态文件是否已更新
3. 补充遗漏的生态连接
4. 测试路由是否能正确触发

### 类型 D：重构迭代（改变结构）

**触发**：SKILL.md 太长 / 结构不合理 / references 需要重组织
**流程**：
1. 读取当前完整 SKILL.md
2. 重新规划结构（Phase 拆分 / references 重组）
3. 渐进披露优化（主文件精简，详细内容下放 references）
4. 全文重写（不是微调）

## 1.25 倍递增规则

**原理**：每轮迭代的修改量必须 ≥ 上一轮的 1.25 倍。

**为什么是 1.25 倍**：
- 太低（如 1.1 倍）→ 容易凑数
- 太高（如 2 倍）→ 后期不可持续
- 1.25 倍是经验最优值：足够有挑战，又不会导致无限膨胀

**计算方式**：
- 第 1 轮 ≥ 20 处修改
- 第 2 轮 ≥ ceil(20 × 1.25) = 25 处
- 第 3 轮 ≥ ceil(25 × 1.25) = 32 处
- 第 N 轮 ≥ ceil(prev × 1.25) 处

**"处修改"的定义**：
- 新增一个 Phase/Section = 3 处
- 新增一个参考技能引用 = 1 处
- 修改一个已有的 Phase = 2 处
- 新增一个 references/ 文件 = 3 处
- 删除冗余内容 = 1 处
- 更新路由表 = 1 处
- 提炼一条黄金法则 = 2 处

## 黄金法则提炼

**什么是黄金法则**：从本轮迭代中提取的、适用于所有未来技能创建的永久性规则。

**格式**：
```markdown
## 黄金法则 #N：[一句话标题]

**来源**：[本轮迭代的哪个发现]
**规则**：[可执行的规则，不是描述]
**反模式**：[违反时的具体表现]
```

**示例**：
```markdown
## 黄金法则 #1：触发词数量 ≥ 10

**来源**：deep-research v3 路由表只有 1 个触发词导致不生效
**规则**：每个技能的路由条目必须包含 ≥ 10 个触发词（同义词 + 口语 + 英文）
**反模式**：只有 1-2 个触发词 → 用户得手动指定技能名
```

## 搜索饱和检测

**信号**：
1. 连续 3 轮未找到新的参考技能
2. 新读的参考技能提取不出新的设计模式
3. 迭代修改主要在"改措辞"而非"改结构/加功能"

**处理**：
- 检测到饱和 → 通知用户"搜索已饱和，建议最后整理一轮后结束"
- 用户说"继续"→ 进入方向转换（换搜索关键词/换参考来源）
- 用户说"停"→ 产出最终版
