# 认知框架武器库（技能创建专用）

> 来源：百大认知书籍（62 本）。每个框架在技能创建中的具体用法。
> 编号对应"百大认知书籍"书架编号。

---

## 核心框架（每次创建必用）

### T1：双系统理论（#003《思考，快与慢》）
**原理**：System 1（直觉/快速）vs System 2（分析/慢速）
**在技能创建中**：判断技能是否必要时——System 1 的第一反应"需要"往往是真需求；但设计工作流必须用 System 2 仔细推敲
**反模式**：用 System 1 设计 Phase 流程（"应该没问题"→ 实际跑不通）

### T2：认知负荷理论（#007《学习之道》）
**原理**：工作记忆容量有限（7±2 项），超载则学习效率骤降
**在技能创建中**：SKILL.md ≤ 300 行不是武断——是认知负荷上限；Phase 数量 ≤ 7（工作记忆上限）
**反模式**：一个 skill 15 个 Phase、1000 行——AI 读了前 300 行就忘了后面的

### T3：WYSIATI（#003《思考，快与慢》）
**原理**：你只看到你想看到的（What You See Is All There Is）
**在技能创建中**：问题消解阶段必须检查——用户描述的需求是否只是冰山一角？Phase 0 的"假设错误"检测就是对抗 WYSIATI
**反模式**：用户说"造个天气技能"就直接造，没问"你是要查天气还是发天气推文？"

### T4：锚定效应（#002《影响力》）
**原理**：第一个信息会锚定后续所有判断
**在技能创建中**：第一个触发词的设计决定了路由准确性——如果第一个触发词是"创建技能"但用户实际说"造一个"，路由就偏了
**实操**：触发词列表的第一个词必须是用户最高频的口语表达

### T5：奥卡姆剃刀（#011《精要主义》）
**原理**：如无必要，勿增实体
**在技能创建中**：Phase 能砍就砍，reference 能合就合，触发词不是越多越好（过多导致误触发）
**反模式**：为了"全面"加了 8 个 Phase，其中 3 个从不被触发

## 设计框架（设计 Phase 时用）

### T6：刻意练习（#008《刻意练习》）
**原理**：有效练习 = 明确目标 + 即时反馈 + 适中难度
**在技能创建中**：每轮迭代必须有明确的改进焦点（不是"全面改进一点"），必须有反馈机制（质量评分）
**反模式**：迭代了 5 轮但每轮改的是同一个地方

### T7：反脆弱（#016《反脆弱》）
**原理**：好的系统在压力下变得更好，不是不坏
**在技能创建中**：技能在边界输入（超长/超短/异常格式）下应该有更好的错误处理，而不是崩溃
**反模式**：正常输入能用，输入一个空字符串就卡死

### T8：间隔效应（#009《认知天性》）
**原理**：间隔学习比集中学习效果好 2-3 倍
**在技能创建中**：迭代不要一次性做 10 轮——间隔几小时甚至几天，每次回来都能发现新问题
**反模式**：一口气迭代 10 轮，越改越差

### T9：检索式练习（#009《认知天性》）
**原理**：主动回忆比被动复习效果好
**在技能创建中**：设计 Phase 时要求 AI 主动回忆已有技能的设计模式，而不是被动列出

### T10：损失厌恶（#003《思考，快与慢》）
**原理**：人对损失的敏感度是收益的 2 倍
**在技能创建中**：删除一个 Phase 比新增一个 Phase 难 2 倍——AI 倾向于保留不需要的东西
**实操**：定期问"这个 Phase 最近 5 次创建有被用到吗？"

## 评估框架（Phase 3 质量门禁用）

### T11：第一性原理（#021《创新者的窘境》）
**原理**：回到最基本的事实，从零推理
**在技能创建中**：评估技能质量时——不看"参考了 20 个 skill"，看"这个技能解决了什么基本问题"
**反模式**：堆砌参考来源但核心问题没解决

### T12：费曼检验（#012《费曼学习法》）
**原理**：如果你不能用简单语言解释，说明你没真正理解
**在技能创建中**：skill 的 description 必须用简单语言写清——AI 路由时只看 description，写复杂了反而匹配不到
**反模式**：description 写了 500 字但说不清"这个 skill 干什么"

### T13：二八法则（#010《高效能人士的七个习惯》）
**原理**：80% 的产出来自 20% 的输入
**在技能创建中**：不要为覆盖 100% 的场景把 skill 做得臃肿——抓住核心 20% 的高频场景，其余走 fallback
**反模式**：为了覆盖冷门场景加了 5 个额外 Phase

### T14：预验尸（#003《思考，快与慢》）
**原理**：在开始之前先想象"如果失败了，原因会是什么"
**在技能创建中**：Phase 0 问题消解 + Phase 3 质量门禁的本质就是预验尸——先想"做错了会怎样"，再决定"怎么做对"
**反模式**：不问"这个技能可能怎么失败"就直接开始构建
