## 连接 1：路由表注册

**文件**：`E:\ai产出文件\牛马\mutual\mutual\skill-routing-table.json`

**操作**：新增路由条目

**格式**：
```json
{
  "id": "r0XX",
  "triggers": ["触发词1", "触发词2", "更多触发词..."],
  "context": ["适用场景1", "适用场景2"],
  "skill": "skill-name",
  "mcp": null,
  "priority": N,
  "auto": true,
  "confidence": 0.8,
  "note": "路由说明"
}
```

**触发词设计规则**：
- 从用户真实对话中提取，不是猜的
- 包含动词（创建/造/做）和名词（技能/skill/工具）
- 包含同义词和口语表达
- 初始 confidence = 0.8，根据使用数据调整

**去重检查**：写入前检查路由表中是否已有相同触发词。如果有，判断是合并还是覆盖。

## 连接 2：今日记忆更新

**文件**：`memory/{today}.md`

**操作**：Edit 追加

**模板**：
```markdown
## [HH:MM] 创建/迭代技能：[技能名]
- 类型：[新建/迭代 vN→vM]
- 参考技能：[N 个]（[列出核心的 3-5 个]）
- 核心机制：[1-2 句话]
- 文件路径：~/.newmax/skills/[name]/
- 文件清单：[SKILL.md + _meta.json + references/...]
- 文件大小：[N] 个文件 / [X] KB
- 路由：r0XX（[M] 个触发词）
- 生态副作用：[列出更新的生态文件]
- 黄金法则：[如果有]
- 是否迭代：[待用户决定 / 已完成 N 轮]
```

## 连接 6：进化信号

**文件**：`self-evolution/evolution-calendar.md`（如果存在）

**操作**：检查是否需要添加进化任务

**触发条件**：
- 某个技能被迭代 ≥3 次 → 该技能应建立独立 SOP
- 某类技能创建请求频繁出现 → 应建立技能模板
- 某个参考技能被反复引用 → 应升级为核心参考

## 连接 7：长期记忆候选

**文件**：`memory/long-term.md`

**操作**：提候选（不直接写入）

**候选条件**：
- 创建过程中发现了新的设计模式
- 用户表达了新的偏好
- 踩了新的坑

**格式**：
```
📋 记忆候选
- 类型：[教训/偏好/决策/知识]
- 内容：[一句话]
- 建议写入：memory/long-term.md § [章节]
- 置信度：[高/中/待确认]
请回复：写入 / 修改后写入 / 不记录
```

## 跨工作区同步

如果新技能适用于多个工作区：

1. **确认适用范围**：全部 5 个工作区 / 仅当前 / 仅 mutual
2. **同步路径**：
   - 全部 → 注册到知识中枢 `02-共享规则/` + 更新各工作区 CLAUDE.md
   - 仅当前 → 只更新当前工作区的路由表
3. **同步约束**：参考 `li-sync` 的"真源+分发"架构
