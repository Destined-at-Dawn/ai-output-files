# 弃用模板与恢复机制

> 用途：Phase 4 软删除的详细模板

---

## DEPRECATED.md 模板

```markdown
# DEPRECATED

弃用日期：{YYYY-MM-DD}
弃用原因：{一句话}
替代方案：{新技能名 + 路径}（如有）
原因详情：{为什么弃用}

## 迁移指南
如果你之前使用本技能，现在应该：
1. {具体步骤}
2. {具体步骤}
```

---

## 弃用流程（详细）

```
Step 1：确认弃用原因
  - 被更好的技能替代？
  - 功能已合并到其他技能？
  - 不再需要？

Step 2：创建 DEPRECATED.md（使用上方模板）

Step 3：从路由表中移除触发词
  - 不物理删除，标注 deprecated
  - 保留记录以便恢复

Step 4：更新 lifecycle-log.md

Step 5：如果有替代技能
  → 更新替代技能的 _meta.json 记录继承关系
```

---

## 恢复机制

如果弃用后发现还需要：

```
1. 删除 DEPRECATED.md
2. 恢复路由表中的触发词
3. 记录到 lifecycle-log.md：
   "恢复：{技能名}，原因：{为什么}"
```

---

## 弃用检查清单

- [ ] 弃用原因已明确记录
- [ ] DEPRECATED.md 已创建
- [ ] 路由表已更新（触发词标注 deprecated）
- [ ] lifecycle-log.md 已记录
- [ ] 如有替代技能，继承关系已更新
