# 融合模板与命名规则

> 用途：Phase 1 技能融合的详细模板和命名决策

---

## 融合记录格式

```markdown
## [日期] 融合：{技能A} + {技能B} → {新技能}

### 原因
{为什么融合}

### 功能取舍
| 功能 | 来源 | 保留/删除 |
|------|------|----------|
| {功能1} | {技能A/B} | 保留/删除 |

### 新技能路径
~/.newmax/skills/{新技能名}/SKILL.md

### 原技能状态
- {技能A}：DEPRECATED → {新技能}
- {技能B}：DEPRECATED → {新技能}
```

---

## 融合命名决策树

```
两个技能有明显主次？
├── 是 → 保留主技能名，副技能 DEPRECATED
└── 否 → 用新名称（如 li-skillfusion）
```

**版本号规则**：`max(版本A, 版本B) + 0.1`

示例：v1.0 + v2.0 → v2.1

---

## 功能矩阵模板

```markdown
| 功能 | 技能 A | 技能 B | 保留哪个 | 理由 |
|------|--------|--------|----------|------|
| {功能1} | ✓ | ✓ | {A/B/合并} | {理由} |
| {功能2} | ✓ | ✗ | A | {理由} |
| {功能3} | ✗ | ✓ | B | {理由} |
```

---

## 融合后检查清单

- [ ] 新 SKILL.md 包含两者核心功能
- [ ] 触发词已合并（无遗漏）
- [ ] 两个原技能均有 DEPRECATED.md
- [ ] 路由表已更新
- [ ] lifecycle-log.md 已记录
- [ ] Write 后 Read 验证通过
