# 微技能规范

> 用途：Phase 3 微技能创建的完整规范

---

## 目录结构

```
~/.newmax/skills/{workspace}-{功能名}/
├── SKILL.md          (< 150 行，极简)
├── .workspace-only   ← 空文件，标记为工作区专用
└── _meta.json
```

---

## 微技能 SKILL.md 模板

```markdown
---
name: {workspace}-{功能名}
description: "{一句话描述} [仅限 {工作区名} 工作区]"
workspace_only: true
parent_skill: "{如果从某个技能拆分而来}"
---

# {功能名}

> 本技能仅在 {工作区名} 工作区生效。

## 触发条件
{触发词列表}

## 执行流程
{简化的 3-5 步流程}

## 与其他技能的关系
{不重复说明，指向全局技能}
```

---

## 路由隔离机制

`.workspace-only` 标记确保：
- 只在指定工作区的路由表中生效
- 不出现在全局技能推荐中
- 不与同类全局技能冲突

---

## 微技能 vs 全局技能决策

| 条件 | 选择 |
|------|------|
| 多个工作区都需要 | 全局技能 |
| 只有 1 个工作区需要 | 微技能 |
| 功能很窄（< 50 行） | 微技能 |
| 功能复杂（> 150 行） | 全局技能 |

---

## 微技能命名规范

```
格式：{工作区名}-{功能名}
示例：创作-文风检查、竞赛-格式校验
```

**注意**：工作区名用中文（与工作区目录名一致），功能名用中文或英文均可。
