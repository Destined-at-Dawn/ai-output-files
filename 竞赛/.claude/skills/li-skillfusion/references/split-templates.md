# 拆分模板与命名规则

> 用途：Phase 2 技能拆分的详细模板和决策

---

## 拆分命名规则

```
新技能名称 = {原技能名}-{模块名}
```

示例：`deep-research-security` → `li-bestskill-security-review`

---

## 模块边界识别指南

可拆分的模块通常满足：
- 以 Phase/Flow 为单位，有独立入口和出口
- 与其他模块的耦合度低（不共享大量状态）
- 有独立的触发条件或使用场景

**不可拆分的信号**：
- 模块间有大量数据传递
- 拆分后两个模块都需要完整上下文才能工作
- 拆分后任一模块 < 50 行（太小无意义）

---

## 拆分后引用段模板

原 SKILL.md 中添加：

```markdown
## 已拆分模块

以下功能已拆分为独立技能：
- **{模块名}** → `~/.newmax/skills/{新技能名}/SKILL.md`
  - 拆分原因：{一句话}
  - 拆分日期：{YYYY-MM-DD}
```

---

## 拆分检查清单

- [ ] 新技能 SKILL.md 自包含（不依赖原技能的上下文）
- [ ] 原技能已移除拆分内容
- [ ] 原技能添加了"引用"段
- [ ] 路由表已更新（新技能的触发词）
- [ ] lifecycle-log.md 已记录
- [ ] Write 后 Read 验证通过
