# 竞赛项目工作区架构规范（CRITICAL — 最高优先级）

> **优先级**：最高（与 chinese-path-safety.md 同级）
> **创建日期**：2026-06-13 | **v2.0 更新**：2026-06-14
> **根因**：竞赛管理文件（outputs/memory/SOP/skills）被错误放在 proj-xxx 临时目录，导致架构混乱

---

## 核心铁律

**每个竞赛项目有且仅有一个主文件夹，所有管理文件必须放在主文件夹内。**

---

## 一、主文件夹唯一性

### 定义

| 项目 | 主文件夹路径 |
|------|-------------|
| YOLO 绝缘子检测竞赛 | `E:\ai产出文件\牛马\竞赛\竞赛\Yolo算法比赛\` |
| 其他竞赛（示例） | `E:\ai产出文件\牛马\竞赛\竞赛\{竞赛名}\` |

### 铁律

- ✅ **主文件夹**：存放所有管理文件（outputs、memory、SOP、skills、CLAUDE.md 等）
- ❌ **proj-xxx 目录**：只是临时辅助，**禁止**存放管理文件
- ❌ **projects/ 子目录**：只是任务跟踪，**禁止**作为主文件夹

---

## 二、文件分类规则

### 管理文件（必须在主文件夹）

| 文件类型 | 路径 | 说明 |
|----------|------|------|
| `outputs/` | `{主文件夹}/outputs/` | 论文、报告、调研结果 |
| `memory/` | `{主文件夹}/memory/` | 记忆文件、复盘记录 |
| `SOPs/` | `{主文件夹}/SOPs/` | 项目 SOP |
| `skills/` | `{主文件夹}/skills/` | 项目级 skill（如 competition-local.md） |
| `CLAUDE.md` | `{主文件夹}/CLAUDE.md` | 项目约束 |
| `.claude/rules/` | `{主文件夹}/.claude/rules/` | 项目规则 |
| `归档/` | `{主文件夹}/归档/` | 双归档备份 |

### 代码文件（可在其他位置）

| 文件类型 | 路径 | 说明 |
|----------|------|------|
| 训练脚本 | `D:/AI竞赛数据集/Yolo算法比赛/` | 可在 D 盘 |
| 数据集 | `D:/AICompData/` | 可在 D 盘 |
| 训练结果 | `D:/AI竞赛数据集/runs/` | 可在 D 盘 |
| ONNX 导出 | `D:/AI竞赛数据集/runs/*/weights/` | 可在 D 盘 |

```

2. {主文件夹}\CLAUDE.md                       ← 项目约束

```

### Skill 读取规则

```
1. C:\Users\13975\.newmax\skills\             ← 全局 skill（通用方法论）

```

**铁律**：通用方法论从全局 skill 获取，具体数据从项目级 skill 获取，两者缺一不可。

## 四、proj-xxx 目录的定位

### 允许存放

- 任务跟踪（task.json）
- 临时文件（会话中间产物）
- 项目计划（project.md）— 仅用于 MCP 工具读取

### 禁止存放

- ❌ outputs/（管理文件）
- ❌ memory/（记忆文件）
- ❌ SOPs/（项目 SOP）
- ❌ skills/（项目级 skill）
- ❌ CLAUDE.md（项目约束）
- ❌ .claude/rules/（项目规则）

## 五、新竞赛项目启动检查清单

收到新竞赛任务时，必须先确认主文件夹：

```python
# 检查清单
1. 主文件夹是否存在？
   - 不存在 → 创建 `{主文件夹}/`
   - 存在 → 继续

2. 主文件夹内是否有以下目录？
   - outputs/
   - memory/
   - SOPs/
   - skills/
   - .claude/rules/
   - 缺少 → 创建

3. 是否误用了 proj-xxx 目录？
   - 检查 proj-xxx 下是否有管理文件
   - 有 → 移动到主文件夹
```

## 五B、proj-xxx ↔ 主文件夹自动感知协议（2026-06-14 新增）

> **问题**：AI 从 proj-xxx 收到竞赛任务时，不知道管理文件应该写到主文件夹。
> **本节解决**：AI 在 proj-xxx 工作时，如何自动找到并写入主文件夹。

### 自动映射表

| proj-xxx 所属 | 对应主文件夹 |
|--------------|-------------|
| `projects/proj-*`（工作区根目录下） | 该工作区的**当前活跃竞赛主文件夹**（如 `Yolo算法比赛/`） |
| 新竞赛项目（尚无主文件夹） | 先创建主文件夹，再执行任务 |

### 感知协议（每次从 proj-xxx 发起任务时强制执行）

```
Step 0: 识别当前位置
        → 如果 cwd 含 projects/proj- → 触发感知协议
        → 如果 cwd 已在主文件夹内 → 跳过

Step 1: 查映射表
        → 找到对应主文件夹路径
        → 如果映射表中无对应 → 问用户确认主文件夹

Step 2: 设置写入目标
        → 所有管理文件（outputs/memory/SOPs/skills/CLAUDE.md）写入主文件夹
        → proj-xxx 只保留 task.json + project.md（MCP 工具必需）

Step 3: 执行任务
        → 脚本/代码可以在任意位置
        → 产出报告/记忆/SOP 必须写主文件夹
```

### 铁律

- ❌ **禁止**在 proj-xxx 下创建 outputs/、memory/、SOPs/、skills/
- ✅ **必须**在主文件夹下创建以上目录
- ✅ proj-xxx 的 project.md 中**必须**标注对应主文件夹路径

## 六、违规检测信号

如果出现以下任何情况，说明本规则被违反了：

- [ ] proj-xxx 目录下有 outputs/、memory/、SOPs/、skills/ 等目录
- [ ] 竞赛管理文件分散在多个位置
- [ ] AI 说"我在 proj-xxx 目录下创建了 outputs"——违规
- [ ] 无法确定哪个是主文件夹

## 七、两级 Memory 读写优先级（2026-06-14 新增）

> **问题**：竞赛区存在两级 memory（工作区级 `竞赛/memory/` + 项目级 `Yolo算法比赛/memory/`），AI 不知道读哪个、写哪个。

### 读取优先级（从高到低）

| 优先级 | 路径 | 何时读 |
|--------|------|--------|
| **P0** | `{主文件夹}/memory/{今天日期}.md` | 每次对话必读（项目级今日记忆） |
| **P1** | `{主文件夹}/memory/long-term.md` | 每次对话必读（项目级长期记忆） |
| **P2** | `{主文件夹}/memory/lessons-learned.md` | 每次对话必读（项目级教训） |
| **P3** | `{工作区根目录}/memory/{今天日期}.md` | 按需（工作区级，跨项目通用） |
| **P4** | `{工作区根目录}/memory/long-term.md` | 按需（工作区级，跨项目通用） |

### 写入规则

| 内容类型 | 写入位置 | 理由 |
|----------|---------|------|
| 项目进展、训练结果、竞赛决策 | `{主文件夹}/memory/{日期}.md` | 项目专属 |
| 项目级教训（如 YOLO 特有踩坑） | `{主文件夹}/memory/lessons-learned.md` | 项目沉淀 |
| 跨项目通用教训（如 Goal 模式问题） | `{工作区根目录}/memory/{日期}.md` | 工作区共享 |
| 长期记忆归档 | 由 longmemory skill 处理 | 遵循 F3 |

### 铁律

- **项目相关记忆只写项目级**，不污染工作区级
- **跨项目通用教训写工作区级**，不重复写项目级
- **禁止同一教训同时写两级**——选对的那一级写

## 八、memory → skill/rule 反向优化矩阵（2026-06-14 新增）

> **问题**：教训记录到 memory 后，没有触发对应的 skill/rule 更新，教训只进不出。

### 反向优化触发条件

每次写入 memory 时（教训/决策/发现），执行以下判断：

```
写入 memory/{日期}.md
  → 这条教训涉及哪个领域？
    ├─ 训练/数据集 → 检查 competition-yolo skill 是否需要更新
    ├─ Goal 模式 → 检查 goal-mode-anti-repeat.md 是否需要更新
    ├─ 架构/文件管理 → 检查 competition-workspace-architecture.md 是否需要更新
    ├─ Skill 使用 → 检查对应 SKILL.md 是否需要更新
    ├─ SOP 流程 → 检查对应 SOP 迭代日志是否需要追加
    └─ 通用工程法则 → 检查 .claude/rules/ 对应规则是否需要更新
```

### 优化矩阵

| 教训来源 | 可能需要更新的文件 | 更新方式 |
|----------|-------------------|---------|
| 训练失败/数据问题 | `competition-yolo/SKILL.md` | Edit 追加避坑指南 |
| Goal 模式踩坑 | `goal-mode-anti-repeat.md` | Edit 追加铁律 |
| 架构违规 | `competition-workspace-architecture.md` | Edit 追加规则 |
| Skill 路由漏调 | `skill-routing-table.json` | Edit 添加触发词 |
| SOP 流程缺陷 | `SOPs/对应SOP.md` | Edit 迭代日志 |
| 通用踩坑 | `.claude/rules/对应规则.md` | Edit 追加规则 |
| 同类教训 ≥3 次 | CLAUDE.md F编号 | 升级为铁律 |

### 铁律

- **写入 memory 后必须执行反向优化判断**（不一定要更新，但必须判断）
- **判断结果记录在当日 memory 末尾**：`反向优化：[已更新 xxx / 无需更新 / 待确认]`
- **同类教训 ≥3 次**：必须升级为独立规则或 Skill

## 十一、认知科学支撑

| 原理 | 来源 | 在本规则中的应用 |
|------|------|-------------------|
| **单一真相源** | 系统思维 | 主文件夹是竞赛项目的唯一真相源，所有管理文件必须在此 |
| **路径依赖** | 认知心理学 | 一旦文件放错位置，后续所有操作都会基于错误路径，导致混乱加剧 |
| **心智模型** | 认知科学 | 清晰的架构规范帮助 AI 建立正确的心智模型，减少决策错误 |
