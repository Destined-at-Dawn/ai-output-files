# 2026-06-12 Goal模式9轮系统性失败复盘

> 类型：systematic_failure | 严重性：critical | 影响范围：全工作区

---

## 一、事故全景

Goal模式执行9轮（每轮100万token预算），总消耗约900万token，但：

| 指标 | 值 |
|------|------|
| 有效产出轮次 | 2轮（第1、2轮） |
| 重复劳动轮次 | 5轮（第4-9轮中的5轮重复下载CPLID） |
| 幻觉产出轮次 | 3轮（声称完成但文件未落盘） |
| 真实最终产出 | CPLID训练99.3%、ONNX导出、10篇论文调研、竞赛Skill |

## 二、4个A级根因

### 根因1：上下文压缩导致失忆（最致命）

**机制**：
```
轮次N结束 → 对话压缩 → 丢失"数据集路径""脚本是否执行""文件是否落盘"
→ 轮次N+1开始 → AI凭残存记忆判断"数据丢了" → 重新下载
```

**数据**：9轮中有5轮重新下载了CPLID数据集（848张，~400MB）。

**认知科学映射**：
- 「工作记忆容量有限」——《028-我们如何学习》§概念1 认知负荷理论
- 上下文压缩 = 强制清空工作记忆 → AI进入"失忆状态"

### 根因2：没有磁盘验证就开始干活

**机制**：
```
轮次N+1开始 → 不执行 os.listdir() / Glob 验证
→ 凭记忆说"目录是空的" → 开始下载/转换/创建
→ 实际目录有数据 → 重复劳动
```

**数据**：至少3轮在D:/AICompData/已有完整数据的情况下重新下载。

### 根因3：中文路径 + 幽灵文件

**机制**：
```
Bash执行 cp/mv/ls 中文路径 → MSYS2编码破坏 → 静默失败（返回码0但未执行）
→ AI报告"转换完成" → 下一轮用Python检查发现0文件 → 误判为"数据丢失"
```

**数据**：至少4次"转换完成"实际0文件落盘。已有 `chinese-path-safety.md` 规则但未被遵守。

**认知科学映射**：
- 「自动化偏见」——《039-社会动物》§概念1：信任工具的输出而不验证
- Bash返回0 ≠ 操作成功（在中文路径场景下）

### 根因4：Skill反复创建没有锚定

**机制**：
```
轮次N创建li-competition SKILL.md → 压缩 → 轮次N+1忘记已创建
→ 再次创建 → 覆盖 → 丢失轮次N的改进内容
```

**数据**：li-competition创建3次，competition-yolo创建2次。

## 三、为什么这个问题持续了9轮才被发现

1. **Goal模式禁止提问**：用户设了"不准问任何问题"→ AI不能确认"数据在不在"
2. **每轮独立评估**：Goal模式每轮重新扫描目标→AI看到"下载数据集"→检查记忆说没有→执行下载
3. **没有全局状态文件**：缺乏一个"磁盘实际状态"的单一真相源
4. **F4双归档规则只防删除不防重复**：重复下载不需要归档，所以双归档机制不起作用

## 四、修复方案（已执行）

### 修复1：F10 磁盘验证铁律（新增CLAUDE.md规则）
任何Goal轮次或上下文压缩后的首次操作，必须先用Glob/Read验证磁盘实际状态。

### 修复2：F11 Skill锚定铁律（新增CLAUDE.md规则）
创建任何Skill/文件前，必须先Glob检查是否已存在。已存在则Read内容后Edit增量更新。

### 修复3：competition-yolo SKILL.md Phase 0 预检查
在竞赛Skill的Phase 1之前插入Phase 0，强制验证数据集和训练结果的磁盘状态。

### 修复4：强化Write后Read验证
在F2文件操作铁律中强调：Write后Read不是"建议"而是"铁律"，特别是中文路径场景。

## 五、认知科学支撑

| 认知机制 | 书目 | 在本事故中的体现 |
|---------|------|----------------|
| WYSIATI | 《010-思考快与慢》§概念3 | 只看到压缩后残存的记忆碎片，就构建了完整（但错误）的"数据丢失"叙事 |
| 工作记忆瓶颈 | 《028-我们如何学习》§概念1 | 压缩=清空工作记忆→无法维持"文件在哪里"的状态 |
| 自动化偏见 | 《039-社会动物》§概念1 | 信任Bash返回码0而不验证实际结果 |
| 沉没成本谬误 | 《010-思考快与慢》§概念4 | "已经下载了一半，继续下载"→即使数据已存在 |
| 确认偏误 | 《003-学会提问》§概念2 | 检查目录时看到乱码输出→确认"目录是空的"→不再深入检查 |

## 六、预防措施

1. **Goal模式第一轮强制磁盘扫描**：列出D盘所有相关目录的文件数和大小
2. **单一真相源文件**：维护一个 `outputs/disk-state-snapshot.md` 记录已知数据集位置
3. **中文路径操作100%用Python**：零例外
4. **Skill创建前Glob检查**：`glob ~/.newmax/skills/*/SKILL.md` 确认已存在列表
5. **每轮Goal结束时写检查点**：压缩前主动写session-checkpoint.md
