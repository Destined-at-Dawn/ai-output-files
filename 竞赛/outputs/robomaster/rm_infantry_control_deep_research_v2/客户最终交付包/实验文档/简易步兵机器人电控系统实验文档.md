# 简易步兵机器人电控系统实验文档

## 一、实验目的

本实验基于 STM32 HAL 软件架构，设计一个简易步兵机器人电控系统。通过本实验，需要掌握：

- RoboMaster 步兵机器人电控系统的基本组成
- 主控任务调度与模块化工程结构
- X 型四轮全向底盘运动学
- PID 闭环控制的基本写法
- 云台 yaw/pitch 目标角控制
- 急停、限幅、Pitch 限位等安全保护
- STM32 HAL 工程中应用层与硬件适配层的分离方式

## 二、实验硬件资源

推荐硬件：

- STM32 开发板一块
- 四路直流霍尔编码器电机，或 RoboMaster 常见 C620/M3508 电机
- Yaw/Pitch 两轴云台电机
- IMU 陀螺仪模块
- 按键若干
- L1/L2 指示灯
- 电机驱动器或 CAN 电调

如果暂时没有完整硬件，也可以先完成软件仿真链路：按键输入、目标速度、目标角度、PID 输出和安全状态均可通过调试变量观察。

## 三、系统总体结构

```text
按键/串口/遥控器
      |
      v
remote.c 生成 remote_cmd_t
      |
      v
app.c 定时调度
      |
      +--> gimbal.c: 目标角 -> IMU反馈 -> PID -> 云台输出
      |
      +--> chassis.c: vx/vy/wz -> 轮速解算 -> PID -> 底盘输出
      |
      +--> safety.c: 急停、遥控丢失、限幅状态
      |
      v
bsp_port.c: GPIO / TIM / CAN / 编码器 / IMU
```

## 四、模块说明

### 1. 主控调度模块

`app.c` 是系统调度中心。它不直接写复杂控制逻辑，只负责按周期调用：

```c
Remote_Update(now_ms);
Gimbal_Control(cmd, dt_s);
Chassis_Control(cmd, yaw_feedback, dt_s);
```

这样设计的原因是：后续如果底盘、云台、遥控器分别调试，不会互相污染代码。

### 2. 遥控输入模块

`remote.c` 负责按键消抖、短按、长按和命令生成。

按键功能：

- `key1`：短按切换小陀螺模式，长按进入跟随云台模式
- `key2`：Yaw 目标角增加
- `key3`：Pitch 目标角增加
- `key_stop`：急停/解除急停

### 3. 底盘控制模块

底盘输入为 `vx/vy/wz`：

- `vx`：前后速度
- `vy`：左右速度
- `wz`：旋转角速度

X 型四轮全向轮解算：

```text
w1 = +vx - vy - wz * K
w2 = +vx + vy + wz * K
w3 = -vx + vy - wz * K
w4 = -vx - vy + wz * K
```

每个轮子再经过 PID 速度闭环，输出给电机。

### 4. 云台控制模块

云台控制链路：

```text
目标角 - IMU反馈角 -> PID -> 云台电机输出
```

Yaw 使用角度环绕处理，避免 179 度到 -179 度时误差突变。Pitch 目标角限制在 `-20°` 到 `30°`，避免撞机械限位。

### 5. PID 模块

PID 模块提供三个接口：

- `PID_Init()`：初始化参数
- `PID_Calc()`：计算输出
- `PID_Reset()`：急停或模式切换时清空内部状态

PID 输出和积分都有限幅，避免失控。

### 6. 安全保护模块

`safety.c` 集中管理安全状态：

- `robot_stop`：整车停机
- `remote_lost`：遥控失联
- `pitch_out_of_range`：Pitch 越界
- `motor_output_limited`：电机输出触碰限幅

安全保护的原则是：机器人不是先追求能动，而是先保证能停。

## 五、实验步骤

1. 新建 STM32 HAL 工程，完成时钟、GPIO、TIM、CAN、UART 初始化。
2. 加入本交付包中的 `Core/Inc` 和 `Core/Src`。
3. 在 `main.c` 中调用 `App_Init()` 和 `App_Loop()`。
4. 修改 `bsp_port.c`，先接入 `HAL_GetTick()` 和按键。
5. 观察 `remote_cmd_t` 是否能正常切换 STOP/NORMAL 和底盘模式。
6. 接入 L1/L2 指示灯，验证呼吸灯和模式灯。
7. 观察底盘目标轮速 `wheel_target[]`，确认运动学方向正确。
8. 低功率悬空测试电机输出，确认方向无误。
9. 接入编码器反馈，调试底盘速度 PID。
10. 接入 IMU 角度，调试云台 yaw/pitch PID。
11. 测试急停、Pitch 限位、输出限幅。

## 六、测试现象

预期现象：

- 上电后默认 STOP，电机不自动转动。
- L1 以呼吸灯形式提示程序运行。
- L2 根据模式以不同频率闪烁。
- 按下 stop 后所有电机输出清零。
- key1 能切换底盘模式。
- key2/key3 能改变 yaw/pitch 目标角。
- Pitch 目标角不会超过 `-20°` 到 `30°`。

## 七、常见问题

### 问题 1：电机方向反了

原因：电机安装方向或编码器方向和软件编号不一致。

解决：先不要改 PID，先确认轮子编号和电机方向。必要时在 `BSP_SetMotorOutput()` 中调整某一路电机符号。

### 问题 2：PID 输出很大但电机不动

原因：可能是 PWM 没接、CAN 没发出去、电机未上电、编码器反馈为 0。

解决：先用固定小输出测试电机，再接 PID。

### 问题 3：云台一直抖

原因：PID 参数过大、IMU 角度方向反了、目标角变化太快。

解决：先只开 P，确认输出方向，再逐步加 D。

### 问题 4：按键按一次触发多次

原因：机械按键抖动。

解决：检查 `REMOTE_DEBOUNCE_MS` 是否生效。
