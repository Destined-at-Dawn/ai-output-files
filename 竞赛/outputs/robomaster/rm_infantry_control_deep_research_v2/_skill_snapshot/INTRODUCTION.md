# Deep-Research v4.0 — 深度调研智能体

> 调研是贯穿所有工作的底层能力。不是"搜竞品"的专属动作。

---

## 一句话定位

**6 大场景 × 多平台搜索 × YouTube 字幕提取 × 1.25 倍递增迭代 × 一鱼多吃生态联动** —— 从信息采集到知识沉淀的全链路智能体。

---

## 为什么需要这个 Skill

大多数 AI 调研是这样的：用户说"帮我搜一下" → AI 搜 3 个链接 → 给一段总结 → 结束。下次对话，一切归零。

**问题不是 AI 搜不到，而是：**

1. **不读画像** — 每次调研都重新问"你要什么深度"，不知道用户偏好外网优先、要点开网页、要道法术器穿透
2. **不产出文件** — 调研结论留在对话里，对话结束就丢失
3. **不喂回生态** — 调研完了不更新记忆、不更新画像、不登记产出，系统越用越笨
4. **不迭代** — 搜完就停，不问"要不要继续深入"，不递增搜索范围
5. **信息源单一** — 只搜 Google/中文社区，漏掉 GitHub 源码、X 原创讨论、Reddit 深度体验、YouTube 实机演示

Deep-Research 解决的是**信息获取的复利问题**：每次调研都让下一次更精准、更高效、更有深度。

## 核心能力

### 双模式入口

- **调研模式**（触发词：调研/搜/找/查/竞品/有哪些/怎么学/什么工具好）→ 完整 7 Phase 流程
- **迭代模式**（触发词：迭代/继续优化/再深入/不要节约token/续跑）→ 读已有报告，1.25 倍递增

两个模式独立触发、互相引用。调研完必问"要不要迭代"，迭代时自动读上次调研。

### 6 大场景自适应

| 场景 | 信号词 | 调研重点 | 输出侧重 |
|------|--------|---------|---------|
| 学习 | 高数/大物/电路/四级/考试 | 思维方式、应试技巧、学习路线 | 结构化笔记 + 路线图 |
| 自媒体 | 小红书/公众号/爆款/变现 | 爆款拆解、平台算法、成功案例 | 拆解报告 + 可复用模板 |
| 竞赛 | FPGA/RTL/硬件/代码/方案 | 已有解决方案、GitHub 项目 | 方案对比 + 推荐路径 |
| 产品 | 竞品/需求/市场/用户 | 竞品分析、需求验证、市场机会 | 竞品卡片 + 差距矩阵 |
| 技能 | skill/工具/插件/效率 | 好用的 AI skill、工具组合 | 推荐 + 组合方案 |
| 学术 | 论文/研究/方向/文献 | 研究方向、创新缺口 | 分级报告 + 研究提案 |

### 多平台搜索矩阵

每个平台执行两轮：**5 热门 + 5 新鲜**。

- **Round 1（热门）**：按 stars/upvotes/views 排序，找最主流的
- **Round 2（新鲜）**：按发布时间倒序 + 质量筛选，找最新高质量的

**信息源头质量阶梯**：
```
一手来源（最高）  → GitHub 源码 / X 原创讨论 / Reddit 深度帖 / 学术论文
二手整理（中等）  → YouTube 视频字幕 / 产品官网 / 博客教程
三手转述（参考）  → 中文社区 / 知乎 / 掘金
```

### YouTube 字幕提取（必执行）

每次调研必须在 YouTube 搜索相关视频 → 提取字幕 → 保存到本地 → 做道法术器分析。

### 1.25 倍递增迭代

- 第 1 轮 ≥ 20 个修改
- 第 2 轮 ≥ 25 个修改
- 第 3 轮 ≥ 32 个修改
- ...每轮是上一轮的 1.25 倍

**不是凑数量**：每轮必须提炼 1 条黄金法则（永久性规则），搜索饱和时自动检测并询问是否停止。

### 一鱼多吃（8 个生态副作用）

每次调研完成后自动写入：

| # | 副作用 | 写入位置 |
|---|--------|---------|
| ① | 调研报告 | `outputs/{主题}-调研报告-{日期}.md` |
| ② | 今日记忆 | `memory/{today}.md` |
| ③ | 调研画像更新 | `memory/research-profile.md` |
| ④ | 使用日志 | `memory/skill-usage-log.md` |
| ⑤ | 产出登记 | `artifact-registry.md` |
| ⑥ | 记忆候选 | 待用户确认后写入 `memory/long-term.md` |
| ⑦ | 进化信号 | `evolution-calendar.md`（发现新模式时） |
| ⑧ | 路由进化 | `skill-routing-table.json`（发现新触发词时） |

## 工作流总览

```
Phase 0  生态预热        → 读画像 + 读历史 + 场景判断
Phase 1  调研澄清        → 问题消解 + 行动锁 + 镜像复述
Phase 2  多平台搜索      → 5 热门 + 5 新鲜 + YouTube 字幕
Phase 3  深度分析        → 道法术器穿透 + 刚需三问 + 认知框架
Phase 4  报告产出        → 竞品卡片 + 差距矩阵 + 边界声明
Phase 5  一鱼多吃        → 8 个生态副作用自动写入
Phase 6  迭代（可选）    → 1.25 倍递增 + 黄金法则 + 饱和检测
Phase 7  必问迭代        → "要迭代吗？给你 3 小时窗口"
```
