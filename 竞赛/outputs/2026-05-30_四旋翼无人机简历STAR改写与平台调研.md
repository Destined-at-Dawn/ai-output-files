# 四旋翼无人机项目简历 STAR 改写与平台调研

> 生成日期：2026-05-30
> 对象定位：大一、双非、机械设计方向学生
> 使用场景：竞赛/项目经历、实习简历、作品集项目页

## 结论先行

这段经历应该写成“机械设计学生完成四旋翼无人机数字样机逆向建模与装配验证”，不要写成“成熟工程师交付数字孪生系统”。

核心原因：
- 大一学生的可信优势是：软件上手快、能把二维图转三维、能做装配约束、能发现干涉/重心/安装问题。
- 简历语言要让面试官愿意追问：“你怎么建模、怎么装配、怎么验证？”而不是质疑：“你真的做了飞行动力学和数字孪生闭环吗？”
- STAR 法则在简历里要压缩成“背景/任务隐含在项目名，行动和结果写进 bullet”，不能写成长段故事。

推荐最终版如下：

## 最终简历版本

项目名称：四旋翼无人机数字样机逆向建模与装配验证

- 面向四旋翼无人机数字化设计与仿真验证需求，基于二维图纸与结构参考，使用 SolidWorks 完成机架、旋翼桨叶、飞控仓等核心零部件的三维逆向建模，搭建整机多层级装配模型。
- 按动力、结构、控制三类子系统拆解整机结构，梳理电机座、机臂、飞控仓、桨叶等关键部位的装配关系与配合约束，优化零部件定位方式，降低后续样机加工与装配返工风险。
- 结合 DFM 思维完成干涉检查、装配约束校核与重心分布分析，基于运动仿真验证旋翼安装空间和整机姿态变化趋势，为物理样机制造提供可复用的数字样机与装配依据。

如果有真实数据，可以升级为数据版：

- 使用 SolidWorks 重建四旋翼无人机 XX 个零件、完成 X 级装配结构，输出三维模型、装配爆炸图与关键零件工程图，用于支撑后续样机加工。
- 针对机臂连接、飞控仓安装、电机座定位等关键结构，完成 X 处干涉排查与 X 轮装配约束调整，使整机重心偏移控制在 XX mm 范围内。
- 按动力、结构、控制三大子系统建立模型命名与装配层级规范，沉淀可复用建模流程，提升后续改型效率。

## STAR 拆解

S：Situation，背景
- 四旋翼无人机需要从二维图纸或参考结构转化为可验证的三维数字样机。
- 对大一机械设计学生来说，项目价值不是“发明无人机”，而是把结构理解、CAD 建模、装配验证串成一个闭环。

T：Task，任务
- 重建机架承力结构、旋翼桨叶、飞控仓等核心部件。
- 完成整机装配关系，检查干涉、重心、运动空间和制造装配可行性。

A：Action，行动
- 用 SolidWorks 做逆向建模和多层级装配。
- 按动力、结构、控制三个子系统拆分模型。
- 用 DFM 思维检查公差、装配顺序、加工可行性。
- 用运动仿真和重心分析辅助验证结构方案。

R：Result，结果
- 得到可用于展示、复盘、改型和加工沟通的三维数字样机。
- 支撑后续物理样机制造前的结构风险排查。
- 形成“二维图纸 -> 三维建模 -> 装配验证 -> 仿真检查 -> 样机依据”的项目闭环。

## 原文问题诊断

原文优点：
- 技术关键词完整：SolidWorks、逆向建模、DFM、装配、运动仿真、重心分布。
- 项目链路完整：二维图纸、三维样机、核心零部件、整机装配、仿真验证。

主要风险：
- “数字孪生交付保障”过重，更像企业级交付，不像大一学生项目。
- “飞行动力学参数收敛”容易被追问控制模型、气动模型、推力曲线、PID 调参。如果实际只是 SolidWorks 运动仿真，不建议这样写。
- “攻克关键技术难点”太满，建议改成“解决二维拓扑提取、特征重构、装配约束建立等建模难点”。
- “确保飞行姿态稳定性”如果没有实飞或控制仿真数据，不建议写“确保”，改成“辅助评估”更稳。

## 为什么这样改

简历不是把项目写得最大，而是把“可信能力”打穿。

对大一双非机械设计学生，最强信号不是学校背景，而是：
- 能做出完整作品，而不是只学过软件。
- 能解释结构为什么这么设计，而不是只会拉伸、旋转、打孔。
- 能把装配、加工、仿真连起来，而不是孤立建模。
- 能把项目沉淀成图纸、模型、装配树、仿真截图、问题清单。

## 调研后的写法原则

1. 先写项目结果，再写过程。
   - 弱：负责四旋翼无人机建模。
   - 强：使用 SolidWorks 完成四旋翼无人机核心零件建模与整机装配验证。

2. 先写个人动作，再写团队背景。
   - 弱：团队完成了无人机数字化设计。
   - 强：独立重建机架、桨叶、飞控仓等核心零部件，并完成整机装配关系梳理。

3. 能量化就量化，不能量化就写交付物。
   - 数字：XX 个零件、X 级装配、X 轮迭代、X 处干涉。
   - 交付物：三维模型、工程图、装配爆炸图、仿真截图、问题清单。

4. 避免超出实际能力边界。
   - 慎用：数字孪生、飞行动力学参数收敛、确保飞行稳定。
   - 可用：数字样机、运动仿真、重心分析、装配验证、辅助样机制造。

5. 为面试追问准备证据链。
   - 模型树截图。
   - 装配层级截图。
   - 干涉检查截图。
   - 重心分析截图。
   - 关键零件工程图。
   - 一页项目复盘：问题、方案、结果、反思。

## 面试回答草稿

面试官问：你这个项目到底做了什么？

回答：
我主要做的是四旋翼无人机的数字样机逆向建模和装配验证。项目开始只有二维图纸和部分结构参考，我先把整机拆成动力、结构、控制三个子系统，再用 SolidWorks 重建机架、桨叶、飞控仓等核心部件。建模完成后，我重点做了装配约束、干涉检查和重心分析，目的是在样机加工前提前发现安装空间、结构定位和重心分布问题。这个项目让我从单纯画零件，过渡到按整机装配关系思考机械设计。

面试官问：你最难的地方是什么？

回答：
最难的是二维图纸到三维特征的还原。二维图里有些结构关系不直观，比如机臂和电机座的连接、飞控仓内部安装空间、桨叶旋转空间等。我不是直接照着画，而是先确定装配基准和功能关系，再建零件特征，最后放回整机装配里检查是否干涉。这个过程让我意识到，机械建模不是把外形画出来，而是要让结构能装、能改、能解释。

面试官问：你怎么体现 DFM？

回答：
我主要从加工和装配两个角度考虑。加工上，我避免过多复杂曲面和不必要的小特征；装配上，我会关注零件定位、螺钉安装空间、飞控仓可维护性，以及电机座和机臂连接是否容易装配。虽然项目还不是企业级制造，但我已经开始用“后续能不能加工、能不能装、能不能维护”的标准检查模型。
