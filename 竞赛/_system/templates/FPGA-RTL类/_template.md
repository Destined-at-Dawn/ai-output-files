# FPGA-RTL 类交付模板

> **适用**：FPGA/Verilog/VHDL 开发板竞赛或客户交付
> **来源**：从 SOP-12「FPGA RTL 客户交付一把过」提取通用部分
> **使用方法**：复制本模板 → 替换 `{占位符}` 为实际项目信息 → 按 SOP-06 执行开发

---

## 1. 交付物清单

| # | 交付物 | 格式 | 必须/可选 |
|---|--------|------|----------|
| 1 | RTL 源代码 | .v / .sv / .vhd | 必须 |
| 2 | 约束文件 | .xdc / .sdc | 必须 |
| 3 | 构建脚本 | .tcl | 必须 |
| 4 | Bitstream | .bit + .ltx | 必须（综合后） |
| 5 | 实验文档 | .md + .docx + .pdf | 必须 |
| 6 | README | .md | 必须 |
| 7 | 工程文件（可复现） | .xpr / 完整工程目录 | 建议 |
| 8 | 答辩材料 | 参见 SOP-05 | 按需 |
| 9 | 自我学习文件夹 | — | 按需 |

## 2. 代码修改前检查

- [ ] 读 README、top、XDC、Tcl、核心 RTL、历史交接记录
- [ ] 找最新客户反馈（优先级 > 旧 README > AI 假设）
- [ ] 确认：reset 极性、LED 有效电平、I2C/外设地址、时钟频率、SMA 连接点
- [ ] 先归档，不直接覆盖可用版本

---

## 3. RTL 验证清单

- [ ] Synthesis：0 error，0 critical warning
- [ ] Implementation/bitgen：bitstream completed
- [ ] Timing：all constraints met
- [ ] `.bit` 和 `.ltx` 均存在
- [ ] RTL/Tcl/XDC/README 无乱码（ASCII 注释优先）
- [ ] ILA probe 足够定位问题，不挤爆 BRAM
- [ ] 单总线/寄生供电器件：验证冷启动和 warm reset

## 4. 文档章节结构

| 章节 | 内容 | 参考 SOP |
|------|------|---------|
| 实验目的 | 要解决什么问题 | SOP-11 |
| 硬件资源 | 开发板型号、FPGA 芯片、外设列表 | SOP-06 |
| 系统总体结构 | 架构图、模块划分、数据流 | SOP-04 |
| 模块说明 | 每个模块的职责、接口、关键参数 | SOP-04 |
| 管脚表 | 与 XDC 严格一致 | SOP-06 |
| 实验步骤 | 从打开 Vivado 到看到现象的全流程 | SOP-11 |
| 实验现象 | 真实截图/波形，不伪造 | — |
| 常见问题 | 包含客户实际反馈的问题 | — |
| 实验结论 | 性能指标、改进方向 | — |

**文档铁律**：
- 工程路径、`.xpr`、`.bit`、`.ltx` 路径必须真实存在
- 原理图来自真实资料，不用虚构图片
- 实验现象写「应该看到什么」，不伪造未验证的结果

## 5. 打包检查

每个功能单独一个 ZIP，中文功能名。

ZIP 内必须有：
- [ ] DOCX（实验文档）
- [ ] README（客户先看）
- [ ] `rtl/` 或 `src/`（源代码）
- [ ] `xdc/`（约束文件）
- [ ] Tcl 脚本
- [ ] `.bit` + `.ltx`（如重新综合）
- [ ] 工程文件（如要求可复现）

**铁律**：没有重新生成 bitstream → 不放旧 `.bit/.ltx` 冒充新版。

## 6. 已知坑（FPGA 通用）

| 坑 | 说明 |
|----|------|
| DAC/ADC 命名 ≠ 实际地址 | 逻辑名 DAC1/DAC2 不能只信名字，要以板上实测为准 |
| I2C 总线吞吐瓶颈 | ADC读 + DAC写共享总线，不能承诺芯片标称采样率 |
| 输入波形范围 | 建议 2.0~2.4Vpp、offset 1.65V，不要满幅避免削峰 |
| ILA 深度 | 必须覆盖事件周期，慢波形不能用过短窗口判断「没动」 |
| 源码编码 | ASCII 注释优先，中文注释需验证编码一致 |
