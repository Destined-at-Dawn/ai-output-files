**教训2：没有磁盘验证就开始干活**
- 现象：多轮声称"数据丢了"但实际在D:/AICompData/
- 根因：凭压缩后残存记忆判断，不验证
- 修复：F10强制验证——Glob确认文件存在+Read确认内容非空

**教训4：Skill反复创建没有锚定**

## 00:15 Goal 模式全流程重写（v4.0）

### 用户指令
用/li-improve 帮我复盘，避免下次再犯，修改对应的 skill 和 SOP 和 memory 和 CLAUDE

### 核心改造
从"防重复检查清单"升级为"完整 Goal 模式生命周期协议"：

**改造成 5 状态状态机**：
```
INIT → PRE_CHECK → RUNNING → RECONCILE → CP_WRITE
```
- INIT：分析目标、格式化 prompt
- **PRE_CHECK（必须先做）**：磁盘验证（Glob）+ 资产盘点（Read CP）+ 路由扫描（skill）+ 去重检查（对比历史 CP）
- RUNNING：逐任务执行，每个任务完成后立即写 CP
- RECONCILE：正文覆盖度核对 + 未覆盖项识别
- CP_WRITE：写 CP02/CP03 + 消融表

**新增 CP 产出格式**：
```markdown
## CP_{NN} — {时间戳}
- done: [...]
- failed: [...]
- created_files: [...]
- created_dirs: [...]
- key_decisions: [...]
- datasets: {name: {path, images, classes, format}}
- training: {run: {map50, map50_95, epochs, status}}
- next_actions: [...]
```

**新增中断续跑协议**：
- 超时/429 → 不是失败，是正常中断
- 续跑 prompt 标准模板（从 CP 恢复，不是从零开始）
- 续跑第一件事：读 CP + 磁盘验证

**新增 F12 铁律**（CLAUDE.md）
- 8 条 Goal 模式铁律：先验证再动手 / 有 CP 不重新开始 / 中断不等于失败 / 不重复已有产出 / 每轮必须写 CP / CP 完成度自评 ≤60% 不用 complete / Skill 创建前必须 Glob 检查 / 路由扫描不豁免
