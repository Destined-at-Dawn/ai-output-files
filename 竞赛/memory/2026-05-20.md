## Bug #1: cnt 多驱动

- **现象**：Vivado 综合报 multi-driver error
- **根因**：`cnt` 在两个 always 块中赋值（计数器块 + 主状态机块）
- **修复**：加 `cnt_clr` 信号桥接，计数器只在一个块里驱动

## Bug #2: TD_STATUS 读的是垃圾值

- **现象**：touch_valid 始终为 0
- **根因**：S_RD_STA_6 未捕获 i2c_rd_data，S_CHECK 读到的是旧值 0x70
- **修复**：S_RD_STA_6 加 `buf_td <= i2c_rd_data`，S_CHECK 用 `buf_td[3:0]`

## Bug #3: 坐标映射只有 0 和 1

- **现象**：数码管只显示 0 或 1
- **根因**：`touch_x <= buf_xh[3:0]` 只取高4位，0~479 的高4位只有 0 或 1
- **修复**：用 DSP48E1 乘法器 `raw_x * 65536 >> 21` + 四舍五入

## Bug #4: 坐标映射常量算错（三轮修正）

- 第一轮：21845 → 最大输出 5（错误）
- 第二轮：65536 → 最大输出 14（截断丢 15）
- 第三轮：65536 + 2^20 偏移 → 四舍五入，0~15 精确（正确）
- **教训**：映射常量必须手算边界值验证（0→0, max→15），不要只算中间值

## Bug #5: wd_rst 信号从未赋值

- **现象**：看门狗永远不触发
- **根因**：`wire wd_rst = (wd_cnt >= T_WD);` 这一行漏写了
- **修复**：加一行赋值

## Bug #6: touch_ctrl.v 被 GBK 编码操作清空

- **现象**：Python 转 GBK 编码时文件变成 0 字节
- **根因**：GBK 编码异常 + 没有备份
- **修复**：从上下文重建 408 行代码
- **教训**：GBK 文件修改前必须先备份，用"备份→修改→验证→删备份"原子操作

## Bug #7: COORD_SWAP 横屏坐标交换

- **发现**：厂商 ctp.c 在横屏模式下 0x03→Y、0x05→X（和 datasheet 标签反的）
- **修复**：加 `COORD_SWAP` 参数，上板后一个参数切换

## Bug #8: SDA 三态门被综合成 push-pull

- **现象**：偶尔看到 SCL/SDA 有短暂波形，然后瞬间变全高
- **根因**：`assign i2c_sda = sda_oe ? sda_out : 1'bz;` Vivado 综合成 push-pull
- **修复**：改为 open-drain 写法 `(sda_oe & ~sda_out) ? 1'b0 : 1'bz`
- **教训**：I2C SDA 从第一天就应该用 open-drain

## Bug #9: CMD 编码不匹配（致命）

- **现象**：SCL/SDA 一直高电平，I2C 完全不工作
- **根因**：i2c_master CMD_START=0，touch_ctrl CMD_START=1，全部差 1
- **修复**：i2c_master 加 CMD_IDLE=0，其余全部 +1，对齐到 touch_ctrl
- **教训**：多模块共享常量必须逐一比对，不能假设一致

## Bug #10: 全局 ACK 错误检测制造假阳性

- **现象**：偶尔有 I2C 波形然后瞬间变全高
- **根因**：`if (i2c_done && !i2c_ack)` 每个周期都检查，done 脉冲时 ack_err 是上一条命令的残留
- **修复**：删掉全局检测，在 S_RD_STA_1 和 S_RD_XY_1 就地检查

## 关键决策记录

1. **RST/INT 硬件默认**：不暴露到 FPGA 端口，类比 lcd_rst/lcd_bl
2. **横屏方向**：长边左右(X=480)，短边上下(Y=320)，左下(0,0)→右上(F,F)
3. **PMOD 引脚确认**：朱工确认 SCL/SDA 在 LCD 40-pin 插排上，不在 PMOD
4. **初始化延迟**：300ms→200μs（调试用），调通后改回
5. **轮询间隔**：50ms→10ms（调试用）
