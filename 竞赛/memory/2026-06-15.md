# 2026-06-15 竞赛工作区日志

## 皮影戏项目 — 第五部分重写

### 背景
用户小黎审阅了附件2任务书的第五部分（汇演展望），做出关键战略决策：
**决赛只做一个角色的自动化（蔡俊—机长），用音效和光效弥补角色数量的简化。**

### 产出
- `outputs/§5-汇演展望-单人物音效光效.md`（7215字符），完整替换了原"四角色多人物联动"方案

### 核心技术决策

**硬件架构（三个子系统）**：
- 动作：Arduino Uno + PCA9685 + 4路SG90金属齿舵机（颈/肩/肘/腰）
- 音效：DFPlayer Mini + microSD + PAM8403功放 + 8Ω3W扬声器
- 光效：WS2812B灯带（60灯珠/1米）+ FastLED库

**音效设计**：
- 5个MP3音频文件（开场BGM、引擎启动、起飞爬升、提示音、谢幕BGM）
- DFPlayer Mini通过SoftwareSerial通信，BUSY引脚硬件握手同步

**光效设计**：
- 5种模式：暖光常亮（3000K）、冷光常亮（6000K）、呼吸闪烁、熄灭、冷暖渐变
- 30fps刷新率保证渐变平滑

**代码架构**：
- 五层分离：config.h → motion层 → audio层 → light层 → choreography层
- 事件驱动编排，时间轴触发

**预算**：硬件约100元，重量<300g，代码增量约300行

### 战略逻辑
决赛现场3-4天，随机组队。与其摊薄精力做四角色半成品，不如把一个角色做透。
"一人、一灯、一影、一腔"的纯粹 → 数字化应强化而非稀释。
