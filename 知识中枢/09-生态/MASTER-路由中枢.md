## 一、问题背景

牛马AI有4个工作区（创作/求职/个人/竞赛），每个工作区下有多个项目子目录。

**同一类文件散落各处：**

| 文件类型 | 工作区级 | 项目级 | 总计 |
|----------|---------|--------|------|
| CLAUDE.md（约束） | 4个 | 21个 | **25个** |
| MEMORY.md（记忆索引） | 4个 | 7个 | **11个** |
| long-term.md（长期记忆） | 4个 | 13个 | **17个** |
| SOPs/（标准流程） | 4个目录 | 3个目录 | **7个目录** |

**核心矛盾：** 文件太多读不完 → 浪费context → 降低运行质量。

## 二、文件层级关系（谁听谁的）

```
MASTER.md（本文件）          ← 最高路由中枢，定义优先级和索引
  └── 工作区 CLAUDE.md       ← 工作区级约束（1800行/650行/430行/230行）
        └── 项目 CLAUDE.md   ← 项目级约束（100~375行，仅覆盖本项目）
```

### 优先级规则

```
场景1：AI在工作区根目录执行任务
  → 读工作区 CLAUDE.md

场景2：AI在项目子目录执行任务
  → 先读工作区 CLAUDE.md（全局约束）
  → 再读项目 CLAUDE.md（项目约束，覆盖/补充全局）
  → 两者冲突时，项目级优先（但不能违反全局铁律）

场景3：AI需要跨工作区信息
  → 读本文件 MASTER.md 路由到对应工作区

场景4：AI需要个人信息
  → 直接读 telos（`C:\Users\13975\.claude\telos\`），不读任何工作区的个人信息副本
```

### 读取策略（节省context）

```
不是所有文件都要读！按需读取：

开局必读（每次对话开始）：
  1. 本文件 MASTER.md（100行，快速定位）
  2. 当前目录的 CLAUDE.md（工作区级或项目级）

按需读取（遇到对应场景才读）：
  - memory/MEMORY.md → 需要恢复上下文时
  - memory/long-term.md → 需要长期记忆时
  - SOPs/SOP总索引.md → 需要执行标准流程时
  - 具体SOP文件 → 匹配到SOP后才读

禁止一次性全读 → 浪费context → 降低质量
```

## 六、技能组协调系统（v2.0 新增）

> 技能组 = 多个 skill 围绕同一意图，按预定顺序或并行自动触发的协调机制。

| 文件 | 用途 | 路径 |

### 4个技能组

| 技能组 | 触发场景 | 成员 | 执行模式 |
|--------|---------|------|---------|
| **Core Loop** | 每次对话必触发 | li-sync(R08) + li-memory(L2) + **session-summary**(收尾) | 条件并行 |
| **Workspace Builder** | 新建工作区/项目 | scaffold-workspace → li-sync → li-memory → session-summary | 串行 |
