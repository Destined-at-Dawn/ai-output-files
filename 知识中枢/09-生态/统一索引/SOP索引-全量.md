## 一、SOP层级关系

```
工作区 SOPs/SOP总索引.md     ← 本工作区SOP入口
  └── SOPs/{具体SOP}.md      ← 具体执行流程
        └── 项目 SOPs/       ← 项目级SOP（仅本项目）
```

### 读取策略

| 场景 | 该读什么 |
|------|---------|
| 执行任务 | 先读工作区 SOP总索引.md → 匹配SOP → 读具体SOP文件 |
| 项目内执行 | 先查项目级SOPs/ → 没有则回退到工作区级 |
| 跨工作区 | 读本文件 → 路由到目标工作区SOP |

## 认知科学支撑：SOP索引的认知设计（百大认知书籍）

| 认知机制 | 来源 | 在SOP索引中的应用 |
|---------|------|----------------------|
| **外部化程序性记忆** | 022-认知神经科学 §概念1 | SOP索引=程序性记忆的外部化目录——"遇到X情况该执行什么流程"不再依赖AI的"记忆"，而是查索引→秒级定位→零遗忘 |
| **组块化检索** | 022-认知神经科学 §概念1 | SOP按类型分组(跨区/Git/隐私等)=组块化——AI不需要遍历全部SOP，只需先确定类型再检索→两步定位→检索时间÷3 |
| **清单效应** | 027-清单革命 §概念1 | SOP索引=操作清单的清单——每个SOP本身是清单，SOP索引是"哪些清单适用于当前场景"的元清单→双层清单=双层安全网 |
| **识别优于回忆** | 009-认知天性 §概念1 | 索引="识别式"检索(看到名字就能判断是否相关)vs"回忆式"检索(凭记忆想有哪些SOP)→识别准确率>回忆准确率→SOP被使用的概率↑ |
| **认知经济性** | 025-认知负荷理论 §概念1 | 一行SOP索引=一个SOP的"最小可识别单元"——不展开全文、不解释用途，只给名称+路径+一句话→信息密度最大化→阅读时间最小化 |
