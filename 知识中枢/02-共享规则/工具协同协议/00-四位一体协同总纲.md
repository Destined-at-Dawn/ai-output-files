## 三、核心协同协议

### 协议 1：进场必问 Hermes

**任何工具开始新任务前**：
```
你 → Hermes："我要做 X"
Hermes → 返回 ≤500 字上下文：
  1. 相关项目卡
  2. 最近相关产出
  3. 待解决问题
  4. 其他工具最近做了什么
```

### 协议 2：交接必写 Handoff

**格式**：`handoffs/{项目名}-{from}-{to}-{任务简称}.md`

Handoff 文件结构（三栏）：
1. **{from} 已完成**：产出路径 + 一句话描述
2. **{to} 需要做**：具体任务清单
3. **约束 & 注意事项**：技术限制/设计偏好/文件位置

### 协议 3：产出必注册

**任何工具产出可复用文件后** → 追加一行到 `artifact-registry.md`：
```
| {日期} | {产出路径} | {一句话描述} | {谁产出的} | {关联项目卡} |
```

## 四、任务流转路线

### 路线 A：内容创作（Newmax 端到端）
```
用户需求 → Newmax（分析+创作） → 产出注册
```

### 路线 B：技术项目（Codex + WorkBuddy + Newmax）
```
用户需求 → Newmax（需求分析） → Handoff → Codex（代码+文档）
    → Handoff → WorkBuddy（展示页） → Handoff → Newmax（推广文案）
```

### 路线 C：数据报告（Codex + WorkBuddy）
```
用户需求 → Codex（数据处理+分析） → Handoff → WorkBuddy（可视化看板）
```

### 路线 D：飞书内容（Newmax + WorkBuddy）
```
用户需求 → Newmax（内容创作） → Handoff → WorkBuddy（飞书文档化+推送）
```

## 六、边界铁律（跨工具共享）

1. **不做别人的事**：每个工具严守自己的主战场（见各自职责规范 §禁止事项）
2. **不跳过交接**：工具 A → 工具 B 必须写 handoff，不能不声不响
3. **不重复造轮子**：动手前先查 artifact-registry.md，确认没有已有产出
4. **不丢失上下文**：handoff 必须包含足够信息让接手工具独立工作
