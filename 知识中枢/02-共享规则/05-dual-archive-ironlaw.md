# R05 — 双归档铁律（真源 v1.0）

> **此规则优先级高于所有其他操作规则。违反=数据丢失=最严重错误。**
> 来源：小黎明确要求——"所有删除文件保存两份，第一份全局归档，第二份原工作区归档"

---

## 标准文本

### 归档目录

| 归档位置 | 路径 |
|---------|------|
| **全局归档** | `E:\ai产出文件\牛马\归档\` |
| **本工作区归档** | `{{工作区路径}}\归档\` |

### 删除前六步（缺一不可）

1. **`ls` 检查**：列出目录/文件的完整内容
2. **识别原工作区**：根据被删除文件的路径判断属于哪个工作区
3. **第一份→全局归档**：`cp -r` 复制到 `E:\ai产出文件\牛马\归档\`（全局备份）
4. **第二份→工作区归档**：`cp -r` 复制到 `{{工作区路径}}\归档\`（没有则 `mkdir -p` 新建）
5. **AskUserQuestion 确认**：展示清单 + 双归档路径 + 验证状态
6. **验证双归档完整性**：ls 确认两份一致后才执行删除

### 删除后报告（强制）

每次删除完成后必须输出：

```
✅ 删除完成
- 删除的文件/目录：[清单]
- 全局归档：E:\ai产出文件\牛马\归档\{文件名} — 已验证
- 工作区归档：{{工作区路径}}\归档\{文件名} — 已验证
```

### 禁止

- 不得在未检查内容的情况下删除任何目录
- 不得在未归档的情况下删除任何文件
- 不得在未获得用户确认的情况下删除任何文件
- **不得只归档一份**——必须全局 + 工作区两份都归档
- **不得删除后不报告**——必须输出双归档路径 + 验证状态

### 各工作区归档路径速查

| 工作区 | 归档路径 |
|--------|---------|
| 创作 | `E:\ai产出文件\牛马\创作\创作\归档\` |
| 个人 | `E:\ai产出文件\牛马\个人\个人\归档\` |
| 竞赛 | `E:\ai产出文件\牛马\竞赛\竞赛\归档\` |
| 求职 | `E:\ai产出文件\牛马\求职\求职\归档\` |

## 认知科学支撑（百大认知书籍）

| 认知机制 | 来源 | 在双归档铁律中的应用 |
|---------|------|----------------------|
| **损失厌恶** | 010-思考快与慢 §概念4 | 双归档=利用损失厌恶的逆向——"如果只有一份备份，丢失时的痛苦>两份备份多占的空间"。两份备份=损失概率降至近零→用户焦虑消除 |
| **冗余设计** | 046-反脆弱 §概念1 | 全局归档+工作区归档=冗余设计的工程化——单点故障=脆弱；双点备份=抗脆弱。归档策略遵循"3-2-1备份原则"的简化版 |
| **确认偏误防御** | 010-思考快与慢 §概念3 | "删除前六步"=防御确认偏误——AI可能"觉得"文件已经备份好了→确认偏误导致跳过验证。六步强制流程=打断"觉得"→强制"确认" |
| **操作性条件反射** | 016-习惯的力量 §概念1 | "删除后报告"=操作性条件反射——删除→报告→用户确认→正强化→"删除必报告"成为自动行为。不报告→用户追问→负强化→行为消失 |
| **预承诺策略** | 051-WOOP §概念1 | "禁止未归档删除"=预承诺策略——在删除冲动之前先"绑住自己的手"(归档)。类似于"先把零食锁起来"——决策时的冷静>行动时的冲动 |
