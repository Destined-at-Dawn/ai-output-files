## 三、同步流程（4 步，不可跳过）

```
Step 1：识别 — 本轮对话中是否读到小黎的新个人信息？
  → 是 → 继续
  → 否 → 跳过同步

Step 2：读取 — Read telos 对应文件当前内容
  → 确认文件存在且可读

Step 3：追加 — Edit 在对应区域追加新信息
  → 格式：`- {新信息}（{日期}，来源：{当前工作区名}）`
  → 在文件末尾「变更记录」表追加：`| 日期 | 变更摘要 | 来源：{工作区名} |`

Step 4：验证 — Read 确认追加内容已落盘
  → 验证通过 = 同步完成
  → 验证失败 = 重新执行 Step 3
```

## 四、禁止行为

- 禁止只更新本工作区而不更新 telos
- 禁止覆盖 telos 文件——只允许追加和修改
- 禁止删除 telos 中的已有信息（除非用户明确要求）
- 禁止在本工作区 CLAUDE.md 中硬编码小黎的个人信息而不同步到 telos
- 禁止 Write 后不 Read 验证

## 五、telos 与本工作区 memory 的关系

| 系统 | 存什么 | 谁维护 |
|------|--------|--------|
| **telos/** | 全局个人信息（身份/目标/偏好/教训） | 所有工作区共同维护 |
| **memory/long-term.md** | 本工作区的长期记忆（项目决策/工具选择） | 本工作区 |
| **memory/{日期}.md** | 本工作区的每日进展 | 本工作区 |

**关键区别**：telos 是「我是谁」，memory 是「我在这个项目里做了什么」。两者互补，不替代。

## 六、示例场景

### 场景 A：小黎提到新经历
> 小黎："我最近在帮老师做一个电力系统仿真项目"
1. 识别 → 新经历，需要同步
2. Read `telos/identity.md`
3. Edit 追加：`- 正在参与电力系统仿真项目（2026-05-09，来源：{工作区名}）`
4. Read 验证

### 场景 B：小黎调整目标
> 小黎："我改主意了，不想去国网了，想去做芯片设计"
1. 识别 → 目标调整，需要同步
2. Read `telos/goals.md`
3. Edit 修改+变更记录追加
4. Read 验证

### 场景 C：AI 被批评
> 小黎："你每次回复都太长了，以后简洁点"
1. 识别 → 用户偏好变化
2. Read `telos/learned.md`
3. Edit 追加偏好条目
4. Read 验证

### 场景 D：没有新信息
> 小黎："帮我查一下 FPGA 的时序约束怎么写"
1. 识别 → 技术问题，不涉及新个人信息
2. 跳过同步，直接回答问题

## 认知科学支撑（百大认知书籍）

| 认知机制 | 来源 | 在个人信息同步SOP中的应用 |
|---------|------|----------------------|
| **单一真相源** | 008-协同智能 §概念1 | telos=分布式认知系统中的"共享工作记忆"。5个工作区=5个认知节点，每个节点只看到小黎的一部分。telos=节点间的"信息总线"→确保每个节点看到完整的、一致的小黎 |
| **信号检测理论** | 022-认知神经科学 §概念2 | 11种信号类型表=信号检测的分类器——"身份变化"(高信号)vs"技术问题"(噪音)。分类器偏向"宁可多捕获"(高recall)而非"精准捕获"(高precision)——漏掉一条个人信息>>多记录一条无关信息 |
| **编码特异性** | 009-认知天性 §概念1 | 追加格式`{信息}（{日期}，来源：{工作区名}）`=编码特异性——每条信息都有"时间+空间"标签。日后检索时，只要回忆起相似情境→就能精确定位 |
| **即时反馈** | 016-习惯的力量 §概念1 | 4步同步流程(识别→读取→追加→验证)=即时反馈回路。Step 4 Read验证=反馈——确认操作成功→建立信任→习惯固化。不验证→不确定→焦虑 |
| **认知-情感边界** | 026-笛卡尔的错误 §概念1 | "心理状态"信号的捕获("我最近压力大")=认知-情感边界的应用——用户的情绪状态影响工作节奏和决策质量。记录情绪=外部化情感→AI能"理解"而非"忽略"小黎的情感状态 |
