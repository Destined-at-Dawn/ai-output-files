## 同步执行流程

当 `000-总索引-Index.md` 的 `书籍总数` 变化时：

```
Step 1: 记录旧值和新值（如 62→63）
Step 2: Grep 搜索所有工作区的旧值
  → 搜索模式：`62本`、`62 本`、`62本书`、`62 本书`、`62本完整`
Step 3: 对每个匹配结果：
  → 判断是否是百大认知相关上下文（排除无关的"62"）
  → Edit 替换为新值
Step 4: 更新 CITATION_TRACKING.md 的统计段
Step 5: Read 验证每个修改过的文件
Step 6: 输出同步报告
```

## 认知科学支撑（百大认知书籍）

| 认知机制 | 来源 | 在书数同步中的应用 |
|---------|------|----------------------|
| **一致性偏见** | 010-思考快与慢 §概念5 | 19个文件书籍数量不一致=一致性偏见的反面——每个文件维护者只"看到自己的数字"并认为是对的。统一真相源(000-总索引)=打破每个节点的"自我一致性"→强制全局一致 |
| **变化盲视** | 043-看不见的猩猩 §概念1 | 书籍数量变化="变化"——如果只更新一个文件，其他18个文件中的旧数字="变化盲视"→没人注意到不一致。15文件同步清单=对抗变化盲视→确保每个变化都被注意到 |
| **版本控制认知** | 018-算法之美 §概念1 | "标注来源+日期"=版本控制的认知等价——不是"62本"而是"62本(v52.0, 2026-05-10)"。版本标签=时间戳→AI能判断"这个数字是最新的还是过时的"→避免用旧数字做决策 |
| **自动化偏差** | 023-认知觉醒 §概念3 | Grep批量搜索+自动替换=自动化偏差的正向利用——人类手动搜索15个文件容易遗漏；Grep自动化=零遗漏。但需要"判断是否是百大认知上下文"=人机协作→完全自动化不可靠 |
