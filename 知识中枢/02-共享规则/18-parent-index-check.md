# R18：父级索引检查规则（Parent Index Check）

> 版本：v1.0 | 创建：2026-06-01
> 维护方：创作工作区
> 适用：所有工作区
> 来源：教训 E27 — 文件存在不等于能被找到

---

## 铁律

**任何新建/修改文件后，必须检查其父级索引是否已更新引用。**
**写入后未被索引引用的文件 = 不存在。**

---

## 一、什么是「父级索引」

| 索引类型 | 示例 | 位置 |
|----------|------|------|
| 目录树 | CLAUDE-xxx.md 中的目录结构 | 子模块路由文件 |
| README | 目录下的 README.md | 每个目录 |
| 路由表 | CLAUDE.md 中的路由表 | 根目录 |
| SOP总索引 | SOPs/SOP总索引.md | SOPs/ |
| 映射表 | skill-rules.json | .claude/ |
| 关键文件引用表 | CLAUDE.md 第二节 | 根目录 |

---

## 二、检查流程

1. 新建/修改文件
2. 问自己：这个文件会被谁读到？
3. 找到它的父级索引（目录树/README/路由表）
4. 检查索引是否已列出这个文件
5. 未列出 → 更新索引
6. 验证：Read 索引文件确认条目存在

---

## 三、什么情况必须检查

| 操作 | 必须检查的索引 |
|------|--------------|
| 新建约束文件 | CLAUDE-xxx.md 目录树 + 路由表 |
| 新建记忆文件 | 目录树 + README |
| 新建经验库文件 | INDEX.md + 目录树 |
| 新建SOP | SOP总索引 + 路由表 |
| 新建Skill | skill-rules.json + 路由表 |
| 修改文件名 | 所有引用旧文件名的索引 |

---

## 四、与其他规则的关系

| 规则 | 关系 |
|------|------|
| R16 根目录优先架构 | R16定义文件放哪里，R18确保放了之后能被找到 |
| F2 文件操作铁律 | F2要求Write后Read验证，R18要求Write后索引验证 |
| F6 教训回写 | 教训文件落盘后也必须更新INDEX |

---

## 五、违规信号

- 新建了文件但目录树/README里没有列出
- 新建了SOP但总索引没有追加
- 新建了Skill但路由表没有更新
- 修改了文件名但旧引用没有更新
