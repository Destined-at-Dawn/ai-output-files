## 二、核心铁律（从失败中学到的教训）

### 铁律1：先搜后建（搜索优先）
**教训**：2026-05-22 创建 li-devil 时，没有搜索已有的学术DA agent（8维挑战+反讨好协议），导致从零手写了一个低质量版本。

**强制流程**：
1. 搜索 `C:\Users\13975\.newmax\skills\` — 已安装的 skill
2. 搜索 `E:\ai产出文件\牛马\*downloads\*` — 学术研究 skill 包（常有高质量 agent）
3. 搜索 `E:\ai产出文件\牛马\知识中枢\` — 共享规则和 SOP
4. 搜索 `E:\ai产出文件\牛马\个人\个人\百大认知书籍\` — 认知科学理论支撑
5. 搜索各工作区 `SOPs/` — 已有 SOP 中可能内嵌 skill 逻辑

### 铁律2：理论先于实现（百大认知先行）
**教训**：li-devil v1 没有认知科学理论支撑，触发词和频次是凭感觉写的。

**强制流程**：
1. 确定这个 skill 解决的是什么认知问题（如"系统1替代启发式"、"非注意盲视"）
2. 从百大认知书籍中找到对应理论和原句
3. 将理论写入 SKILL.md 的"认知科学理论支撑"段
4. 每个设计决策必须能引用具体的认知科学概念

### 铁律3：用工具不手写
**教训**：2026-05-22 直接手写 SKILL.md，没有用 `skill-creator` 的6步流程和模板。

**强制流程**：
1. 简单 skill（<100行）：可直接手写，但必须遵循 YAML frontmatter 规范
2. 复杂 skill（>100行）：必须参考 `li-mindcoach-SKILL.md` 或 `逐字稿清洗-SKILL.md` 的结构
3. 涉及决策/判断的 skill：必须包含"反讨好协议"（防止AI讨好用户）
4. 涉及频繁使用的 skill：必须包含"自我进化模块"

### 铁律4：备份先行
**教训**：多次覆写导致数据丢失。

**强制流程**：
1. 优化已有 skill 前，必须先将旧版复制到 `E:\ai产出文件\牛马\归档\{date}-{skillname}-v{N}/`
2. 新建 skill 前，必须先搜索确认不存在同名 skill

## 三、执行步骤

### Step 1: 需求分析（2分钟）
- **操作**：明确 skill 要解决什么问题、在什么场景触发、用户是谁
- **输出**：一句话需求描述 + 触发场景列表
- **自检**：这个需求是否有现成 skill 能覆盖？

### Step 2: 搜索现有资源（5分钟，铁律1）
- **操作**：
  1. 搜索 `~/.newmax/skills/` 中是否已有同类 skill
  2. 搜索 downloads/ 中的高质量 agent 模板
  3. 搜索百大认知书籍中的相关理论
  4. 搜索各工作区 SOPs/ 中的相关流程
- **输出**：资源清单（已有 skill / 可参考模板 / 理论支撑）
- **自检**：有没有现成轮子？如果有，是升级还是复用？

### Step 3: 理论锚定（3分钟，铁律2）
- **操作**：从百大认知书籍中找到 2-5 个核心概念作为设计依据
- **输出**：理论支撑表（设计要素 → 书籍编号 → 核心机制）
- **自检**：每个设计决策能引用具体书籍的具体概念吗？

### Step 4: 编写 SKILL.md（铁律3）
- **操作**：
  1. YAML frontmatter（`name:` + `description:` 必填，description 含触发词）
  2. Profile 段（author/version/language/scope/tone）
  3. 认知科学理论支撑段
  4. 触发方式（显式 + 隐式 + 手动）
  5. 执行流程（分步编号）
  6. 反模式
  7. 联动（与其他 skill 的关系）
  8. 版本历史
- **特殊模块**（根据 skill 类型添加）：
  - 决策/判断类 skill → 反讨好协议
  - 高频使用 skill → 自我进化模块
  - 跨工作区 skill → 路由注册

### Step 5: 验证与注册
- **操作**：
  1. `mcp__skill-handler__Skill(skill="{name}")` 验证 skill 可被识别
  2. 更新 `skill-catalog-9cat.md` 将新 skill 注册到对应分类
  3. 更新知识中枢 `跨工作区快速参考.md`（如果是全局 skill）
  4. 更新工作区 CLAUDE.md（如果需要路由引用）
- **输出**：注册确认 + 路由测试结果

### Step 6: 记录与进化
- **操作**：
  1. 写入 memory 文件记录创建/优化过程
  2. 如果有自我进化模块，初始化日志文件
  3. 如果涉及教训回流（R09），更新教训→skill 映射表

## 七、决策表：创建 vs 优化 vs 复用

| 场景 | 动作 | 原因 |
|------|------|------|
| 搜索后发现完全匹配的已有 skill | 复用 + 通知用户 | 不重复造轮子 |
| 搜索后发现功能相似但不完全匹配 | 优化已有 skill | 增量迭代优于重建 |
| 搜索后发现有高质量 agent 模板可参考 | 融合创建 | 站在巨人肩膀上 |
| 搜索后确认不存在 | 从零创建（遵循6步流程） | 确认是新需求后才新建 |
| 用户反馈"没触发" | 先诊断：是触发词问题还是路由问题 | 对症下药 |
| skill 有触发词但不准 | 内置模式采集模块，从真实对话中采集迭代（PK-SKILL-006） | 触发词是长出来的，不是设计出来的 |

## 认知科学支撑：Skill 创建的认知原理（百大认知书籍）

| 认知机制 | 来源 | 在本SOP中的应用 |
|---------|------|----------------|
| **搜索优先=对抗确认偏误** | 010-思考快与慢 | 系统1倾向"我想到的第一个方案就是对的"，强制搜索打破这个倾向 |
| **理论锚定=对抗知识错觉** | 043-看不见的猩猩 | "我觉得这样设计对"=知识错觉，必须有外部理论验证 |
| **反讨好协议=对抗社交偏见** | 052-积极心理学 | AI训练优化社交和谐而非智力严谨，需要显式协议对抗 |
| **自我进化=反脆弱设计** | 046-反脆弱 | skill 不是静态的，要从使用反馈中变得更强（凸性响应） |
| **备份先行=否定法** | 046-反脆弱 | Via Negativa——先确保不造成损失（备份），再追求收益（优化） |
