## 四、执行后标准收尾（出水管，必做）

1. 记录本次执行路径（与SOP决策表对比）
2. 如有偏差 → 记录到SOP迭代日志
3. 更新 `memory/{YYYY-MM-DD}.md` 当日记录
4. 如有重要教训 → 更新 `自我进化/做得差的避免/`（附「固化规则」段）
5. 教训文件必须包含：目标文件 + 目标段落 + 可直接粘贴的新规则文本

### 进水管（每个 SOP 的 Step 0）
- **先读 `自我进化/跨系统索引.md`** — 按SOP编号查表，找到关联案例
- 读取 `自我进化/做得差的避免/` + `做得好的鼓励/` 最近文件
- 铁律：禁止"读了不用"
- **维护铁律**：新增自我进化案例时，必须同步更新跨系统索引.md + 对应SOP的「关联教训」表

## 六、自学习框架（R09 反馈闭环）

> 「教训只记不改 = 白记。」

### 6.1 反馈闭环流程

```
执行 SOP → 记录实际路径
  ↓
对比 SOP 决策表 → 找到偏差
  ↓
偏差 = 新知识？→ 写入 SOP「迭代日志」
偏差 = 失败？→ 写入 自我进化/做得差的避免/ + SOP 迭代日志
偏差 = 优化？→ 写入 自我进化/做得好的鼓励/ + SOP 最佳实践
  ↓
3 次同类偏差 → 升级为 SOP 铁律
```
