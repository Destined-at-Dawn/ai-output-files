## 〇、场景定位

| SOP | 场景 | 核心区别 |
|-----|------|---------|
| **SOP-07** | 是什么？为什么？ | 结论先行，1句话定义 → 深层解析 → 行动建议 |
| **SOP-10** | 怎么用？怎么操作？ | 一句话定义 → 手把手实操步骤 → 避坑指南 |

**分流规则**：
- 用户说"是什么/原理/讲讲" → SOP-07
- 用户说"教我/怎么操作/一步步来/只学X方法" → SOP-10
- 灰色地带（如"教我ILA"）→ SOP-10（先一句话定义，再操作教学）

## 一、执行流程

### Step 0: 自我进化预检

```
读 `自我进化/跨系统索引.md` → 查"概念解释/操作教学"相关案例
→ 有案例则读取，提取可复用模式
→ 在输出中体现教训（禁止"读了不用"）
```

**关联案例**：
- 术语平替+操作锚点（Xilinx BRAM解析_成功案例）
- 三级渐进法（Vivado安装引导_成功案例）
- ILA 方法二教学（2026-05-16，本对话）

### Step 1: 一句话定义（不超过20字）

> 用户问"教我ILA" → 先说：**ILA = FPGA 内置的片上逻辑分析仪，用来抓内部信号波形。**

### Step 2: 手把手操作步骤

**格式要求**：
- 每步有编号 + 标题
- 每步有"在哪操作"（菜单路径/按钮位置）
- 关键步骤有截图提示（如"点击这里"）
- 复杂步骤拆成子步骤

**实操教学模板**：
```
Step N: {操作名称}
  - 位置：{菜单路径}
  - 操作：{具体动作}
  - 关键：{注意事项}
  - 验证：{怎么确认做对了}
```

### Step 3: 避坑指南

**格式**：`现象 → 原因 → 解决`

**来源**：
- 优先从 `自我进化/做得差的避免/` 提取
- 其次从 SOP-06 避坑指南提取
- 最后从 claude-skill-verilog 的 Verilog pitfalls 提取

### Step 4: 用户反馈收集

- 问用户是否需要更深入讲解某个步骤
- 问用户是否需要配图/截图
- 记录用户的学习进度到当日记忆文件

## 二、教学铁律

| 规则 | 说明 |
|------|------|
| **一句话定义先行** | 操作教学前必须先给一句话定义，不能上来就甩步骤 |
| **步骤可验证** | 每个关键步骤必须有验证方法（"怎么确认做对了"） |
| **避坑必读** | 操作教学必须包含至少1个避坑指南 |
| **工具特定规则引用** | 教 Vivado/Vivado 操作时必须引用 claude-skill-verilog 的特定规则 |
| **记录教学进度** | 用户学习情况记录到 `memory/{YYYY-MM-DD}.md` |

## 三、与 SOP-07 的分工

SOP-07 是"概念解释"，只讲"是什么/为什么"。
SOP-10 是"操作教学"，讲"怎么用/怎么做"。

**混合场景处理**（如"教我ILA"）：
1. 先用 SOP-07 的"一句话定义"格式给定义（≤20字）
2. 立即转入 SOP-10 的手把手操作流程
3. 不需要走完整的 SOP-07 流程

## 五、认知科学支撑（百大认知书籍）

> **为什么需要这一节？** "教我怎么操作"是最常见的学习需求，但90%的操作教学都是低效的——因为忽视了学习的神经科学机制。理解"学习如何发生"的物理原理，才能设计出"大脑真正能学会"的操作教学。

### 5.1 操作学习的四大支柱

| 支柱 | 来源 | 在操作教学中的应用 |
|------|------|-------------------|
| **注意力** | 057-我们如何学习 §概念1 | 学习的第一道门是注意力。Step 1一句话定义的功能=强制调动注意力资源。没有注意力的"学习"不会产生突触变化——物理意义上等于没有学习 |
| **主动参与** | 057-我们如何学习 §概念2 | 被动看教程=几乎没有学习效果。Step 2手把手操作必须是"用户跟着做"而非"用户看着AI做"。每个步骤必须有用户的物理动作（点击/输入/连接） |
| **错误反馈** | 057-我们如何学习 §概念3 | 无错误的学习=无学习。Step 3避坑指南不是"预防错误"而是"利用错误"——每个坑都是高质量的预测误差→突触重塑。AI应鼓励用户"故意犯小错"来加速学习 |
| **记忆巩固** | 057-我们如何学习 §概念4 | 短期操作记忆需要通过睡眠重播固化为长期程序记忆。Step 4学习进度记录=为睡眠重播提供"素材"——当日学了什么→当夜大脑重播什么 |

### 5.2 刻意练习框架在操作教学中的应用

| 刻意练习要素 | 来源 | 在SOP-10中的应用 |
|------------|------|-----------------|
| **明确目标** | 017-刻意练习 §概念2 | 每次操作教学必须有可验证的终点——不是"学会了ILA"而是"能独立用ILA抓到一个指定信号的波形" |
| **舒适区边缘** | 017-刻意练习 §概念3 | 教学难度应略高于用户当前能力。太简单→天真练习→不增长能力。AI应根据用户反馈动态调整步骤详细程度 |
| **即时反馈** | 017-刻意练习 §概念4 | 每个关键步骤必须有"验证方法"（§一 Step 2的模板要求）。无反馈的操作练习=在黑暗中射箭→不知道偏了还是正了 |
| **心理表征构建** | 017-刻意练习 §概念1 | 操作教学的终极目标不是"用户会操作"而是"用户建立了Vivado操作的心理表征"→能迁移到新场景。每次教学结束追问："如果换个FPGA型号，你会怎么调整？" |

### 5.3 学习科学优化策略

| 策略 | 来源 | 具体做法 |
|------|------|---------|
| **检索式练习** | 009-认知天性 §概念1 | 教完操作后，不要让用户"再看一遍教程"，而是关掉教程让用户凭记忆重新做一遍。做不出来=暴露了真正不懂的环节→精确打击 |
| **交错练习** | 028-我们如何学习 §概念4 | 不要连续教同类操作（如连续教3种协议），应混搭不同类型（UART→SPI→I2C交错）。虽然感觉更难，但辨别能力和迁移能力大幅提升 |
| **必要难度** | 009-认知天性 §概念2 | 教学中故意留白——不把所有细节都讲透，让用户自己摸索关键步骤。"感觉学得很吃力"=学习正在发生的信号 |
| **分布式刻录** | 028-我们如何学习 §概念1 | 30小时的操作训练分散在多个不连续区块（中间至少隔1-2天），利用遗忘过滤噪音、睡眠固化记忆。禁止集中式"一天学完" |
| **环境解耦** | 028-我们如何学习 §概念4 | 操作技能不应绑定在特定环境（特定电脑/特定Vivado版本）。每次教学后追问："换一台电脑你能做到吗？"→促进技能与环境解耦 |

### 5.4 操作教学中的常见认知陷阱

| 陷阱 | 认知原因 | 正确做法 | 来源 |
|------|---------|---------|------|
| "看教程都会，自己做就忘" | 流畅度错觉+被动学习 | 强制"关教程独立操作"→暴露真实掌握程度 | 009-认知天性 §概念3 |
| "步骤太长记不住" | 工作记忆超载 | 每个关键步骤≤3个子操作，超过则拆分 | 025-认知负荷理论 §概念1 |
| "跟着做没问题，换个场景就不会" | 技能绑定在特定上下文 | 教完后给一个"变体场景"让用户独立完成 | 028-我们如何学习 §概念4 |
| "一次学太多，第二天全忘" | 没有睡眠巩固窗口 | 每次教学≤90分钟，中间必须有休息或隔天 | 057-我们如何学习 §概念4 |

| 认知机制 | 来源 | 在概念学习SOP中的应用 |

| **概念层次** | 025-认知负荷理论 §concept4 | 学习="概念→操作→应用"三层——先理解"是什么"(概念)→再学会"怎么做"(操作)→最后"在项目中用"(应用)→三层递进→每层都是下层的基础 |
