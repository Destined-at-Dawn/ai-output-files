# 教训：落盘铁律 + memory 覆写陷阱

## 两条教训

### 1. 高价值内容必须落盘
"说了但没写"等于没说。对话回复是临时的，会被上下文窗口吞掉，文件才是持久的。

**自检清单**（任一命中必须落盘）：
- 数据盘点 / 框架矩阵 / 流程改进 / 选题规划
- 新规则 / 新SOP / 新模式
- 工程决策 / 架构变更

### 2. memory 写入必须"读-追加-写"
`write_project_file` 是**覆写**不是追加。直接写入新内容会丢失全部历史。

**正确流程**：Read 当前内容 → 拼接新旧 → Write → Read 验证文件大小

## 跨工作区影响
所有工作区的 memory 管理都受此影响。已固化为共享规则 R04（memory-append-ironlaw），但实操中仍频繁被违反。
