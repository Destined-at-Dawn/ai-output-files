# 教训：隐私脱敏必须覆盖全部输出通道

## 核心教训
排查报告中引用原始隐私值 = 在公开仓库里又泄露一遍。

**所有输出通道一视同仁必须脱敏**：
- 代码文件 ✅
- README / 文档 ✅
- 排查报告 / 诊断建议 ✅
- 对话回复（涉及公开内容时）✅

## 延伸：历史清理
- `git rm --cached` 只移除当前版本，旧 commit 中仍可被任何人翻到
- 隐私清理必须用 `git filter-repo` 覆盖全部历史

## 开源清单（缺一不可）
README + License + .gitignore + Release + 封面图（可发现性）

## 适用范围
任何涉及 GitHub 公开发布的项目（创作/竞赛/个人）。
