# 教训：proj-ID 目录 vs 语义目录的路径混乱

## 发生了什么
MCP `write_project_file` 的实际写入路径不等于 AI 预期的路径。proj-1778067689138-1wcqv9 目录下的文件实际被写入了 `projects/跨校协作/outputs/`，导致同一个项目的产出分散在两个目录中。

## 根因
- AI 启动时可能绑定到 proj-ID 目录（系统生成的随机ID）
- 但 MCP 工具的实际写入路径由项目配置决定，可能指向语义目录（如 `projects/跨校协作/`）
- 没有机制在会话开始时验证"我当前的写入路径到底是哪里"

## 修复方案
1. **以语义目录为权威**：proj-ID 目录只是系统占位，语义目录（如 `跨校协作/`、`竞赛全流程/`）才是真实工作目录
2. **会话启动自检**：第一次 write 操作前，先用 list_project_files 确认实际路径
3. **写入后 Read 验证**：不能假设路径正确，写入后必须读取验证

## 适用场景
- 任何使用 MCP project-tools 的工作区
- 任何存在 proj-ID 和语义目录并存的项目
- 新项目初始化时，优先创建语义目录

## 跨工作区影响
此教训影响所有5个工作区，因为 MCP 工具在每个工作区中行为一致。
