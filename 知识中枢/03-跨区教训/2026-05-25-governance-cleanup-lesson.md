# 治理文件清理教训

> 日期：2026-05-25
> 来源：mutual 工作区治理文件清理

## 问题

上一轮28次迭代生成了大量治理文件（178个），包括：
- FINAL-AUDIT-REPORT.md
- COMPLETE-FILE-LISTING.md
- R16-summary.md
- capability-matrix.md
- health-check.md
- 等等...

这些文件本身就是在制造混乱——**治理规则比业务规则还多**。

## 教训

1. **治理文件不是越多越好**：过多的治理文件会增加认知负荷
2. **文件数量不等于质量**：28次迭代的文件数量增长不代表真正的改进
3. **归档前必须备份**：删除任何文件前必须先归档
4. **每轮迭代要有不同的方法论**：复制粘贴不是迭代

## 正确做法

1. **精简治理**：只保留必要的治理文件
2. **统一入口**：用index.md作为唯一入口
3. **分层治理**：核心规则放 .claude/rules/（Claude Code 自动加载），SOP放SOPs/，记忆放memory/
4. **定期清理**：定期清理过期的治理文件
