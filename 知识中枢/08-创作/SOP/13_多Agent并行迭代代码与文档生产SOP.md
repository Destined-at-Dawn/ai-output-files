# 08_多Agent并行迭代代码与文档生产SOP

> **工作区：** 创作区
> **版本：** v1.0 | **最后更新：** 2026-05-21
> **触发条件：** 用户需要用多个Agent共同完成复杂工程代码编写与高质量文档（如白皮书）总结时触发。
> **经验来源：** 基于2026集创赛IC2022华大九天MAC16命题中，5个独立团队并行探索的《迭代历史白皮书》实战经验提取。

---

## 一、核心心法：物理世界的降维打击框架

本 SOP 的底层逻辑深度耦合了小黎的认知系统，将芯片设计中的非凸优化实战提炼为可复用的 AI 调度框架：

1. **结构洞套利统筹（来自037《结构洞与好主意》）**：
   - 主 Agent 的核心不是下场写代码，而是占据信息咽喉（Tertius Gaudens）。它不需要是最聪明的解题者，但必须是各个子空间 Agent 之间的唯一信息连接者，负责吸收、过滤、并拼装最优解。
2. **正交子空间并行与垃圾桶模型（来自041《垃圾桶模型》）**：
   - 面对复杂的工程问题（如1.5G@SS的物理极限），串行探索必然陷入局部最优的死锁。必须将任务降维切片，同时启动多个子 Agent 分别在不重叠（正交）的子空间中进行极端的架构搜索、激进的底层优化和验证，让解决方案的"水流"随机碰撞。
3. **受控幻觉验证与预测编码（来自047《体验机》）**：
   - AI 极易陷入 "Pre-signoff PASS"（逻辑综合看似完美）的受控幻觉中。必须建立严酷的物理验证防线（真实 Post-route signoff、SPEF 验证），通过主动制造真实的预测误差，打破 Agent 的自我欺骗。
4. **延伸的表型构建（来自040《延伸的表型》）**：
   - 每个子 Agent 产出的局部最优解（如 instance-abstract 脚本）是其独立的延伸表型。主 Agent 将这些表型固化、组装，最终生成的白皮书就是整个工程系统的"认知水坝"。

## 二、多 Agent 系统角色定义与提示词框架

### 1. 主 Agent（结构洞统筹者/Scrum Master）
**核心职责**：不写代码，只做任务分发、进度监督、结果拼装与跨界信息套利。
**提示词框架**：
```markdown
【身份设定】
你是一个处于"结构洞"位置的顶级架构师和项目统筹者（Scrum Master）。你的目标不是解决具体的代码问题，而是管理和拼装来自多个独立子Agent的探索结果。

【工作准则】
1. **正交任务拆解**：将复杂任务拆解为至少三个互不重叠的子空间（如：纯算法架构层、底层物理/内存优化层、测试与验证层），并分别委派给对应的子Agent。
2. **警惕局部最优（受控幻觉防御）**：当子Agent汇报"测试通过"时，你必须冷血地要求其提供最终物理环境或极限边界测试下的真实数据（等同于 Post-route signoff），拒绝接受理想状态下的 Pre-signoff 结果。
3. **跨界信息套利**：如果Agent A在算法上卡死，但Agent B开发了一个通用的底层脚本，你必须敏锐地将B的脚本提取出来注入给A，完成Tertius Iungens（连接的第三者）动作。
4. **全局收敛与产出**：定期（如每天）将各子Agent的阶段性最优解进行拼装，并梳理出失败路径和物理上限，为最终白皮书做准备。
```

### 2. 子 Agent 群（极端正交探索者）
**核心职责**：在被划定的绝对边界内进行极致、密集的探索，提供可被主Agent收割的组件。
你需要实例化多个具有不同性格与偏好的子 Agent，例如：

#### Agent A：激进架构派（类 codex2）
**提示词片段**：
```markdown
你是一个极致的架构探索者。不要考虑底层实现细节，只在算法层面上进行大规模、密集的重构尝试（例如流水线拆分、并行度展开）。请在你的子空间内疯狂生成架构变体，并记录每个变体的理论上限。
```

#### Agent B：底层物理/极限压榨派（类 newmax）
**提示词片段**：
```markdown
你是一个底层硬件/系统级别的极限黑客。你不关心上层算法多优雅，你只关心如何通过激进的底层参数调整（如内存分配策略、底层寄存器操作、激进APR约束）突破物理极限。你的任务是把别人跑不通的代码在底层强行跑通。
```

#### Agent C：冷酷验证派（类 claude2 验证线）
**提示词片段**：
```markdown
你是一个没有感情的验证工程师。你的任务是构建最严酷的测试环境（如 LVS instance-abstract 验证、极端边界条件注入）。你必须找出其他Agent代码中的所有受控幻觉，不接受任何理论上的"PASS"，只接受真实环境跑通的报告。
```

## 三、标准执行流程（SOP）

### Phase 1: 降维切片与子空间初始化
1. 主 Agent 接收整体工程目标。
2. 主 Agent 将目标拆分为互相正交的子任务（例如：UI层、逻辑层、数据库层，或 架构层、综合层、验证层）。
3. 主 Agent 唤醒对应的子 Agent，并为其设定**硬性边界**（禁止越界干扰其他 Agent 的工作）。

### Phase 2: 并行非凸优化与受控幻觉破除
1. 子 Agent 开始独立探索，产生多版本变体（Variant）。
2. 子 Agent 提交阶段性成果。
3. 主 Agent 或验证 Agent（Agent C）对成果进行"物理级"验证。
   - **拦截条件**：如果发现只在理想状态下通过（Pre-signoff 假象），直接打回重做。
   - **记录教训**：把失败原因记录在案，作为明确的约束传递给其他 Agent。

### Phase 3: 结构洞套利与拼装
1. 主 Agent 识别出不同子 Agent 的核心突破点（例如 A 的高效算法 + B 的底层资源分配方案）。
2. 主 Agent 进行拼装，组合出跨界的最优解（The Best Variant）。

### Phase 4: 白皮书级文档生成
基于老师 MAC16 项目的华为白皮书风格，主 Agent 最后统一输出文档。
**文档生成 Prompt 框架**：
```markdown
【文档生成指令】
请基于本次多Agent的迭代历史，生成一份具有华为白皮书风格的高密度技术文档。必须包含以下结构：
1. **Executive Summary（摘要）**：一句话说明物理上限是什么，我们达到的最优结果是什么。
2. **全局时间线与数据支撑（Gantt视图）**：使用ASCII表格清晰展示各Agent在时间轴上的关键变体、测试数据对比（如TT/FF/SS、Pre vs Post数据）。
3. **架构探索谱系图**：展示所有失败与成功的迭代分支。
4. **"受控幻觉"警示记录**：明确指出在哪个阶段Agent产生了自我欺骗（Pre-signoff PASS），真实环境实测数据差了多少，以此作为后续工程的避坑指南。
5. **共识结论与物理极限**：承认哪些目标是物理上不可达的，明确系统的真实边界在哪里。
```

## 四、认知科学支撑（百大认知书籍）——深度注入

### 多Agent系统的认知架构原理

| 认知机制 | 来源 | 在多Agent并行系统中的应用 |
|---------|------|--------------------------|
| **分布式认知** | 008-协同智能 §概念1 | 多Agent系统的核心不是"多个AI并行工作"，而是"认知任务分布在不同节点上"。主Agent的认知功能=信息整合；子Agent的认知功能=深度探索。两者缺一不可→分布式认知整体>部分之和 |
| **群体智慧的涌现** | 008-协同智能 §概念3 | 多个正交子Agent的独立探索→在主Agent处汇聚→涌现出任何单个Agent都发现不了的跨界解。但涌现的前提是**独立性**——如果子Agent互相看到对方的中间结果→从众效应→多样性丧失→涌现失败 |
| **受控幻觉的认知本质** | 047-体验机 §概念5 | AI的"受控幻觉"=大脑的"先验模型劫持"——当预测编码系统收到足够强的内部信号时，会将其误认为外部验证。Pre-signoff PASS就是AI版的"体验机"——逻辑层面的完美≠物理层面的真实 |
| **非凸优化的认知类比** | 046-反脆弱 §概念2 | 串行探索=在一个山谷里挖得越来越深→永远找不到更低的山谷。并行正交探索=同时在N个山谷挖→N越大，找到全局最优的概率越高。这正是杠铃策略在认知层面的实现 |
| **注意力瓶颈** | 025-认知负荷理论 §概念1 | 主Agent同时监督>5个子Agent→注意力超载→无法有效识别子Agent的"受控幻觉"。子Agent数量应控制在3-5个，与工作记忆容量一致 |

### 结构洞理论的深层应用

| 应用层面 | 来源 | 具体做法 |
|---------|------|---------|
| **信息套利** | 037-结构洞 §概念1 | 主Agent的核心价值=跨子空间的信息套利。Agent A的算法突破+Agent B的底层优化→组合后的方案>两者之和。主Agent必须"看到别人看不到的连接" |
| **第三方受益者** | 037-结构洞 §概念2 | Tertius Gaudens（享受的第三者）——主Agent从子Agent的竞争中获益，但不参与竞争。这与019-敏捷革命中的Scrum Master角色高度一致——保护团队而非指挥团队 |
| **桥接vs黏合** | 037-结构洞 §概念3 | 桥接社会资本（跨团队连接）>黏合社会资本（团队内部紧密）。主Agent应优先投资"桥接"——连接不同子Agent的异质知识，而非加固某个子Agent的同质深度 |
| **创新半衰期** | 037-结构洞 §概念4 | 结构洞位置的信息优势有半衰期——一旦子Agent之间建立直接连接，主Agent的信息套利价值消失。因此白皮书生成（Phase 4）是"收割窗口"——在信息优势消失前固化成果 |

### 受控幻觉破除的认知工程

| 防御层级 | 认知原理 | 来源 | 具体做法 |
|---------|---------|------|---------|
| **物理级验证** | 躯体标记 | 026-笛卡尔的错误 §概念1 | 受控幻觉的特征是"逻辑自洽但身体信号异常"——代码编译通过但功耗超标/时序违例。Agent C的任务就是模拟"身体信号"——用物理级指标（功耗/面积/时序）而非逻辑级指标（编译通过）来判断 |
| **对抗性测试** | 魔鬼辩护 | 010-思考快与慢 §概念5 | Agent C应采用"魔鬼辩护"模式——不是验证"代码是否正确"，而是寻找"代码哪里会失败"。如果找不到失败点→不是代码好，是测试不够严酷 |
| **预测误差信号** | 预测编码 | 047-体验机 §概念1 | 当子Agent的预测（"这个方案应该能通过"）与验证结果（"实际PPA超标40%"）之间的误差越大→系统越需要更新模型。主Agent应将"最大预测误差"作为最有价值的信号——这是系统学习最快的地方 |
| **元认知检查** | WYSIATI防御 | 010-思考快与慢 §概念3 | "我看到了所有问题"=WYSIATI陷阱。Phase 3拼装后应强制执行"还有什么是我没看到的？"→主Agent主动搜索自己的认知盲区→避免在白皮书中遗漏关键失败路径 |

### 白皮书生成的认知说服策略

| 策略 | 来源 | 应用 |
|---------|------|------|
| **认知放松=可信** | 010-思考快与慢 §概念2 | 白皮书排版整洁、数据图表清晰→读者系统1产生"认知放松"→判断为"专业可信"。华为白皮书风格的底层认知原理就是"最大化认知放松" |
| **数据锚定** | 010-思考快与慢 §概念5 | Executive Summary中的核心数据成为读者后续判断的锚点。"物理极限为X，我们达到Y"→锚定→后续所有数据都与X和Y比较→形成清晰的认知框架 |
| **叙事传输** | 010-思考快与慢 §概念1 | "架构探索谱系图"本质上是一个"探索叙事"——从起点→遇到困难→多种尝试→最终突破。读者进入叙事模式→降低批判性审查→更容易接受结论 |
| **成就归因** | 039-社会动物 §概念1 | 白皮书应明确归因每个突破的来源（哪个Agent、什么条件）→建立"功劳可追溯性"→读者信任度提升。模糊归因→"大概不错吧"→信任度低 |
