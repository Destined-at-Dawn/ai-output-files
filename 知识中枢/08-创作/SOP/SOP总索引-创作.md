## 二、消息路由规则

> 每次收到消息时，按以下顺序匹配：

### 2.1 输入类型优先（强制）

```
IF 消息包含链接/图片/文件/引号内容 → 输入分发SOP（路由到对应主题SOP）
ELSE → 进入主题匹配
```

### 2.2 主题匹配（10类）

```
IF 说"写/创作/灵感/公众号/小红书" → SOP-01 内容创作
IF 说"链接/文章/采集/文风" → SOP-02 文章采集
IF 说"图/封面/配图/信息图" → SOP-03 视觉素材
IF 说"PPT/答辩/汇报/演讲" → SOP-04 PPT制作
IF 说"视频/运镜/分镜/Seedance" → SOP-05 视频内容
IF 说"skill/机制/映射" → SOP-06 系统管理
IF 说"保存/整理/归档" → SOP-07 文件管理
IF 说"学习/认知/拆解" → SOP-08 知识学习
IF 说"论文/文献/arXiv" → SOP-09 科研论文
IF 说"自我介绍/规划/竞赛/面试" → SOP-10 个人发展
IF 检测到个人信息变动信号 → SOP-11 个人信息同步
IF 说"搜论文/找文献/知网/文献检索/数据库/AI检索" → SOP-12 高校文献检索
ELSE → 回退到 CLAUDE.md §十二 skill映射表
```

### 2.3 SOP-00 拦截层

> **SOP-00包裹所有SOP**。在匹配到任何SOP后，先过SOP-00清晰度检查。

- 清晰度≥6分 → 直接执行匹配的SOP
- 清晰度<6分 → 生成3个改进提示词变体，让用户选择
- **绕过条件**：含链接/图片/文件/引号文本/用户说"快"/"急"/"马上"/消息>50字且有明确动词

## 三、执行后标准收尾

每次SOP执行完毕后：

1. 记录本次执行路径（与SOP决策表对比）
2. 如有偏差，记录到SOP的迭代日志
3. 同一偏差出现3次 → 触发SOP更新
4. 更新 `memory/{YYYY-MM-DD}.md` 当日记录
