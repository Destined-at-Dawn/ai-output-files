# 2026-05-20 触摸屏工程综合症状 — I2C 多个致命Bug复盘

> **严重度**：🔴 致命（多个Bug叠加）
> **影响**：SCL/SDA 偶尔有波形瞬间变全高 → 调试 2 天无法定位
> **教训来源**：触摸屏工程 `D:\AMD\touchscreen\`
> **关联 Bug 编号**：#8（SDA push-pull）、#10（全局ACK检测假阳性）

---

## 发生了什么

上板后观察到：按复位键偶尔能看到 SCL/SDA 有短暂波形，然后瞬间变全高。

## 根因分析（两个Bug叠加）

### Bug #8：SDA 三态门被综合成 push-pull

**代码**：
```verilog
// 错误写法
assign i2c_sda = sda_oe ? sda_out : 1'bz;
```

**问题**：Vivado 综合时，behavioral tri-state 可能被推到 IOB。如果 IOB 不支持三态（或综合工具判断为 push-pull 更优），FPGA 在 ACK 阶段（`sda_oe=0`）仍然驱动 SDA=高电平，与 slave 的 ACK（拉低）总线冲突。

**正确写法**：
```verilog
// open-drain 写法
assign i2c_sda = (sda_oe & ~sda_out) ? 1'b0 : 1'bz;
```

**真值表**：
| sda_oe | sda_out | 行为 | I2C语义 |
|--------|---------|------|---------|
| 0 | x | 高阻（释放） | ACK阶段：slave可以拉低 |
| 1 | 0 | 驱动低 | 发送 0 bit |
| 1 | 1 | 高阻（释放） | 发送 1 bit：pull-up拉高 |

### Bug #10：全局 ACK 错误检测制造假阳性

**代码**：
```verilog
// 错误写法：每个周期都检查
if (i2c_done && !i2c_ack && state != S_CHECK && state != S_PROCESS
             && state != S_RD_STA_6 && state != S_RD_XY_9)
    state <= S_ERROR;
```

**问题**：`done` 脉冲到达时，`ack_err` 可能是**上一条命令**（CMD_START）的残留值。CMD_START 不产生 ACK，ack_err 不会被 slave 更新 → 假阳性 → 状态机被推到 S_ERROR → 等 200μs → S_IDLE → 轮询 → 再发命令 → 又被拦截 → 循环。

**正确做法**：删掉全局检测，在每个关键 I2C 步骤**就地检查**：
```verilog
S_RD_STA_1: begin  // WRITE device address
    if (i2c_done) begin
        if (!i2c_ack) begin
            state <= S_ERROR;    // 就地检查，CMD_START已清除ack_err
        end else begin
            state <= S_RD_STA_2; // 正常继续
        end
    end
end
```

## 为什么两个Bug叠加后更难发现

1. Bug #8 让 ACK 阶段 SDA 总线冲突 → slave 不 ACK → ack_err=1
2. Bug #10 的全局检测捕获了这个 ack_err → 状态机跳到 S_ERROR
3. S_ERROR 回 S_IDLE 重试 → 又碰到 Bug #8 → 循环
4. 偶尔有波形 = 命令成功发了几个字节（Bug #8 在某些情况下综合正确）
5. 瞬间变全高 = 全局检测触发 S_ERROR（Bug #10）

```
### 7. I2C SDA 必须用 Open-Drain 铁律
- 禁止 behavioral tri-state 描述 I2C SDA
- 必须用 open-drain 写法：assign sda = (sda_oe & ~sda_out) ? 1'b0 : 1'bz;

### 8. 禁止全局错误检测铁律
- 禁止在状态机外用"每个周期都检查"的方式检测错误
- 必须在每个具体 I2C 命令完成的 case 分支内就地检查 ack_err
```

## 触发条件

- 使用 I2C/SPI 等需要三态总线的协议
- 状态机中有"全局错误检测"模式
- Vivado 综合（其他综合工具可能行为不同）

## 防范措施

1. I2C SDA 从第一天就用 open-drain 写法，不要用 behavioral tri-state
2. 错误检测放在具体命令完成的 case 分支内，不要"全局每周期检查"
3. 上板后用 ILA 看 SDA 在 ACK 阶段的电平——如果 master 仍然驱动高，就是 push-pull 问题
4. XDC 中 SDA 引脚确保有外部上拉电阻（4.7kΩ）
