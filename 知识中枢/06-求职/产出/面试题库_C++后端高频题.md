# 🎯 C++后端面试高频题库

> **使用方法**：先看题目自己答，再看参考答案。重点理解「为什么」，而不是背答案。

## 一、C++基础（必问，决定第一印象）

### 1.1 智能指针（几乎必问）
**Q：shared_ptr、unique_ptr、weak_ptr的区别和使用场景？**

**参考答案**：
- **shared_ptr**：共享所有权，引用计数。适用于多个指针指向同一对象的场景（如：观察者模式）
- **unique_ptr**：独占所有权，不可复制，只能移动。适用于明确单一所有权的场景（如：工厂方法返回值）
- **weak_ptr**：不增加引用计数的「弱引用」。用于解决**循环引用**问题（如：双向链表的前驱/后继指针）

**追问**：shared_ptr的线程安全性？
- 引用计数的增减是原子操作（线程安全）
- 但**指向的对象本身不是线程安全的**，需要额外加锁

### 1.2 虚函数与多态（必问）
**Q：虚函数的实现原理？**

**参考答案**：
- 每个有虚函数的类都有一个**虚函数表（vtable）**，存储虚函数的地址
- 对象内存的**前8字节**（64位系统）存储指向vtable的指针（vptr）
- 调用虚函数时，通过vptr找到vtable，再通过偏移量找到函数地址
- **动态绑定**：运行时确定调用哪个函数（通过vtable）
- **静态绑定**：编译时确定（普通函数）

**追问**：虚析构函数的作用？
- 基类指针指向派生类对象时，如果析构函数不是虚函数，只会调用基类析构函数，导致**派生类资源泄漏**
- 所以基类的析构函数**必须**声明为virtual

### 1.3 内存管理（高频）
**Q：new/delete和malloc/free的区别？**

**参考答案**：
| 特性 | new/delete | malloc/free |
|------|-----------|-------------|
| 语言 | C++运算符 | C库函数 |
| 构造/析构 | **会**调用构造/析构函数 | **不会** |
| 返回类型 | 具体类型指针 | void* |
| 内存大小 | 自动计算 | 需手动指定 |
| 失败处理 | 抛出std::bad_alloc | 返回NULL |

**追问**：内存泄漏怎么检测？
- **Valgrind**：`valgrind --leak-check=full ./program`
- **AddressSanitizer**：编译时加 `-fsanitize=address`
- **智能指针**：优先使用shared_ptr/unique_ptr，避免手动管理

### 1.4 C++11新特性（高频）
**Q：移动语义和右值引用的作用？**

**参考答案**：
- **右值引用（T&&）**：绑定到临时对象（右值）
- **移动语义**：将资源「转移」而非「复制」，避免深拷贝的开销
- **std::move()**：将左值转换为右值引用，触发移动构造/赋值

**例子**：
```cpp
std::vector<int> v1 = {1, 2, 3};
std::vector<int> v2 = std::move(v1); // v1的资源转移到v2，v1变为空
```

**追问**：完美转发（Perfect Forwarding）？
- `std::forward<T>(arg)` 保持参数的左值/右值属性
- 用于模板函数中，将参数「原样」转发给另一个函数

## 二、网络编程（后端核心）

### 2.1 TCP三次握手/四次挥手（必问）
**Q：为什么是三次握手，不是两次？**

**参考答案**：
- **两次握手的问题**：客户端的「历史重复连接」会被服务器误认为是新连接
- **第三次握手的作用**：客户端确认收到了服务器的SYN+ACK，防止「已失效的连接请求」被服务器接受

**四次挥手**：
- 客户端发送FIN → 服务器ACK → 服务器发送FIN → 客户端ACK
- **为什么四次**：TCP是全双工，每个方向需要单独关闭
- **TIME_WAIT状态**：客户端等待2MSL（最大报文段生存时间），确保服务器收到最后的ACK

### 2.2 Epoll（后端必问）
**Q：select、poll、epoll的区别？**

**参考答案**：
| 特性 | select | poll | epoll |
|------|--------|------|-------|
| 最大连接数 | 1024 | 无限制 | 无限制 |
| 数据结构 | fd_set（位图） | pollfd数组 | **红黑树+就绪链表** |
| 拷贝开销 | 每次调用都拷贝fd集合 | 同select | **只在epoll_ctl时拷贝** |
| 触发模式 | 水平触发（LT） | 水平触发 | **LT/ET都支持** |
| 时间复杂度 | O(n) | O(n) | **O(1)** |

**追问**：ET（边缘触发）和LT（水平触发）的区别？
- **LT**：只要fd就绪，每次epoll_wait都会返回 → 编程简单，但可能重复通知
- **ET**：只在fd状态变化时通知一次 → 编程复杂（必须循环读到EAGAIN），但性能更高

### 2.3 Reactor模式（加分项）
**Q：Reactor模式的原理？**

**参考答案**：
- **核心思想**：一个线程（Reactor）负责监听所有fd的IO事件，分发给对应的Handler处理
- **组成**：
  - Reactor：事件循环，调用epoll_wait
  - Acceptor：处理新连接
  - Handler：处理读写事件
- **变种**：
  - 单Reactor单线程：Redis（6.0之前）
  - 单Reactor多线程：Nginx
  - 主从Reactor多线程：Netty、Memcached

## 三、数据库（必问）

### 3.1 MySQL索引（高频）
**Q：为什么用B+树，不用B树/红黑树/哈希？**

**参考答案**：
- **B+树 vs B树**：B+树的叶子节点用链表连接，**范围查询更快**
- **B+树 vs 红黑树**：B+树的**树高更低**（3层可存储2000万条数据），IO次数更少
- **B+树 vs 哈希**：哈希不支持**范围查询**和**排序**

**追问**：聚簇索引和非聚簇索引的区别？
- **聚簇索引**：叶子节点存储**完整的数据行**（InnoDB的主键索引）
- **非聚簇索引**：叶子节点存储**主键值**（需要回表查询）

### 3.2 事务隔离级别（必问）
**Q：四种隔离级别分别解决什么问题？**

**参考答案**：
| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|---------|------|-----------|------|
| READ UNCOMMITTED | ❌ | ❌ | ❌ |
| READ COMMITTED | ✅ | ❌ | ❌ |
| REPEATABLE READ（默认） | ✅ | ✅ | ❌ |
| SERIALIZABLE | ✅ | ✅ | ✅ |

**追问**：InnoDB如何解决幻读？
- **MVCC（多版本并发控制）**：读取历史版本快照
- **间隙锁（Gap Lock）**：锁住记录之间的间隙，防止插入

### 3.3 Redis（高频）
**Q：Redis的数据结构和使用场景？**

**参考答案**：
| 数据结构 | 使用场景 |
|---------|---------|
| String | 缓存、计数器、分布式锁 |
| Hash | 存储对象（用户信息） |
| List | 消息队列、最新消息 |
| Set | 去重、共同好友 |
| ZSet | 排行榜、带权重的队列 |

**追问**：缓存穿透、缓存击穿、缓存雪崩的区别？
- **缓存穿透**：查询**不存在的数据**，缓存和数据库都查不到 → 布隆过滤器
- **缓存击穿**：**热点key过期**，大量请求直接打到数据库 → 互斥锁/永不过期
- **缓存雪崩**：**大量key同时过期**，数据库压力骤增 → 随机过期时间

## 四、操作系统（加分项）

### 4.1 进程与线程（必问）
**Q：进程和线程的区别？**

**参考答案**：
- **进程**：资源分配的基本单位（独立地址空间、文件描述符）
- **线程**：CPU调度的基本单位（共享进程的地址空间）
- **区别**：
  - 进程切换开销大（需要切换页表、刷新TLB）
  - 线程切换开销小（只需要切换栈和寄存器）
  - 进程间通信需要IPC（管道、消息队列、共享内存）
  - 线程间通信可以直接读写共享变量（需要同步机制）

**追问**：线程同步的方式？
- **互斥锁（Mutex）**：同一时间只有一个线程访问临界区
- **条件变量（Condition Variable）**：等待某个条件成立
- **信号量（Semaphore）**：控制并发数量
- **原子操作（Atomic）**：无锁编程

### 4.2 虚拟内存（加分项）
**Q：虚拟内存的作用？**

**参考答案**：
- **地址隔离**：每个进程有独立的虚拟地址空间，防止互相干扰
- **内存扩展**：通过页面置换，可以使用比物理内存更大的空间
- **共享内存**：多个进程可以映射同一物理页（如：共享库）

**追问**：页面置换算法？
- **LRU（最近最少使用）**：淘汰最久未使用的页面
- **LFU（最不经常使用）**：淘汰使用次数最少的页面
- **FIFO（先进先出）**：淘汰最先进入的页面（性能差）

## 五、设计模式（加分项）

### 5.1 单例模式（高频）
**Q：如何实现线程安全的单例模式？**

**参考答案**：
```cpp
// 方法1：Meyers' Singleton（C++11推荐）
class Singleton {
public:
    static Singleton& getInstance() {
        static Singleton instance; // C++11保证线程安全
        return instance;
    }
private:
    Singleton() = default;
    ~Singleton() = default;
    Singleton(const Singleton&) = delete;
    Singleton& operator=(const Singleton&) = delete;
};

// 方法2：双重检查锁（C++11之前）
class Singleton {
public:
    static Singleton* getInstance() {
        if (instance == nullptr) {
            std::lock_guard<std::mutex> lock(mtx);
            if (instance == nullptr) {
                instance = new Singleton();
            }
        }
        return instance;
    }
private:
    static Singleton* instance;
    static std::mutex mtx;
};
```

### 5.2 工厂模式（高频）
**Q：简单工厂、工厂方法、抽象工厂的区别？**

**参考答案**：
- **简单工厂**：一个工厂类，根据参数创建不同产品 → 违反开闭原则
- **工厂方法**：每个产品对应一个工厂 → 符合开闭原则
- **抽象工厂**：创建一系列相关产品 → 适用于产品族

1. **链表**：反转、合并、环检测
2. **二叉树**：遍历、深度、路径和
3. **排序**：快排、归并、堆排（手写）
4. **动态规划**：背包、最长子序列
5. **字符串**：KMP、最长回文子串

### 6.2 面试技巧
- **先说思路**：不要直接写代码，先解释你的思路
- **边界条件**：空指针、空数组、溢出
- **时间复杂度**：主动分析时间和空间复杂度
- **优化方向**：如果时间允许，提出优化方案
