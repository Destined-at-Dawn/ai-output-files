# 🎤 自我介绍模板：3个版本

> **使用原则**：不要背稿，要「自然地讲出来」。准备3个版本，根据面试官的反应灵活切换。

---

## 一、30秒版（快速版，用于HR面/电话面）

```
面试官您好，我叫[姓名]，[学校][专业]的[年级]学生。

我主要做C++后端开发，熟悉网络编程和数据库。
独立开发过一个基于Epoll的高并发HTTP服务器，支持10000+并发连接。

我对贵公司的[具体业务/技术栈]很感兴趣，希望能加入团队。
```

**适用场景**：HR面、电话筛选、时间紧张时

---

## 二、1分钟版（标准版，用于技术面）

```
面试官您好，我叫[姓名]，来自[学校][专业]。

我主要的技术栈是C++后端开发，包括：
- 熟悉C++11/14/17特性，掌握STL、智能指针、多态等核心机制
- 熟悉Linux系统编程，有网络编程和高并发服务器的实战经验
- 熟悉MySQL和Redis，了解索引优化和缓存策略

项目方面，我独立开发过两个主要项目：

第一个是基于Epoll的高并发HTTP服务器，我从零实现了HTTP协议解析、线程池、数据库连接池，最终支持了10000+并发连接，QPS达到3000。

第二个是基于Protobuf的RPC框架，实现了服务注册发现、负载均衡和异步调用，让我对分布式系统有了更深的理解。

除了技术，我还参加过蓝桥杯竞赛并获得省级奖项，这锻炼了我的算法能力和问题解决能力。

我对贵公司的[具体业务/技术方向]很感兴趣，希望能在这里进一步提升自己的技术能力。
```

**适用场景**：技术面、项目面、正式面试

## 三、2分钟版（详细版，用于深入面试/老板面）

```
面试官您好，我叫[姓名]，来自[学校][专业]。

我从[大一/大二]开始学习C++，到现在已经有[X]年的开发经验。
我的技术栈主要集中在C++后端开发，包括网络编程、高并发、数据库等方向。

先说说我的技术基础：
- C++方面：我熟悉C++11/14/17的新特性，比如智能指针、移动语义、Lambda表达式
- 网络方面：我理解TCP/IP协议栈，熟悉Socket编程和HTTP协议
- 数据库方面：我熟悉MySQL的索引优化和事务隔离级别，也了解Redis的常见数据结构和使用场景

项目方面，我做过两个比较完整的项目：

第一个是基于Epoll的高并发HTTP服务器。
这个项目的背景是，我想深入理解网络编程和高并发的原理，所以从零开始实现了一个HTTP服务器。
我遇到了几个关键问题：如何处理大量并发连接？如何高效地解析HTTP请求？如何优化数据库访问？
我的解决方案是：使用Epoll实现IO多路复用，设计线程池处理请求，引入数据库连接池和Redis缓存。
最终，这个服务器支持了10000+并发连接，QPS达到3000，数据库操作延迟降低了60%。

第二个是基于Protobuf的RPC框架。
这个项目的背景是，我想学习分布式系统的核心概念，所以实现了一个轻量级的RPC框架。
我实现了服务注册发现（基于ZooKeeper）、自定义协议设计、负载均衡和异步调用。
这个项目让我对分布式一致性、序列化、网络通信有了更深的理解。

除了项目，我还参加过蓝桥杯竞赛并获得省级奖项。
这个经历锻炼了我的算法能力和代码实现能力，让我能够快速地将思路转化为代码。

我对贵公司的[具体业务/技术方向]很感兴趣。
我了解到贵公司在[具体技术领域]有很深的积累，这正是我想要深入学习的方向。
我相信我的技术基础和学习能力，能够快速融入团队并为项目做出贡献。
```

**适用场景**：深入技术面、老板面、时间充裕时

## 四、自我介绍的「心法」

### 4.1 结构要清晰
- **开场**：姓名+学校+专业（5秒）
- **技术栈**：你擅长什么（10秒）
- **项目经验**：你做过什么，解决了什么问题（30秒-1分钟）
- **为什么选这家公司**：你对公司的了解和兴趣（10秒）

### 4.2 项目要「讲故事」
```
❌ 我做了一个HTTP服务器
✅ 我想深入理解网络编程，所以从零实现了一个HTTP服务器，遇到了XX问题，用XX方法解决
```

### 4.3 要有「记忆锚点」
- **数字**：10000+并发、3000 QPS、60%延迟降低
- **技术词**：Epoll、Reactor、Protobuf、ZooKeeper
- **故事**：从零开始、独立解决、性能优化

### 4.4 要「自然」，不要「背稿」
- **练习方法**：对着镜子讲，或者录音回听
- **目标**：听起来像「聊天」，不是「背书」
- **技巧**：可以有停顿、可以有「嗯」「就是说」这样的口头语

### 4.5 要「自信」，不要「自大」
```
❌ 我精通C++，我什么都会
✅ 我熟悉C++的核心特性，有实战经验，还在持续学习
```

## 五、自我介绍的「加分技巧」

### 5.1 提前了解公司
- 看公司官网、技术博客、GitHub
- 了解公司的技术栈、业务方向
- 在自我介绍中提到「我了解到贵公司在XX方面...」

### 5.2 准备「钩子」
- 在自我介绍中留下「可追问」的点
- 例如：「我在做XX项目时遇到了一个有趣的问题...」
- 面试官大概率会问：「什么问题？你怎么解决的？」

### 5.3 准备「反问」
- 面试结束时，面试官会问「你有什么问题吗？」
- **好问题**：
  - 「这个岗位的主要工作内容是什么？」
  - 「团队的技术栈是什么？」
  - 「公司对新人的培养机制是怎样的？」
- **不要问**：
  - 「加班多吗？」（显得怕吃苦）
  - 「薪资多少？」（HR面再问）

## 六、不同场景的「灵活切换」

### 6.1 如果面试官很忙（HR面/电话面）
- 用30秒版
- 重点讲「我有什么能力」「我对公司感兴趣」

### 6.2 如果面试官是技术主管（技术面）
- 用1分钟版
- 重点讲「项目细节」「技术深度」

### 6.3 如果面试官是老板（终面）
- 用2分钟版
- 重点讲「解决问题的能力」「学习能力」「为什么选这家公司」

### 6.4 如果面试官追问项目细节
- 展开讲项目中的「技术难点」和「解决方案」
- 用STAR法则：背景→任务→行动→结果

---

**记住**：自我介绍的唯一目的是「让面试官对你感兴趣」。只要他愿意继续问你问题，你就成功了。
