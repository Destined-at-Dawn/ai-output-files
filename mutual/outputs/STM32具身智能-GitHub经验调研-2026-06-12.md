## 核心结论（30秒版）

1. **不是缺仓库，是缺"专业固件范式"**。社区共识：Claude 写 STM32 小块代码够用，但大块易错——根因是 AI 默认堆 HAL 调用，而专业固件靠**状态机 + Active Object + RTOS** 架构。具身智能机器人控制天然是状态机，这点最该喂给你的 AI。
2. **调试设备 = 实时观测工具链**：SWD/ITM/SEGGER RTT 让你不停机看变量，是"调试设备"的核心抓手（对标你 FPGA 侧的 ILA）。
3. **单点最高杠杆**：把下面 S 级三个仓库的精华提炼成一个 STM32 skill（或增强 Codex 的 hardware-design），让 AI 带着范式工作。

## 道法术器穿透

- **道（Why）**：具身智能 = 感知+决策+控制，STM32 是控制层的实时执行体。AI 要帮你写好它，前提是脑子里有"专业固件长什么样"，否则只会堆 HAL。
- **法（How）**：调试驱动开发——SWD/RTT 实时观测，复现→假设→插桩→修复→回归（= li-debug 循环）。先能"看见"设备状态，才能调。
- **术（What）**：寄存器级 + HAL 双轨理解；状态机/Active Object 架构；MISRA-C 编码规范。
- **器（With What）**：cpq 裸机指南（调试）、Miro 课程（架构）、SEGGER RTT（观测）、awesome-embedded（索引）、STM32Cube（参考）。
