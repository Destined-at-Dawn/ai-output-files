## 🧠 在开始之前：你电脑为什么越来越卡

### 旧世界：你的电脑正在被你"喂胖"

打开你的 C 盘看看——还剩多少空间？

如果你的 C 盘使用率超过 **80%**（也就是"可用空间不到总容量 1/5"），你不是一个人。每一个用电脑超过半年的人，都会遇到这个情况。

**电脑变卡，不是因为"老化了"，而是因为你从来没清理过。**

### 这些"垃圾"从哪来的

你不知道的是，你每次用电脑，都在悄悄往硬盘里塞东西：

- **你打开微信**：别人发的每张图片、每个文件，自动存进一个隐藏文件夹（动辄 3-5 GB）
- **你安装和卸载软件**：卸载只删了表面那层，安装包、临时文件、更新包全留在磁盘深处
- **你用浏览器**：Chrome/Edge 的缓存文件每天都在涨（几百 MB → 几 GB）
- **你写代码**（如果会的话）：npm、pip、pnpm 这些工具每次安装包都会缓存一份，半年下来能攒 5-10 GB
- **你编辑视频/图片**：剪映、PR 这些软件的缓存和自动保存文件比你想象的大的多

这些文件**你根本看不到**——它们藏在 AppData、.cache、Temp 这些"系统角落"里。Windows 不会主动告诉你它们在哪儿，也不会帮你清理。

> 💬 **用一句话理解**：就像你搬家之后，旧房子的快递包裹、外卖小票、装修边角料还堆在新家柜子里——你平时看不见，但它们占了 1/3 的面积。

### 新世界：一次扫描 + 分级清理 = 电脑恢复出厂速度

用 Storage Analyzer 之后，你会得到：

- ✅ **命令行直接出结果**——扫描完就在终端里列出所有绿灯项目 + 预计释放空间
- ✅ **三色分级清单**——🟢 能直接删的（你不需要判断）、🟡 需要你判断的、🔴 绝对不能动的
- ✅ **一次确认全清**——看完清单按 Y，所有绿灯项自动清理，不用逐项勾选

## 📊 Step 3：看懂三色分级（你不需要背，看一眼就懂）

你已经看到终端里的🟢🟡清单了。理解一下这三色是什么意思——这是 Storage Analyzer 的核心设计：

### 🟢 绿色 = 安全清理（工具直接帮你清）

这一类是**确定不会影响系统运行的文件**。你按 Y 的时候清理的就是这些。常见的有：

| 项目 | 大概大小 | 是什么 | 删了有影响吗 |
|------|---------|--------|------------|
| Temp 临时文件 | 3-10 GB | Windows 和应用产生的临时文件 | ❌ 完全没影响 |
| 浏览器缓存 | 1-5 GB | Chrome/Edge 的网页缓存 | ❌ 没影响，只是网页会重新加载 |
| npm / pip / pnpm 缓存 | 3-8 GB | 编程工具的安装包缓存 | ❌ 没影响，下次安装会重新下载 |
| Playwright 浏览器 | 1-2 GB | 自动化测试的浏览器包（不用的话） | ❌ 没影响 |
| Windows 更新缓存 | 2-5 GB | 系统更新后留下的安装包 | ❌ 没影响 |
| 各种应用更新包 | 1-3 GB | 软件自动更新下载的安装包 | ❌ 没影响 |

> 💡 **这些由 295 条规则自动识别，你不需要逐项判断。工具说"这是绿的"，它就是安全的。**

### 🟡 黄色 = 需要你手动判断（不包含在一次清理中）

这一类**不会在按 Y 时被清理**。但它们是"大户"——往往比绿色项还大。需要你**另外手动处理**：

| 项目 | 大概大小 | 怎么手动处理 |
|------|---------|------------|
| 微信/QQ 文件 | 3-10 GB | 打开 `WeChat Files` 文件夹，删掉旧聊天里的图片/文件 |
| 剪映/PR 缓存 | 3-8 GB | 打开剪映 → 设置 → 缓存管理 → 清理 |
| Ollama / AI 模型 | 5-20 GB | Win+R 输入 `cmd` → `ollama list` 看哪些模型在用 → `ollama rm 模型名` 删掉不用的 |
| 下载文件夹 | 1-5 GB | 打开下载文件夹，旧安装包（.exe/.msi）删掉 |
| 夸克/百度网盘缓存 | 2-5 GB | 打开对应 App → 设置 → 清理本地缓存 |

> 💡 **黄色项 = 有云端备份的删、已发布的视频项目缓存删、不用的 AI 模型删。拿不准的，保留不动。**

### 🔴 红色 = 不要动

这一类是**系统文件、工作数据、重要配置**。**不会出现在清理清单里**——295 条规则内置了白名单，直接把它们排除了。

| 项目 | 为什么不会出现 |
|------|------------|
| Windows 系统文件 | 白名单硬排除（`C:\Windows\System32` 等） |
| Program Files 里的软件 | 白名单硬排除 |
| 工作项目文件夹 | 不在缓存/临时目录的路径里 |
| 照片/视频文件夹 | 不在清理目标路径里 |

> ⚠️ **原则：不确定能不能删的，默认不动。宁可多留，不要误删。Storage Analyzer 的设计贯彻了这个原则——它只自动清理它 100% 确定是垃圾的东西。**

### 为什么可以"看都不看就删"

因为🟢绿色项的判断不是靠"猜"，是靠 **295 条硬编码规则**：

- 规则里明确定义了"什么是缓存""什么是临时文件""什么是安全的"
- `C:\Windows\Temp` → 规则 #001 → 绿色
- `C:\Users\xxx\AppData\Local\npm-cache` → 规则 #047 → 绿色
- `C:\Windows\System32` → 白名单 → 根本不会出现在清单里

**这不是 AI 猜的，是人工审查过每条路径后写死的规则。绿色 = 绝对安全。**

### 误删了怎么办

打开回收站 → 右键文件 → 「还原」。搞定。

> 💡 **清理完用两天电脑，确认没问题，再去回收站清空。这条建议和旧版本一样。**

## 🆘 遇到问题怎么办

### 问题 1：提示"python 不是内部命令"

**原因**：你的电脑没装 Python，或者装了但没加到系统 PATH。

**解决（选一个方向）**：

**方向 A：装 Python（推荐，5 分钟）**
1. 打开 https://www.python.org/downloads/
2. 点击黄色「Download Python 3.x.x」按钮
3. 运行下载的安装包
4. **⚠️ 关键一步**：勾选底部的「Add Python to PATH」→ 再点 Install Now
5. 装完之后，关掉所有命令行窗口，重新打开，输入 `python --version` 确认

**方向 B：用 Windows 自带的磁盘清理（也能用，但少清 50%）**
1. Win+R → 输入 `cleanmgr` → 回车
2. 选择 C 盘 → 确定
3. 勾选「临时文件」「回收站」「缩略图」→ 确定
4. 大概能清 3-5 GB（对比 Storage Analyzer 的 10-30GB）

### 问题 2：直接清理模式（菜单 1）我不放心，想看清单再决定

**解决**：选菜单里的 `2. 扫描 + 分析 + 网页报告`。这个模式会生成交互式网页，你可以逐项查看、逐项勾选、选择清理方式。和旧版完全一样。

> 💡 第一次用建议先选 2（网页报告模式）感受一下。用熟了之后就选 1（直接清理），快很多。

### 问题 3：扫描到一半卡住了

**解决**：正常。某些系统目录（如 `C:\Windows\WinSxS`）包含几十万个文件，统计大小需要时间。等 2-3 分钟，不要关。

> 💡 **判断是否真卡死的方法**：打开任务管理器（Ctrl+Shift+Esc），找到 `python.exe`，看 CPU 使用率。如果是 0% 且持续超过 5 分钟，才是真卡死了。关掉重来即可——扫描不修改任何文件，重来不会有任何副作用。

### 问题 4：清理之后某个软件打不开了

**解决**（概率很低，但不排除）：

这种情况通常是因为清理了 🟡 黄色项里的东西。比如：
- 清了某个软件的缓存 → 软件首次启动会重建缓存（慢一点但能正常用）
- 清了大模型文件 → 重新下载模型（`ollama pull xxx`）

**如果真出问题了**：
1. 检查回收站，把相关的文件「还原」
2. 重装那个软件（数据不会丢，只是重装程序本身）

### 问题 5：我是 Mac，能用吗

**操作步骤**：
1. 解压压缩包
2. 打开终端（聚焦搜索 `Terminal`）
3. `cd` + 把解压后的文件夹拖进终端 → 回车
4. `bash 一键清理.sh` → 后续和 Windows 一样

> 如果提示「Permission denied」：先 `chmod +x 一键清理.sh`

> Mac 版小黎没实测过（原版代码里写好了 macOS 的扫描路径），如果遇到问题群里喊一声，小黎去修。

## 🔗 进阶：如果你想了解更多

### 295 条识别规则是怎么工作的

`auto_analyze.py` 内置了 295 条规则，覆盖了 200+ 种应用和缓存目录。每条规则定义了"这个目录是什么"和"应该分到哪个颜色"。

规则的匹配方式有 6 种：
- **exact**：精确路径匹配（如 `C:\Windows\Temp`）
- **path**：路径包含匹配（如 路径含 `\npm-cache\`）
- **regex**：正则表达式匹配（如 路径含版本号模式 `\d+\.\d+\.\d+`）
- **keyword**：关键词匹配（如 文件夹名含 `cache`）
- **size**：大小阈值匹配（如 超过 1GB 的临时文件）
- **age**：时间阈值匹配（如 超过 90 天的下载文件）

这保证了：即使你装了什么冷门软件产生了垃圾文件，只要它的缓存路径匹配了这些规则，就能被正确识别。

### 安全机制：为什么不会误删系统文件

Storage Analyzer 有**三层防护**：

1. **只读扫描**——扫描阶段不写任何东西，纯读取文件夹大小
2. **白名单机制**——系统关键目录（`C:\Windows\System32`、`C:\Program Files` 等）被硬编码为 🔴 红色，永远不会出现在可选清理列表里
3. **回收站回退**——默认删除方式是"移到回收站"，误删可以还原
