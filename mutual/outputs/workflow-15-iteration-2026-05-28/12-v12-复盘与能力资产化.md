# V12 复盘与能力资产化

## 用户反馈

“我觉得你还可以优化得更好。项目做完不能只是文件多了，能力也要增长。”

## 本轮问题判断

高质量工作流不只产出文件，还要把经历转化为能力资产。

## 解决方案

每个项目结束必须做 4 层复盘：

1. 事实复盘：做了什么
2. 能力复盘：提升了什么
3. 证据复盘：有什么可展示材料
4. 迁移复盘：下次如何复用

## 复盘模板

```markdown
# 项目复盘

## 1. 事实
- 目标：
- 完成产出：
- 未完成：

## 2. 能力
- 技术能力：
- 文档表达：
- 面试表达：
- 项目管理：

## 3. 证据
- 代码：
- 文档：
- 截图/演示：
- 简历 bullet：

## 4. 迁移
- 可复用模板：
- 下次避免：
- 是否沉淀为 SOP：
```

## 沉淀规则

- 一次经验：写项目复盘
- 两次复用：写 SOP 草案
- 三次有效：升级 SOP
- 跨区适用：进入共享协议候选

## 本轮仍未解决

如果用户没有主动复盘，系统可能跳过。下一轮设计自动提醒和 Hermes 监控。
