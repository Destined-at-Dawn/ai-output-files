# V06 多 Agent 分工与质检链

## 用户反馈

“我觉得你还可以优化得更好。Claude、Codex、Hermes 约束不同，怎么协作？”

## 本轮问题判断

不要让不同 Agent 复刻彼此。应该让它们遵守同一协议，但执行不同职责。

## 解决方案

建立三段式质检链：

```text
Claude/Newmax：主控与语义质量
Codex：工程实现与文件验证
Hermes：定时监控与长期自动化
```

## 分工表

### Claude/Newmax

- 拆任务
- 定策略
- 写文档
- 判断是否沉淀记忆
- 维护项目卡

### Codex

- 读写文件
- 代码实现
- 路径审计
- 运行验证
- 输出问题清单

### Hermes

- 定时抓取
- 日报/周报
- 趋势监控
- 长期任务提醒
- 自动收集候选素材

## 质检规则

- Claude 产出重要文档后，Codex 抽查路径、结构、重复、可执行性。
- Codex 做工程修改后，Claude 做语义审查与使用场景转化。
- Hermes 只做低风险长期监控，不直接改核心规则。

## 本轮仍未解决

不同 Agent 写入记忆可能冲突。下一轮需要定义记忆分层与候选机制。
