# V02 项目卡最小化与生命周期

## 用户反馈

“我觉得你还可以优化得更好。项目卡如果也越来越多，最后还是乱。”

## 本轮问题判断

项目卡必须是“导航索引”，不是“项目全文”。它只负责定位，不负责承载全部内容。

## 解决方案

项目卡只保留 8 个字段：

1. 项目 ID
2. 项目目标
3. 当前状态
4. 工作区分工
5. 关键产出路径
6. 决策记录
7. 下一步行动
8. 关闭条件

## 项目状态

```text
Inbox      刚出现的想法，尚未立项
Active     正在推进
Blocked    卡住，需要外部条件
Reusable   已完成，可复用
Archived   结束，不再推进
```

## 关闭机制

每张项目卡必须有关闭条件：

- 面试项目：完成简历 bullet + 面试 Q&A + 项目文档
- 技术项目：代码可运行 + README + 复盘
- 内容项目：发布稿 + 素材来源 + 复盘

## 落地规则

```markdown
项目卡超过 120 行时必须拆：
- 项目卡保留索引
- 详细内容放回对应工作区
- 项目卡只保留路径和摘要
```
