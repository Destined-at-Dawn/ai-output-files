# V09 轻启动与按需加载协议

## 用户反馈

“我觉得你还可以优化得更好。启动读太多会慢，但读太少又断上下文。”

## 本轮问题判断

启动协议不能一刀切。日常小任务和跨区大项目需要不同加载深度。

## 解决方案

使用三级启动：

```text
L1 轻启动：日常问答、小改动
L2 项目启动：涉及项目、跨区产出、连续任务
L3 高风险启动：删除、批量修改、规则变更、记忆沉淀
```

## L1 轻启动

读取：

- 当前工作区简短说明
- 今日摘要
- 用户本次消息

## L2 项目启动

追加读取：

- 项目上下文卡
- artifact-registry 中相关产出
- 当前工作区 SOP

## L3 高风险启动

追加读取：

- 底层铁律
- 读写/归档规则
- 记忆候选协议
- 相关历史决策

## 判断规则

```text
用户问概念/让写一段话 → L1
用户说“这个项目/面试/简历/竞赛” → L2
用户说“删除/同步/改规则/长期记住/批量” → L3
```
