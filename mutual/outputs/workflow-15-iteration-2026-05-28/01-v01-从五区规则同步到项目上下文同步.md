# V01 从五区规则同步到项目上下文同步

## 用户反馈

“我觉得你还可以优化得更好。五区不同步会让我信息断裂。”

## 本轮问题判断

当前矛盾不是“要不要同步”，而是“同步什么”。五区完全同步规则会制造维护负担；五区完全不同步会制造项目断裂。

## 解决方案

把同步对象从“规则全文”改为“项目上下文”：

- 规则只同步底层铁律：安全、读写、归档、记忆候选、产出注册。
- 项目上下文必须跨区同步：目标、当前状态、关键路径、决策、下一步。
- 每个跨区项目建立一张 `Project Context Card`。
- 所有工作区处理该项目时，先读项目卡，再进入本区专业流程。

## 落地结构

```text
mutual/project-context/
  PROJECT-YYYY-topic.md

竞赛区：技术实现、实验、代码
创作区：调研、文档、PPT、表达
求职区：简历、面试、公司定制
个人区：学习复盘、长期能力沉淀
```

## 关键规则

```markdown
任何跨区任务开始前：
1. 先查 mutual/project-context/ 是否已有项目卡
2. 有则读取；没有则创建
3. 产出完成后把路径写回项目卡
4. 只把稳定结论沉淀为 SOP 或记忆
```
