# V07 记忆候选池与长期记忆门禁

## 用户反馈

“我觉得你还可以优化得更好。我要越用越懂我，但不能乱记。”

## 本轮问题判断

“越用越懂我”的关键不是所有信息都进长期记忆，而是能区分临时状态、项目事实、长期偏好和规则。

## 解决方案

建立四层记忆：

```text
Daily Log       当日事实，自动追加
Project Memory  项目事实，写入项目卡
Memory Candidate 长期记忆候选，等待确认
Long-term Memory 已确认长期偏好/决策/教训
```

## 写入规则

### 自动写入

- 当日任务记录
- 文件路径
- 项目状态
- 失败记录

### 候选确认

- 用户长期偏好
- 重要人生/职业决策
- 反复出现 3 次以上的教训
- 会影响多个工作区的工作流规则

### 禁止直接写入长期记忆

- 一次性情绪
- 临时想法
- 未验证判断
- Agent 自己推断的用户偏好

## 候选格式

```markdown
## MEM-CAND-YYYYMMDD-001
- 候选内容：
- 来源对话/任务：
- 适用范围：
- 证据：
- 失效条件：
- 建议写入位置：
```

## 本轮仍未解决

记忆候选太多也会膨胀。下一轮需要建立“工作流改进收件箱”和批处理节奏。
