# V11 跨区冲刺流程

## 用户反馈

“我觉得你还可以优化得更好。面试项目会同时涉及技术、文档、简历、模拟面试。”

## 本轮问题判断

跨区项目不应该按工作区碎片推进，而应该按短周期冲刺推进。

## 解决方案

每个跨区项目进入 7 天冲刺：

```text
Day 1 定义目标与项目卡
Day 2 技术实现/证据补齐
Day 3 调研与文档包装
Day 4 简历转化
Day 5 面试问答
Day 6 公司定制
Day 7 复盘与注册
```

## 每日输出

- 当天具体产出
- 写回项目卡
- 登记关键 artifact
- 标记阻塞

## 冲刺看板

```markdown
## Sprint Backlog
- [ ] 技术证据
- [ ] 项目 README
- [ ] 简历 bullet
- [ ] 面试 Q&A
- [ ] 公司定制版本
- [ ] 复盘
```

## 好处

你不再问“我该在哪个工作区干活”，而是问“今天这个冲刺阶段需要哪个工作区支援”。

## 本轮仍未解决

冲刺结束后如何沉淀为长期能力，还需要复盘协议。下一轮解决。
