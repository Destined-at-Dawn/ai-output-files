# V08 工作流改进收件箱

## 用户反馈

“我觉得你还可以优化得更好。我经常突然想到一个优化点，不能每次都五区同步。”

## 本轮问题判断

工作流优化想法需要先进入低成本收件箱，不能一出现就升级为规则。

## 解决方案

建立 `workflow-inbox/`，存放所有未验证优化想法。

## 优化想法模板

```markdown
## WF-CAND-YYYYMMDD-001
- 想法：
- 触发场景：
- 想解决的问题：
- 适用范围：全局 / mutual / 个人 / 创作 / 求职 / 竞赛
- 风险：
- 最小验证方式：
- 验证状态：未验证 / 试用中 / 有效 / 放弃
- 是否升级：否 / SOP / 共享协议 / 底层铁律
```

## 升级路径

```text
一次想法 → workflow-inbox
两次复用 → SOP 草案
三次稳定有效 → 正式 SOP
跨区高频有效 → 共享协议
涉及安全/覆盖/删除 → 底层铁律
```

## 批处理节奏

- 每周一次清理 inbox
- 不在任务中临时大改规则
- 工作流改进和当前任务分离
