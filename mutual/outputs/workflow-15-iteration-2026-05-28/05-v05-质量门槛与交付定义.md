# V05 质量门槛与交付定义

## 用户反馈

“我觉得你还可以优化得更好。全自动不能变成自动生产垃圾。”

## 本轮问题判断

自动化的核心风险是把低质量产出快速规模化。必须先定义“什么叫完成”。

## 解决方案

所有重要产出必须通过 `Definition of Done`。

## 通用完成标准

```markdown
一份产出可标记 usable，必须满足：
1. 目标明确：解决哪个问题
2. 可直接使用：不是半成品说明
3. 来源清楚：关键判断有来源或依据
4. 结构完整：开头结论，正文分层，结尾下一步
5. 复用入口明确：已登记到 artifact-registry
```

## 面试项目完成标准

- 有项目一句话介绍
- 有技术难点
- 有个人贡献
- 有量化结果或可展示证据
- 有 10 个面试追问与回答

## 技术项目完成标准

- 能运行或说明不能运行的原因
- 有 README
- 有验证记录
- 有已知问题
- 有下一步

## 文档产出完成标准

- 有目标读者
- 有结论
- 有来源
- 有可执行建议
- 无明显空话

## 本轮仍未解决

质量门槛需要有人检查。下一轮设计多 Agent 质检分工。
