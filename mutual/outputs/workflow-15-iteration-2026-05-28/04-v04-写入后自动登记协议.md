# V04 写入后自动登记协议

## 用户反馈

“我觉得你还可以优化得更好。注册表靠我手动维护，肯定坚持不了。”

## 本轮问题判断

凡是靠用户手动补登记的流程，长期都会失败。登记必须变成 Agent 写文件后的默认动作。

## 解决方案

增加“写入后自动登记协议”：

```text
写入重要文件
  → 判断是否可复用
  → 可复用则写入 artifact-registry
  → 关联项目卡
  → 在最终回复中告知登记结果
```

## 可复用判断

满足任意一条就登记：

- 未来面试/简历/答辩会用
- 是项目说明、README、PPT、复盘
- 是调研结论或资料清单
- 是代码工具或脚本
- 是跨工作区会引用的素材

## 不登记内容

- 临时草稿
- 单次对话中间推理
- 无法复用的碎片
- 已经被更高质量版本替代的文件

## 登记粒度

不要登记每个小文件，只登记“能被复用的单元”。例如一个项目 30 张截图，只登记“项目截图包”。

## 本轮仍未解决

Agent 可能把低质量产出也登记进去。下一轮需要加入“质量门槛”。
