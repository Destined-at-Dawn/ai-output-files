# V10 任务入口路由器

## 用户反馈

“我觉得你还可以优化得更好。我不想自己判断该去哪个工作区。”

## 本轮问题判断

用户不应该承担路由成本。系统应该根据任务类型推荐工作区和启动等级。

## 解决方案

建立 `task-router.md`。

## 路由规则

```text
代码/硬件/实验/仿真 → 竞赛区 + Codex
调研/文档/PPT/表达稿 → 创作区 + Claude
简历/面试/公司研究 → 求职区 + Claude
长期学习/认知复盘/习惯 → 个人区 + Claude
跨区项目/工作流优化/注册表 → mutual + Claude/Codex
定时监控/日报/趋势 → Hermes
```

## 路由输出格式

```markdown
建议入口：
- 工作区：
- Agent：
- 启动等级：
- 需要读取：
- 预计产出：
- 是否登记：
```

## 特殊规则

如果任务跨越 2 个以上工作区，必须先进入 mutual 建项目卡；不要直接在某个业务区开干。

## 本轮仍未解决

项目推进中仍可能频繁切换工作区。下一轮设计“跨区冲刺流程”。
