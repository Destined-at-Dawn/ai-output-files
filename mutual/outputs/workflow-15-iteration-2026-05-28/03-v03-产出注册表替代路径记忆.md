# V03 产出注册表替代路径记忆

## 用户反馈

“我觉得你还可以优化得更好。我不想记得每个文件放在哪里。”

## 本轮问题判断

跨区协作不能依赖人脑记路径，也不能依赖 Agent 临时搜索。需要一个统一的产出注册表。

## 解决方案

建立 `artifact-registry.md`，所有可复用产出都登记。

## 注册字段

```markdown
## ART-YYYYMMDD-001
- 名称：
- 所属项目：
- 所属工作区：
- 类型：代码 / 文档 / 简历 / 调研 / PPT / 面试材料 / 复盘
- 状态：draft / usable / deprecated / archived
- 路径：
- 摘要：
- 可复用场景：
- 上次更新：
```

## 使用规则

- Agent 需要素材时，先查注册表，不先全盘搜索。
- 文件移动后，只更新注册表路径。
- 废弃文件不删除记录，只改状态为 `deprecated`。
- 重要产出必须绑定项目卡。

## 价值

这会把“文件散落”变成“资产可寻址”。以后求职区写简历，不需要知道项目在哪，只需要查“该项目有哪些 usable 资产”。

## 本轮仍未解决

注册表如果靠手动维护会漏。下一轮需要设计“写入后的自动登记协议”。
