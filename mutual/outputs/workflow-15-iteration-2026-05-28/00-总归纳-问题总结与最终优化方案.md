# 总归纳：问题总结与最终优化方案

## 结论

你当前工作流的问题不是“缺同步”，而是“同步对象错误”。

最终方案：

> 五区不要同步完整规则，要同步项目上下文、产出注册、决策记录、工作流候选和长期记忆候选。

## 当前核心问题

### 1. 五区规则完全同步会制造维护灾难

规则越多，越容易出现漂移、重复、冲突。同步全部规则会让 5 个工作区都变成难维护的副本。

### 2. 五区完全不同步会制造学习断裂

你的真实任务经常跨区：竞赛做项目、创作做文档、求职做转化、个人做复盘。如果上下文不共享，会降低学习效率。

### 3. 绝对路径引用不够稳定

只靠路径会在文件移动、改名、归档后失效。需要产出注册表。

### 4. 自进化权限过大

Agent 可以自动记录，但不应该自动修改长期规则和核心记忆。否则“越用越懂你”会变成“越用越乱”。

### 5. 自动化缺少质量门槛

全自动如果没有 Definition of Done，会快速生产低质量文件。

```text
求职/
  简历、面试、公司研究、能力转化
```

## 最小落地顺序

### 第 1 步：先建项目上下文卡

解决跨区任务断裂。

### 第 2 步：再建产出注册表

解决文件散落和路径记忆问题。

### 第 3 步：再建 workflow-inbox

解决你突然想到优化点时不知道放哪的问题。

### 第 4 步：再加轻启动/项目启动/高风险启动

解决每次都读太多或读太少的问题。

### 第 5 步：最后接 Hermes 自动巡检

让长期任务自动提醒，但不让 Hermes 直接改核心规则。

## 最终三条铁律

```text
跨区项目先建项目卡
重要产出必须登记
长期记忆必须候选确认
```

## 你以后怎么用

### 场景：准备一个面试项目

1. 在 mutual 创建项目卡
2. 竞赛区做技术实现
3. 创作区写项目说明、调研、PPT
4. 求职区转成简历 bullet 和面试 Q&A
5. 个人区做能力复盘
6. 所有产出路径写回项目卡和注册表

### 场景：突然想到一个优化工作流的想法

1. 不改五区规则
2. 写入 workflow-inbox
3. 标记适用范围和最小验证方式
4. 验证后再升级 SOP 或共享协议
