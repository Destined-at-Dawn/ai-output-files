# 微信聊天分析 SOP

> 端到端流水线：密钥提取 → 数据库解密 → 聊天导出 → 社群过滤 → 心理蒸馏

---

## 阶段 1：密钥提取与数据库解密

### 前置条件
- Windows 系统
- 微信已登录且正在运行
- 管理员权限可用

### 工具链
| 工具 | 用途 | 获取方式 |
|------|------|---------|
| wx_key v2.1.8 | 从微信进程内存提取密钥 | 已包含在可复刻包 tools/wx_key/ |
| WeChatDataAnalysis | 用密钥解密 DB 文件 | GitHub 搜索 |

### 步骤

**Step 1.1：定位微信数据目录**
```
默认路径：D:\data\wechat\xwechat_files\wxid_{你的wxid}
包含：db_storage/（加密DB）+ msg/（附件）+ config/
```

**Step 1.2：提取密钥**
1. 确保微信正在运行
2. 以管理员身份运行 wx_key.exe
3. 自动检测微信进程并提取密钥
4. 密钥显示在界面上（一串 hex 字符）

**Step 1.3：解密数据库**
1. 运行 WeChatDataAnalysis
2. 输入密钥 + 数据目录路径
3. 选择输出目录（建议 Temp 目录，路径无中文）
4. 等待解密完成（16 个 DB 文件，约 450MB）

**Step 1.4：验证解密结果**
```python
python -c "
import sqlite3, os
db_dir = r'解密输出目录'
for f in os.listdir(db_dir):
    if f.endswith('.db'):
        conn = sqlite3.connect(os.path.join(db_dir, f))
        tables = conn.execute('SELECT name FROM sqlite_master WHERE type=\"table\"').fetchall()
        print(f'{f}: {len(tables)} tables')
"
```

### 阶段 1 出口门禁
- [ ] 密钥已提取
- [ ] 16 个 DB 文件已解密
- [ ] 表数量验证通过（预期 ~500）
- [ ] 解密副本已备份到安全位置

## 阶段 3：社群过滤

**使用 Skill**：community-filter

### 目标
从群聊消息中提取真正有价值的资源（文件/链接/方法论），过滤噪音。

### 关键数据
- 信息密度 < 2%（99%+ 的群聊消息是噪音）
- 拍一拍占社交群 40%+（最大噪音源）
- 价值信号：文件分享 > 公众号文章 > 链接 > 方法论 > 决策判断

### 阶段 3 出口门禁
- [ ] 噪音模式已过滤（拍一拍/表情/短文本）
- [ ] 价值信号已提取
- [ ] 资源清单已生成

---

## 阶段 4：心理蒸馏

**使用 Skill**：wechat-distiller + niuma-voice-dna

### 目标
从聊天记录中蒸馏人格画像、思维框架、表达 DNA。

### 分析维度
1. 情绪模式（情绪词频/波动曲线/触发词）
2. 社交模式（主动vs被动/消息长度/回复速度）
3. 认知模式（问句比例/抽象vs具象/逻辑连接词）
4. 表达 DNA（高频词/句式偏好/标点习惯）
5. 关系模式（语气差异/亲密称呼/话题深度）

### 阶段 4 出口门禁
- [ ] 五维分析完成
- [ ] 边界声明齐全（高/中/低置信度分类）
- [ ] 输出格式符合模板

## 9 条浓缩铁律

1. **密钥只在内存中**：必须从运行中的微信进程提取
2. **微信必须正在运行**：提取密钥前不能关闭微信
3. **Temp 目录清理风险**：解密后立即复制到安全位置
4. **表名会变**：先查 sqlite_master，不要硬编码
5. **zstd 必须解压**：magic bytes 28 B5 2F FD
6. **sender_id 因聊天而异**：不同聊天中同 ID 代表不同人
7. **Bash heredoc 含中文必炸**：一律用 Python 写文件
8. **拍一拍占 40%+ 噪音**：分析前必须过滤
9. **信息密度 < 2%**：先去噪再分析
