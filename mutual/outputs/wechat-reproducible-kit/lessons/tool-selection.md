# 工具选型理由

## 为什么用 wx_key 而不是其他工具

| 工具 | 方式 | 优势 | 劣势 |
|------|------|------|------|
| **wx_key v2.1.8** | DLL Hook 注入 | 最稳定，支持新版微信 | 需要管理员权限 |
| wechat-cli | Hex 模式匹配 | 不需要注入 | 新版微信 0/19 匹配失败 |
| pywxdump | 多种方式 | 功能最全 | 安装复杂，依赖多 |
| chatlog | 内存扫描 | CLI 友好 | 需要微信运行 |
| pysqlcipher3 | SQLCipher | 可离线解密 | 需要密钥，安装需编译 |

**结论**：wx_key 是最可靠的密钥提取工具。其他工具在新版微信（3.x / xwechat）上兼容性不佳。

## 为什么用 Python 而不是 PowerShell/Bash

| 场景 | Python | PowerShell | Bash |
|------|--------|-----------|------|
| 中文路径 | ✅ 原生 Unicode | ⚠️ 通过 Bash 启动时炸 | ❌ heredoc 必炸 |
| SQLite 操作 | ✅ 标准库 | ⚠️ 需要额外模块 | ❌ 不支持 |
| zstd 解压 | ✅ pip install | ❌ 无原生支持 | ❌ 无原生支持 |
| 文件编码 | ✅ open(encoding=) | ⚠️ 默认 UTF-16 | ❌ 取决于 locale |

**结论**：Python 是唯一能可靠处理中文路径 + SQLite + zstd 的语言。

## 为什么用 MD 格式导出而不是 JSON/CSV

| 格式 | AI 可读性 | 人类可读性 | 编辑便利性 |
|------|----------|-----------|-----------|
| **Markdown** | ⭐⭐⭐ | ⭐⭐⭐ | ⭐⭐⭐ |
| JSON | ⭐⭐ | ⭐ | ⭐⭐ |
| CSV | ⭐ | ⭐⭐ | ⭐⭐⭐ |

**结论**：MD 格式同时满足 AI 分析（蒸馏/过滤）和人类浏览（直接打开看）的需求。

## 为什么解密 DB 放创作区而不是 Temp

| 位置 | 持久性 | 中文路径风险 | 备份便利性 |
|------|--------|------------|-----------|
| **创作区** | ⭐⭐⭐ | ⚠️ 需要 Python 处理 | ⭐⭐⭐ Git 管理 |
| Temp | ⭐（随时被清理） | ✅ ASCII 路径 | ⭐ 无备份 |
| 用户桌面 | ⭐⭐ | ⚠️ 可能有空格 | ⭐⭐ |

**结论**：创作区是权威副本位置。Temp 只做临时工作副本。
