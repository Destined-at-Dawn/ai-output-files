# 社群信息过滤器 v2

> 噪音是信息的敌人。90% 的群聊消息是无价值的。
> 本 skill 的目标：从 21,000 条消息中筛出 288 个真正有用的资源。

**详细方法论见 references/ 目录。本文件是执行骨架。**

## Phase 0：环境确认

### 必须确认的文件
1. 已导出的聊天 MD 文件路径（默认：`个人/projects/20260425-成长历程/成长历程/交往/`）
2. 解密 DB 是否可用（Temp 目录下 `wechat-decrypted-*`）
3. 百大认知书籍目录（`个人/百大认知书籍/`）

### 模式判断
- **资源扫描模式**：用户说"列出资源" / "有什么资源" / "资源清单" → Phase 1
- **内容过滤模式**：用户说"过滤群聊" / "筛选信息" / "有什么有价值" → Phase 2
- **深度分析模式**：用户说"分析群聊" / "社群洞察" / "信息趋势" → Phase 3

## Phase 1：资源扫描（快扫）

**目标**：提取所有链接/文件/文档/视频引用，生成清单。

### 过滤流水线

```
原始消息（21,000+）
  → Step 1: 去噪（8 类噪音模式，见 references/noise-patterns.md）
  → Step 2: 信号提取（URL、文件引用、公众号文章、视频引用）
  → Step 3: 按群分组 + 按类型分类
  → Step 4: 输出资源清单 JSON + MD
```

### 噪音过滤规则（必须按序执行）

| 优先级 | 过滤规则 | 匹配模式 |
|--------|---------|---------|
| 1 | 系统消息 | speaker == "[系统]" |
| 2 | 媒体占位 | text 以 [图片]/[表情]/[视频]/[语音]/[文件] 开头 |
| 3 | VoIP 通话 | text 含 `<voipmsg` |
| 4 | 拍一拍 | text 含 "拍了拍" 且 len < 120 |
| 5 | 红包 | text 含 "CDATA[微信红包]" |
| 6 | 版本限制 | text 含 "版本不支持展示该内容" |
| 7 | 加群通知 | text 含 ("通过扫描" + "加入群聊") 或 ("邀请" + "加入了群聊") |
| 8 | 短消息 | len(text) < 15 |

### 信号提取规则

| 信号类型 | 匹配模式 | 价值等级 |
|----------|---------|---------|
| 文件分享 | `[25769803825/6] filename.ext` | ⭐⭐⭐ |
| 公众号文章 | `[21474836529/5] title` | ⭐⭐ |
| 视频/文章 | `[17179869233/4] title` | ⭐⭐ |
| 直接链接 | `https?://` | ⭐⭐ |
| 文档引用 | `.pdf/.doc/.docx/.ppt/.pptx/.xlsx/.zip` | ⭐⭐⭐ |

## Phase 2：内容过滤（深度扫描）

**目标**：在去噪后的消息中，识别有价值的内容讨论。

### 价值信号（必须是讨论内容，不是关键词匹配）

| 信号 | 判断标准 | 示例 |
|------|---------|------|
| 方法论分享 | 讲解"怎么做"的过程 | "我用这个方法2天学完一学期的课" |
| 工具推荐 | 推荐具体工具+使用方式 | "GeminiCLI教程，10分钟装好" |
| 经验教训 | 分享踩坑和解决方案 | "我花了两小时踩的坑" |
| 深度讨论 | >60字的认真回复 | 讨论商业模式、技术方案 |
| 决策依据 | 明确的判断和理由 | "这个赛道gemini不行，编程这个赛代够呛" |

### 反模式（这些不是有价值内容）

- ❌ 闲聊中的"分享"（"找我分享" "分享日常"）
- ❌ "工作"指代群聊（"工作群" "工作消息"）
- ❌ 泛化的鼓励（"你真棒" "加油"）
- ❌ 重复的 bug 报告（"main.log" ×16）

## Phase 3：深度分析（百大认知交叉）

**目标**：将有价值内容与百大认知框架交叉，输出行动建议。

### 交叉矩阵

| 百大认知书籍 | 在社群中的信号 | 典型映射 |
|-------------|--------------|---------|
| 《深度工作》 | 专注力讨论、干扰管理 | "沉浸式深度工作" |
| 《原子习惯》 | 习惯养成、打卡 | "坚持" "微习惯" |
| 《精要主义》 | 取舍决策 | "不做什么" "优先级" |
| 《反脆弱》 | 试错、踩坑 | "失败" "犯错" |
| 《认知觉醒》 | 元认知、自我觉察 | "认知" "觉醒" |
| 《稀缺》 | 忙、焦虑 | "时间不够" "焦虑" |

### 输出格式

```markdown
## 社群信息分析报告

### 核心发现（3 条以内）
1. [最有价值的发现]
2. [第二有价值的发现]
3. [第三有价值的发现]

### 资源清单（按价值排序）
#### ⭐ 必看
- [资源名]：[为什么重要] | 来源：[群名]

#### 🔵 建议看
- [资源名]：[简述] | 来源：[群名]

#### ⚪ 可忽略
- [资源类型]：[为什么没价值]

### 百大认知关联
- 《XX》在 [群名] 中出现了 N 次，典型场景：[描述]

### 行动建议
1. [具体可执行的建议]
```

## Phase 4：DB 更新触发（与 wechat_db_update 联动）

当解密 DB 更新时自动触发：
1. 重新扫描所有注册群聊
2. 对比上次扫描结果，提取新增资源
3. 更新资源索引文件
4. 生成增量报告

### 增量报告格式
```markdown
## [日期] 社群信息增量
- 新增资源：N 个
- 重点资源：[top 3]
- 百大认知新增关联：[描述]
```
