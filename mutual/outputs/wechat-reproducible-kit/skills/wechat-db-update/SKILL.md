## 触发条件

以下任一条件满足即触发：
1. 用户说"更新微信" / "刷新聊天" / "同步微信数据库"
2. 用户提供新的解密 DB 路径
3. 用户说"我开了微信"（暗示要提取密钥 + 解密）

---

## Phase 0：变更检测

**目标**：确认 DB 是否有更新，避免重复分析。

### 检测方法
1. 定位解密 DB 目录（按 wechat-analysis Phase 0 探测）
2. 检查 `message_0.db` 的修改时间
3. 与上次分析时间对比（存储在 `创作/output/wechat_analysis/last_update.json`）
4. 如果有新消息 → 进入 Phase 1
5. 如果无变化 → 告知用户"DB 未变化，无需更新"

### 上次更新记录格式
```json
{
  "last_update": "2026-06-09T18:00:00",
  "db_path": "E:\\ai产出文件\\牛马\\创作\\创作\\output\\databases\\wxid_3nvidmluot0a22",
  "total_messages": 339086,
  "total_sessions": 492,
  "exported_chats": ["Camellia", "米蛋糕", ...]
}
```

## Phase 1：增量导出

**目标**：只导出新增消息，不重复处理已有文件。

### 步骤
1. 读取上次导出的最后时间戳
2. 查询 DB 中每个会话的新增消息数
3. 对增量 > 100 条的会话重新导出
4. 对已有 MD 文件追加新消息（不覆写）

## Phase 2：社群过滤增量

**目标**：在新增消息中提取有价值资源。

### 步骤
1. 读取上次资源清单（`outputs/precious_resources.json`）
2. 对新增消息运行 community-filter Phase 1-2
3. 去重后追加到资源清单
4. 生成增量资源报告

### 输出格式
```markdown
## [日期] 社群信息增量
- 新增消息：N 条（覆盖 X 个群）
- 新增资源：Y 个
- 重点资源：
  1. [资源名] — [来源群] — [价值说明]
  2. ...
- 百大认知新增关联：[描述]
```

## Phase 3：文风DNA更新

**目标**：从小黎的新增消息中提取最新文风特征。

### 步骤
1. 提取小黎在所有会话中的新增消息
2. 与已有文风DNA基线对比
3. 如果风格变化 > 15% → 更新DNA文件
4. 追加到 `创作/output/writing_dna/` 目录

### DNA更新触发条件
- 新增消息 > 500 条
- 或距离上次更新 > 7 天
- 或用户主动要求

## 反模式

1. **全量重新处理**：每次更新都重新处理所有消息 → 应该增量处理
2. **不检查变更**：DB 没变也跑一遍全套 → 浪费资源
3. **忽略去重**：同一资源被重复添加到清单 → 增量报告质量下降
4. **不记录时间戳**：下次不知道从哪里开始增量 → 每次都全量
5. **单独跑每个skill**：手动串联容易遗漏 → 本 skill 自动编排
