#### B. 语气要求

- 像一个踩过坑的大三学长，在给学弟学妹发微信
- 直接、有判断力、不讨好
- 会说"我踩过这个坑""蓝哥原话："
- 口语化但不水，每句话有信息量
- ★ 安心话术："不用担心！""整个过程大约需要 X 分钟"（bijitongbu 风格）

#### C. 格式化规则

**必须加粗的**：
1. 关键概念首次出现（如 **AI分身**、**微创新**）
2. 工具/产品名（如 **Claude Code**、**秘塔AI搜索**）
3. 动作指令（如 **复制下面的提示词**、**点击下载**）
4. 核心结论/金句
5. 章节引用（如 **"启动方式"** 小节）
6. ★ UI 元素名称（如 **设置**、**第三方插件**、**安全模式**）

**标记系统**：
- `> 💡 本节概要` — 步骤摘要
- `> ⚠️ 注意` — 警告/防坑
- `💬 例子：` — 场景化示例
- `> 一句话记住：` — 核心观点提炼
- `❌` — 错误示范（v2.0 新增）
- `✅` — 成功确认/前置条件（v2.0 新增）
- `> **👉 点这里展开「xxx」**` — 可折叠区域（v2.0 新增）

**排版**：
- 表格用于对比/分类/工具汇总（不堆数据）
- 有序列表用于步骤
- 无序列表用于平级选项
- `---` 分隔独立段落
- ★ 代码块用于文件/目录结构图（树形符号 └── ├──）

#### D. 内容设计

每个步骤必须包含：
1. **价值前置** — 为什么要做这一步
2. **具象例子** — 身份+场景+对话+结果
3. **启动方式** — 复制粘贴就能用的提示词（AI相关步骤）
4. **产出检查** — 做完手上有什么
5. **衔接说明** — 怎么接上下一步
6. ★ **错误示范** — ❌ 做错了长什么样 + 为什么错（v2.0 新增）
7. ★ **成功确认** — ✅ 做完这步你应该看到什么（v2.0 新增）

全篇必须有：
- 至少3处 "> 一句话记住："
- 至少3条 "⚠️ 防坑"
- 至少1个真实的场景比喻
- 至少1个引用来源（蓝哥原话/真实案例/数据）
- ★ 至少2处 ❌ 错误示范（v2.0 新增）
- ★ 至少1处 👉 可折叠区域（v2.0 新增）
- ★ 3-5个术语的基础概念说明（v2.0 新增）

#### E. 词汇红线

**绝对禁止**：
赋能、深耕、抓手、闭环、对齐、拉齐、打通、链路
综上所述、由此可见、毋庸置疑、值得注意的是、与此同时
可能、大概、应该、好像、基本上、差不多

**用这些替换**：
- "赋能" → 直接说"能帮你做什么"
- "综上所述" → 删掉，直接说结论
- "可能" → 给确定判断，不确定说"取决于"
