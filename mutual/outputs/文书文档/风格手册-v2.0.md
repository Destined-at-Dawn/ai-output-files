## 一、整体定位

**这不是知识库文档，这是"行动手册"。**

核心差异：
- 知识库文档 = 告诉你"这是什么"
- 行动手册 = 带着你"做一遍"，每步有产出

每一篇的目标是：**读者离开页面时，手上多了一个真东西。**

## 五、排版规范

### 标题层级
- `#` 仅用于页面主标题（一个页面只有一个 H1）
- `##` 用于大章节
- `###` 用于步骤/子主题
- **加粗** 用于关键概念、工具名、动作指令

### 加粗规则
以下内容**必须加粗**：
1. 关键概念首次出现（如 **AI分身**、**微创新**）
2. 工具/产品名称（如 **Claude Code**、**秘塔AI搜索**）
3. 动作指令（如 **复制下面的提示词**、**点击下载**）
4. 核心结论/金句
5. 章节名称（在正文中引用时）
6. ★ UI 元素名称（如 **设置**、**第三方插件**、**安全模式**）← v2.0 新增

### 标题写法规范（v2.0 新增）
- 大章节标题用**陈述句**："安装步骤""配置 API 密钥"
- 子步骤标题用**动词短语**："搜索 `bijitongbu`，找到 Biji Tongbu"
- 可以用**疑问句做引导标题**："不想每次手动点同步？开启自动同步"
- 可以在标题里**加括号注释**："方法二：手动下载安装（默认折叠 · 海外网络不佳时推荐）"

### 分隔符
- 用 `---` 分隔独立段落
- 章节间用空行 + 标题自然分隔
- 不使用表情符号做分隔符

### 列表
- 无序列表用于平级选项
- 有序列表用于步骤
- 表格用于对比、分类、工具汇总

## 六、内容设计原则

### 1. 价值前置
每个段落开头先说"为什么你要看这个"和"看完你能得到什么"。

### 2. 认知对比
"你以前是 XXX → 现在用这套方法变成 YYY"
用 before/after 制造认知落差，让读者觉得"不学就亏了"。

### 3. 具象化
每说完一个抽象概念，立刻跟一个**带场景的例子**：
- "💬 举个例子：你是大二市场营销专业的..."
- 例子必须包含：身份+场景+对话+结果

### 4. 防坑设计
每节末尾必须有 "⚠️ 防坑" 小节，写 1-3 条该步骤最容易翻车的点。

### 5. 信任建立
- 引用真实案例（有名有姓有学校）
- 承认局限（"AI 目前做不到..."）
- 披露灰色地带（"以下内容不是教你这么做，是让你知道这个世界长什么样"）

### 6. 情感设计
- 承认痛点："大部分人论文翻车不是写不好，是题目选错了"
- 正常化挫折："初稿被打回来是正常的，90%的论文初稿都会被要求修改"
- 建立动量："恭喜你，到这里你已经超过了90%的人"

### ★ 7. 错误优先（v2.0 新增 — 来自 bijitongbu）
**先展示错误，再给正确做法。** bijitongbu 的教程在每个易错步骤都会先画一个 ❌ 错误示范，再给 ✅ 正确做法。

为什么有效：
- 认知心理学的"负面偏差"——人对错误信息的注意力比正确信息高 30%
- "做错了长什么样"比"做对了长什么样"更容易记住
- 减少"我觉得我做对了其实没做对"的假阳性

### ★ 8. 概念前置（v2.0 新增 — 来自 bijitongbu）
**操作步骤之前，先用 Q&A 格式解释关键概念。** bijitongbu 在教程开头放"基础概念说明"，用"什么是 X？"的格式解释 3-5 个术语。

为什么有效：
- 新手看到陌生术语会卡住，卡住 = 流失
- Q&A 格式比段落式解释更易扫读
- 概念解释和操作步骤分开，不打断操作流

### ★ 9. 可折叠兜底（v2.0 新增 — 来自 bijitongbu）
**主流程保持简洁，复杂/边缘情况放进可折叠区域。** 用 `> **👉 点这里展开「xxx」**` 标记折叠区。

适用场景：
- 备选安装方法
- 兜底方案（主方法不生效时）
- 进阶配置
- 非主流环境的特殊处理

## 七、词汇选择

### 🟢 用这些
| 类别 | 推荐词 |
|------|--------|
| 称呼读者 | 你（不称"用户""读者"） |
| 动词 | 打开、复制、粘贴、输入、点击（不写"执行""操作"） |
| 量词 | 一篇教程、一套方法、一个工具 |
| 连接词 | 但、不过、所以、因为（不用"然而""因此"） |
| 程度词 | 极、非常、特别（不用"较为""相对"） |

### 🔴 避免这些
| 禁用 | 替代 |
|------|------|
| 赋能、深耕、抓手、闭环、对齐 | 直接说"能帮你做什么" |
| 综上所述、由此可见、毋庸置疑 | 删掉，直接说结论 |
| 值得注意的是、与此同时 | 删掉，直接说内容 |
| 可能、大概、应该、好像 | 给确定判断，不确定就说"取决于" |
| AI味儿套话 | 想象你在给学弟发微信 |

## 十四、小黎真实文风 DNA（v3.0 新增 — 从创作工作区逆向提取）

> 以下技法和风格特征来自小黎的真实朋友圈、社群分享、创作工作区的品牌档案。
> 写飞书教程时，这些特征必须体现——否则读者会觉得"这不是小黎写的"。

### 14.1 语气特征

| 特征 | 正确示例 | 错误示例 |
|------|---------|---------|
| kaomoji 穿插 | "说实话有点燃ฅ^•ﻌ•^ฅ" | "本次分享内容丰富" |
| #话题标签当小标题 | "#今天在实验室 #一个发现" | "一、实验室收获" |
| 暴露真实情绪 | "说实话有点燃🔥" | "本次活动圆满成功" |
| 短句有呼吸感 | "我试了。跑通了。真的好用。" | "经过反复测试，该工具确实能够有效提升工作效率" |
| 口语缓冲 | "说实话""怎么说呢""还真别说" | "众所周知""不可否认" |

### 14.2 内容结构特征

| 特征 | 说明 | 来源 |
|------|------|------|
| 先讲经历再提炼 | 不是上来就"你是否..."，而是先讲自己踩的坑 | 品牌档案 |

| 桥接句必须有 | 每段故事后加"这跟你的关系是……" | SOP-16 铁律7 |
| 自嘲稀释 | GPA3.9后加"但也天天熬夜赶工" | jc技法 |
| 认知对比 | "以前你：面完就算了 / 以后你：每次面试自动存档" | 行动手册DNA |

### 14.3 绝对禁忌（从品牌档案+教训档案提取）

**禁止出现的词**：赋能、深耕、抓手、闭环、对齐、拉齐、打通、链路、综上所述、由此可见、毋庸置疑、值得注意的是、与此同时

**禁止出现的句式**：
- "在当今社会""随着科技的发展"等 AI 流水线开头
- "首先/其次/最后/综上所述"等作文体
- 排比句堆砌（"无论A、B还是C"）
- 说教句式（"我想讲的是""我想告诉你"）
- "让我们一起""记得点赞"等升华式结尾
- 营销号模板（"你是否还在为XX焦虑？"）

**社群分享教训（写教程时也要遵守）**：
- ❌ 自嗨式输出：只讲自己的故事，没有桥接到"这对你有什么用"
- ❌ 消息拆太碎：信息轰炸不是娓娓道来
- ❌ 术语不解释：Harness Engineer、本地 Agent、配置包、SOP 等必须加括号解释
- ❌ 结尾潦草+红包引流：破坏"利他真诚"人设

### 14.4 可直接套用的写作公式

**公式1·坦诚+反转**："坦白说，[缺点]——但这恰恰说明[反转]"
**公式2·金句对比**："[A]是[负面]，[B]是[正面]"
**公式3·疲惫+温暖**："[真实状态]——但[温暖的一句话]"
**公式4·自问推进**："[陈述]——但[自问]？"
**公式5·数字锚定**：开头第一句用具体数字制造锚点
**公式6·故事三段式**："起点→转折→现在"天然叙事弧
