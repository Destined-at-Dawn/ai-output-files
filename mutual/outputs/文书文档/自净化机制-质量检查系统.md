# 自净化机制 — 质量检查系统 v2.0

> 每篇教程写完后必须跑完四级检查才能交付。
> 来源：锦成16篇逆向 + bijitongbu 23篇逆向 + 内容创作SOP最佳实践

---

## 四级检查流程

```
L1 格式检查 → L2 逻辑检查 → L3 风格检查 → L4 可读性检查
    ↓              ↓              ↓              ↓
  通过/修复      通过/修复      通过/修复      通过/修复
                                              ↓
                                         质量报告
```

## L1 格式检查（骨架完整性）

### 检查项

| # | 检查项 | 通过标准 | 权重 |
|---|--------|---------|------|
| 1.1 | 标题格式 | `行动手册 X · 动词+场景+结果` | 必须 |
| 1.2 | 价值声明 | 第一屏可见"做完能得到什么" | 必须 |
| 1.3 | 预计耗时 | 有，且拆分为子步骤耗时 | 必须 |
| 1.4 | 前置条件 | ✅ 列表，3-5项 | 必须 |
| 1.5 | 概念说明 | Q&A 格式，类比解释，≥2个术语 | 必须 |
| 1.6 | 步骤结构 | 每步有 💡概要 + 操作 + ⚠️防坑 | 必须 |
| 1.7 | FAQ | 有"遇到问题怎么办"章节 | 必须 |
| 1.8 | 下一步 | 有多路径分流 | 必须 |
| 1.9 | 金句 | 有"一句话记住"总结 | 必须 |
| 1.10 | 视觉标记 | 使用了 💡/⚠️/💬/❌/✅ 中 ≥3 种 | 必须 |

### 修复方法
缺失哪个就补哪个，不重写整篇。

## L2 逻辑检查（步骤衔接）

### 检查项

| # | 检查项 | 通过标准 | 权重 |
|---|--------|---------|------|
| 2.1 | 概念→操作顺序 | 概念说明在操作步骤之前 | 必须 |
| 2.2 | 步骤衔接 | 每步末尾有"怎么接上下一步" | 重要 |
| 2.3 | 产出可验证 | 每步有"做完手上有什么"，且可具体验证 | 必须 |
| 2.4 | 错误示范 | 每个易错步骤有 ❌ 标记 | 重要 |
| 2.5 | 成功确认 | 每步有 ✅ "你应该看到什么" | 重要 |
| 2.6 | 逻辑闭环 | 从"为什么"到"怎么做"到"做完了"形成闭环 | 必须 |
| 2.7 | 素材覆盖 | 输入素材中的每个操作都有对应步骤 | 必须 |

### 修复方法
- 缺少衔接 → 在步骤末尾补"怎么接上下一步"
- 缺少错误示范 → 逆向思考"做错了会怎样"，补 ❌ 段
- 逻辑跳跃 → 补充中间过渡步骤

## L3 风格检查（一致性）

### 检查项

| # | 检查项 | 通过标准 | 权重 |
|---|--------|---------|------|
| 3.1 | 语气一致 | 全篇像同一个人写的，学长口吻 | 必须 |
| 3.2 | 加粗覆盖 | 关键概念+工具名+UI元素+动作指令全部加粗 | 必须 |
| 3.3 | 禁词检测 | 无赋能/闭环/综上所述/值得注意的是等 | 必须 |
| 3.4 | 比喻密度 | 每大节 ≥1 个生活化比喻 | 重要 |
| 3.5 | 例子密度 | 每步 ≥1 个 💬 例子 | 重要 |
| 3.6 | 防坑密度 | 全篇 ≥3 条 ⚠️ 防坑 | 必须 |
| 3.7 | 表格用法 | 表格只用于对比/分类，不堆数据 | 重要 |
| 3.8 | 称呼一致 | 全篇用"你"，不用"用户""读者" | 必须 |

### 禁词扫描列表
```
赋能、深耕、抓手、闭环、对齐、拉齐、打通、链路
综上所述、由此可见、毋庸置疑、值得注意的是、与此同时
可能、大概、应该、好像、基本上、差不多
然而、因此、故而、是以
用户、读者（应改为"你"）
```

### 修复方法
- 禁词 → 直接替换（见风格手册词汇选择章）
- 语气不一致 → 统一为学长口吻重写相关段落
- 缺少比喻/例子 → 补充

## L4 可读性检查（读者体验）

### 检查项

| # | 检查项 | 通过标准 | 权重 |
|---|--------|---------|------|
| 4.1 | 信息密度 | 每段 ≤ 4 行，无大段纯文字 | 重要 |
| 4.2 | 扫读友好 | 30秒扫读能看到所有关键操作步骤 | 重要 |
| 4.3 | 段落长度 | 单段 ≤ 150 字 | 建议 |
| 4.4 | 移动端适配 | 表格 ≤ 5列，代码块 ≤ 60字符宽 | 建议 |
| 4.5 | 视觉层次 | 标题/加粗/标记形成清晰的视觉层次 | 重要 |
| 4.6 | 折叠使用 | 复杂/边缘情况用 👉 折叠区，主流程简洁 | 建议 |

### 修复方法
- 信息过密 → 拆段，加标记
- 表格过宽 → 精简列数或改为列表
- 扫读不友好 → 加粗关键词 + 加标记

## 质量评分

### 评分规则

| 级别 | 权重 | 满分 |
|------|------|------|
| L1 格式 | 25% | 25分 |
| L2 逻辑 | 30% | 30分 |
| L3 风格 | 25% | 25分 |
| L4 可读性 | 20% | 20分 |
| **总分** | **100%** | **100分** |

### 评级

| 评级 | 分数 | 含义 |
|------|------|------|
| ⭐⭐⭐⭐⭐ | 90-100 | 可直接发布 |
| ⭐⭐⭐⭐ | 80-89 | 小修后发布 |
| ⭐⭐⭐ | 70-79 | 需要修订 |
| ⭐⭐ | 60-69 | 需要重写部分内容 |
| ⭐ | < 60 | 需要重写 |

### 一票否决项
以下任何一项不通过 = 总分不超过 60：
- 标题格式错误
- 无价值声明
- 无操作步骤
- 有禁词
- 语气不一致

## 反馈闭环

### 用户反馈记录格式

```markdown
## [日期] [教程标题]
- 质量评分：[分数]
- 用户反馈：[原文]
- 需要调整：[具体调整项]
- 已应用到：[风格手册/SOP/记忆文档]
```

### 反馈处理规则

| 反馈类型 | 处理方式 |
|---------|---------|
| 格式问题 | 更新 L1 检查清单 |
| 内容问题 | 更新 L2 检查清单 |
| 风格偏好 | 记录到记忆文档 + 更新风格手册 |
| 新元素需求 | 评估 → 纳入风格手册 |
| 重复反馈 ≥2次 | 升级为 SOP 规则 |

## 月度优化

### 每月最后一天执行

1. **质量趋势回顾**
   - 本月教程平均质量评分
   - 高频扣分项 Top 3
   - 与上月对比

2. **风格手册更新**
   - 有无新的风格源值得纳入？
   - 有无需要删除的过时规则？

3. **记忆文档清理**
   - 过期偏好标记 `[需刷新]`
   - 低置信记录评估

4. **效率优化**
   - 模板是否需要更新？
   - 快速参考卡是否准确？

## Skill 调用质量检查（v3.0 新增）

### Skill 调用前检查

| # | 检查项 | 通过标准 | 权重 |
|---|--------|---------|------|
| S1 | 输入完整性 | 素材包含足够的操作细节（截图描述 + 文字说明） | 必须 |
| S2 | 内容类型判断 | 正确判断为认知/实操/参考类型 | 必须 |
| S3 | Skill 选择 | 选择了最优的 skill 组合 | 重要 |
| S4 | 提示词质量 | 使用了优化后的提示词（prompt-optimizer） | 建议 |

### Skill 调用后检查

| # | 检查项 | 通过标准 | 权重 |
|---|--------|---------|------|
| S5 | 风格一致性 | 输出符合风格手册 v2.0 规范 | 必须 |
| S6 | AI 味检测 | 通过 chinese-natural-voice-revision 检测 | 必须 |
| S7 | 传播力评估 | 标题/开头具有传播力（如需要） | 建议 |
| S8 | 多格式适配 | 支持飞书/公众号/小红书等多平台（如需要） | 建议 |

### Skill 调用质量评分

| 级别 | 权重 | 满分 |
|------|------|------|
| 调用前检查 | 40% | 40分 |
| 调用后检查 | 60% | 60分 |
| **总分** | **100%** | **100分** |

### Skill 调用失败处理

| 失败类型 | 处理方式 |
|---------|---------|
| Skill 不存在 | 检查安装状态，尝试安装或使用替代 skill |
| Skill 调用超时 | 重试 1 次，失败则降级到原生能力 |
| 输出质量不达标 | 用 chinese-natural-voice-revision 修复，或重新生成 |
| 风格不一致 | 用风格手册 v2.0 重新应用风格 |
