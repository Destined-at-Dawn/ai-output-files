## 一、系统概述

### 这是什么
一套完整的飞书教程写作工作流——从素材输入到成品输出，包含风格规范、质量自检、记忆沉淀、效率工具。

### 为谁服务
小黎（及同辈互助计划成员），用于批量产出高质量飞书教程。

### 核心目标
**输入：** 图片 + 文字素材
**输出：** 逻辑严谨、结构清晰、有加粗有风格的飞书教程

## 三、完整工作流（6 步）

```
┌──────────┐   ┌──────────┐   ┌──────────┐
│ 1. 输入  │ → │ 2. 结构  │ → │ 3. 风格  │
│  收集    │   │  搭建    │   │  填充    │
└──────────┘   └──────────┘   └──────────┘
                                    │
┌──────────┐   ┌──────────┐   ┌──────────┐
│ 6. 归档  │ ← │ 5. 交付  │ ← │ 4. 自检  │
│  沉淀    │   │  发布    │   │  修复    │
└──────────┘   └──────────┘   └──────────┘
```

### Step 1：输入收集

**做什么：** 把用户给的素材整理成结构化输入

**输入格式：**
```
【标题】行动手册 X · [具体动作]
【核心概念/痛点】[2-3句话]
【操作素材】[图片描述 + 操作文字]
【注意事项】[需要提醒的点]
【目标读者】[新手/进阶/所有人]
【术语清单】[需要解释的术语，逗号分隔]
```

**检查清单：**
- [ ] 素材包含足够的操作细节（截图描述 + 文字说明）？
- [ ] 目标读者明确？
- [ ] 有明确的"做完能得到什么"？

**产出：** 结构化输入表

---

### Step 2：结构搭建

**做什么：** 按 v2.0 骨架搭出教程框架

**必须包含的结构块：**
1. 标题（行动手册 X · 动词+场景+结果）
2. 价值声明 + 预计耗时（拆分）
3. 开始前的准备（✅ 前置条件）
4. 基础概念说明（Q&A 格式，类比解释术语）
5. 核心概念（认知对比 + 一句话记住）
6. 分步操作（每步 = 💡概要 + 操作 + 💬例子 + ❌错误 + ✅确认 + ⚠️防坑）
7. 遇到问题怎么办（Q&A + 排查链接）
8. 进阶配置（链接矩阵）
9. 工具与资源（对比表格）
10. 下一步（多路径分流）
11. 一句话记住（全篇总结）
12. 技术支持 + 安全说明（如需要）

**产出：** 教程骨架（标题 + 各节标题 + 关键内容占位）

---

### Step 3：风格填充

**做什么：** 按风格手册填充内容

**填充顺序：**
1. 先写价值声明和概念对比（吸引读者继续看）
2. 再写操作步骤（核心价值）
3. 补充 ❌ 错误示范 + ✅ 成功确认
4. 补充 💬 例子和 ⚠️ 防坑
5. 写 Q&A 和进阶链接
6. 最后写下一步分流

**风格要点：**
- 语气：学长发微信，不是产品手册
- 加粗：关键概念 + 工具名 + UI元素 + 动作指令
- 标记：💡/⚠️/💬/❌/✅/👉 六种标记覆盖
- 禁忌：无AI套话（赋能/闭环/综上所述）

**产出：** 完整教程初稿

---

### Step 4：自检修复

**做什么：** 跑四级质量检查

详见 `自净化机制-质量检查系统.md`

**四级检查：**
1. **格式检查** — 骨架完整性 + 标记规范
2. **逻辑检查** — 步骤衔接 + 产出可验证
3. **风格检查** — 语气一致 + 加粗规范 + 禁词检测
4. **可读性检查** — 信息密度 + 扫读友好 + 移动端适配

**产出：** 质量报告 + 修复后的教程

---

### Step 5：交付发布

**做什么：** 输出最终版本并发布到飞书

**发布方式（二选一）：**

**方式A：自动发布（推荐）**
```
python skills/feishu-doc-writer/scripts/write_feishu_doc.py create \
  --title "教程标题" \
  --from-md outputs/tutorial.md
```
自动创建飞书文档 + 写入全部内容，返回文档 URL。

**方式B：手动粘贴**
- 复制 Markdown 内容 → 粘贴到飞书文档
- 适用于需要在飞书内手动微调格式的场景

**飞书发布注意事项：**
- 自动发布支持：标题(H1-H6)、段落、列表、代码块、引用、分割线、加粗/斜体/链接
- 表格目前以文本形式写入，需要手动在飞书内调整
- 图片需要先上传到飞书获取 token 后才能写入

**产出：** 飞书文档 URL + 质量报告

---

### Step 6：归档沉淀

**做什么：** 记录本次写作数据到记忆文档

**记录内容：**
- 本次教程的标题、字数、质量评分
- 使用了哪些新增元素
- 用户反馈（如有）
- 发现的新风格特征（如有）

详见 `记忆文档-用户偏好与历史.md`

**产出：** 更新后的记忆文档

## 四、自净化机制概述

### 四级检查

| 级别 | 检查内容 | 工具 |
|------|---------|------|
| L1 格式 | 骨架完整性、标记规范、加粗覆盖率 | 检查清单 |
| L2 逻辑 | 步骤衔接、产出可验证、概念→操作顺序 | 逻辑审查 |
| L3 风格 | 语气一致、禁词检测、比喻/例子密度 | 风格比对 |
| L4 可读性 | 信息密度、段落长度、移动端友好 | 可读性评分 |

### 反馈闭环

```
用户反馈 → 记录到记忆文档 → 更新风格手册 → 下次写作自动应用
```

### 月度优化

每月最后一天：
1. 回顾本月所有教程的质量评分趋势
2. 统计高频扣分项 → 更新检查清单
3. 检查是否有新的风格源值得纳入
4. 清理过期的偏好记录

## 七、使用方式

### 快速版（3步）
1. 读 `快速参考卡.md`（30秒）
2. 把素材按输入格式整理好
3. 把 `写作系统提示词-v2.0.md` + 素材一起发给 AI

### 完整版（6步）
按上面的 Step 1-6 完整执行

### 批量版（多篇教程）
1. 先读 `风格手册-v2.0.md` 建立风格认知
2. 批量整理所有素材
3. 逐篇按 Step 1-6 执行
4. 最后统一做月度质量回顾
