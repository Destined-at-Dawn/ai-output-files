## 二、Skill 分层架构

### 第一层：输入处理（读取 + 理解）

| Skill | 路由 ID | 功能 | 调用时机 |
|-------|---------|------|---------|
| **feishu-doc-reader** | r048 | 读取飞书文档原文（docx/doc/sheet/slide） | 用户给飞书链接时 |
| **transcript-cleaner** | r053 | 逐字稿清洗 + 道法术器拆解 | 用户给录音/逐字稿时 |

**调用逻辑：**
```
用户给飞书链接 → feishu-doc-reader 读取 → 结构化 JSON
用户给录音/逐字稿 → transcript-cleaner 清洗 → 结构化内容
用户给图片+文字 → 直接进入第二层
```

---

### 第二层：内容生成（核心创作）

| Skill | 路由 ID | 功能 | 调用时机 |
|-------|---------|------|---------|
| **jc-jiaocheng** | r050 | 写「读完就能动」的教程（9 种公式） | **主 skill，每次必调** |
| **doc-coauthoring** | r049 | 三阶段共创（收集→迭代→读者测试） | 复杂文档/需要深度共创时 |
| **blog-post-writer** | r009 | 公众号文章风格（800-1200 字） | 需要公众号同步发布时 |
| **writing-plans** | r051 | 工程实现计划（精确到文件路径） | 教程涉及代码/工程操作时 |

**调用逻辑：**
```
素材输入 → jc-jiaocheng（主引擎）
  ├── 简单教程 → 直接用 jc-jiaocheng
  ├── 复杂教程 → jc-jiaocheng + doc-coauthoring（共创模式）
  ├── 含代码 → jc-jiaocheng + writing-plans（工程模式）
  └── 需公众号 → jc-jiaocheng + blog-post-writer（双发布）
```

## 三、文风 DNA 集成

### DNA 来源（双源融合）

| 来源 | 文档数 | 核心贡献 |
|------|--------|---------|
| **锦成飞书知识库** | 16 篇 | 行动手册结构、认知对比、工具矩阵、路径分流 |
| **bijitongbu.site** | 23 篇 | 前置条件、Q&A 概念、错误优先、成功确认、可折叠兜底 |

### DNA 核心要素（已编码到风格手册 v2.0）

1. **人设**：过来人学长（不是老师，是踩过坑的人）
2. **语气**：直接、不讨好、有判断力（像群里跟学弟学妹聊天）
3. **结构**：价值声明 → 认知对比 → 分步操作 → 工具矩阵 → 路径分流
4. **标记**：💡/⚠️/💬/❌/✅/👉 六种视觉标记
5. **公式**：9 种表达公式（概念三层、新旧世界、工具安装五步等）
6. **禁忌**：无 AI 套话（赋能/闭环/综上所述/值得注意的是）

### DNA 自动注入流程

```
素材输入 → jc-jiaocheng 生成初稿
  → 风格手册 v2.0 自动应用
  → chinese-natural-voice-revision 降 AI 味
  → 最终输出符合 DNA 的教程
```

## 八、质量保障机制

### 四级自检（每次生成后必跑）

| 级别 | 检查内容 | 工具 | 通过标准 |
|------|---------|------|---------|
| L1 格式 | 骨架完整性、标记规范、加粗覆盖率 | 检查清单 | 100% 骨架完整 |
| L2 逻辑 | 步骤衔接、产出可验证、概念→操作顺序 | 逻辑审查 | 无断链 |
| L3 风格 | 语气一致、禁词检测、比喻/例子密度 | 风格比对 | 禁词 0 出现 |
| L4 可读性 | 信息密度、段落长度、移动端友好 | 可读性评分 | ≥ 80 分 |

### 反馈闭环

```
用户反馈 → 记录到记忆文档 → 更新风格手册 → 下次写作自动应用
```

### 月度优化

每月最后一天：
1. 回顾本月所有教程的质量评分趋势
2. 统计高频扣分项 → 更新检查清单
3. 检查是否有新的风格源值得纳入
4. 清理过期的偏好记录
