# Kami × 小黎工作流融合分析

> 日期: 2026-05-22 | 类型: 工具集成 | 优先级: 高

## 结论

Kami 是一个**约束驱动的文档设计系统**，解决"AI生成内容排版散乱"的核心痛点。它不是UI框架，而是一套**印刷级文档规范**——暖色羊皮纸底色、墨蓝强调色、衬线体层级、9种文档模板、14种图表基元。已成功安装并配置了你的专属品牌档案。

**核心价值**: 你的四大工作区（竞赛/求职/个人/创作）产出的文档将从"纯Markdown散乱输出"升级为"统一设计语言的专业PDF/PPT"。

---

## 一、Kami 解决了什么问题

### 问题诊断

你的工作流中有**5类高频输出文档**，目前的问题各不相同:

| 文档类型 | 当前状态 | 痛点 |
|---------|---------|------|
| 竞赛PPT/方案 | 手动排版，风格不统一 | 每次比赛都从零开始，视觉不专业 |
| 简历 | Markdown转PDF，排版简陋 | 学历劣势下，视觉专业度是加分项 |
| 小红书图文 | 截图+文字，无设计感 | 信息密度高但排版杂乱，影响传播 |
| 科研日报/长文档 | 纯文本，无可视化 | 数据堆砌无结构，难以快速提取 |
| 项目方案/one-pager | 无模板，靠感觉 | 竞赛评审/导师审阅时缺乏专业感 |

### Kami 的解法

Kami 的设计哲学: **约束即自由**。10条不变量（详见下文）锁死设计空间，让你不用做设计决策，内容自然好看。

## 二、Kami 核心能力拆解

### 2.1 设计系统 — 10条不变量

| # | 不变量 | 为什么重要 |
|---|--------|-----------|
| 1 | 页面背景 `#f5f4ed` 暖羊皮纸，**永远不用纯白** | 视觉温度，降低屏幕阅读疲劳 |
| 2 | 唯一强调色: 墨蓝 `#1B365D`，**没有第二种彩色** | 单色系统=零设计决策成本 |
| 3 | 所有灰色必须暖色调（黄棕底），**禁止冷灰** | 保持视觉一致性 |
| 4 | 英文: 衬线体全部; 中文: 标题衬线+正文无衬线 | 层级清晰 |
| 5 | 衬线字重锁定500，**不用bold** | 克制=高级感 |
| 6 | 行高: 标题1.1-1.3，正文1.4-1.55 | 印刷级节奏 |
| 7 | 字间距: 中文正文0.3pt，英文正文0 | CJK呼吸空间 |
| 8 | Tag背景必须纯色hex，**不用rgba** | WeasyPrint渲染兼容 |
| 9 | 深度用ring shadow或whisper shadow，**不用硬阴影** | 纸张感 |
| 10 | 印刷模板**不用斜体** | 中文排版规范 |

## 三、工作流融合策略

### 3.1 竞赛工作区（最大受益者）

**场景**: iCAN RoboRacer + 皮影戏数字化

| 原流程 | 新流程 (Kami) |
|--------|--------------|
| 手动做PPT，风格每次不同 | `slides` 模板 + 品牌档案自动填充 |
| 方案文档纯文本 | `one-pager` 模板，墨蓝强调+架构图 |
| 作品集没有统一风格 | `portfolio` 模板，4:3图片裁切+成果bullets |

**具体SOP更新**:
1. 竞赛答辩PPT → 触发词"做PPT/答辩slides" → Kami slides-weasy模板
2. 项目方案 → 触发词"方案/one-pager" → Kami one-pager模板
3. 架构图 → 触发词"架构图" → Kami architecture.html图表基元
4. 流程图 → 触发词"流程/决策流" → Kami flowchart.html图表基元

### 3.2 求职工作区（简历是杀手锏）

**场景**: 实习投递 + 考研复试

| 原流程 | 新流程 (Kami) |
|--------|--------------|
| Markdown简历转PDF，排版简陋 | `resume` 模板 + resume-writing.md写作规范 |
| 每次手动调格式 | 品牌档案自动填充姓名/学校/GPA/联系方式 |

**Kami 简历写作规范（resume-writing.md）**:
- 每个bullet: Action + Scope + Result + Business Outcome
- 量化一切: "GPA 3.896/4.0 (Top 5%)" > "成绩优异"
- 技术栈用tag组件，不用纯文字罗列

### 3.3 个人工作区（长文档+决策报告）

**场景**: 百大认知 × 数字先知 + 考研规划

| 原流程 | 新流程 (Kami) |
|--------|--------------|
| 数字先知决策报告纯文本 | `equity-report` 模板 + candlestick/line图表 |
| 考研规划Markdown | `long-doc` 模板 + timeline里程碑 |
| 项目文档无设计 | `changelog` 模板记录版本迭代 |

### 3.4 创作工作区（小红书+开源）

**场景**: 小红书内容 + 开源项目运营

| 原流程 | 新流程 (Kami) |
|--------|--------------|
| 小红书图文无设计感 | `one-pager` 模板导出PNG → 发布 |
| 开源README无视觉 | `landing-page` 模板做项目官网 |
| 版本更新无记录 | `changelog` 模板生成版本日志 |

## 五、WeasyPrint 环境检查

Kami 依赖 WeasyPrint 渲染PDF。需要确认环境:

```bash
# 检查 WeasyPrint 是否已安装
weasyprint --version

# 如未安装:
pip install weasyprint

# 字体确保（中文文档需要）
bash ~/.newmax/skills/kami/scripts/ensure-fonts.sh
```

**注意**: WeasyPrint 在 Windows 上需要 GTK3 运行时。如果遇到渲染问题，fallback 到 PPTX 输出（slides.py）。
