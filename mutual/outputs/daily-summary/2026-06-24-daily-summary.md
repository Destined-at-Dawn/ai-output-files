## 关键收获与洞察

### 1. 用户体验敏感度极高

用户对色彩饱和度、视觉"高级感"、交互灵动性有明确预期。AI 推荐降低饱和度的方案被直接否决——"时刻保持高饱和，给人阳光向上的感觉"。

### 2. "先搜索再推荐"是铁律

用户明确表示：footer 方案不能在已有信息里重新组装，必须先搜索真实优秀案例（国内外工程师个人网站），带案例回来让用户选。这是 think-before-act 规则的具体场景化。

### 3. 填充语是信任杀手

用户对"你说得对""我之前的方案是闭门造车"等 AI 自我批评式填充语极度反感。要求：直接去做，不要铺垫。已写入全局禁令。

### 4. 独立 EXE 安装的更新陷阱

Claude Code 和 Codex CLI 都是独立 EXE 而非 npm 包。npm install -g 更新的只是另一份副本，不会影响 PATH 优先级更高的独立安装版本。更新必须走 GitHub Releases 下载替换。

### 5. Skill 生命周期管理需要更强的追踪

li-content 被合并到 li-analyze 后，引用它的路由规则和 SOP 没有同步清理，导致用户查找时发现"skill 消失了"。skill 合并/弃用后的路由清理需要自动化。
