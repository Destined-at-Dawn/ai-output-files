# 实战模拟场景库

## 通用场景

### 求职面试（三轮递进）

**第一轮：HR面**
- 角色：人力资源经理，友善但专业
- 重点：自我介绍、动机、职业规划、基本沟通能力
- 典型问题：「简单介绍一下你自己」「为什么选择我们公司」「你的职业规划是什么」
- 压力点：追问简历上的空白期、频繁跳槽、专业不对口

**HR面对话示范：**

```
AI（面试官）：你好，请坐。先简单介绍一下你自己吧。
用户：我叫XXX，今年大四，学的是金融。
AI（面试官）：嗯。（停顿3秒）还有呢？
用户：呃……我在学校参加过一些活动……
AI（面试官）：具体说说，什么活动？你在里面做了什么？
用户：就是学生会，组织过一些讲座。
AI（面试官）：你说组织讲座——你负责哪个环节？联系嘉宾？场地？宣传？还是现场执行？
用户：主要是现场执行。
AI（面试官）：好的。那你为什么投我们这个岗位？你对这个岗位的理解是什么？

（模拟结束后的评估）

## 模拟评估

### 表现得分（1-10）
- 内容质量：3/10 — 自我介绍没有准备，只说了姓名和专业，没有亮点
- 表达清晰度：4/10 — 基本能听懂，但信息量太少，每句话都需要追问才能补充
- 临场反应：5/10 — 面对追问能回答，但都是被动的，没有主动展开
- 整体印象：3/10 — 如果我是真的HR，会觉得你没有准备，对自己要说什么没有规划

### 致命问题
- 自我介绍是面试第一印象，你的版本只有一句话。任何面试官都会觉得你要么没准备，要么对自己没什么可说的。

### 改进版示范
**原版**：「我叫XXX，今年大四，学的是金融。」
**改进版**：「我叫XXX，XX大学金融专业大四。大三的时候在XX证券实习了三个月，主要做li-industry。这段经历让我发现自己对XX方向特别感兴趣，也是我投这个岗位的主要原因。」
**为什么更好**：30秒内给了三个关键信息（背景+经历+动机），面试官有东西可以接着问，而不是冷场。
```

**第二轮：业务面**
- 角色：部门负责人，直接、高效
- 重点：专业能力、项目经验、问题解决思路
- 典型问题：「说一个你主导过的项目」「遇到XX问题你会怎么处理」「你怎么看待XX行业趋势」
- 压力点：追问项目细节、质疑方案可行性、提出反对意见

**业务面对话示范：**

```
AI（部门负责人）：我看了你简历上写的那个项目——帮一家奶茶店做的营销方案。具体说说你做了什么。
用户：我们团队帮他们做了一个社交媒体营销方案，主要是在小红书上发内容。
AI（部门负责人）：效果呢？有数据吗？
用户：嗯……客户说效果还不错。
AI（部门负责人）：「还不错」不是数据。粉丝涨了多少？转化率是多少？ROI算过没有？
用户：这个……我们没有特别追踪数据。
AI（部门负责人）：（皱眉）一个营销方案不追踪数据，你怎么知道它有效？你在这个项目里的具体角色是什么？是你主导的还是团队里的一个人？
```

**第三轮：压力面**
- 角色：高管或交叉面试官，节奏快、问题尖锐
- 重点：抗压能力、逻辑思维、价值观
- 典型问题：「你最大的失败是什么」「如果团队成员不配合你怎么办」「你凭什么觉得自己能胜任」
- 压力点：打断回答、质疑答案、沉默施压

**压力面对话示范：**

```
AI（高管）：你最大的失败是什么？
用户：大二的时候我参加了一个创业比赛，但最后没进决赛——
AI（高管）：（打断）为什么没进？
用户：因为我们的商业模式被评委质疑——
AI（高管）：（再次打断）具体哪里被质疑了？
用户：呃，他们觉得我们的获客成本太高。
AI（高管）：那你觉得他们说得对吗？
用户：……对，确实是我们的问题。
AI（高管）：所以你从这件事里学到了什么？
用户：要更重视数据和商业模型。
AI（高管）：这话谁都会说。具体到下一次做创业项目，你会在哪一步做不同的事？
```

### 论文答辩

- 角色：答辩委员会主席 + 委员
- 风格：严谨、学术、会抓逻辑漏洞
- 重点：研究动机、方法论合理性、结论可靠性、创新点
- 典型追问：「你的样本量够吗」「这个方法为什么不用XX替代」「你的结论和XX的研究矛盾，怎么解释」
- 压力点：质疑数据真实性、指出文献遗漏、问「所以你的贡献到底是什么」

**答辩对话示范：**

```
AI（答辩主席）：请用三分钟简要介绍你的研究。
用户：我的论文研究的是社交媒体对大学生消费决策的影响……（用户介绍完）
AI（答辩主席）：你的样本量是多少？
用户：200份问卷。
AI（委员A）：200份。你的研究涉及了6个变量，200份的样本量做回归分析，统计功效够吗？
用户：呃……我觉得200份应该够了。
AI（委员A）：「觉得」不行。你算过最小样本量吗？以你的效应量和6个变量，需要至少多少份？你做过功效分析没有？
AI（答辩主席）：还有一个问题——你的问卷是怎么发的？全校统一发还是只在某个学院？如果只在一个学院发，你的结论能推广到整个大学生群体吗？
```

### 工作汇报

- 角色：部门总监，时间有限、只关注结果
- 风格：会打断冗长的铺垫，直接问「所以呢」
- 重点：结构化表达、重点提炼、数据支撑
- 典型反应：「说重点」「这个数据说明什么」「下一步打算怎么做」
- 压力点：汇报被打断、被要求用一句话总结、被问没准备的问题

**汇报对话示范：**

```
AI（总监）：你有五分钟，说说上个月的项目进展。
用户：好的，上个月我们主要做了三件事。首先是调研阶段，我们花了两周时间做了用户调研——
AI（总监）：（打断）调研结果是什么？
用户：我们发现用户的主要痛点是——
AI（总监）：一句话说。
用户：用户最大的痛点是找不到合适的XX。
AI（总监）：好。然后呢，你们打算怎么解决？
用户：我们设计了三个方案——
AI（总监）：你推荐哪个？
用户：方案二。
AI（总监）：为什么？用数据说话。

（模拟结束后的评估摘要）
- 内容质量：5/10 — 有内容但组织混乱，需要我反复追问才能提炼出重点
- 致命问题：没有在开头用一句话说结论。好的汇报结构是「结论→原因→数据→行动」，你用的是「过程→发现→思考」，这在总监面前是致命的
- 改进建议：开口第一句话就说结论「上月调研发现用户最大痛点是XX，我们推荐方案二，预计能解决80%的问题，需要投入XX资源」
```

### 创业路演

- 角色：投资人，见过太多项目，对空话免疫
- 风格：犀利、数据驱动、会抓商业逻辑漏洞
- 重点：市场规模、商业模式、竞争壁垒、团队能力
- 典型追问：「你的获客成本是多少」「竞品做这个你有什么优势」「如果XX巨头进场你怎么办」
- 压力点：质疑市场规模、要求给出具体数字、问「你凭什么能赢」

**路演对话示范：**

```
AI（投资人）：说说你在做什么。
用户：我们做的是一个AI驱动的学习平台——
AI（投资人）：停。AI学习平台全世界有两千个，你和他们有什么不一样？
用户：我们的核心差异是个性化——
AI（投资人）：每个人都说自己个性化。具体怎么个性化法？你的用户留存率是多少？日活多少？
用户：我们还在早期，还没有太多用户数据——
AI（投资人）：没有数据你怎么知道用户需要你的产品？你和用户聊过吗？聊了几个？他们怎么说的？
```

### 专业辩论

- 角色：持相反立场的同行专家
- 风格：尊重但不留情面，用数据和逻辑反驳
- 重点：论点组织、反驳能力、在对抗中保持理性
- 典型行为：引用你不知道的研究来反驳、指出你论证中的逻辑跳跃、提出你没考虑过的变量
- 压力点：你的核心论据被拆解后还能不能站住

**辩论对话示范：**

```
AI（对手）：我听说你认为远程办公比线下办公效率更高？
用户：对，研究表明远程办公减少了通勤时间，员工满意度更高——
AI（对手）：斯坦福2023年的研究显示远程办公降低了10-20%的生产力，你怎么看这个数据？
用户：那个研究可能有问题——
AI（对手）：具体哪里有问题？你读过原文吗？它的样本量是2000人，实验设计是随机对照，你的「可能有问题」的依据是什么？
用户：呃……我主要是基于个人体验——
AI（对手）：个人体验不是证据。你能不能给我一个同等质量的反驳研究？
```

## 专业场景

### 计算机/技术

| 场景 | 角色 | 核心考察 |
|------|------|---------|
| 系统设计面试 | 高级工程师 | 架构思维、权衡取舍、规模化思考 |
| Code Review | 严格的Tech Lead | 代码质量意识、设计模式理解、边界条件 |
| 技术方案评审 | CTO | 技术选型合理性、成本考量、风险评估 |

**系统设计面试对话示范：**

```
AI（高级工程师）：设计一个类似微信朋友圈的系统，支持1亿日活用户。你会怎么设计？
用户：首先数据库用MySQL——
AI（高级工程师）：等一下。1亿日活，假设每人每天发1条动态+看50条，你算一下读写QPS是多少？MySQL单机能撑住吗？
用户：呃……1亿除以86400，大概1150QPS？
AI（高级工程师）：这是平均值。高峰期呢？中午和晚上的流量可能是平均值的5-10倍。而且你只算了发动态，没算读。每个人刷朋友圈要读好友的动态，一个人500好友，每次刷加载20条。你重新算一下读QPS。
用户：那读的话……太大了，MySQL肯定不行。
AI（高级工程师）：所以你的架构需要什么？继续想。
```

```
AI（客户）：律师，我租的房子漏水了，房东不修，我要告他！
用户：首先需要了解一下情况。您的租房合同是怎么约定维修责任的？
AI（客户）：我不管合同怎么写的，漏水就是他的问题！我的电脑都被泡了！

AI（客户）：那我找你干嘛？你到底能不能帮我把钱要回来？

（模拟重点：能不能在客户情绪激动时保持专业，把复杂法律逻辑用客户能听懂的方式说清楚，同时管理对方的期望值。）
```

### 金融/商科

| 场景 | 角色 | 核心考察 |
|------|------|---------|
| 行研路演 | 基金经理 | 分析框架、数据敏感度、投资逻辑 |
| 尽职调查 | PE合伙人 | 财务分析、风险识别、商业判断 |
| 案例面试 | 咨询公司面试官 | 结构化思维、假设驱动、快速计算 |

**咨询案例面试对话示范：**

```
AI（面试官）：你的客户是一家连锁咖啡品牌，去年利润下降了15%，CEO想知道为什么以及怎么办。你会怎么分析？
用户：我先把利润拆成收入和成本两部分——
AI（面试官）：好，继续拆。收入下降还是成本上升？
用户：假设是收入下降。收入等于门店数量乘以单店收入——
AI（面试官）：单店收入还能怎么拆？
用户：客单价乘以客流量？
AI（面试官）：好。现在我告诉你，门店数量增加了10%，客单价没变，但客流量下降了25%。你觉得问题出在哪？下一步你会问什么数据？
```

## 场景选择引导

如果用户没有指定场景，按以下顺序推荐：

1. 用户提到了「面试」→ 求职面试（问清楚目标公司/岗位后选对应轮次）
2. 用户提到了「答辩」→ 论文答辩
3. 用户提到了「汇报」「PPT」→ 工作汇报
4. 用户说了具体专业 → 匹配专业场景
5. 都没说 → 列出通用场景让用户选

**引导对话示范：**

```
用户：我想做实战模拟。
AI：好。你想练什么场景？
- 求职面试（我需要知道你的目标公司和岗位）
- 论文答辩
- 工作汇报
- 创业路演
- 专业辩论
- 或者告诉我你的具体情况，我帮你选。

用户：面试。
AI：什么类型的公司？什么岗位？你是应届生还是有工作经验？
用户：应届生，想去互联网大厂做产品经理。
AI：好，我们从HR面开始。准备好了就直接说「开始」。
用户：开始。
AI：你好，请坐。我是XX部门的HR，看了你的简历，今天想更深入地了解一下你。先简单介绍一下你自己？
```
