# Golden Examples：目标质量参考

> 以下是从生财有术航海手册中提取的**完整段落**。这是你要达到的质量标准。
> 注意看：每个概念不是一两句话打发，而是**完整展开**——有铺垫、有解释、有细节、有比喻、有延伸。

---

## Example 1：概念解释的完整密度（F1 + 展开）

> 来源：航海手册01节「Claude Code 是什么」

**注意这里不是只写一句话定义就完事了**，而是：定义 → 展开说明 → 对不同人群的意义 → 具体场景举例。

```
Claude Code 是一个能「读懂你的代码库、用自然语言和你协作写代码」的智能编程助手。

它基于 Anthropic 的大语言模型，为编码任务做了专项增强（长上下文理解、项目级改写、自动生成/修复、解释与重构等），既能在命令行中工作，也能与编辑器/项目工具链配合，完成从 0 到 1 的构建与日常维护。

Claude Code 作为 AI 编程工具的一种，让我们普通人也有机会做出 AI 产品。

过去要把想法变成能用的产品，要有专门的程序员去完成需求。但现在不一样了。即使你对代码一窍不通，也能借助 AI 编程工具快速把想法落地。这种变化不是渐进式的改良，而是颠覆性的重构。

就在生财来说，这里有太多生动的例子，不少圈友从来没写过一行代码，却靠着 Cursor、Claude Code 这些工具，实实在在地做出了自己的应用。有人用它开发了海外工具站，有人搭建了自动化工作流，有人甚至做出了能赚钱的小程序。

这里面的差距已经不是效率高一点那么简单了。会用 AI 编程和不会用 AI 编程的人，生产力的差距可能是十倍、甚至百倍的量级。一个懂得如何和 AI 协作的人，可以在一天内完成过去一个小团队一周的工作量。更关键的是，这种能力正在重新定义「谁能创造价值」，技术不再是壁垒，想法和执行力才是。

有了 Claude Code 这样的 AI 工具，你不再需要把 80% 的精力花在写代码上，而是可以专注于真正重要的事情，那就是创意、产品设计、用户体验。代码实现？交给 AI。你负责想清楚「要做什么」和「为什么这么做」，AI 负责怎么做，极大解放你的时间精力。
```

**拆解这段为什么好：**
- 不是甩一个定义就走，而是从定义**自然过渡**到「这意味着什么」
- 用真实案例说服（「有人用它开发了海外工具站」），不是空洞说「很厉害」
- 最后落到学员能获得的具体好处（「你负责想清楚要做什么，AI 负责怎么做」）
- 语气像在和你聊天，不是在念教科书

## Example 2：「为什么学」的完整展开（F2）

> 来源：航海手册01节「为什么要用 Claude Code」

```
过去要成为开发者，你得先翻厚厚的编程书，啃下复杂的专业术语和概念，再折腾各种框架配置。学习周期动辄半年起步，投入的时间成本高得惊人。很多人就是在这个阶段放弃了，不是因为不够聪明，而是因为这条路实在太陡峭。

但 AI 时代彻底改写了这个游戏规则。现在的核心能力不再是记住编程语法，而是学会和 AI 对话。你需要做的是清晰地描述你的需求，用最自然的语言告诉 Claude Code 你想要什么。AI 会基于你的描述生成代码，给出多种实现方案。你的工作变成了从这些方案中挑选最合适的那个，然后继续对话，比如：「这里的逻辑能不能调整一下」「能不能加个导出功能」。就这样一轮轮迭代，项目在你和 AI 的协作中逐渐成型。

对于不同阶段的人，Claude Code 发挥的用处也不同：

对新手：
- 上手快：不会写也能「跑起来」——把需求讲清楚，它给出能运行的初稿。
- 学得快：让它逐行解释、对比写法、指出 bug 的成因，「边用边学」比死记语法高效。
- 错误不恐惧：报错时直接请它定位并给修复方案与测试样例。

对有经验开发者：
- 效率倍增：样板/重复工作交给它，人专注架构、边界与质量。
- 原型极致快：半天做完过去一周的雏形；可以并行尝试多条技术路径。
- 质量护栏：让它做代码走查、补注释与测试，减少「技术债」。
```

**拆解这段为什么好：**
- 旧世界不是抽象说「以前很难」，而是具体到「翻厚厚的编程书」「学习周期半年起步」
- 新世界不是画饼，而是具体描述了一个使用场景（对话→挑方案→迭代）
- 分人群好处是**具体的**，不是「提高效率」这种废话

## Example 3：概念串讲 + 总比喻（F3 + F4）

> 来源：航海手册01节「安装 Claude Code 时常见的概念」

**注意入口处的降低恐惧声明：**

```
在开始安装 Claude Code 之前，我们需要先理解几个会反复出现的核心名词。它们听起来陌生，但其实各自扮演着非常清晰的角色。理解这些，就相当于理解了 Claude Code 运行的底层逻辑。

⚠️ 注意：以下都是名词概念解释，方便大家在安装 Claude Code 时，对这个名词有基本了解，不做重要学习内容
```

**然后每个概念的解释是这个密度的（不是一两句话）：**

```
Git 是什么

Git 是一个代码版本管理系统，可以记录你对代码的每一次修改，就像「时间机器」。你写的每一段代码都能被追溯、回退或对比，不管团队协作还是个人项目，都能防止「手滑改崩」或「忘了昨天改了啥」。

在 Claude Code 的世界里，Git 的作用非常重要：当你让 Claude 修改、生成或重构代码时，它能自动帮你记录改动历史，让你随时对比 AI 修改前后的版本。Claude 甚至可以直接读取 Git 提交记录，帮你理解整个项目的变更脉络。

可以这样理解：Git 负责记忆，Claude Code 负责思考。两者配合，你的开发就既安全又高效。
```

```
终端（Terminal）是什么

终端是电脑里最基础的「命令窗口」，可以用文字和电脑对话。我们平时用鼠标点开软件，而程序员更习惯在终端里用命令来操作电脑，比如创建文件夹、安装程序、运行项目。终端的出现早于图形界面，是所有系统的「底层入口」。

在 Claude Code 的学习中，终端是你和 AI 打交道的舞台。因为 Claude Code 是一个命令行工具，它没有可视化界面，所以我们要在终端里输入命令来安装它、启动它、登录账号，甚至和它对话。你可以把终端理解为「AI 的语音通道」，你在这里发出指令，它就在这里响应。

每个系统都有自带的终端：macOS 的 Terminal、Windows 的 PowerShell 或 CMD。只要掌握一些最基础的命令，你就能操控整个编程世界。
```

**每个概念都是三段式：通用定义 → 和 Claude Code 的关系 → 比喻/总结。不是一句话定义就完事。**

**最后用总比喻把所有概念串起来：**

```
💡 Claude Code 的运行，就像一场合作演出：
- 终端是舞台，命令行是你发出的指令；
- Node.js 是引擎，提供 Claude Code 运行的动力；
- Claude Code 是演员，负责理解你的需求、生成和修改代码；
- VS Code 是后台工作室，用来查看、编辑和测试这些代码；
- Git / GitHub 是记录员，保存每次修改，让项目可追溯、可协作；
- GLM 4.6 / Claude 模型 是大脑，让 Claude Code 具备智能思考与编程能力。

一句话总结：你通过终端发出命令 → Claude Code 借助 Node.js 运行 → 调用 AI 模型生成代码 → 在 VS Code 编辑 → 用 Git/GitHub 保存与协作
```

## Example 4：实操安装步骤的完整密度（F5 + F7）

> 来源：航海手册02节「安装 Git」

```
💡 Q：为什么需要安装 Git？
A：
1. Claude Code 在所有系统上都需要识别、管理、保存代码项目的版本记录，而这项能力是 Git 提供的，Windows 系统默认是没有 Git 的，必须手动安装，否则 Claude Code 无法识别项目的版本信息。
2. 后面代码的存档管理，也是需要 Git，开发基础环境必备，有了 Git，Claude Code 才能安全、完整地理解并管理你的项目。

PS：注意，装 Git 的时候按默认路径装，不要自己改安装目录，一旦改了，后面就会报错

第一步：下载
直接点击官网下载地址即可：https://git-scm.com/downloads
然后选择「Windows」，点击蓝色字体 Click here to download

⚠️ 注意：虽然软件只有 64 兆不到，但官网下载速度很慢，需要等待。

当然也可以去网盘下载也可以
夸克网盘链接：https://pan.quark.cn/s/xxx
百度网盘链接：https://pan.baidu.com/s/xxx

第二步：安装
直接打开，下一步

💡 这里的安装目录强烈建议使用默认
后面涉及到环境变量识别的问题，还需要单独设置。
并且 Git 不占用 C 盘多大空间

📸 此处有截图：每一个「下一步」都有对应截图

第三步：验证是否安装成功
Shell: git --version

注意：因为电脑型号不一样，所以显示的紫色字体也不一样，依据自己的电脑型号来，如果不确定可以截图问 AI 是否安装成功。

下图是 ARM 处理器，显示 CLANGARM64 ～
如果电脑是 Intel 或者 AMD 处理器，紫色字体会显示 MINGW64 ～
```

**拆解这段为什么好：**
- Q&A 开头让你知道「为什么」，不是无脑执行
- **防错提醒**在操作前就说了（「按默认路径装，不要自己改」），不是出错后补救
- 下载给了**3个源**（官网 + 夸克 + 百度），不会因为一个源慢而卡住
- 安装目录用 💡 高亮提醒，视觉上就能看到
- 验证步骤预判了**视觉差异**（「你看到的可能不一样，没关系」）
- 每一步都有截图（我们用 📸 标注需要补充的位置）

## Example 6：操作速查表的完整密度（F9）

> 来源：航海手册03节

```
这里给大家介绍关于使用 Claude Code 的一些常用操作，如果之前从来没有使用过 Claude Code，这个板块可以仅做了解学习，看看即可，不需要马上掌握就可以用，具体等到第二章的案例，就会在实操中使用这些操作的。

基础入门操作：

| 操作名称 | 功能是什么 | 什么时候用 | 如何操作 |
|---------|-----------|-----------|---------|
| 自动编辑模式 | 让 Claude 自动修改代码，不用每次确认 | 写代码、修 bug、调逻辑时 | Mac：Shift + Tab 一次 / Windows：Alt + M 切换模式 |
| @ 指定文件 | 告诉 Claude 只处理某个文件 | 修改单个文件或定位错误 | 在对话里输入 @文件名 即可 |
| 粘贴图片 | 让 Claude 看图帮你调界面或排查错误 | 前端调界面、查页面错误 | Mac：Ctrl + V（不是 Cmd + V）/ Windows: Alt + V |
| 常用命令 | 管理 Claude 的状态和上下文 | 清空对话、切换模型、诊断问题 | /clear 清空上下文；/compact 清理但保留记忆；/status 查看状态；/doctor 检测安装问题；/model 切换模型 |
```

**注意开头的那段话——先降低压力，再给内容。不是直接甩一个表格。**

## 内容密度标准

从以上 examples 总结出的**密度标准**：

| 内容类型 | 每个概念/步骤的最低密度 |
|---------|---------------------|
| 概念解释 | 至少3段：通用定义 → 和当前主题的关系 → 比喻或总结。不低于150字。 |
| 为什么学 | 至少2段旧世界 + 2段新世界 + 分人群好处（至少2类人群）。不低于300字。 |
| 安装步骤 | 每个工具：Q&A + 防错提醒 + 分步骤 + 验证 + 报错。每个步骤都要说清楚操作细节。 |
| 概念串讲 | 每个概念独立3段式 + 最后总比喻。4个概念以上要用总比喻串联。 |
| 操作速查 | 入口降压声明 + 表格（4列：名称/功能/场景/操作）+ 按难度分层。 |

**铁律：宁可写多让用户删减，不要写少让用户补充。教程的首要目标是「新手读完就能做」，信息不足的教程毫无价值。**
