# 飞书组件使用指南

教程输出为 Markdown 时，用以下标记提示飞书排版。

## 组件对照表

| 组件 | Markdown 写法 | 用在哪 | 作用 |
|------|-------------|--------|------|
| 高亮块 Callout | `> 💡 内容` | 概要、路线图、关键总结、Q&A | 视觉锚点，告诉学员「这块重要」 |
| 警告块 | `> ⚠️ 内容` | 注意事项、降低焦虑、前置说明 | 降低心理负担 |
| 表格 | 标准 Markdown 表格 | 对比、速查、分系统对照 | 结构化信息，一眼扫完 |
| 代码块 | 三个反引号 | 命令、配置文件、代码 | 可直接复制执行 |
| 有序列表 | `1. 2. 3.` | 步骤 | 明确顺序 |
| 无序列表 | `- ` | 并列要点 | 罗列信息 |
| 加粗 | `**文字**` | 关键词、验证点、一句话定义 | 扫读锚点 |
| 分割线 | `---` | 板块之间 | 视觉分隔 |

## 使用原则

1. **每800字至少一个视觉锚点**（高亮块/表格/代码块），避免大段纯文字
2. **概要和路线图必须用高亮块**，让学员第一眼看到结构
3. **步骤用有序列表**，不用无序列表（顺序很重要）
4. **命令必须用代码块**，方便复制
5. **关键验证点加粗**，学员扫读时能抓到

## 飞书特有注意事项

- 飞书支持折叠块，但 Markdown 无法直接表示。需要折叠的内容用 `<details>` 标签标注，提醒用户在飞书中手动转为折叠块
- 飞书的高亮块颜色有多种，Markdown 中统一用 `> 💡` 和 `> ⚠️` 区分，飞书中可手动调色
- 飞书表格支持合并单元格，Markdown 不支持。需要合并的用注释标注
