# 逐字稿清洗与道法术器拆解（专业版）

## 触发条件

当检测到以下任一情况时，自动触发本技能：

- 用户发送包含"系统音："、"主持人："、"嘉宾："、"分享者："等发言人标记的文本
- 用户明确提到"逐字稿"、"会议纪要"、"录音转录"、"讲座记录"、"分享拆解"
- 文本长度超过500字且包含口语化表达、语气词、重复断句
- 用户要求"洗稿"、"整理会议纪要"、"总结会议内容"、"道法术器拆解"

## 核心能力

1. **发言人识别**：自动识别不同发言人并按段落组织，支持多发言人场景
2. **术语校正**：修正AI名词、商业用语的大小写和拼写错误（100+词条）
3. **语气词过滤**：去除"对吧"、"就是"、"然后"等冗余表达，保留核心信息
4. **科学理论识别**：自动识别逐字稿中提到的科学理论（荷塘效应、羊群效应、复利效应等40+理论）
5. **结构化输出**：按生财有术风格的"道法术器"框架生成会议纪要
6. **长文本处理**：支持万字以上长文本的清洗和总结
7. **价值评估**：提供"值不值得学/做"的四维打分（钱/关系/技能/影响力）
8. **多格式输出**：支持Markdown、HTML（动画）、Word、PPT格式

## 道法术器拆解优化（提升准确度）

### 拆解前分析

**先回答三个问题：**
1. **这是分享还是对话？**
   - 分享型（嘉宾主讲）：重点提取嘉宾观点，主持人/听众内容作为背景
   - 对话型（多人讨论）：平衡提取各方观点，标注观点冲突

2. **内容类型是什么？**
   - 方法论分享 → 法层权重提高
   - 案例复盘 → 术层权重提高
   - 认知升级 → 道层权重提高
   - 工具推荐 → 器层权重提高

3. **时间结构？**
   - 线性叙事（按时间顺序）→ 按时间线组织
   - 主题分类（跳跃式）→ 按主题聚类组织

### 道层提取规则（核心认知）

**什么是真正的"道"：**
- ✅ 跨场景复用的底层逻辑
- ✅ 改变世界观的一句话
- ✅ 不是具体做法，而是判断标准
- ❌ 不是方法论（那是法层）
- ❌ 不是操作步骤（那是术层）
- ❌ 不是工具推荐（那是器层）

**提取信号词：**
- "本质上..."、"底层逻辑是..."、"核心在于..."
- "不是...而是..."（对比结构）
- "永远记住..."、"最重要的是..."
- 比喻、类比（说明底层原理）

**质量检查：**
- 这个认知能用在其他领域吗？（跨场景检验）
- 去掉这句话，整个分享还成立吗？（必要性检验）
- 三年后再看，这句话还管用吗？（时间检验）

### 法层提取规则（方法论框架）

**什么是真正的"法"：**
- ✅ 可迁移的框架/模型
- ✅ 有命名（如"631原则"、"三步法"）
- ✅ 包含判断标准或决策逻辑
- ❌ 不是具体操作（那是术层）
- ❌ 不是单次案例（那是术层）

**提取信号词：**
- "我总结了一个...方法"
- "可以分为X个阶段/层次"
- "关键是把握...原则"
- 数字命名（三步法、五维模型、631原则）

**法层结构模板：**
```
**[方法论名称]**

[一句话说明适用场景和核心思想]

1. [维度1/步骤1]：[一句话说明]
   - [关键要点]
   - [判断标准]
2. [维度2/步骤2]：[一句话说明]
   - [关键要点]
3. [维度3/步骤3]：[一句话说明]

→ [结论：什么时候用/怎么用/注意事项]
```

### 术层提取规则（具体策略）

**什么是真正的"术"：**
- ✅ 可直接执行的操作
- ✅ 包含具体数字、工具、流程
- ✅ 单次案例的详细拆解
- ❌ 不是通用框架（那是法层）
- ❌ 不是底层认知（那是道层）

**提取信号词：**
- "我当时是这样做的..."
- "具体步骤是..."
- "用了X天/花了X钱"
- 时间线描述（先...然后...最后...）

**术层结构模板：**
```
**【案例标题：做了什么+结果如何】**

- **背景**：[一句话说明场景]
- **具体动作**：
  1. [步骤1：用了什么工具/花了多少时间/花了多少钱]
  2. [步骤2：具体怎么操作]
  3. [步骤3：关键转折点]
- **结果**：[数据支撑：收入X万/涨粉X万/ROI X%]
- **关键要点**：[如果复制这个案例，必须注意什么]
```

### 器层提取规则（工具与资源）

**什么是真正的"器"：**
- ✅ 具体工具名称（软件、平台、书籍、课程）
- ✅ 使用场景说明
- ❌ 不是抽象能力（那是术层）

**提取信号词：**
- "推荐一个工具..."
- "用的是..."
- "可以在...上找到"

**器层结构：**
```
| 工具 | 用途 | 使用场景 |
|------|------|----------|
| **[工具名]** | [一句话说明功能] | [什么时候用] |
```

### 四维打分规则（内容驱动）

**基于文本内容动态评分，而非固定模板：**

```python
# 评分逻辑示例
if "月入" in text or "收入" in text or "变现" in text:
    钱_短期 = 4-5
if "长期" in text or "复利" in text or "积累" in text:
    钱_长期 = 4-5
if "人脉" in text or "连接" in text or "圈子" in text:
    关系_短期 = 3-4
```

**评分标准：**
- 5分：明确数据支撑（月入10万、4亿播放、ROI 300%）
- 4分：有数据但不够具体（月入几万、播放量很高）
- 3分：提到相关点但没有数据
- 2分：隐约相关
- 1分：未提及

## 脚本工具

- **clean_transcript.py**：Python脚本，用于命令行批量处理逐字稿文件
  - 用法：`python clean_transcript.py <文件路径> [标题] --quick`
  - 快速模式：`--quick` 跳过确认直接处理
  - 详细模式：默认，先确认再处理

- **generate_html.py**：生成HTML网页
  - 用法：`python generate_html.py <markdown文件路径> --template modern-light`
  - 支持模板参数：`modern-light`（默认）、`cyberpunk-dark`、`minimal-elegant`、`retro-earth`

- **generate_docx.py**：生成Word文档
  - 用法：`python generate_docx.py <markdown文件路径>`

- **generate_ppt.py**：生成PPT演示文稿
  - 用法：`python generate_ppt.py <markdown文件路径>`

## 注意事项

1. **快速模式默认开启**：除非用户明确说"详细模式"或"先确认"，否则直接处理
2. **道法术器边界清晰**：每句话归类前先问"这是认知/框架/操作/工具？"
3. **数据必须真实**：所有数字必须来自原文，不编造
4. **金句质量优先**：宁缺毋滥，10条高质量 > 15条凑数
5. **行动必须可执行**：每个行动项都要能落地，不泛泛而谈
6. **自动保存必须成功**：保存失败时提示用户，不静默失败
