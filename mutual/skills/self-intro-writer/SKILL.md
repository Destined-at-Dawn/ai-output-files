# 自我介绍撰写 Skill

## 执行流程

### Step 1：确认场景

用户触发后，确认以下信息：

1. **场景**（必须确认，如果用户没说清楚则问）：
   - A：微信私聊 - 线下认识后加好友
   - B：微信私聊 - 线上加好友（视频号/社群/活动引流）
   - C：社群/活动群自我介绍
   - D：朋友圈自我介绍
   - E：社交平台 Bio
   - F：商务合作场景
   - 其他：用户自定义

2. **对方是谁**（可选，用于定制「能提供的价值」部分）：
   - 如果用户提到了对方的身份/行业，调整内容侧重
   - 没提就用通用版

3. **特殊要求**（可选）：强调某个方面、去掉某个信息、语气偏好等

### Step 2：读取资料

1. 读取 `references/jincheng-profile.md` 获取个人资料
2. 读取 `references/scenarios.md` 获取对应场景的写作要点
3. 如需参考范例，读取 `references/examples.md`

### Step 3：生成 2-3 个版本

根据场景生成 2-3 个版本，每个版本有不同侧重：

| 版本 | 侧重 |
|------|------|
| 版本 A | 偏实力展示（数据、成果） |
| 版本 B | 偏价值提供（能帮对方什么） |
| 版本 C | 偏个性人设（记忆点、温度） |

每个版本标注：
- 字数
- 适合什么类型的对方
- 和其他版本的区别

### Step 4：用户选择+微调

用户选一个版本后：
- 可以说「把A的开头+B的资源部分组合」
- 可以说「加上/去掉某个信息」
- 可以说「语气再松弛一点」

直到用户满意为止。

## 写作铁律

1. **不写废话**：「互相学习」「期待链接」如果不跟具体方向就是废话，要么删掉要么补上方向
2. **数据>形容词**：有数据用数据，没数据就用具体事实，绝不用「能力很强」「经验丰富」
3. **长度匹配场景**：私聊<150字，群聊200-400字，Bio<50字
4. **每条信息都有目的**：写之前问「这条信息帮对方记住我/找到我/信任我中的哪一个？」都不是就删
5. **文风松弛**：不端着、不卑不亢、像说话不像写简历。不用「您好」用「你好」，不用「鄙人」用「我」
6. **有记忆锚点**：每个版本至少有一个让人一下记住的点（04年/休学/某个硬数据/某个故事）
7. **需求要具体**：需求是给别人找你的理由，越具体对方越知道要不要找你
