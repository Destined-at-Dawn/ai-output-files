# 清理模式库 — 从实战中提炼的可复用模式

> 来源：2026-06-22 mutual 工作区清理（45项 -> 21项）

---

## 模式 1：timestamp 碎片检测

**特征**：文件名以数字开头 + 含 "Pasted"，是用户在聊天界面粘贴内容时自动生成的临时文件。

**检测**：
```python
if f[0:4].isdigit() and "Pasted" in f:
    # 这是 timestamp 碎片
```

**来源**：NewMax / Claude Code 的粘贴行为会在工作目录生成 `{timestamp}-Pasted-{n}.txt`。

---

## 模式 2：根目录膨胀诊断

**经验值**（从 5 个工作区实测）：

| 根目录项数 | 状态 | 动作 |
|-----------|------|------|
| <= 15 | 健康 | 不动 |
| 16-25 | 轻度 | 只做卫生清理 |
| > 25 | 重度 | 卫生 + 结构 + 死链接全流程 |

**膨胀的典型原因**（按频率排序）：
1. timestamp 碎片积累（最常见）
2. 项目文件溢出到根目录
3. 归档目录没移到全局位置
4. 旧版目录残留（如 `自我进化/` 没合并到 `self-evolution/`）
5. 临时目录没清理（`.tmp/`、`drafts/`）

## 模式 3：错位目录识别

**判断标准**：这个目录的职责是否属于根目录？

| 目录 | 该在根目录？ | 正确位置 | 判断依据 |
|------|------------|---------|---------|
| `CLAUDE.md` | 是 | 根目录 | 入口文件 |
| `memory/` | 是 | 根目录 | 核心目录 |
| `outputs/` | 是 | 根目录 | 核心目录 |
| `projects/` | 是 | 根目录 | 核心目录 |
| `skills/` | 是 | 根目录 | 核心目录 |
| `.claude/` | 是 | 根目录 | 隐藏配置 |
| `self-evolution/` | 是 | 根目录 | 核心目录 |
| `归档/` | **否** | `E:\ai产出文件\牛马\归档\` | 全局统一 |
| `.tmp/` | **否** | 归档或删除 | 临时文件 |
| `conversations/` | **否** | `memory/conversations/` | SRP: 对话属于记忆 |
| `drafts/` | **否** | `outputs/drafts/` | SRP: 草稿属于产出 |
| `handoffs/` | **否** | 删除（如为空） | 已废弃机制 |
| `unified-index/` | **否** | `_system/unified-index/` | SRP: 系统归系统 |
| `自我进化/` | **否** | 合并到 `self-evolution/` | 命名统一 |
| `文书文档/` | **否** | `outputs/文书文档/` | SRP: 文档属于产出 |
| `skill-calls/` | **否** | `memory/skill-calls/` | SRP: 调用记录属于记忆 |
| `scaffold-workspace/` | **否** | `skills/scaffold-workspace/` | SRP: 模板属于技能 |

## 模式 4：死链接分类

扫描到旧路径引用后，必须分三类：

| 类别 | 判断标准 | 处理 |
|------|---------|------|
| **配置引用** | 文件在 .claude/rules/、skills/、templates/、shared-rules/ 中 | 必须修复 |
| **历史记录** | 文件在 memory/2026-*.md、outputs/daily-summary/ 中 | 不动 |
| **其他工作区** | 文件描述的是创作/求职/竞赛/个人工作区的结构 | 不动（除非也要清理那个工作区） |

**易误判的场景**：
- `memory/long-term.md` 中提到 `归档/` — 可能是"归档"这个动词/概念，不是路径引用
- `file-registry.md` 中的 `归档/` — 如果描述的是其他工作区，那是准确的

---

## 模式 5：防根因复发

**铁律**：清理完工作区后，必须检查产生"乱"的根因是否还在。

常见根因：
1. **scaffold 模板** 还在教新工作区建 `归档/` 和 `自我进化/` -> 修模板
2. **共享规则** 还在引用旧路径 -> 修 shared-rules/
3. **li-manage** 没有检查根目录卫生 -> 考虑在 li-manage 中加入卫生检查
4. **定时任务** 没有定期清理 -> 考虑建立定期卫生检查任务
