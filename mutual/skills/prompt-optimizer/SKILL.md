# Prompt Optimizer — 基于BROK框架的提示词优化器

> **核心理念：** 好的提示词不是"说得多"，而是"说得准"。
> **方法论：** BROK = Background（背景） + Role（角色） + Objective（目标） + Key Result（关键结果）
> **来源：** 何锦成BROK框架 v2.2 + 牛马AI优化

---

## 核心概念

**BROK框架四要素：**

| 要素 | 问的是什么 | 缺了会怎样 |
|------|-----------|-----------|
| **B - Background** | 你是谁？什么场景？有什么约束？ | AI不知道上下文，输出泛泛而谈 |
| **R - Role** | AI要扮演什么角色？什么专业水平？ | AI默认"通用助手"模式，缺乏专业深度 |
| **O - Objective** | 你到底想要什么？具体到什么程度？ | AI猜你的意图，答非所问 |
| **K - Key Result** | 什么样的输出算"好了"？ | AI不知道什么时候该停，质量不可控 |

**一句话记忆：** `没有B是瞎猜，没有R是外行，没有O是乱打，没有K是白干。`

---

## 工作流程

### Step 1：接收原始提示词

用户给出一段粗糙的提示词或任务描述。可能是：
- 一句话："帮我写个方案"
- 一段描述："我想让AI帮我分析这个数据..."
- 一个现有提示词："这个提示词效果不好，帮我改"

### Step 2：三维度目标猜测（必须执行，不可跳过）

基于用户输入，从3个不同维度猜测用户的真实目标，用 AskUserQuestion 让用户确认：

**猜测维度：**
1. **功能维度** — 用户想让AI做什么具体的事？
2. **场景维度** — 用户在什么工作/生活场景下需要这个？
3. **质量维度** — 用户期望的输出是什么形式和水平？

**示例：**

用户输入："帮我写个提示词分析用户反馈"

猜测：
1. 功能：你想让AI批量分析用户反馈并提取关键问题
2. 场景：你在做产品迭代，需要从海量反馈中找到优先修复的bug
3. 质量：你希望AI输出结构化的分析报告，包含问题分类+频次+严重度

**如果用户补充信息** → 回到Step 2重新猜测
**如果用户确认或选择** → 进入Step 3

### Step 3：BROK拆解分析

用中括号 `[分类]` 标注用户提供的每条信息属于BROK的哪个维度：

```
原始信息："我是一家电商公司的运营负责人，公司年销售额5000万，
主要客户是25-35岁女性，最近客诉率上升了15%，
我想让AI帮我分析客诉原因并给出改进方案"

拆解后：
B（背景）：电商公司运营负责人[角色]，年销售额5000万[公司规模]，
          25-35岁女性[目标客群]，客诉率上升15%[当前问题]
R（角色）：需要一个数据分析专家+客服质量顾问[AI角色]
O（目标）：分析客诉原因并给出改进方案[任务目标]
K（关键结果）：（缺失 → 需要补充）
```

**补充缺失维度：**

| 缺失维度 | 怎么补 |
|----------|--------|
| 缺B | 追问："你在什么行业/公司规模/面对什么问题？" |
| 缺R | 根据O自动推荐："这个任务需要AI扮演xx专家角色，x年经验水平" |
| 缺O | 追问："你最终想要什么产出？报告？代码？方案？" |
| 缺K | 追问："什么样的结果算'好'？要包含哪些内容？格式是什么？" |

### Step 4：生成优化提示词

去掉中括号和分类标注，将BROK四要素拼接成流畅的完整提示词。

**优化原则：**

| 原则 | 说明 | 示例 |
|------|------|------|
| **具体化** | 模糊词→具体数字/标准 | "降低成本" → "将采购成本降低8%" |
| **可衡量** | 每个K都有明确指标 | "分析报告" → "包含Top5问题分类+频次+严重度评分的报告" |
| **有边界** | 明确什么做、什么不做 | "分析客诉" → "聚焦近30天的产品质量类客诉，排除物流问题" |
| **有格式** | 指定输出格式和结构 | "给出建议" → "用Markdown表格输出：问题/原因/建议/优先级" |
| **有示例** | 复杂任务给一个示例 | 在提示词末尾附一个输入→输出示例 |

### Step 5：输出最终结果

按以下格式输出：

```
### 分析结果（BROK拆解）
- B（背景）：[拆解后的背景信息]
- R（角色）：[优化后的角色定义]
- O（目标）：[具体化的任务目标]
- K（关键结果）：[可衡量的产出标准]

### 优化后的提示词
[完整的、可直接使用的提示词，不含中括号和分类标注]

### 优化说明
- 补充了什么：[列出从用户输入推断出的信息]
- 改进了什么：[列出关键优化点]
- 注意事项：[使用这个提示词时的注意点]

### 更多建议
- [进一步优化的方向]
- [可能需要的迭代调整]
```

## 高级技巧

### 技巧1：多角色协作提示词

当任务复杂时，拆分为多角色协作：

```
你是一个由以下专家组成的分析团队：
1. 数据分析师（负责数据清洗和统计）
2. 行业专家（负责业务解读）
3. 报告撰写者（负责输出结构化报告）

请按以下步骤协作：
Step 1: 数据分析师对输入数据进行...
Step 2: 行业专家基于分析结果解读...
Step 3: 报告撰写者整合为最终报告...
```

### 技巧2：Few-shot示例嵌入

复杂任务给1-2个示例比描述100字规则更有效：

```
输入示例：[具体输入]
输出示例：[期望输出]

现在请处理：[实际输入]
```

### 技巧3：约束条件显式化

把"不要做什么"也写进去：

```
要求：
- 必须基于数据说话，禁止编造
- 每个结论必须标注数据来源
- 不要使用"可能""也许"等模糊词
```

### 技巧4：思维链（Chain of Thought）

引导AI分步思考：

```
请按以下步骤分析：
1. 首先，识别问题的核心是什么
2. 然后，列出所有可能的原因
3. 接着，用排除法筛选最可能的原因
4. 最后，针对每个原因给出解决方案
```

## 触发判定规则

### 何时主动触发

以下场景**必须主动调用此skill**：

1. **用户对AI回复不满意** — "这个回答不好"/"AI没理解我的意思"/"重新来"
2. **用户描述了一个复杂任务** — 长段文字 + "帮我xxx" → 建议用BROK框架结构化
3. **用户说"提示词/prompt"** — 任何相关提及
4. **用户发了一段"要求"** — 想让AI按特定方式执行 → 帮其结构化为提示词

### 不触发的情况

- 用户只是在聊天/闲聊
- 用户已经在执行具体任务（不需要优化提示词）
- 用户明确说"不用优化"

## 示例

### 示例1：简单优化

**用户输入：** "帮我写个提示词让AI分析销售数据"

**Step 2 目标猜测：**
1. 你想让AI从销售数据中找出趋势和异常
2. 你在做月度/季度复盘，需要AI生成销售分析报告
3. 你想建立一个可复用的销售数据分析提示词模板

**Step 3 BROK拆解：**
- B：（缺失 → 需要补充：什么行业？什么数据？什么时间范围？）
- R：数据分析师
- O：分析销售数据
- K：（缺失 → 需要补充：输出什么格式？包含哪些维度？）

**Step 5 优化后的提示词：**
```
你是一名资深数据分析师，擅长从销售数据中发现业务洞察。

请分析我提供的销售数据，完成以下任务：
1. 数据概览：总销售额、订单量、客单价、同比/环比变化
2. 趋势分析：识别销售额的上升/下降趋势，标注拐点
3. 异常检测：找出偏离均值超过2个标准差的数据点
4. Top分析：销售额Top10产品/品类/区域
5. 建议：基于分析结果给出3条可执行的业务建议

输出格式：Markdown，包含数据表格和关键发现的文字总结。
数据来源请在每个结论后标注。
```
