# 生态集成协议（Ecosystem Integration Protocol）

> 每次调研/迭代完成后的"一鱼多吃"写入清单。
> 本文件是 SKILL.md Phase 4.2 和 Phase 6 的详细参考。

---

## 写入清单（7 个生态副作用）

### ① 调研报告（outputs/）

**路径**：`outputs/{主题关键词}-调研报告-{YYYY-MM-DD}.md`
**格式**：见 SKILL.md Phase 4.1 报告结构
**验证**：PowerShell `Test-Path` + 文件大小 > 5KB
**什么时候写**：Phase 4.2（调研完成）+ Phase 5 终止时（迭代最终版）

### ② 今日记忆（memory/{today}.md）

**方式**：Edit 追加到文件末尾
**内容**：
```markdown
## [HH:MM] 调研完成：[主题]
- 模式：调研/迭代第N轮 | 平台：[列表] | 搜索：N次 | 点开：N个
- 核心结论：[1-2句]
- 报告：outputs/[文件名]（XX KB）
- 迭代：待决定/第N轮/已终止
```
**什么时候写**：每次调研完成 + 每轮迭代完成

### ③ 调研画像（memory/research-profile.md）

**方式**：Edit 追加/修改
**更新内容**：
- 历次调研索引表：新增一行（日期/主题/模式/轮次/平台/报告路径）
- 平台偏好：如果本次用了新平台 → 追加到首选平台
- 关注领域：如果本次发现新方向 → 追加到关注领域
- 调研习惯：如果用户改变了深度偏好 → 更新
- 痛点和禁忌：如果用户纠正了 → 更新
**什么时候写**：Phase 4.2（调研完成）+ Phase 5 终止

### ④ 使用日志（memory/skill-usage-log.md）

**方式**：Edit 追加到文件末尾
**内容**：
```markdown
## [YYYY-MM-DD] 技能调用记录
- 用户意图：[用户原始消息摘要]
- 路由决策：deep-research [调研/迭代]模式
- 执行：[平台数]平台 [搜索数]次 [点开数]网页
- 结果：成功/失败/部分成功
- 用户反馈：满意/纠正/待观察/要求迭代
- 路由调整：无需调整/新增触发词/降低置信度
```
**什么时候写**：Phase 4.2

### ⑤ 产出登记（artifact-registry.md）

**方式**：Edit 追加到表格末尾（如果文件存在）
**内容**：
```
| [YYYY-MM-DD] | [主题]调研报告v[N] | outputs/[文件名] | deep-research | 调研/竞品分析/需求验证 |
```
**什么时候写**：Phase 4.2 + Phase 5 终止
**如果文件不存在**：跳过（不在调研过程中创建新文件）

### ⑥ 长期记忆候选（memory/long-term.md）

**方式**：展示候选给用户确认（mcp__ask-user__ask_user）
**候选条件**（满足任意一条）：
- 用户明确表示某个方向是重点
- 发现了高频+高痛+有付费的刚需
- 某个竞品的关键数据（用户数、融资、技术栈）
- 用户在调研中的纠正（"不是这个方向"/"这个更重要"）
**什么时候写**：Phase 4.3（问迭代时顺便问）

### ⑦ 进化信号（evolution-calendar.md）

**方式**：Edit 追加（如果发现进化信号）
**信号**：
- 同类调研 ≥3 次 → "该领域应建立独立 skill"
- 某个方向迭代超过 5 轮 → "该方向应升级为长期项目"
- 路由表需要调整 → 记录待调整项
**什么时候写**：Phase 6

## 写入优先级

| 优先级 | 副作用 | 条件 |
|--------|--------|------|
| 🔴 必须 | ① 调研报告 | 每次 |
| 🔴 必须 | ② 今日记忆 | 每次 |
| 🟡 建议 | ③ 调研画像 | 每次（有新信息时） |
| 🟡 建议 | ④ 使用日志 | 每次 |
| 🟢 按需 | ⑤ 产出登记 | 文件存在时 |
| 🟢 按需 | ⑥ 长期记忆候选 | 高价值发现时 |
| 🟢 按需 | ⑦ 进化信号 | 发现模式时 |
