# 竞赛全流程支持

覆盖竞赛从选题到答辩的六大阶段，每个阶段都有标准SOP、硬约束和技能调用链。

## 适用竞赛类型

| 竞赛类型 | 适配度 | 说明 |
|---------|--------|------|
| 集创赛（集成电路） | ⭐⭐⭐ | 完整适配，含FPGA/IC设计专用流程 |
| 电子设计大赛 | ⭐⭐⭐ | 硬件+软件全流程 |
| 跨校协作/创新训练营 | ⭐⭐⭐ | 文创+科技双轨制 |
| 大创（创新创业项目） | ⭐⭐ | 方案设计+申报书撰写 |
| ROS/机器人竞赛 | ⭐⭐ | 技术调研+代码实现 |
| 其他工科竞赛 | ⭐⭐ | 通用流程适用 |

## 六大阶段总览

```
阶段1：选题与调研 ──→ 阶段2：方案设计 ──→ 阶段3：编码实现
     │                    │                    │
     ▼                    ▼                    ▼
  调研报告.md          任务书/方案.md         代码+测试
                                                 │
阶段6：答辩准备 ←── 阶段5：报告撰写 ←── 阶段4：集成测试
     │                    │                    │
     ▼                    ▼                    ▼
  答辩PPT+模拟          提交材料.pdf         测试报告.md
```

## 阶段1：选题与调研

### 1.1 选题评估

收到竞赛通知后，按以下维度评估选题：

| 维度 | 评估内容 | 权重 |
|------|---------|------|
| **可行性** | 团队技术能力能否实现？时间够不够？ | 🔴 最高 |
| **创新性** | 评委看重什么？往届获奖作品的创新点在哪？ | 🟡 高 |
| **完整性** | 能否做出完整可演示的系统？ | 🔴 最高 |
| **匹配度** | 与团队成员专业/兴趣/经历的匹配程度 | 🟡 高 |
| **可答辩** | 评委可能会问什么？能否答得上来？ | 🟡 高 |

**选题决策SOP：**
```
1. 读竞赛通知 → 提取评分标准和赛道要求
2. 搜往届获奖作品 → 分析共性特征
3. 列出3-5个候选选题 → 按5个维度打分
4. 与团队讨论 → 选定最终选题
5. 写选题论证文档 → 存入 01-调研/
```

### 1.2 技术调研

**调研框架（调用 `AI方法论学习器`）：**

```
Step 1: 定义调研范围
  → 核心技术是什么？需要调研哪些方向？

Step 2: 多源搜索
  → GitHub（搜关键词+stars排序）
  → arXiv/IEEE（搜最新论文）
  → B站/YouTube（搜教程和Demo）
  → 知乎/掘金（搜实战经验）

Step 3: 对比分析
  → 列出3-5个候选方案
  → 每个方案评估：成熟度/难度/成本/可扩展性
  → 写对比分析表

Step 4: 选定方案+写调研报告
  → 存入 01-调研/{调研主题}.md
```

**调研报告模板：**
```markdown
# {调研主题}

## 调研背景
为什么要做这个调研？解决什么问题？

## 候选方案对比
| 方案 | 成熟度 | 难度 | 成本 | 可扩展性 | 综合评分 |
|------|--------|------|------|---------|---------|

## 推荐方案
选哪个？为什么？

## 参考资源
- [GitHub项目名](URL) — 一句话说明
- [论文标题](URL) — 核心贡献
```

## 阶段2：方案设计

### 2.1 技术方案设计

```
Step 1: 系统架构图
  → 用mermaid画系统架构（感知层/计算层/执行层/交互层）
  → 每层标注核心技术栈

Step 2: 技术路线选择
  → 从调研报告中选定方案
  → 标注为什么选这个方案（评委可能追问）

Step 3: 模块分解
  → 将系统拆解为独立模块
  → 每个模块标注：负责人/技术栈/接口/预计工期

Step 4: 风险评估
  → 列出技术风险+非技术风险
  → 每个风险标注：发生概率/影响程度/应对方案
```

### 2.2 文档撰写

**竞赛文档输出流程：**
```
1. 调 五步写作法 拿骨架
2. AI独立撰写初稿（引用调研材料）
3. 调 khazix-writer 去AI味（四层检查）
4. 自检：匿名+去AI味+引用格式+技术一致性
5. 存档到对应目录
```

## 阶段3：编码实现

### 3.1 代码开发规范

| 规范 | 说明 |
|------|------|
| **版本控制** | 必须用Git，每天commit，commit message写清楚改了什么 |
| **模块化** | 每个功能独立模块，接口清晰，方便换人维护 |
| **注释** | 关键函数必须写注释，说明输入/输出/用途 |
| **测试** | 每个模块写单元测试，集成后写集成测试 |
| **文档** | README写清楚：怎么安装、怎么运行、目录结构 |

### 3.2 硬件开发流程（FPGA/嵌入式）

```
1. 调用 hardware-design 技能 → 编码规范+时序约束
2. RTL编写 → Verilator Lint → 仿真测试
3. 综合 → 时序分析 → 迭代优化
4. 上板测试 → 记录问题 → 修复
```

### 3.3 软件开发流程

```
1. 环境搭建（记录版本号和依赖）
2. 核心功能开发 → 单元测试
3. 模块集成 → 集成测试
4. 性能优化 → 压力测试
5. 打包部署 → 演示环境搭建
```

## 阶段4：集成测试

### 4.1 测试清单

```
□ 硬件模块单独测试通过
□ 软件模块单独测试通过
□ 软硬件联调通过
□ 异常情况处理（断电/超时/输入错误）
□ 演示流程走通3遍以上
□ 计时：完整演示在X分钟内完成
```

### 4.2 问题记录

每个Bug用以下格式记录：
```markdown
### Bug #{编号}: {标题}
- **现象**：用户看到什么
- **复现步骤**：1. xxx 2. xxx 3. xxx
- **根因**：真正的原因
- **修复**：怎么修的
- **验证**：修复后测试通过
- **教训**：下次怎么避免
```

## 阶段5：报告撰写

### 5.1 竞赛文档清单

| 文档 | 用途 | 技能链 |
|------|------|--------|
| 作品简介 | 评委第一眼看到的 | `五步写作法` → `khazix-writer` |
| 技术报告 | 详细技术方案 | `五步写作法` → `AI方法论学习器` → `khazix-writer` |
| 答辩稿 | 口头汇报内容 | `五步写作法` → `thinking-coach` 模拟答辩 |
| 演示视频脚本 | 视频拍摄指导 | `五步写作法` → `khazix-writer` |

### 5.2 写作硬约束（竞赛专用）

**匿名要求（违反=成绩为0）：**
- ❌ 正式评审材料中严禁出现学校名/校徽/队员姓名/老师姓名
- ❌ 文件属性/页眉页脚中不得有学校信息
- ✅ 用"某高校学生团队"、"团队成员A/B/C"

**去AI味检查（调用 `khazix-writer`）：**
- ❌ 不用「本质上」「意味着」「不可否认」「综上所述」
- ❌ 不用「首先…其次…最后」机械过渡
- ❌ 不用排比句堆砌
- ❌ 不用「在当今…的时代」开头
- ✅ 段落长短有变化
- ✅ 像人说话而不是像写报告

**技术路径一致性：**
- 描述的技术路径 = 实际建造的系统路径
- 不写没做过的功能/技术
- 不写评委追问时答不上来的细节

**文案可落地原则：**
- 只描述能做出来的功能
- 技术描述用功能，不用实作细节
- 舞台效果不列具体清单
- 不固化数量（写"多关节"不写"12个关节"）

## 阶段6：答辩准备

### 6.1 PPT制作

```
调用 ppt-generator-pro 技能

结构：
1. 封面（作品名+一句话亮点）
2. 问题与背景（为什么做这个）
3. 技术方案（核心创新点，配架构图）
4. 实现效果（演示截图/视频片段）
5. 团队分工（简要）
6. 总结与展望

要求：
- 每页不超过3个要点
- 多图少字
- 动画简洁，不花哨
```

### 6.2 答辩模拟

```
调用 thinking-coach 技能（实战模拟模式）

流程：
1. 读取技术报告+PPT → 生成评委可能问的问题（20-30个）
2. 按难度排序：基础验证 → 技术深度 → 创新点 → 边界情况
3. AI扮演评委，逐个追问
4. 记录回答质量：✅好 / ⚠️需改进 / ❌答不上
5. 生成改进报告：哪些问题需要补答案
```

### 6.3 答辩话术准备

```
开场（30秒）：一句话说清楚做了什么+创新点
技术介绍（2分钟）：架构图→核心模块→关键技术
演示（2分钟）：走一遍完整流程，提前排练3遍以上
总结（30秒）：成果+展望+感谢

常见追问准备：
- 为什么选这个方案而不是XX？
- 遇到的最大困难是什么？怎么解决的？
- 如果时间再多一个月，你会改进什么？
- 这个项目你具体负责了什么？
```

## 全流程技能调用链

| 阶段 | 必须调用的技能 | 触发时机 |
|------|--------------|---------|
| 选题调研 | `AI方法论学习器` + Web搜索 | 调研开始时 |
| 方案设计 | `五步写作法` + `khazix-writer` | 写方案文档时 |

| 报告撰写 | `五步写作法` → `khazix-writer` | 写任何文档时 |
| 答辩PPT | `ppt-generator-pro` | 做PPT时 |
| 答辩模拟 | `thinking-coach`（实战模拟） | 准备答辩时 |
| 会议纪要 | `deep-review` | 团队讨论后 |
| 复盘 | `li-improve` | 每个里程碑后 |
| 任务管理 | `smart-task-planner-skill` + `li-plan` | 项目启动时 |
