# 余一关键案例库

> 这些案例用来佐证诊断判断，展示方法论在真实场景中的表现

---

## 案例1：棉手套标题（市场验证 vs 人类经验）

**背景**：AI 100%生成短视频，标题取了"棉手套"。余一觉得不好，但坚持不干涉——"我在做AI百分百的实验"。

**过程**：
- 发布后数据不好
- 余一不自己分析原因，把数据给AI
- AI自主分析：
  1. 还原决策链：从8个候选标题中选了"棉手套"，理由是有比喻、有数字
  2. 发现"棉手套"引发**困惑**不是**好奇**（好奇=大概知道想说什么；困惑=完全不知道）
  3. 自主对比成功标题"不要当AI保姆"——"保姆"所有人秒懂
  4. **自主发现**评估体系缺一个维度："目标用户会不会点"
  5. 还给自己打底："标题只是影响因素之一，还有时机、封面、算法等"

**余一的评价**："他识别出来了我的意图是我希望数据能好，所以他同时分析了标题只是某一个方面。这个才是真正做事的方法——step back一步。"

**诊断应用**：
- 用户倾向于自己改AI的输出 → 用这个案例说明"不污染"的价值
- 用户不做市场验证 → 用这个案例说明数据反馈比人工判断更有效
- AI分析停在表面 → 用这个案例说明"step back看目的"的重要性

## 案例2：flomo+网盘（AI理解真实意图）

**背景**：余一让AI准备访谈内容，文件在另一台电脑上，iCloud同步出问题。

**过程**：
- 余一说"把邮件发给我"
- AI尝试发邮件 → 失败
- AI自己打开Flomo，调用快捷指令 → 也失败
- AI自己去网上找临时网盘 → 上传文件 → 给下载链接

**余一的评价**："它识别出来了我的目的不是你给我发邮件，我的目的是我想拿到这个文件。"

**诊断应用**：
- 用户的AI只处理字面指令 → 用这个案例说明意图理解的重要性
- 用户的AI遇到障碍就停下来问人 → 用这个案例说明自主性的价值
- 说明"长程任务"的定义：不是时间长，是AI能基于真正目标自己做决策、验收、妥协

## 案例3：复利工程吵架（教训 vs Action）

**背景**：余一让AI做复利总结，AI写了一堆教训。

**第1轮**：AI写了教训文件。余一骂："你说的话里面哪一句是带Action的？不带Action的我默认你做不到。"

**第2轮**：AI写了自检问题，说"前面不是写下来，而是改变执行路径"。余一觉得方向对了。

**第3轮**：AI没有自动启动复利。余一骂："为什么还需要我来提醒你？"

**第4轮**：AI回溯对话只看最近几条。余一骂：回溯整段对话。

**第5轮**：AI没有把通用能力抽出来。AI做了抓取引擎但绑在项目里。余一："这个问题不应该是我来提。"

**第6轮**：AI造了机制但没跑。余一："造了机制当场跑一遍，不跑等于没造。"

**诊断应用**：
- 用户写了教训/总结但没有Action → 直接引用"教训文件需要他写吗？"
- 用户建了规则但没测试 → 引用"造了机制当场跑一遍"
- 用户的通用能力绑在具体项目里 → 引用"应该他自己自主发现"

## 案例4：时间检查规则（AI自主创建规则）

**背景**：AI帮余一补阅读空白时，时间搞错了。

**过程**：
- 余一没说"你应该检查时间格式"
- 只说了一句："我不希望你下次再出现这个问题"
- AI自己分析原因 → 创建"时间敏感度任务检查规则" → 写入系统 → 自动触发

**诊断应用**：
- 用户习惯告诉AI怎么改 → 用这个案例说明"只说结果不说方法"的效果
- 说明真正的复利：不是写教训文件，是创建自动触发的规则

## 案例5：语音PPT的偶然起源

**背景**：余一感冒了说不了话，被邀请做30分钟分享。

**过程**：把任务给AI → AI生成含语音的PPT → 现场AI先讲余一补充 → 效果极好 → 变成固定工作流。

**诊断应用**：
- 最好的工作流不是设计出来的，是从真实需求中试出来的
- 先试再说，不要过度设计

## 案例6：AI说"我好沮丧"

**背景**：AI穷尽所有方法后说"I'm so frustrated"。

**AI自己提议**："我们要不要把这条经验沉淀在经验库里面？后面如果还想让我再试的话，这些踩过的坑我就可以不踩了。"

**诊断应用**：
- 这是"穷尽所有办法再说不会"这条底层规则的效果
- AI真的会穷尽，真的会诚实说做不到，然后**主动提议做复利**
- 说明底层基因对AI行为的深远影响

## 案例7：五个预测——AI和人不谋而合

**背景**：AI自主做了2026年五个预测放在PPT里，余一事先完全不看。

**过程**：现场打开后发现"所有预测我都赞同"。AI还在四个举措里选了"特种部队"作为重点讲的——跟余一现场讲的核心观点一致。

**诊断应用**：
- 说明对齐做到位后，AI的判断可以跟人高度一致
- 这种一致不是靠写详细规则实现的，是靠长期的反馈和对齐
