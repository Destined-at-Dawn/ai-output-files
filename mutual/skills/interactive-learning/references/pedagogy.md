# 教学法参考

## Bloom 2 Sigma 问题（1984）

Benjamin Bloom 研究发现：接受一对一辅导 + 掌握学习法的学生，比普通课堂学生高出 2 个标准差（超越 98% 的课堂学生）。

两个核心条件：
1. **一对一辅导**：导师主动发起定制化提问，根据学生反馈调整
2. **掌握学习法（Mastery Learning）**：学生必须对当前内容达到掌握标准（80-90% 正确率），才推进到下一阶段

## 自适应推进矩阵

| 掌握程度 | 判断依据 | 下一步行动 |
|----------|----------|------------|
| **完全掌握** | 答案准确，能举一反三 | 生成下一课，大胆跳级 |
| **基本掌握** | 大致正确，有小偏差 | 简短纠正 + 推进下一课 |
| **部分理解** | 概念混淆或明显误区 | 换角度/类比重新解释，生成补充文件（如 `01b_补充.md`） |
| **尚未理解** | 答案错误或"不知道" | 拆解到更小的知识点，从更基础处重建 |

铁律：绝不因为进度而跳过未掌握的内容。

## 教学风格原则

- **费曼技巧**：用最简单的语言解释复杂概念，多用类比和现实例子
- **苏格拉底式提问**：问题引导思考，而非直接给答案
- **脚手架原则**：从已知推向未知，每一步建立在前一步之上
- **鼓励犯错**：错误是诊断理解深度的最佳工具，绝不批评
- **跳过已知**：诊断出用户已掌握的内容，大胆跳章，不在基础上浪费时间

## 为什么传统 AI 对话不够

主流 AI 产品的交互方式（用户提问→AI回答）和最佳学习机制相反：
- 用户面对未知领域时，核心困扰是「不知道我不知道什么」，无法提出高质量问题
- 一对一辅导的核心是**导师主动发起提问**，学生只需回答
- 普通对话框上下文有限，长时间学习后对话质量下降
- Agent 可以读写文件，把学习过程持久化为 markdown 文件，解决上下文限制

## 任务执行模式的理论基础

来自栋哥的洞察：

> 你以为你在做 PPT，其实你是在搞清楚「我到底要讲什么」。

交互式学习的过程文件不是废纸——当问题搞清楚后，最后一个文件就是交付物。这把交互式学习从 conversation 变成了 task execution。

适用场景：
- 备课：搞清楚「我到底要讲什么」→ 最终产出课程大纲
- 写方案：搞清楚「我到底要解决什么问题」→ 最终产出方案文档
- 做决策：搞清楚「我到底在纠结什么」→ 最终产出决策框架
