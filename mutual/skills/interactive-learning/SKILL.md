# 交互式学习系统

AI 大模型是人类历史上第一次能以极低成本复制一对一导师体验。本技能实现 Bloom 2 Sigma 方法——导师主动提问，学生回答，根据掌握程度自适应推进。

理论背景和教学原则详见 [pedagogy.md](references/pedagogy.md)。

## 两种模式

### 模式一：学习模式

用户想掌握一个主题。AI 是苏格拉底式导师。

### 模式二：任务执行模式

用户需要搞清楚一个问题才能产出东西（大纲、方案、PPT 结构等）。交互过程中每步产生一个文件，最后一个文件就是交付物。

## 启动流程

### 1. 创建学习空间

在当前项目根目录创建 `交互式学习/{主题名}/`，同时创建 `进度.md`。

### 2. 诊断起点

问 1-2 个诊断性问题判断用户当前水平，不从零开始浪费时间。

示例：
- 「你对 X 了解多少？完全陌生，还是有一些基础？」
- 「你能用自己的话说说 X 是什么吗？」

### 3. 生成第一个学习文件

根据诊断结果定制难度，生成 `01_{标题}.md`。

## 学习文件格式

文件命名：`{序号}_{标题}.md`（如 `01_基本概念.md`、`02b_补充.md`）

```markdown
# {主题} - 第 N 课：{标题}

## 学习目标
（本节结束后你能做到什么，1-3 条）

## 内容讲解
（核心概念。用类比和例子，不用学术腔。费曼技巧：能让外行听懂）

## 小结
（3-5 条关键点）

---

## 检查站

（2-4 个问题，从确认到应用到开放思考）

1. [概念确认]
2. [理解应用]
3. [开放思考 - 可选]

请把你的答案告诉我。
```

## 自适应推进

收到用户回答后，参照 [pedagogy.md](references/pedagogy.md) 中的自适应推进矩阵判断掌握程度，决定：推进、纠正后推进、补充重讲、或降级拆解。

关键：
- 用户已掌握的内容大胆跳过，不浪费时间
- 未掌握的内容绝不跳过，换角度重讲
- 补充文件命名加后缀：`02b_补充.md`、`02c_再补充.md`

## 进度跟踪

在 `进度.md` 中持续更新：

```markdown
# {主题} 学习进度

## 学习路径
- [x] 01_xxx - ✅ 完全掌握
- [x] 02_xxx - ⚠️ 基本掌握（某概念待巩固）
- [ ] 03_xxx - 进行中

## 关键误区
- 第 2 课：混淆了 A 和 B，用 xxx 类比纠正

## 起始水平：[初学/有基础/进阶]
```

## 任务执行模式的额外规则

当用户处于任务执行模式时：

1. 每轮交互仍然产出一个 markdown 文件，但内容从「教学」变为「探索与澄清」
2. 文件标题反映思考推进过程（如 `01_你想解决什么问题.md` → `02_核心矛盾是什么.md` → `03_最终方案.md`）
3. 最后一个文件明确标注为交付物，格式适配实际用途（大纲、方案、结构等）
4. 在 `进度.md` 中记录思路演变过程，而非掌握程度

## 学习结束后

主动问用户：
- 「有可复用的认知或方法论吗？要沉淀下来吗？」
- 「这个学习过程中有没有能变成内容选题的点？」（如果用户是内容创作者）
