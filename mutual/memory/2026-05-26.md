### ask-user MCP 彻底修复 ✅
- **时间**：23:00-23:30
- **起因**：用户反馈 ask-user MCP 完全不工作，重启无数次无效
- **根因**（双重问题）：
  1. `ask-user-mcp` 是纯 HTTP 服务器（StreamableHTTPServerTransport），但配置用了 `"command"`（stdio）→ 协议不匹配
  2. 缺少端口参数 → `parseInt("-y")` = NaN → 服务器启动崩溃
- **修复**：
  - MCP 配置从 `"command"` 改为 `"type": "http"`, `"url": "http://127.0.0.1:7878/mcp"`
  - 启动命令补全端口参数 `7878`
  - 后台启动服务器（端口 7878 在监听）
  - 注册表 HKCU Run 键设置开机自启
  - HTTP initialize 握手测试通过 ✅
- **教训**：PK-INFRA-016（协议类型不能混用）+ PK-INFRA-017（npx 参数位置踩坑）已写入 long-term.md
- **之前说"重启就行"是错的**——根因是配置错误，不是用户没重启。教训：排查问题要从配置本身入手，不能甩锅给用户操作。
