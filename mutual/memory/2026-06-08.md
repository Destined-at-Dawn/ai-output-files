## 搜索质量问题修复（li-bestskill + li-improve 双升级）

### 问题发现
用户指出 li-improve 创建时的外部研究只找到 13★ 以下的小项目，而他用简单关键词 `li-improve` 就能找到 HyperAgents（2.6K★）和 peterskoett（641★）。

### 根因
- li-bestskill 的搜索策略用了**精确匹配**思维（`claude-code li-improve skill`）
- 只搜了 skills.sh / awesome-claude-skills 小生态，没搜 GitHub 全局
- 没有按 stars 排序，没有质量自检（结果 <100★ = 搜索方式有误）

### 修复内容
1. **li-bestskill 搜索策略更新**：
   - 新增"宽泛优先"原则（先搜核心概念，不加平台限定）
   - 新增 stars 排序规则（GitHub 搜索必须 `&s=stars&o=desc`）
   - 新增质量自检（结果 <100★ → 换宽泛关键词重搜）

2. **li-improve v1.0 → v1.1 重大升级**：
   - **Hook 驱动激活**：activator.sh（UserPromptSubmit）+ error-detector.sh（PostToolUse）
   - **递归度追踪**：Pattern-Key 去重 + Recurrence-Count + First/Last-Seen
   - **自动 Skill 提取**：extract-skill.sh + 5 项质量门禁
   - **元认知自修改层**：每月 1 次检查改进流程本身的效率
   - **知名项目参考库**：5 个项目（Letta 23.2K★ / HyperAgents 2.6K★ / peterskoett 641★ / DGM 2.1K★ / GPTSwarm 1.0K★）

### 教训 [高置信]
- **搜索"灵感发现"必须用宽泛关键词**，精确匹配 = 过滤掉所有高质量跨生态项目
- **搜索结果 <100★ = 搜索方式有误**，必须换更宽泛的词重搜
- **peterskoett（641★）的 hook 驱动 + 递归度追踪**比我们之前的"AI 记得要做"可靠 10 倍
- **HyperAgents（2.6K★）的元认知自修改**——改进流程本身也应该被优化

### 未来所有 skill 创建/融合前的搜索标准
1. 先用宽泛关键词搜 GitHub 全局（按 stars 排序，看 Top 10）
2. 读 Top 3 的 README/SKILL.md，提取核心机制
3. 再精确搜 Claude Code 生态（skills.sh / awesome-claude-skills）
4. 合并两个搜索结果，选择性吸收到新 skill 中

## 搜索策略二次修复（li-bestskill v1.1 → v1.2）

### 问题
用户指出之前的"优化"只写了 GitHub 搜索策略，其他 12+ 平台完全没有搜索标准。
用户说："你不能只去搜索github，任何有很好的skill下载的网站你都应该去查看寻找。你的看似的优化，却进一步使我们的更改变得很差。"

### 根因
- 搜索策略铁律（第 160-185 行）只覆盖了 GitHub 的排序规则
- 其他 12+ 平台（skills.sh / Agent Skills Hub / MCP Registry / ToolSDK / AIHot / 腾讯 / 阶跃等）没有搜索策略
- "宽泛优先"原则只适用于 GitHub，其他平台怎么办完全没写
- references/ 目录为空——没有详细的平台搜索指南

### 修复
1. **SKILL.md 搜索策略段重写**（v1.1 → v1.2）：
   - 三大搜索场景（灵感发现 / 具体需求 / 探索发现）
   - 四阶段全平台搜索策略（阶段1:宽泛全局 → 阶段2:中文生态 → 阶段3:社区论坛 → 阶段4:精确补充）
   - 各平台排序/质量指标表
   - 搜索质量自检 3 条铁律

2. **references/platform-search-guide.md 创建**（194 行）：
   - 10 个平台的具体搜索技巧和质量判断
   - 社区/论坛搜索方法（Reddit/HN/知乎/微信公众号）
   - 搜索优先级总结
   - 搜索质量自检清单

### 严重教训 [最高置信]
- **"优化"不等于"在旧框架上叠加限制"**——真正的优化是打开视野，不是缩窄路径
- **只写一个平台的策略 = 告诉 AI 其他平台随缘 = 漏掉大部分优质 skill**
- **搜索策略必须覆盖所有有优质 skill 的平台**，不只是 GitHub
- **references/ 目录不应该是空的**——详细指南放 references/，主文件放框架
