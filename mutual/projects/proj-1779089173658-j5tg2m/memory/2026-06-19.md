## mempalace MCP "Server disconnected" 修复

**根因**：`E:\python.exe`（Python 3.13.14）PATH 优先级高于 WindowsApps 的 Python 3.11.9，但 3.13 没装 mempalace 包。Newmax 解析 `python` 命令时命中 3.13 → `ModuleNotFoundError: No module named 'mempalace'` → 进程即刻退出 → Server disconnected。

**修复**：`.newmax/.mcp.json` 中三个 MCP server（markitdown-mcp / mempalace / agentmail）的 `command` 从 `"python"` 改为完整路径 `"C:/Users/13975/AppData/Local/Microsoft/WindowsApps/python.exe"`。

**附带操作**：mempalace 3.4.0 → 3.4.1 升级。

**教训**：Windows 上存在多个 Python 安装时，MCP 配置必须用完整路径，不能依赖 `python` 解析。
