## 2026-06-21 skill-auto-activation v2.0 升级（联动优先 + 自我学习）

### 用户诉求（三条核心）
1. 技能联动调用 > 单打独斗：一个任务应触发多 skill 协作链
2. 自我学习：用户不满 → 记录 → 调整联动链 → 下次自动应用
3. 探索性任务强制用技能：禁止原生裸跑

### 执行操作
1. **归档原文件**：`E:\ai产出文件\牛马\归档\2026-06-21-skill-activation-upgrade\skill-auto-activation.md`
2. **升级 `.claude/rules/skill-auto-activation.md`**（v1.0 → v2.0，307行）
3. **路由表新增 linkage 字段**：11 个关键 skill 配置了联动链

### v2.0 核心变更

**三层路由优先级链**：
```
Layer 0: 技能路由表（最高优先）
Layer 1: SOP 索引（li-intent Phase 0）
Layer 2: 触发词语义匹配（li-local-search）
Layer 3: 原生能力（仅简单任务允许）
```

**预定义联动链**（11条）：
- li-xhs → [li-devil]
- li-research → [li-analyze, li-devil]
- li-dbs → [li-analyze, li-devil]
- li-study → [li-analyze, li-memory]
- li-plan → [li-devil, li-manage]
- li-skillcreate → [li-local-search, li-skills-mgmt]
- li-asset → [li-xhs, li-devil]
- li-diagnose → [li-improve, li-infra]
- li-analyze → [li-research, li-devil]
- blog-post-writer → [li-devil]
- li-industry → [li-analyze, li-dbs]

**自我学习闭环**（5步）：
1. 检测不满信号
2. 分类（路由错误/联动不足/深度不够/风格不对）
3. 确定修正方案
4. 当轮应用
5. 写入学习记录 + 更新路由表

**探索性任务铁律**：
- 用户没给明确方向的任务 → 强制调 li-analyze/li-research/li-devil
- 禁止原生裸跑（唯一例外：<3步的简单事实查询）

### 设计决策
- linkage 字段加在路由表 JSON 中（而非规则文件），方便程序化调度
- 联动链不超过 2-3 个辅助 skill（防拖慢主流程超 50%）
- 动态联动兜底：预定义没覆盖时，AI 自主判断增强/质疑/归档三类联动至少选1

## 2026-06-21 R20 全局化（用户纠正：规则只写本地无效）

### 用户纠正
"你写在这个工作区是没用的，你要写写在全局，只要使用了 claude 就应该记住这条规则。"

### 执行操作（全局三层部署）

|------|------|------|---------|
