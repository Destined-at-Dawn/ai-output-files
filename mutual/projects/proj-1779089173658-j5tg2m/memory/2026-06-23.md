# 2026-06-23 记忆

## li-study v1.0 → v2.0 系统级重构（li-skillcreate 联动审核后执行）

### 背景
用户纠正：li-study 的升级不应该是"功能堆砌"（直接把 learn-anything-with-AI 的特性追加为 Phase 7/8/9），应该做体系级设计。li 系列有复杂的体系——模式路由、Progressive Disclosure、自进化、生态联动。

### 执行方式
按 li-skillcreate Phase 6 迭代模式（类型 D：重构迭代）+ li-skillcreate + li-skills-mgmt + li-skillcreate 三工具联动。

### 核心设计决策
1. **模式路由替代线性堆叠**：6 种 Mode（A概念攻克/B概念解剖/C真读检验/D学习路径/E考前冲刺/F项目陪练），每种有独立身份（教练/审视者/看门人/导航员/出题官/陪练搭档）
2. **Progressive Disclosure**：SKILL.md 275 行骨架 + 10 个 references/ 文件（1640 行详细方法论）
3. **融入外部优势但不堆砌**：掌握度 L1-L4 融入每个 Mode 的收尾验收，来源策略融入 Mode D，项目化产出融入 Mode F，输出模板融入 references/
4. **自进化机制**：golden_rules 迭代 + 反馈采集 + 学习画像 + 掌握度追踪
5. **一鱼多吃**：学习完成后的 5 种生态副作用（概念卡片/错题本/项目产出/记忆候选/教训归档）
