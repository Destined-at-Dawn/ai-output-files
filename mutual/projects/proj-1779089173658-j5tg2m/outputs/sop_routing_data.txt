> **版本**：v2.5 | **更新日期**：2026-05-13

## 二、指令匹配规则（v1.3 更新）

### 2.1 优先级链（v1.2 提示词优化+输入类型优先）

```
🔴 最高优先级：提示词质量检查（模糊指令拦截层）

0. 用户发了模糊指令 → 读「00_提示词优化SOP.md」→ 评分→输出3个改进版→用户选择→执行
   - 跳过条件：链接/图片/文件/引号长文本/说"快"/≥50字且有动词/定时任务

⚠️ 次高优先级：输入类型识别（丢东西进来的场景）

1. 用户发了链接/图片/文件 → 读「输入分发SOP.md」→ 自动识别类型 → 调已有 Skill → 衔接主题 SOP
   - 微信公众号链接 → 读「公众号自动化SOP.md」→ 单篇走入口A（mptext.top API优先），批量走入口C（mptext.top）
   - GitHub 链接 → 全能网页采集器 / gh CLI
   - YouTube 链接 → anything-to-notebooklm → transcript-cleaner
   - 图片 → ppocrv5（OCR）/ li-visual（风格解码）
   - 文件 → docx/xlsx/pdf/zerox → 按类型处理
2. 用户说"跑公众号/找爆款" → 直接读「公众号自动化SOP.md」入口B（批量发现模式）
3. 🆕 用户说"批量抓取/学习这个作者" → 直接读「公众号自动化SOP.md」入口C（mptext.top批量模式）
4. 🆕 用户发了视频文稿/长文链接+标注"品读/解剖/分析" → 读「02_文章采集与分析SOP.md」§2.5（百大认知解剖+四层拆解五步闭环）
5. 用户明确说了"做PPT" → 直接读 PPT制作SOP.md
6. 🆕 用户说"模仿我的风格/以我的风格/你觉得我会怎么说/像我一样说/用我的语气/说人话/像我写的" → 调用 `mcp__skill-handler__Skill skill: "niuma-voice-dna"` → 进入风格模式（全局生效，不限工作区）
7. 用户说了灵感/想写 → 内容创作
7. 用户发了引号内容 → 文件与信息管理（引号保存）
8. 用户说了改/优化/帮我 → 内容创作（优化流程）
9. 用户说了技能/映射/系统 → 系统管理
10. 用户说了网站/网页/HTML/落地页/个人主页 → 读 `12_网站与HTML生成SOP.md`
11. 🆕 用户提到信息差/普通人不知道/新工具/纠正误区/值得写/收藏 → 读 `13_信息差采集与分层输出SOP.md`
12. 🆕 用户说"学这个领域/先问三个问题/MIT三问法/建立全局认知" → 读「08_知识学习SOP.md」§三.5（MIT三问法五步执行：核心思维模式→三个分歧→10道自测→归档）→ 自动产出认知地图+选题素材
13. 🆕 用户说"竞赛六步法/选赛/反向对标/组队找老师/论文七步" → 读「10_个人发展SOP.md」§三（竞赛全流程六步法）→ 自动产出参赛策略+对标分析
14. 🆕 写文章/创作时 → 自动从「信息差素材库」§八金句库取材（15条锦成AI课金句+5个认知框架），禁止忽略已有素材

**懒人指令规则**：用户丢链接/图片/文件时，不需要说"帮我提取/分析"，AI 自动走输入分发 SOP。

⑦ 🆕 如读到小黎新个人信息 → 同步更新 telos/ 对应文件（SOP-11）
```

## 三、自学习框架（所有 SOP 共用）

> 「真正的学习不是记住知识，而是改变行为。」——爱因斯坦

### 3.1 自学习闭环（嵌入每个 SOP）

```
执行 SOP → 记录实际路径（不是理想路径）
  ↓
对比 SOP 中的决策表 → 找到偏差
  ↓
偏差 = 新知识？→ 写入对应 SOP 的「迭代日志」
偏差 = 失败？→ 写入 自我进化/做得差的避免/
偏差 = 优化？→ 写入 自我进化/做得好的鼓励/
  ↓
3 次同类偏差 → 更新 SOP 的决策表或流程
```

### 3.2 每个 SOP 的自学习组件

每个 SOP 文件末尾必须包含：

```markdown
## 迭代日志

| 日期 | 触发事件 | 原流程 | 发现的问题 | 更新内容 | 版本 |
|------|---------|--------|-----------|---------|------|
| YYYY-MM-DD | 用户做了X | 步骤A→B→C | 步骤B多余 | 删除步骤B | v1.1 |
```

### 3.3 自动触发 SOP 更新的条件

| 条件 | 动作 |
|------|------|
| 同一 SOP 连续 3 次用户说"不好" | AI 必须重读自我进化记录并提出改进方案 |
| 同一 SOP 某步骤连续 3 次被跳过 | 该步骤标记为「可选」或合并到前一步 |
| 用户指出新场景 | 新增场景行到决策表 |
| Skill PK 产生新默认 | 更新 SOP 的 Skill 选型 |
| memory 中出现新的成功模式 | 追加到 SOP 的「最佳实践」 |

### 3.4 数据来源

| 数据源 | 路径 | 作用 |
|--------|------|------|
| 自我进化/做得好的鼓励/ | `自我进化/做得好的鼓励/` | 成功模式提取 → 写入 SOP 最佳实践 |
| 自我进化/做得差的避免/ | `自我进化/做得差的避免/` | 失败模式提取 → 写入 SOP 避坑清单 |
| memory/{日期}.md | `memory/` | 每日任务进展 → 提取高频操作 |
| memory/long-term.md | `memory/long-term.md` | 持久决策 → 更新 SOP 决策表 |
| 技能调用记录.md | `公众号/自己/技能调用记录.md` | 技能使用数据 → 更新 Skill 选型 |
| 每日完成记录/README.md | `每日完成记录/README.md` | 完成率统计 → 优化流程 |
