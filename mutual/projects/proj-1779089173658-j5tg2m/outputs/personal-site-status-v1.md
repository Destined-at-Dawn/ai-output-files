## 三、服务器决策（ECS e实例 99元/年）

### 最终选择
- 产品：ECS 经济型 e 实例，2核2G
- 价格：99 元/年（**续费同价**，2027 年前锁价）
- 地域：华东 2（上海）
- 镜像：**Ubuntu 22.04 64位**
- 带宽：3M 固定带宽

### 淘汰的选项及原因

| 淘汰项 | 原因 |
|--------|------|
| 轻量应用服务器 68元 | 共享算力，支付回调不稳定；首年优惠，续费暴涨至 400-600 元 |
| 122元全家桶套餐 | 域名已买，OSS 暂不需要，多花 23 元无用 |
| 学生优惠机型 | 续费跳回 600-1000 元；突发性能型 CPU，超过 20% 即限速 |
| Ubuntu 24.04 | ROS2 Jazzy 生态未成熟，多数传感器驱动未适配；Python 3.12 + OpenSSL 3.0 与支付插件不兼容 |
| Ubuntu 20.04 | 标准支持已于 2025年4月结束；ROS2 Foxy/Galactic 停止维护 |

### 附赠权益（已建议领取）
- 主机病毒防护（免费 3 个月）
- 文件备份权益（100GB 免费）

## 四、支付方案决策

### 双轨收款策略

| 站点 | 收款方式 | 理由 |
|------|---------|------|
| lilanyuan.cn | **不挂任何支付** | 保住 ICP 备案 |
| lilianyuan.net | 虎皮椒/蓝兔（ISV 合规接入） | 个人身份证即可，资金 D+1 官方结算到个人银行卡 |

### 朋友中转站思路的借鉴
- 他用的"码支付/免签支付"逻辑：安卓手机挂监听 App → 监听到账通知 → 回调服务器生成密钥 → 用户即时获得积分
- 对你的启示：自动化发货是虚拟产品竞争力的核心——用户付款后必须秒得密钥，不能靠人工发
- 实现路径：虎皮椒支付回调 → 后端更新数据库 → 前端自动展示密钥/兑换码

### 码支付 vs 虎皮椒 决策

| 方案 | 优点 | 缺点 | 适用阶段 |
|------|------|------|---------|
| 码支付（自挂手机） | 零手续费，秒到账 | 需维护手机，有挂机风险 | 成长期（日 >3 单） |
| 虎皮椒 | 官方 ISV，合规，D+1 结算 | 约 1.5% 手续费 + 开户费 | 成熟期（商业化运营） |
| 微信手动转账 | 零手续费 | 流失率高，不可规模化 | 测试期（0-10 客户） |

## 五、技术栈

### 国内站（lilanyuan.cn）
- 服务器：阿里云 ECS Ubuntu 22.04
- Web 服务：Nginx（Docker 容器化）
- SSL：Let's Encrypt 自动续期
- 部署：GitHub Actions → SSH → ECS
- 当前页面：lilanyuan-cn-v3.html（65KB，已在 F:\work\个人建站\）

### 海外站（lilianyuan.net）
- 前端：Cloudflare Pages（免费，全球 CDN，自带 DDoS+WAF）
- 后端：CF Workers / Lisa 美国主机
- 发卡系统：独角数卡 (Unicard)
- 支付：虎皮椒回调接口
- 部署：GitHub push → Cloudflare 自动构建
