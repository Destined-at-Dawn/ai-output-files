## 一、战略逻辑（为什么需要两个版本）

你目前处于一个罕见的交叉点：

| 身份 | 面向人群 | 需要展示的 |
|------|---------|-----------|
| **体制内保底** | 常州电网面试官、皮赛老板、学校老师 | 专业成绩（GPA 3.9）、企业实习硬核成果、合规性 |
| **自媒体前锋** | AI 信息差用户、私域粉丝、潜在合作方 | 变现能力、AI 工具链深度、社群影响力、商业闭环 |

同一张网页无法同时满足这两类人——电网面试官看到"卖 AI 信息差"会觉得你不稳重；自媒体粉丝看到"电网保底"会觉得你不酷。

**解决方案**：一个身份，两个窗口。用不同的域名和内容策略分别承接。

## 四、部署建议

### 方案 A（推荐）：同域名子路径
```
lilanyuan.cn          → 国内版（默认首页）
lilanyuan.cn/creator → 海外版（自媒体向）
```
- 优点：一个域名，两个入口，SEO 权重集中
- 需要：在同一个 HTML 文件内做路径判断，或部署两个独立页面

### 方案 B：独立域名
```
lilanyuan.cn   → 国内版
lilianyuan.net → 海外版（或 lilianyuan.com）
```
- 优点：两个域名完全隔离，国外版可以更激进地做 SEO/广告投放
- 缺点：多一个域名需要续费和管理

### 当前建议
先用**方案 A**（同域名子路径），因为：
1. `lilanyuan.cn` 刚买，暂不需要额外域名开销
2. 国内版的流量可以自然溢出到自媒体版
3. 等海外版验证有足够流量后，再考虑独立域名

### 国内版部署路径（阿里云 OSS + CDN）
```
lilanyuan.cn（阿里云万网已注册）
  ├─ 备案主体：李兰源（个人 ICP 备案，身份证 + 学生证/在读证明）
  ├─ DNS 解析 → 阿里云 CDN（国内加速，CNAME 指向）
  │   └─ 回源 → 阿里云 OSS（存放 index.html，65KB 单文件）
  └─ 费用：OSS 存储几分钱/月，CDN 流量前 10GB/月免费
```
**为什么个人备案而非挂企业**：小黎的职业路径是大厂轮转（特斯拉/蔚来/宁德时代/宇树科技），域名备案必须跟着人走，不能绑在任何一家企业营业执照上。个人备案一次搞定，永远不受换公司影响。

**临时方案（备案期间）**：先在 Cloudflare Pages 或 Lisa 主机做临时展示，备案下来后切到国内 CDN。
