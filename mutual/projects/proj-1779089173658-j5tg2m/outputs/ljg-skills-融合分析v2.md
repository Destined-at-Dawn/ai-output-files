### 1. ljg-think「追本之箭」（67 行）

**做什么**：纵向深钻——给一个观点，像箭一样一路向下钻到不可再分的本质。

**核心设计**：
- 隐喻：一支离弦之箭，只知前进不知后退
- 钻的逻辑：表象→机理→原理→公理，每次切换到更基本的解释框架
- 维度切换：从社会学→心理学→生物学→物理学→数学→逻辑本身
- 写法要求：**写一场下坠，不是分析报告**。有失重感、每层有命名、层间有裂缝、终点要狠（"让人沉默片刻"）
- 三条铁律：纵向不横向、单刀直入、层层惊叹

**为什么高价值**：
- 我们的 li-analyze 是**横向四维展开**（道/法/术/器），他是**纵向深钻**
- 两者互补：道法术器告诉你"这个领域有什么"，追本之箭告诉你"为什么会这样"
- 可以作为 li-analyze 的 Mode B：用户要横向展开→道法术器，用户要纵向深挖→追本模式

**具体融合方案**：
li-analyze 新增 Mode B（追本模式）：
- 触发词：为什么/本质/深挖/钻到底/想透/追本/根源
- 流程：表象→机理→原理→公理（四层下钻）
- 每层命名（2-3字）+ 层间裂缝（一个问题或矛盾）+ 终点（不可再分的元素）
- 保持 li-analyze 的 Phase 0 消解和 Phase 2 百大认知注入

### 2. ljg-rank「降秩引擎」（484 行 — 全仓库最大）

**做什么**：从大量现象中找到"独立的生成器"——能生成所有现象的最小要素集。

**核心设计哲学（来自 David Deutsch《无穷的开始》）**：
- 评判好理论的标准：**解释力 + 难以变更**
- 不是"总结要点"，是"找到生成所有要点的根"
- 好要素 = 解释力（一个要素解释多个现象）+ 难以变更（砍掉或换掉任何一项，整体崩塌）
- 坏要素 = 只描述不解释、被替换后无影响

**完整流程（7 步）**：
1. **铺**：列出所有现象
2. **挂**：建立"现象↔要素"映射矩阵
3. **收**：合并同源要素（归一化等价要素）
4. **递归下沉**：如果要素还能分，继续下沉
5. **砍**：一个一个移除要素，测试剩余要素能否覆盖所有现象
6. **反生成**：测试每个要素——如果没有它，哪些现象无法解释
7. **验证**：预测（生成器能否预测未观察现象）+ 变更（替换后是否崩塌）

**9 种取景框**：目镜/透镜/聚光灯/多棱镜/滤光片/广角/鱼眼/分光镜/棱镜——每种从不同角度看要素

**ASCII 语义矩阵**：用 ASCII 矩阵可视化现象↔要素的关系

**为什么高价值**：
- 我们有 62 本百大认知 + 大量 outputs + 117 个 skill — **信息过载是核心问题**
- ljg-rank 是**唯一能系统性压缩认知体系的工具**
- 没有对应 skill，也没有近似替代

**具体融合方案**：
创建 li-rank 独立 skill：
- 输入：大量文本/笔记/认知书籍
- 流程：铺现象→挂映射→收同源→递归下沉→砍→反生成→验证
- 输出：独立生成器集（最小要素集 + 语义矩阵 + 验证报告）
- 应用场景：压缩百大认知系统、压缩项目记忆、压缩知识体系

### 3. ljg-paper「论文故事化」（426 行）

**做什么**：读论文，当一个故事讲——让不懂领域的聪明人能复述。

**七拍故事骨架**：
1. **主角**：领域困境的具体化身
2. **困境**：利害（why care）+ 水位（base rate：现状到哪，典型一步多大）
3. **旧路**：前人怎么走的，卡在哪
4. **转折**：作者核心想法（"我有一个大胆猜想"）
5. **解法**：机制怎么动 + 证据怎么撑 + 最反直觉的副发现
6. **结局**：论文自身结论
7. **内核**：带得走的新视角 + 检验（拿到生活里哪儿站住哪儿破）

**速读卡**：压缩成三句——一句话/大想法/只记三件事

**三条铁律**：
1. A 必须自给自足——没读过原论文的人也能懂
2. 数字来自论文正文——遇大数必标注来源
3. 术语拉到常识——首次出现必须有通俗解释

**验收**：复述检验 + 动机追问 + 断点检测 + "So What" 检验 + 技术转化 + 知识链路

**为什么高价值**：
- 我们的 li-research 对论文的处理偏**分析/总结**，缺乏"讲故事"的能力
- 论文阅读对于考研/保研/竞赛论文撰写都是核心需求
- 七拍骨架 + 速读卡 + 复述检验 = 完整的论文阅读方法论

**具体融合方案**：
li-research 新增论文故事化模式：
- 触发词：论文/paper/讲论文/把这个讲给我听
- 流程：七拍骨架（主角→困境→旧路→转折→解法→结局→内核）
- 输出：速读卡（三句压缩）+ 故事全文 + 检验段
- 保持 li-research 的来源可信度标记

### 4. ljg-read「完美读书」（273 行）

**做什么**：读完一本书——真正读完，不是翻完。

**核心设计**：
- **二进制标准**：看完能复述 = 真读了（1），做不到 = 假读了（0）。"读懂 80%""基本看完"是自欺
- **三分法**：入书（看作者）→ 读书（讲给自己）→ 出书（看自己）
- **检视**：三句话说清这本书 → 复述不了 = 没读，回头重来
- **深读**：要把书读成自己的语言——每段看到什么，什么改变了，什么引发了什么
- **读后产出**：质疑清单 + 关系网 + 行动清单

**为什么高价值**：
- 我们的 li-study 有费曼+间隔重复+认知负荷，但缺少"读书方法论"
- 62 本百大认知书籍 → 用户怎么读这些书？需要系统方法
- "真读假读二进制"可以作为 li-study 的验收门禁

**具体融合方案**：
li-study 新增读书模式：
- 触发词：读书/读完这本书/真读/假读
- 流程：入书（作者+背景）→ 检视（三句话复述）→ 深读（自己的语言）→ 出书（质疑+关系网+行动）
- 验收：二进制标准——能复述 = 1，不能 = 0

### 5. ljg-plain「白」（105 行）

**做什么**：让聪明的 12 岁孩子能 grok。

**9 条红线**（按优先级排序）：
1. 口语检验 — 你会这样跟聪明朋友说话吗？
2. 零术语 — 术语先大白话落地，再顺带提名称
3. 短词优先 — "进行分析"→"看"
4. 一句一事 — 每句只推进一步
5. 具体 — 名词看得见，动词有力气
6. 开头给理由 — 第一句让人想读第二句
7. 不填充 — 删开场白、拐杖词、夸大象征
8. 信任读者 — 说一遍够了
9. 诚实 — "大概 70%" 比"可能"诚实

**额外磨**：破公式、变节奏、杀金句、查跳跃、查译感

**与我们文风DNA的关系**：
- 我们有禁用词列表（说白了/本质上/综上所述...）+ 五要素
- ljg-plain 的 9 条红线更系统、更可执行
- **建议**：将 ljg-plain 的红线检查融入 voice-dna-auto-inject.md 的自检清单

### 6. ljg-writes「写作引擎」（160 行）

**做什么**：外科手术式写观点——一层层剥到底，1000-1500字。

**核心设计**：
- 姿态：外科医生的手，朋友的口
- 过程：把观点放到台面上 → 切第一刀（反问/追问/翻转）→ 切第二刀 → 切到底 → 合起来看
- **不自标深度**：禁用"再深入一层""最深的一层是""更深地说"。深入是思考行为本身，宣告深度 = 把深度戳破
- 磨：口语检验 + AI 痕迹过滤 + 反风格 + 意外检验
- **中文重写**：初稿出来后，合上它，用中文读者的眼睛再写一遍

**与我们的关系**：
- "不自标深度"是文风DNA的绝佳补充——我们的禁用词列表里缺这条
- "中文重写"两稿机制可以增强 li-xhs 的写作流程
- 五种写作手法（场景代替论证、让步弯道、反问入链、探索性语气、短句做锤子）可以融入内容创作 skill

### 7. ljg-qa「问答提取」（98 行）

**做什么**：把文章的论证骨架翻出来，每根骨头长成一个尖锐的问题。

**核心设计**：
- 不是 FAQ、不是摘要换皮、不是知识点列表
- Q 链有方向：Q1 答完→Q2 自然冒出来，读者走完 = 复现了作者的推理路径
- A 严格四段：结论（一句话）+ 形式化（文字+符号压成一行可视关系）+ 论证步 + 边界
- 形式化不是数学公式，是"思想的几何"

**与我们的关系**：
- 我们的 li-wechat 对文章的分析偏"提取要点"，缺乏"Q 链"思维
- 四段式 A（结论+形式化+论证步+边界）可以增强 li-research 的输出结构
- Q 链思维可以增强 li-study 的学习方法

### 8. ljg-roundtable「圆桌讨论」（133 行）

**做什么**：找 3-5 个真实历史/当代人物，围绕一个议题做结构化辩论。

**核心设计**：
- 选人形成"张力网络"（非简单正反方）+ 至少一位"意外视角"
- 每人必须**忠于其真实思想体系**发言，引用经典著作
- 主持人每轮提炼核心争议点 + 生成 ASCII 思考框架图 + 提出下一层引导问题
- 指令：可（继续）/ 止（结束）/ 深入此节 / 引入新人物
- 结束后生成知识网络 ASCII 图

**与我们的关系**：
- 我们的 li-devil 是"魔鬼代言人"——一个人泼冷水
- ljg-roundtable 是**多人辩论**——可以作为 li-analyze 的子模式
- "意外视角"和"张力网络"设计值得学习

## 三、设计哲学深度对比

### 李继刚的 5 个设计哲学（我们没有的）

#### 1. 隐喻即方法论

**他的做法**：每个 skill 的"你是什么"段落定义了执行姿态：
- ljg-think："你是一支离弦之箭"
- ljg-plain："让人 grok"
- ljg-card："铸 — 内容进去，PNG 出来"
- ljg-writes："外科医生的手，朋友的口"

**我们的做法**：Phase 1 → Phase 2 → Phase 3... 功能清单式

**差距**：隐喻不只是修辞，它约束了 AI 的行为模式。"一支箭"意味着不回头、不横向；"外科医生"意味着冷静精准不抖。这比"Phase 2: 执行 X"更能约束 AI 的输出质量。

#### 2. 极简 vs 完备

**他的做法**：ljg-think 只有 67 行，方法论本身就是全部。没有 references/，没有执行协议，没有门禁。

**我们的做法**：每个 li-skill 有 SKILL.md + references/ + Phase + 门禁 + 执行协议。

**差距**：不是所有 skill 都需要复杂流程。认知类工具（思考/写作/阅读）的方法论本身就很短，加 Phase 反而让它笨重。应该分类：
- **执行类 skill**（数据分析/内容创作/项目管理）→ 需要 Phase + references + 门禁
- **认知类 skill**（思考/写作/概念理解）→ 方法论本身就是全部，极简即可

#### 3. 输出格式即身份

**他的做法**：org-mode + ASCII art 是**硬约束**，不是可选风格。每个 skill 都有：
- "Org-mode 语法：加粗用 `*bold*`，禁止 `**bold**`"
- "ASCII Art：只用基本字符，禁止 Unicode 绘图符号"

**我们的做法**：Markdown 为主，格式约束分散在各处。

**差距**：统一的输出格式 = 读者一眼认出"这是谁写的"。这是一种**品牌一致性**。

#### 4. 红线 > 门禁

**他的做法**：ljg-plain 有 9 条红线（口语检验、零术语、短词优先...），每条必须过。不是"Phase 结束检查"，是"写作过程中的持续约束"。

**我们的做法**：门禁在 Phase 结束时检查。

**差距**：红线是**过程约束**，门禁是**结果约束**。过程约束更早发现问题。

#### 5. "不自标深度"规则

**ljg-writes 的铁律**：禁用"再深入一层""最深的一层是""更深地说"。深入是思考行为本身——下一句的内容让读者自己感到"原来不止这样"。说"我要深入了"反而把深入戳破了。

**我们的缺失**：文风DNA的禁用词列表里没有这条。AI 输出经常出现"更深层次来看""本质上来说"。
