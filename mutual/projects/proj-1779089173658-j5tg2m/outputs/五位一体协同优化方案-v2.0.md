## 一、为什么升级

**核心矛盾**：牛马AI（Newmax）越来越难用了。

| 问题 | 表现 | 根因 |
|------|------|------|
| 启动序列过重 | CLAUDE.md 6 个 Step 每次对话都要跑，占用上下文 | 规则膨胀（150+ 轮迭代堆出来的） |
| Skill 路由失灵 | 109 条路由但 96.5% 沉睡，路由命中率低 | 触发词设计脱离真实对话 |
| MCP 配置脆弱 | 3 次配置事故（路径错/import错/协议混用） | Electron 封装层引入额外复杂度 |
| 上下文压缩不可靠 | Read 工具缓存/幻觉问题反复出现 | Newmax 的文件读取有中间层 |

**升级决策**：
- **Newmax → Claude Desktop**：保留 Claude Code 的所有能力（CLAUDE.md、memory/、.claude/rules/、MCP），去掉 Newmax 封装层的问题
- **+ Marvis**：补齐系统级操控能力（文件/系统/应用/浏览器/搜索/数字资产 6 个 Agent）

## 三、五个工具重新定位

| 工具 | v1.0 角色 | v2.0 角色 | 核心变化 |
|------|----------|----------|---------|
| **Marvis** | （新增） | **系统级操控中枢 + 入口** | 6 个 Agent 协作，本地知识库 RAG，每天免费 1000 万 Token |
| **Claude Desktop** | （替代 Newmax） | **推理 + 内容创作中枢** | 保留 CLAUDE.md/memory/rules 体系，去掉 Newmax 封装层 |
| **Hermes** | 记忆中枢 | **记忆中枢（不变）** | 唯一记忆引擎，扫描文件系统同步 |
| **Codex** | 代码引擎 | **代码引擎（不变）** | gpt-5.5，代码/技术文档/架构设计 |
| **WorkBuddy** | UI 引擎 | **UI 引擎（不变）** | HTML/飞书/数据看板 |

### 每个工具的一句话定义

**Marvis**：「你对电脑说什么，它就做什么」——文件管理、系统操作、浏览器自动化、本地知识库搜索。它是你的系统级双手。

**Claude Desktop**：「你想要什么深度分析，它就给你什么」——继承牛马AI 的全部知识体系（CLAUDE.md + memory/ + .claude/rules/ + skill-routing-table），但用 Claude 原生能力运行，无中间层。

**Hermes**：「它记住你所有的历史，发现你自己都没注意到的矛盾」——唯一记忆引擎，跨工具上下文同步。

**Codex**：「代码的事交给它」——gpt-5.5 写代码、管 Git、做技术文档。

**WorkBuddy**：「好看的东西交给它」——HTML 页面、飞书文档、数据可视化。

## 四、协作协议升级

### 4.1 进场协议（升级版）

**v1.0**：进场必问 Hermes
**v2.0**：进场先问 Marvis（系统状态）+ Hermes（记忆上下文）

```
开始新任务
  → Marvis：检查系统状态（磁盘/进程/网络/相关文件位置）
  → Hermes：获取 ≤500 字项目上下文
  → 选择执行工具 → 开始
```

**为什么 Marvis 排在 Hermes 前面**：
- Hermes 的记忆可能过时（上次同步是几小时前）
- Marvis 实时读系统状态，知道文件真实在哪里、进程是否活着
- 系统状态是一切任务的前提（磁盘满了？代理挂了？文件被锁了？）

### 4.2 工具选择决策树

```
收到任务 →
  ├─ 需要操作系统/文件系统/浏览器？
  │   └─ YES → Marvis
  │            ├─ 文件查找/格式转换/跨端传输 → Marvis File Agent
  │            ├─ 系统设置/性能分析/进程管理 → Marvis Computer Agent
  │            ├─ 网页搜索/自动操作/登录抓取 → Marvis Browser Agent
  │            └─ 本地知识库 RAG 查询 → Marvis Search Agent
  │
  ├─ 需要深度推理/内容创作/Skill管理？
  │   └─ YES → Claude Desktop
  │            ├─ 公众号/小红书/分析报告 → Claude Desktop + li-* skill
  │            ├─ 概念拆解/决策分析 → Claude Desktop + dao-fa-shu-qi
  │            ├─ 深度调研 → Claude Desktop + li-research
  │            └─ 记忆/偏好管理 → Claude Desktop → 写入 memory/
  │
  ├─ 需要写代码/技术文档/Git 操作？
  │   └─ YES → Codex CLI
  │            ├─ Python/Verilog/脚本 → Codex
  │            ├─ 技术文档/API 文档 → Codex
  │            └─ Git commit/PR/分支管理 → Codex
  │
  ├─ 需要做页面/飞书文档/数据看板？
  │   └─ YES → WorkBuddy
  │
  └─ 不确定 → Marvis（最通用，系统级能力兜底）
```

### 4.3 交接协议（不变）

交接文件路径：`handoffs/{项目名}-{from}-{to}-{任务简称}.md`

三栏结构：
1. **{from} 已完成**：产出路径 + 一句话描述
2. **{to} 需要做**：具体任务清单
3. **约束 & 注意事项**

### 4.4 产出注册（不变）

产出可复用文件后 → 追加到 `artifact-registry.md`

## 五、任务流转路线（升级版）

### 路线 A：内容创作（Claude Desktop 端到端）
```
用户需求 → Claude Desktop（分析+创作） → 产出注册
```

### 路线 B：技术项目（Codex + WorkBuddy + Claude Desktop）
```
用户需求 → Claude Desktop（需求分析） → Handoff → Codex（代码+文档）
    → Handoff → WorkBuddy（展示页） → Handoff → Claude Desktop（推广文案）
```

### 路线 C：系统运维（Marvis 独占）
```
用户需求 → Marvis（系统操作） → 结果写入 artifact-registry.md
```
示例：文件整理、磁盘清理、环境配置、软件安装

### 路线 D：深度调研（Marvis + Claude Desktop）
```
用户需求 → Marvis Browser Agent（网页搜集） → 搜索结果写入临时文件
    → Handoff → Claude Desktop + li-research（深度分析） → 调研报告
```

### 路线 E：本地知识库查询（Marvis RAG）
```
用户提问 → Marvis Search Agent（本地文档 RAG） → 检索结果
    → 如果需要深度分析 → Handoff → Claude Desktop
```
**核心场景**：62 本认知书籍、考研资料、项目文档的本地检索

### 路线 F：跨端协作（Marvis 独占）
```
手机指令 → Marvis（PC 端执行） → 结果传回手机
```
示例：手机上说"把桌面截图发给我" → Marvis 截图 → 传输

### 路线 G：飞书内容（Claude Desktop + WorkBuddy）
```
用户需求 → Claude Desktop（内容创作） → Handoff → WorkBuddy（飞书文档化+推送）
```

## 八、Hermes 保持不变的部分

Hermes 作为记忆中枢的职责不变：
- **唯一记忆引擎**：所有工具的记忆写入都通过文件系统，Hermes 扫描同步
- **矛盾检测**：发现不同工具记录的信息矛盾时告警
- **上下文注入**：新任务开始时提供 ≤500 字上下文摘要
- **自动遗忘**：超过 90 天的低置信记忆自动归档

**Hermes 与 Marvis 的分工**：
| 维度 | Hermes | Marvis |
|------|--------|--------|
| 记忆 | 语义级（理解内容含义） | 系统级（知道文件在哪） |
| 状态 | 项目状态（做了什么/要做什么） | 系统状态（磁盘/进程/网络） |
| 矛盾 | 信息矛盾检测 | 文件冲突/锁检测 |
| 时间维度 | 长期记忆（天/周/月） | 实时状态（秒/分钟） |

## 十一、风险与边界声明

### 可声称
- ✅ Marvis 的 6 个 Agent 覆盖了牛马AI 不涉及的 OS 层（证据：Marvis 产品文档 + 36kr 评测）
- ✅ Claude Desktop 比 Newmax 封装层更稳定（证据：原生 vs 封装的通用规律）

### 仅供参考
- ⚠️ Marvis 速度问题：微博签到 3 分钟、图像搜索 200 万 Token（36kr 评测数据），高频使用可能有影响
- ⚠️ Marvis 本地知识库 RAG 质量未知，需要实际测试 62 本认知书籍的检索效果

### 尚不能声称
- ❓ Claude Desktop 是否完全兼容 Newmax 的 MCP 配置和 skill 系统（需要实测）
- ❓ Marvis + Claude Desktop 的协作延迟是否可接受（需要实测）
- ❓ 五位一体后维护成本是否低于四位一体（需要至少 2 周数据）
