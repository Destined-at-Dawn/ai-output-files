# SOP: 文献引用验证流程（学术诚信 — 最高优先级）

> 优先级：最高（与 no-blind-overwrite 同级）
> 创建：2026-06-19
> 根因：academic-research-skills 工具引用错误率约 50%（外部实测反馈）

---

## 铁律

**所有文献引用必须经二次验证。未验证的引用禁止进入正式论文。**

没有例外。没有"这次引用看起来没问题"的例外。没有"时间紧迫先跳过验证"的例外。

---

## 验证流程（每次获取引用后强制执行）

### Step 1: 提取引用信息
从 AI 输出中提取：
- 标题（Title）
- 作者（Authors）
- 年份（Year）
- DOI（如有）
- 期刊/会议（Venue）

### Step 2: 验证方式（按优先级）

#### 优先级 1：DOI 解析（最可靠）
```
验证方式：访问 https://doi.org/{DOI}
成功标准：页面能正常打开，显示的标题/作者与引用一致
信任等级：🔴 高信任
```

#### 优先级 2：标题精确搜索
```
验证方式：Google Scholar 标题精确搜索（用引号包围标题）
成功标准：搜索结果第一条与引用信息完全匹配（标题+作者+年份）
信任等级：🟡 中信任
```

#### 优先级 3：作者+年份+关键词搜索
```
验证方式：Google Scholar 搜索 "作者 年份 关键词"
成功标准：能找到匹配的文献，但可能存在同名文献
信任等级：⚠️ 低信任（需人工确认）
```

### Step 3: 验证结果标记

| 验证状态 | 标记 | 处理方式 |
|----------|------|----------|
| 验证通过（DOI 解析成功） | ✅ [Verified-DOI] | 可直接使用 |
| 验证通过（标题搜索成功） | ✅ [Verified-Title] | 可使用，建议补充 DOI |
| 部分验证（仅找到相似文献） | ⚠️ [Partial-Verified] | 需人工确认后使用 |
| 验证失败（找不到匹配文献） | ❌ [Verification-Failed] | 禁止使用，删除该引用 |
| 未验证（跳过验证步骤） | 🚫 [Unverified] | 禁止进入正式论文 |

## 信任层级体系

### 🔴 高信任（可直接引用）
- 有 DOI 且 DOI 解析成功
- 有标题且 Google Scholar 第一条精确匹配
- 来自正规数据库（IEEE Xplore, ACM DL, arXiv, PubMed 等）

### 🟡 中信任（建议补充验证）
- 有标题但搜索结果在第 2-5 条
- 有 DOI 但页面显示信息不完整
- 来自预印本服务器（需确认是否已正式发表）

### ⚠️ 低信任（需人工确认）
- 仅通过作者+年份找到
- 搜索结果有多条相似文献
- 来自非主流来源（个人网站、博客等）

### ❌ 不信任（禁止使用）
- 验证失败的引用
- 无法找到任何匹配文献
- DOI 解析显示错误页面
- 标题搜索无结果或结果完全不匹配

## 工作流集成

### deep-research 阶段调整

**原流程**：
```
deep-research → 获取文献列表 → 直接使用
```

**新流程（强制）**：
```
deep-research → 获取初始文献列表
    ↓
verification 阶段（新增）
    - 逐条验证引用
    - 标记验证状态
    - 删除验证失败的引用
    ↓
verified-literature（最终可用文献）
    ↓
literature-review（基于验证后的文献）
```

### academic-paper 阶段调整

- 所有引用必须带有验证标记
- 论文参考文献列表只包含 ✅ [Verified-DOI] 或 ✅ [Verified-Title]
- 在 methods 或 acknowledgments 中说明引用验证流程

---

## 禁止模式

| 禁止行为 | 原因 |
|----------|------|
| 直接使用 AI 输出的引用而不验证 | 50% 错误率不可接受 |
| 用"看起来合理"代替实际验证 | AI 能生成看似真实但不存在的引用 |
| 仅验证部分引用（抽样验证） | 每条引用都可能有问题 |
| 验证失败后仍使用该引用 | 学术诚信红线 |
| 时间紧迫时跳过验证 | 质量 > 速度 |

## 例外情况

### 可以跳过验证的场景（极少数）
- 引用的是自己的已发表论文（你已知是真实的）
- 引用的是教科书或经典著作（广泛已知）
- 引用的是官方标准文档（IEEE Std, RFC 等）

### 必须验证的场景（无例外）
- AI 生成的任何引用
- 从其他论文中"转引"的引用（需验证原始文献）
- 任何用于支撑关键论点的引用

---

## 违规检测信号

如果出现以下任何情况，说明本规则被违反了：

- [ ] 论文中有引用但没有验证标记
- [ ] 引用列表中有 [Unverified] 或 [Verification-Failed] 标记
- [ ] 用户问"这个引用验证过吗"时答不上来
- [ ] 引用的 DOI 无法解析或标题搜索无结果

---

## 事故响应

如果发现论文中使用了未验证或验证失败的引用：

1. 立即停止使用该引用
2. 尝试找到正确的文献（可能需要修改引用信息）
3. 如果找不到正确文献，删除该引用并调整论文内容
4. 记录到 negative-results.md 作为教训
5. 重新验证所有其他引用（防止类似问题）
