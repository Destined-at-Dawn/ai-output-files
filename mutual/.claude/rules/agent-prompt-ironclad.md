# Agent Prompt 铁律（最高优先级 — Agent 故障根因修复）

> 来源：5 次 Agent 故障复盘（2026-06-09）
> 根因：Agent 子会话无主对话上下文，prompt 泛化 = Agent 必然跑偏
> 优先级：与 script-safety-check 同级

---

## 铁律 1：Agent Prompt 必须包含三要素

每次调用 Agent 工具，prompt 字段必须明确回答三个问题。缺任何一项 → 禁止启动 Agent。

### 三要素

| 要素 | 含义 | 反面示例 | 正确示例 |
|------|------|----------|----------|
| **具体目标** | Agent 要产出什么 | "搜索 FPGA 文件" | "找到 D:\AMD 下所有 .v 文件中包含 `I2C` 关键字的文件路径和行号" |
| **输出格式** | 返回什么结构 | "分析一下" | "返回路径列表，每行一个绝对路径" |
| **停止条件** | 什么时候该停 | "继续搜" | "搜完 D:\AMD 全部子目录后停止" 或 "找到 3 个匹配就停止" |

### 三要素缺失的后果

| 缺失项 | Agent 行为 | 结果 |
|--------|-----------|------|
| 无具体目标 | Agent 猜测任务范围，过度搜索或跳过关键文件 | 半小时后返回 500 行无关内容 |
| 无输出格式 | Agent 返回自然语言叙述，信息无法被程序使用 | "好像有 XX 文件"——无法定位 |
| 无停止条件 | Agent 继续搜索不必要的内容，超时或浪费 token | 任务超时，已搜到的也被丢弃 |

## 铁律 2：Agent 类型必须匹配任务

| Agent 类型 | 擅长的任务 | 不擅长的任务 | 读文件能力 |
|-----------|-----------|------------|-----------|
| **Explore (quick)** | 按文件名/模式找文件、grep 关键词 | 综合分析、跨文件对比 | 只读片段，会漏内容 |
| **Explore (medium)** | 中等范围搜索 | 深度分析 | 读片段 |
| **Explore (very thorough)** | 大范围搜索 | 写代码 | 读片段 |
| **general-purpose** | 多文件综合分析、写代码、复杂任务 | 快速定位（overhead 大） | 完整读取 |
| **Plan** | 架构设计、实现计划 | 搜索文件（不能 Edit/Write） | 完整读取 |
| **code-reviewer** | 代码审查、安全检查 | 通用任务 | 完整读取 |

**用错类型的代价**：
- Explore 搜代码库找 bug → ok
- Explore 做综合分析 → 它只读片段，必然漏关键信息
- general-purpose 搜文件 → 太慢，overhead 大
- Plan 搜文件 → 不能 Edit/Write，搜了也没用

**决策速查**：
```
找文件/搜关键词 → Explore (quick/medium)
读多个文件做分析 → general-purpose
评估代码质量 → code-reviewer
设计架构方案 → Plan
什么都不确定 → general-purpose（最安全）
```

## 铁律 3：模基混合策略（mimo 用户专用）

**主会话 mimo（免费额度），Agent 按需选模型。**

| Agent 任务类型 | 模型 | 原因 |
|---------------|------|------|
| 纯文本搜索/分析 | 继承（mimo） | 不需要 multimodal，免费额度 |
| 需要读 PDF/图片/文件 | `model: "opus"` | mimo 不支持 multimodal，需切到 Opus |
| 代码生成 | 继承（mimo） | 纯文本任务 |
| 需要准确理解意图 | `model: "opus"` | Opus 理解力更强 |

**调用方式**：
```
# 纯文本 Agent（不指定 model，继承 mimo）
Agent(description="搜索 verilog 文件", prompt="...")

# 需要读 PDF 的 Agent（显式指定 opus）
Agent(description="读取 FPGA 文档", prompt="...", model="opus")
```

**关键**：model 参数是 Agent 工具的原生参数——不需要切换主会话模型，只在这次 Agent 调用中覆盖。

## 铁律 4：Agent 故障三问排查

Agent 失败后，不问"为什么失败了"，问三个具体问题：

```
1. 「Agent 类型对不对？」 → 搜文件用了 general-purpose？分析用了 Explore？
2. 「Prompt 有三要素吗？」 → 具体目标/输出格式/停止条件缺了哪个？
3. 「是不是 multimodal 任务用了 mimo？」 → 需要读 PDF 但 Agent 继承了 mimo？
```

按这个顺序排查，不要跳跃。

## 铁律 5：Agent 上下文注入标准模板

Agent 看不到主对话历史。每次调用前，在 prompt 中注入以下上下文：

```markdown
## 背景
[1-2 句话说清楚这个任务在做什么项目、为什么需要这个信息]

## 你的任务
[具体目标 — 三要素]

## 工作目录
[绝对路径]

## 已知信息
[主对话中已经发现的，Agent 不需要再搜的]

## 约束
- 只读，不修改任何文件
- 搜完 [X] 个目录后停止
- 返回 [具体格式]
```

---

## 违规检测信号

- [ ] Agent prompt 中出现"搜索一下"、"看看"、"分析一下"等泛化词 → 缺三要素
- [ ] Agent 跑完后说"没找到需要的信息" → 目标不够具体
- [ ] 同一个 Agent 任务重复了 2 次以上 → 类型或模型不匹配
- [ ] Agent 返回了 500+ 行但无法直接使用 → 输出格式没指定
- [ ] Agent 超时 → 没设置停止条件
