# 记忆沉淀候选机制（Memory Candidate Protocol）

> AI 不得"偷偷写入"记忆文件。任何记忆写入前必须经过用户确认。

---

## 核心规则

**禁止未经确认直接写入 memory/ 目录的任何文件。**

所有记忆沉淀必须走以下流程：

```
AI 发现值得记录的信息
    → 提出候选（mcp__ask-user__ask_user）
    → 用户确认/修改/拒绝
    → 确认后才写入
```

---

## 触发条件

以下情况触发记忆候选流程：

1. **用户纠正 AI**（"不对，应该是..."、"我喜欢..."、"下次别这样..."）
2. **重复踩坑**（同类问题在本次对话中出现 ≥2 次）
3. **重要决策**（用户做了影响后续工作的选择）
4. **任务完成后**（对话结束前检查是否有值得沉淀的教训）

---

## 候选格式

AI 提出候选时，必须使用以下结构：

```
📋 记忆候选

- 类型：[偏好/教训/决策/知识]
- 内容：[一句话描述]
- 建议写入位置：memory/long-term.md § [对应章节]
- 置信度：[高/中/待确认]

请回复：
1. 写入（原样写入）
2. 修改后写入（告诉我怎么改）
3. 不记录
```

---

## 不需要候选的例外

以下情况可以直接写入（跳过候选流程）：

- **每日记忆 memory/{日期}.md**：AI 可以直接追加当日任务记录
- **项目产出 outputs/**：最终交付物可以直接写入
- **用户明确说"记录下来"**：用户主动要求记录时

---

## 候审区（Pending Candidates）

如果用户在对话中没有回应候选（对话结束），候选信息不写入，但保留在当日记忆的 `## 待审候选` 小节中，下次对话时提醒用户。

---

## 反过度沉淀

**单次未重复的反馈不入库。** 只有以下情况才升级为永久记忆：

- 同类纠正 ≥ 2 次
- 用户明确要求"记住这个"
- 影响跨对话工作的决策

---

## 记忆推动机制（Memory Nudges）— v2 新增

> 来源：Hermes Agent 辩证式记忆系统 + MindStudio 自改进模式
> 目标：让记忆系统更智能，减少人工确认负担

### Nudge 1：会话结束摘要
**触发**：对话即将结束（用户说"再见"/"结束"/长时间无消息）
**动作**：自动生成本次对话的结构化摘要，追加到 `memory/{today}.md`
**内容**：任务完成情况、关键决策、新增教训、待跟进事项

### Nudge 2：模式检测自动持久化
**触发**：同类观察/纠正出现 ≥3 次
**动作**：自动从"待确认"升级为"中置信"，写入 `memory/long-term.md`
**条件**：
- 3 次观察必须跨不同对话（同一次对话内的重复不计）
- 模式必须可明确描述（不能是模糊的"用户似乎喜欢..."）
- 写入时标注 `[中置信] [自动持久化: 3次观察]`

### Nudge 3：定期记忆健康检查
**触发**：每 10 轮对话或每次压缩后
**动作**：
1. 检查 `memory/long-term.md` 中是否有超过 30 天未更新的记忆
2. 标记为 `[需刷新]`
3. 检查是否有矛盾记忆（新信息与旧信息冲突）
4. 检查当前任务是否与某条记忆相关但被跳过

### Nudge 4：记忆过期清理
**触发**：每月 1 日或手动触发
**动作**：
1. 超过 90 天且 `[低置信]` 的记忆 → 归档到 `memory/archive/`
2. 超过 180 天的所有记忆 → 评估是否仍有价值
3. 标记为 `[失效条件: xxx]` 且条件已满足的记忆 → 归档
