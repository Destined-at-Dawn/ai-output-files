# 竞赛项目工作区架构规范（全局版 — 所有竞赛适用）

> **优先级**：最高（与 chinese-path-safety.md 同级）
> **创建日期**：2026-06-14
> **来源**：竞赛区 v2.0 规范的全局同步版（`E:\ai产出文件\牛马\竞赛\竞赛\.claude\rules\competition-workspace-architecture.md`）
> **适用范围**：牛马 AI 生态中所有竞赛项目（YOLO、FPGA、电力巡检等）

---

## 核心铁律

**每个竞赛项目有且仅有一个主文件夹，所有管理文件必须放在主文件夹内。**

---

## 一、架构总览

```
{工作区根目录}/              ← 竞赛工作区
├── {竞赛名}/                ← 主文件夹（唯一）
│   ├── outputs/             ← 产出文件
│   ├── memory/              ← 项目级记忆（优先读写）
│   ├── SOPs/                ← 项目SOP
│   ├── skills/              ← 项目级skill
│   ├── CLAUDE.md            ← 项目约束
│   └── .claude/rules/       ← 项目规则
├── projects/proj-xxx/       ← 临时辅助（只放 task.json + project.md）
├── memory/                  ← 工作区级记忆（跨项目通用教训）
└── CLAUDE.md                ← 工作区约束

D:/{竞赛数据}/               ← 代码和数据（可在其他位置）
├── runs/                    ← 训练结果
└── 数据集/                  ← 训练数据
```

## 二、文件分类规则

### 管理文件（必须在主文件夹）

| 文件类型 | 路径 | 说明 |
|----------|------|------|
| `outputs/` | `{主文件夹}/outputs/` | 论文、报告、调研结果 |
| `memory/` | `{主文件夹}/memory/` | 项目级记忆 |
| `SOPs/` | `{主文件夹}/SOPs/` | 项目 SOP |
| `skills/` | `{主文件夹}/skills/` | 项目级 skill |
| `CLAUDE.md` | `{主文件夹}/CLAUDE.md` | 项目约束 |
| `.claude/rules/` | `{主文件夹}/.claude/rules/` | 项目规则 |

### 代码文件（可在其他位置）

| 文件类型 | 说明 |
|----------|------|
| 训练脚本 | D 盘或其他位置 |
| 数据集 | D 盘或其他位置 |
| 训练结果/runs | D 盘或其他位置 |

## 三、proj-xxx ↔ 主文件夹自动感知协议

### 自动映射表

| proj-xxx 所属 | 对应主文件夹 |
|--------------|-------------|
| `projects/proj-*`（工作区根目录下） | 该工作区的**当前活跃竞赛主文件夹** |
| 新竞赛项目（尚无主文件夹） | 先创建主文件夹，再执行任务 |

### 感知协议（每次从 proj-xxx 发起任务时强制执行）

```
Step 0: 识别当前位置
        → 如果 cwd 含 projects/proj- → 触发感知协议
        → 如果 cwd 已在主文件夹内 → 跳过

Step 1: 查映射表 → 找到对应主文件夹路径
        → 无对应 → 问用户确认

Step 2: 管理文件写主文件夹，proj-xxx 只保留 task.json + project.md

Step 3: 执行任务（代码可在任意位置，产出报告/记忆/SOP 必须写主文件夹）
```

## 四、两级 Memory 读写优先级

### 读取优先级

| 优先级 | 路径 | 何时读 |
|--------|------|--------|
| **P0** | `{主文件夹}/memory/{今天日期}.md` | 每次对话必读 |
| **P1** | `{主文件夹}/memory/long-term.md` | 每次对话必读 |
| **P2** | `{主文件夹}/memory/lessons-learned.md` | 每次对话必读 |
| **P3** | `{工作区根目录}/memory/{今天日期}.md` | 按需（跨项目通用） |
| **P4** | `{工作区根目录}/memory/long-term.md` | 按需（跨项目通用） |

### 写入规则

| 内容类型 | 写入位置 |
|----------|---------|
| 项目进展、训练结果、竞赛决策 | `{主文件夹}/memory/{日期}.md` |
| 项目级教训 | `{主文件夹}/memory/lessons-learned.md` |
| 跨项目通用教训 | `{工作区根目录}/memory/{日期}.md` |

**铁律**：同一教训禁止写两级——选对的那一级写。

## 五、memory → skill/rule 反向优化矩阵

### 触发条件

每次写入 memory 时（教训/决策/发现），执行以下判断：

```
写入 memory/{日期}.md
  → 这条教训涉及哪个领域？
    ├─ 训练/数据集 → 检查 competition-yolo skill
    ├─ Goal 模式 → 检查 goal-mode-anti-repeat.md
    ├─ 架构/文件管理 → 检查 competition-workspace-architecture.md
    ├─ Skill 使用 → 检查对应 SKILL.md
    ├─ SOP 流程 → 检查对应 SOP 迭代日志
    └─ 通用工程法则 → 检查 .claude/rules/ 对应规则
```

### 优化矩阵

| 教训来源 | 可能需要更新的文件 | 更新方式 |
|----------|-------------------|---------|
| 训练失败/数据问题 | competition-yolo SKILL.md | Edit 追加避坑指南 |
| Goal 模式踩坑 | goal-mode-anti-repeat.md | Edit 追加铁律 |
| 架构违规 | competition-workspace-architecture.md | Edit 追加规则 |
| Skill 路由漏调 | skill-routing-table.json | Edit 添加触发词 |
| SOP 流程缺陷 | 对应 SOP.md | Edit 迭代日志 |
| 同类教训 ≥3 次 | CLAUDE.md F编号 | 升级为铁律 |

**铁律**：写入 memory 后必须执行反向优化判断，判断结果记录在当日 memory 末尾。

## 八、认知科学支撑

| 原理 | 来源 | 在本规则中的应用 |
|------|------|-------------------|
| **单一真相源** | 系统思维 | 主文件夹是竞赛项目的唯一真相源 |
| **路径依赖** | 认知心理学 | 文件放错位置 → 后续所有操作基于错误路径 |
| **心智模型** | 认知科学 | 清晰架构 → AI 建立正确心智模型 |
| **反馈闭环** | 控制论 | memory→skill/rule 反向优化 = 教训必须产生行动 |
