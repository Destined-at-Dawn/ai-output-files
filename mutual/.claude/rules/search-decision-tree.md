# 搜索决策树规则（Search Decision Tree）

> 来源：Claude Fable 5 系统提示词逆向分析 + Anthropic 搜索设计哲学
> 创建：2026-06-14
> 洞见：搜索不是一句话"必要时搜索"，而是一套完整的决策流程

---

## 核心原则

**"必要时搜索" = 没给指令。搜索规则必须是决策树，不是模糊建议。**

---

## 一、搜索决策树（每次信息获取前必过）

```
我需要获取外部信息？
  ├─ 这是不变的事实吗？
  │   └─ 是 → 不搜索，直接用已知知识
  │       例：勾股定理、死人生平、物理常数、已发布的国际标准
  │
  ├─ 这是可能变化的状态吗？
  │   └─ 是 → 必须搜索，禁止凭记忆回答
  │       例：某人当前年龄、产品最新价格、股票实时行情、公司当前 CEO
  │
  ├─ 这是我不认识的实体/名词吗？
  │   └─ 是 → 必须搜索，禁止凭记忆编造
  │       例：没听过的公司名、新发布的产品、不熟悉的论文作者
  │
  └─ 这是需要最新信息的问题吗？
      └─ 是 → 必须搜索
          例："最新""2026""最近""当前""现在"
```

---

## 二、搜索深度分级

| 问题复杂度 | 搜索次数 | 示例 |
|-----------|---------|------|
| **简单** | 1 次 | 查一个事实/定义/日期 |
| **中等** | 3~5 次 | 对比 2-3 个方案/验证多个来源 |
| **深度研究** | 5~10 次 | 系统性调研/竞争格局/全量扫描 |

**反模式**：简单问题搜 10 次 → 信息过载；深度问题只搜 1 次 → 结论不可靠。

---

## 三、搜索工具选择

| 工具 | 适用场景 | 限制 |
|------|---------|------|
| `mcp__web-search__web_search` | 实时信息、新闻、外部网页 | 返回摘要，非全文 |
| `WebFetch` | 指定 URL 的页面内容提取 | 部分网站受限 |
| `browser-use` | 需要渲染/登录的页面 | 慢，适合关键页面 |
| `Grep` / `Glob` | 本地文件搜索 | 仅本地 |
| `mcp__mempalace__search` | 语义搜索个人记忆 | 仅已索引内容 |

**原则**：能用 web_search 解决的不用 browser（快 > 慢）；需要全文的用 WebFetch。

---

## 四、搜索质量门禁

搜索完成后的自检：

- [ ] 信息源是否标注？（Source: URL/文件名）
- [ ] 搜索次数是否匹配问题复杂度？
- [ ] 是否搜索了"不认识"的实体？（而非凭记忆编造）
- [ ] 是否搜索了"可能变化"的状态？（而非过时知识）

## 六、负面示例（来自文章洞见#8）

- ❌ "必要时搜索" → 太模糊，等于没给指令
- ❌ "搜索 2025 年最新 iPhone"在 2026 年执行 → 过期，应自动纠正为 2026
- ✅ 搜索决策树 + 深度分级 = 机器可执行的精确指令
