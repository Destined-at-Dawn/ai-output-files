# 工作流改进收件箱（Workflow Inbox）

> 突然想到的优化点、工作流改进想法，先丢这里，不立即改规则。
> 每周清理一次：验证有效的升级为 SOP，无效的标记放弃。

---

## 收件规则

1. **想法出现** → 立即写入本文件（不改五区规则）
2. **标记适用范围**和最小验证方式
3. **每周清理**一次：
   - 未验证的标记为 `过期-未验证`
   - 试用中有效的升级为 SOP 草案
   - 两次复用 → 正式 SOP
   - 跨区高频有效 → 共享协议
   - 涉及安全/覆盖/删除 → 底层铁律

## 升级路径

```
一次想法 → workflow-inbox（本文件）
两次复用 → SOP 草案
三次稳定有效 → 正式 SOP（SOPs/）
跨区高频有效 → 共享协议（知识中枢/02-共享规则/）
涉及安全/覆盖/删除 → 底层铁律（.claude/rules/）
```
