# R03 — 文件操作铁律（真源 v1.1）

> **「承诺」一文不值，只有 tool_use 发起的实际文件操作才算数。**
> Source: 交叉学习自竞赛区（Verilog FPGA CLAUDE.md + 皮影戏 CLAUDE.md）

---

## 标准文本

| 规则 | 说明 |
|------|------|
| **3.1 禁止伪造** | 所有文件操作必须通过 tool_use（Write/Edit/Bash）发起，禁止在纯文本中描述"已写入 xxx"但本轮未发 tool_use |
| **3.2 验证必须** | 每次 Write/Edit 后，必须 Read 同一文件确认内容正确落盘 |
| **3.3 绝对路径审计** | 写入路径必须是 `{{工作区路径}}\`（不是少一层的上层目录）。系统会核对每个声称的绝对路径，虚假路径会被记录 |
| **3.4 禁止空文件** | 写入的内容必须有实际信息，不能写空文件或"今日无更新"占位符 |
| **3.5 文件类型强制调 skill** | 遇到 `.pdf` → **先读 pdf skill 指南**（`mcp__skill-handler__Skill skill: "pdf"`）→ 用 pdfplumber 提取 → 输出到**当前工作区 `outputs/`**。禁止跳过 skill 直接写代码（2026-05-17：pdfplumber 已安装，中文支持优于 PyMuPDF） |

**验证格式：** 每次写文件后，回复中必须注明 `✅ 已验证: {文件路径} — {行数}行/{大小}`
