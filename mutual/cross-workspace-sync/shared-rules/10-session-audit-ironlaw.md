# R10: 技能审计铁律

> 版本：v1.2 | 最后更新：2026-05-11
> 共享级别：L2（全4工作区 + 项目级均需植入）

---

## 规则内容

每次对话结束前，**必须**输出以下格式的技能审计报告。没有例外。不可跳过。不是"建议"，是铁律。

```
📋 技能审计

本次调用：{技能名}（{详情}）、{工具名} ×{次数}

被忽视的：{应该调用但没调用的技能/文件} —— {为什么}

下次改进：{问题/坑/优化方向}
```

---

## 格式铁律

1. 标题与各字段之间**必须有空行**（提升可读性）
2. 各字段（本次调用/被忽视的/下次改进）之间**必须有空行**
3. 不加感叹号、不加鼓励语、不加emoji装饰
4. 一行一条，高信息密度

---

## 安装位置

此规则必须出现在 CLAUDE.md 的**最前面几节**（metadata 之后、§零之前），不能埋在末尾。

理由：AI 读 CLAUDE.md 时前几节的注意力最强，末尾容易被跳过。

---

## 触发条件

| 场景 | 动作 |
|------|------|
| 用户说"总结/复盘/结束/收工/拜拜" | 立即触发 |
| 问题已解决、无新任务 | 自动触发 |
| 每次对话至少触发一次 | 即使对话很短 |
| 用户说"不用总结" | 跳过 |
| 纯闲聊无实质任务 | 简化为一句话 |

---

## 违反后果

输出标注：`⚠️ 本次对话质量不合格：缺少技能审计`

## 同步验证步骤（同步时强制执行）

> ⚠️ **2026-05-11 教训：竞赛区 §1.3 旧版本格式没空行，同步时漏掉了。**
> 根因：Grep 只搜了 F1 标题，没搜「技能审计」关键词的所有变体。

**同步本规则时，必须执行以下三步：**

**Step 1 — Grep 全量搜索（不只搜 F1，不只搜 CLAUDE.md）**
```
搜索模式：「技能审计」
搜索范围：目标工作区全部 .md 文件（CLAUDE.md + memory/long-term.md + projects/ 下所有 CLAUDE.md）
目的：找出所有包含审计格式的位置（F1、§1.3、项目级、memory中的历史模板等）
```
> 2026-05-11 补充：旧格式还藏在 memory/long-term.md 中，AI 读历史记忆时照抄旧模板。

**Step 2 — 逐个验证格式**
对每个匹配位置，检查：
- [ ] 📋 与"本次调用"之间有空行？
- [ ] "本次调用"与"被忽视的"之间有空行？
- [ ] "被忽视的"与"下次改进"之间有空行？
- [ ] 字段名是标准名（本次调用/被忽视的/下次改进），不是旧名（已写入记忆/未调用但可用）？

**Step 3 — 修复不一致项**
任何一处格式不对 → 立即 Edit 修复 → Read 验证

**禁止**：只改 F1 就声称同步完成。
