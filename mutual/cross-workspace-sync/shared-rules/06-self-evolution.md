# R06 — 自我进化机制（真源 v1.0）

> 只统一基础框架。求职区的 Pattern-Key 体系、记录格式模板等扩展机制保留在各工作区的领域专属规则中。

---

## 标准文本（基础框架）

### 目录结构

```
self-evolution/
├── 做得好的鼓励/    ← 用户肯定时，保存该轮对话
└── 做得差的避免/    ← 用户批评时，保存该轮对话
```

### 操作规则

**当用户说"做得好/不错/厉害/满意"时：**
- 将本轮一问一答提取为 `.md` 文件，保存到 `self-evolution/做得好的鼓励/`
- 文件名：`{YYYYMMDD}_{主题}_成功案例.md`

**当用户批评或指出错误时：**
- 将本轮一问一答提取为 `.md` 文件，保存到 `self-evolution/做得差的避免/`
- 文件名：`{YYYYMMDD}_{主题}_失败教训.md`
- 包含：原问题 + AI回复 + 错误原因 + 正确做法
- 写入 memory，防止同类错误重复

### 触发铁律（来自创作区）

> **用户的提问带有问号且内容为指责/批评/否定时，必须无条件记录。**
> 禁止辩解、解释、推卸。只记录，不争论。

### 反馈闭环（R09 联动）

> **教训只记不改 = 白记。** 每条教训必须流向它能影响的 SOP 或 skill。

记录教训后的强制操作：
1. 查 `映射表`（见 R09）确定受影响的 SOP
2. 在 SOP「迭代日志」追加一行
3. 同类教训 3 次 → 升级为 SOP 铁律
4. 影响 skill 行为 → 更新 CLAUDE.md 对应规则段落

完整规则见：`shared-rules/09-feedback-loop.md`
