# 知识库三层结构

> 方法论: 信息差6 — 知识库用三层结构设计

## 层级说明

| 层级 | 目录 | 性质 | 生命周期 |
|------|------|------|----------|
| Layer 0 | layer0-fragments/ | 碎片，快速积累 | 30天审查 |
| Layer 1 | layer1-cards/ | 知识卡，独立可复用 | 长期保留 |
| Layer 2 | layer2-executable/ | Skill/SOP/规则 | 跟随生命周期 |

## 数据流

```
对话/研究 → 碎片(L0) → 提炼 → 知识卡(L1) → 固化 → Skill/SOP(L2)
                                        ↑                        |
                                        └────── 反馈 ←───────────┘
```

## 管理规则

- 碎片: 30天未引用→待清理，90天→归档
- 知识卡: 定期审查，用户纠正即更新
- 可执行层: li-manage负责生命周期
